[38;2;248;248;242mTHE ZSH/ATTR MODULE[0m
[38;2;248;248;242m       The zsh/attr module is used for manipulating extended attributes.  The -h  option[0m
[38;2;248;248;242m       causes  all  commands to operate on symbolic links instead of their targets.  The[0m
[38;2;248;248;242m       builtins in this module are:[0m

[38;2;248;248;242m       zgetattr [ -h ] filename attribute [ parameter ][0m
[38;2;248;248;242m              Get the extended attribute attribute from the specified filename.  If  the[0m
[38;2;248;248;242m              optional argument parameter is given, the attribute is set on that parame‐[0m
[38;2;248;248;242m              ter instead of being printed to stdout.[0m

[38;2;248;248;242m       zsetattr [ -h ] filename attribute value[0m
[38;2;248;248;242m              Set the extended attribute attribute on the specified filename to value.[0m

[38;2;248;248;242m       zdelattr [ -h ] filename attribute[0m
[38;2;248;248;242m              Remove the extended attribute attribute from the specified filename.[0m

[38;2;248;248;242m       zlistattr [ -h ] filename [ parameter ][0m
[38;2;248;248;242m              List  the  extended attributes currently set on the specified filename. If[0m
[38;2;248;248;242m              the optional argument parameter is given, the list of attributes is set on[0m
[38;2;248;248;242m              that parameter instead of being printed to stdout.[0m

[38;2;248;248;242m       zgetattr and zlistattr allocate memory dynamically.  If the attribute or list  of[0m
[38;2;248;248;242m       attributes  grows between the allocation and the call to get them, they return 2.[0m
[38;2;248;248;242m       On all other errors, 1 is returned.  This allows the calling  function  to  check[0m
[38;2;248;248;242m       for this case and retry.[0m

[38;2;248;248;242mTHE ZSH/CAP MODULE[0m
[38;2;248;248;242m       The  zsh/cap  module is used for manipulating POSIX.1e (POSIX.6) capability sets.[0m
[38;2;248;248;242m       If the operating system does not support this interface, the builtins defined  by[0m
[38;2;248;248;242m       this module will do nothing.  The builtins in this module are:[0m

[38;2;248;248;242m       cap [ capabilities ][0m
[38;2;248;248;242m              Change  the shell's process capability sets to the specified capabilities,[0m
[38;2;248;248;242m              otherwise display the shell's current capabilities.[0m

[38;2;248;248;242m       getcap filename ...[0m
[38;2;248;248;242m              This is a built-in implementation of the POSIX standard utility.  It  dis‐[0m
[38;2;248;248;242m              plays the capability sets on each specified filename.[0m

[38;2;248;248;242m       setcap capabilities filename ...[0m
[38;2;248;248;242m              This  is a built-in implementation of the POSIX standard utility.  It sets[0m
[38;2;248;248;242m              the capability sets on each specified filename to the specified  capabili‐[0m
[38;2;248;248;242m              ties.[0m

[38;2;248;248;242mTHE ZSH/CLONE MODULE[0m
[38;2;248;248;242m       The zsh/clone module makes available one builtin command:[0m

[38;2;248;248;242m       clone tty[0m
[38;2;248;248;242m              Creates  a forked instance of the current shell, attached to the specified[0m
[38;2;248;248;242m              tty.  In the new shell, the PID,  PPID  and  TTY  special  parameters  are[0m
[38;2;248;248;242m              changed appropriately.  $! is set to zero in the new shell, and to the new[0m
[38;2;248;248;242m              shell's PID in the original shell.[0m

[38;2;248;248;242m              The return status of the builtin is zero in both shells if successful, and[0m
[38;2;248;248;242m              non-zero on error.[0m

[38;2;248;248;242m              The  target  of clone should be an unused terminal, such as an unused vir‐[0m
[38;2;248;248;242m              tual console or a virtual terminal created by[0m

[38;2;248;248;242m                     xterm -e sh -c 'trap : INT QUIT TSTP; tty;[0m
[38;2;248;248;242m                             while :; do sleep 100000000; done'[0m

[38;2;248;248;242m              Some words of explanation are warranted  about  this  long  xterm  command[0m
[38;2;248;248;242m              line: when doing clone on a pseudo-terminal, some other session ("session"[0m
[38;2;248;248;242m              meant  as  a  unix  session group, or SID) is already owning the terminal.[0m
[38;2;248;248;242m              Hence the cloned zsh cannot acquire the pseudo-terminal as  a  controlling[0m
[38;2;248;248;242m              tty. That means two things:[0m

[38;2;248;248;242m              •      the  job control signals will go to the sh-started-by-xterm process[0m
[38;2;248;248;242m                     group (that's why we disable INT QUIT and TSTP with trap; otherwise[0m
[38;2;248;248;242m                     the while loop could get suspended or killed)[0m

[38;2;248;248;242m              •      the cloned shell will have job control disabled, and the  job  con‐[0m
[38;2;248;248;242m                     trol keys (control-C, control-\ and control-Z) will not work.[0m

[38;2;248;248;242m              This does not apply when cloning to an unused vc.[0m

[38;2;248;248;242m              Cloning  to  a used (and unprepared) terminal will result in two processes[0m
[38;2;248;248;242m              reading simultaneously from the same terminal, with input bytes going ran‐[0m
[38;2;248;248;242m              domly to either process.[0m

[38;2;248;248;242m              clone is mostly useful as a shell built-in replacement for openvt.[0m

[38;2;248;248;242mTHE ZSH/COMPCTL MODULE[0m
[38;2;248;248;242m       The zsh/compctl module makes available two builtin commands. compctl, is the old,[0m
[38;2;248;248;242m       deprecated way to control completions for ZLE.   See  zshcompctl(1).   The  other[0m
[38;2;248;248;242m       builtin  command,  compcall  can  be used in user-defined completion widgets, see[0m
[38;2;248;248;242m       zshcompwid(1).[0m

[38;2;248;248;242mTHE ZSH/COMPLETE MODULE[0m
[38;2;248;248;242m       The zsh/complete module makes available several builtin  commands  which  can  be[0m
[38;2;248;248;242m       used in user-defined completion widgets, see zshcompwid(1).[0m

[38;2;248;248;242mTHE ZSH/COMPLIST MODULE[0m
[38;2;248;248;242m       The zsh/complist module offers three extensions to completion listings: the abil‐[0m
[38;2;248;248;242m       ity to highlight matches in such a list, the ability to scroll through long lists[0m
[38;2;248;248;242m       and a different style of menu completion.[0m

[38;2;248;248;242m   Colored completion listings[0m
[38;2;248;248;242m       Whenever  one of the parameters ZLS_COLORS or ZLS_COLOURS is set and the zsh/com‐[0m
[38;2;248;248;242m       plist module is loaded or linked into the shell, completion lists  will  be  col‐[0m
[38;2;248;248;242m       ored.  Note, however, that complist will not automatically be loaded if it is not[0m
[38;2;248;248;242m       linked in:  on systems with dynamic loading, `zmodload zsh/complist' is required.[0m

[38;2;248;248;242m       The  parameters  ZLS_COLORS and ZLS_COLOURS describe how matches are highlighted.[0m
[38;2;248;248;242m       To turn on highlighting an empty value suffices, in which case  all  the  default[0m
[38;2;248;248;242m       values  given below will be used.  The format of the value of these parameters is[0m
[38;2;248;248;242m       the same as used by the GNU version of the ls command: a colon-separated list  of[0m
[38;2;248;248;242m       specifications  of  the  form `name=value'.  The name may be one of the following[0m
[38;2;248;248;242m       strings, most of which specify file types for which the value will be used.   The[0m
[38;2;248;248;242m       strings and their default values are:[0m

[38;2;248;248;242m       no 0   for normal text (i.e. when displaying something other than a matched file)[0m

[38;2;248;248;242m       fi 0   for regular files[0m

[38;2;248;248;242m       di 32  for directories[0m

[38;2;248;248;242m       ln 36  for  symbolic links.  If this has the special value target, symbolic links[0m
[38;2;248;248;242m              are dereferenced and the target file used to determine the display format.[0m

[38;2;248;248;242m       pi 31  for named pipes (FIFOs)[0m

[38;2;248;248;242m       so 33  for sockets[0m

[38;2;248;248;242m       bd 44;37[0m
[38;2;248;248;242m              for block devices[0m

[38;2;248;248;242m       cd 44;37[0m
[38;2;248;248;242m              for character devices[0m

[38;2;248;248;242m       or none[0m
[38;2;248;248;242m              for a symlink to nonexistent file (default is the value defined for ln)[0m

[38;2;248;248;242m       mi none[0m
[38;2;248;248;242m              for a non-existent file (default is the value defined for fi);  this  code[0m
[38;2;248;248;242m              is currently not used[0m

[38;2;248;248;242m       su 37;41[0m
[38;2;248;248;242m              for files with setuid bit set[0m

[38;2;248;248;242m       sg 30;43[0m
[38;2;248;248;242m              for files with setgid bit set[0m

[38;2;248;248;242m       tw 30;42[0m
[38;2;248;248;242m              for world writable directories with sticky bit set[0m

[38;2;248;248;242m       ow 34;43[0m
[38;2;248;248;242m              for world writable directories without sticky bit set[0m

[38;2;248;248;242m       sa none[0m
[38;2;248;248;242m              for  files with an associated suffix alias; this is only tested after spe‐[0m
[38;2;248;248;242m              cific suffixes, as described below[0m

[38;2;248;248;242m       st 37;44[0m
[38;2;248;248;242m              for directories with sticky bit set but not world writable[0m

[38;2;248;248;242m       ex 35  for executable files[0m

[38;2;248;248;242m       lc \e[ for the left code (see below)[0m

[38;2;248;248;242m       rc m   for the right code[0m

[38;2;248;248;242m       tc 0   for the character indicating the file type  printed after filenames if the[0m
[38;2;248;248;242m              LIST_TYPES option is set[0m

[38;2;248;248;242m       sp 0   for the spaces printed after matches to align the next column[0m

[38;2;248;248;242m       ec none[0m
[38;2;248;248;242m              for the end code[0m

[38;2;248;248;242m       Apart from these strings, the name may also be an asterisk (`*') followed by  any[0m
[38;2;248;248;242m       string.  The  value given for such a string will be used for all files whose name[0m
[38;2;248;248;242m       ends with the string.  The name may also be an equals sign (`=')  followed  by  a[0m
[38;2;248;248;242m       pattern;  the  EXTENDED_GLOB  option will be turned on for evaluation of the pat‐[0m
[38;2;248;248;242m       tern.  The value given for this pattern will be used for all  matches  (not  just[0m
[38;2;248;248;242m       filenames)  whose display string are matched by the pattern.  Definitions for the[0m
[38;2;248;248;242m       form with the leading equal sign take precedence over the values defined for file[0m
[38;2;248;248;242m       types, which in turn take precedence over the  form  with  the  leading  asterisk[0m
[38;2;248;248;242m       (file extensions).[0m

[38;2;248;248;242m       The  leading-equals  form also allows different parts of the displayed strings to[0m
[38;2;248;248;242m       be colored differently.  For this, the pattern has to  use  the  `(#b)'  globbing[0m
[38;2;248;248;242m       flag and pairs of parentheses surrounding the parts of the strings that are to be[0m
[38;2;248;248;242m       colored  differently.   In this case the value may consist of more than one color[0m
[38;2;248;248;242m       code separated by equal signs.  The first code will be used  for  all  parts  for[0m
[38;2;248;248;242m       which  no explicit code is specified and the following codes will be used for the[0m
[38;2;248;248;242m       parts matched by the sub-patterns in parentheses.  For example, the specification[0m
[38;2;248;248;242m       `=(#b)(?)*(?)=0=3=7' will be used for all matches which are at least two  charac‐[0m
[38;2;248;248;242m       ters  long  and  will  use the code `3' for the first character, `7' for the last[0m
[38;2;248;248;242m       character and `0' for the rest.[0m

[38;2;248;248;242m       All three forms of name may be preceded by a pattern in parentheses.  If this  is[0m
[38;2;248;248;242m       given,  the value will be used only for matches in groups whose names are matched[0m
[38;2;248;248;242m       by the pattern given in the parentheses.  For example, `(g*)m*=43' highlights all[0m
[38;2;248;248;242m       matches beginning with `m' in groups whose names  begin with `g' using the  color[0m
[38;2;248;248;242m       code  `43'.   In case of the `lc', `rc', and `ec' codes, the group pattern is ig‐[0m
[38;2;248;248;242m       nored.[0m

[38;2;248;248;242m       Note also that all patterns are tried in the order in which they  appear  in  the[0m
[38;2;248;248;242m       parameter  value until the first one matches which is then used.  Patterns may be[0m
[38;2;248;248;242m       matched against completions, descriptions  (possibly  with  spaces  appended  for[0m
[38;2;248;248;242m       padding),  or  lines  consisting  of a completion followed by a description.  For[0m
[38;2;248;248;242m       consistent coloring it may be necessary to use more than one pattern or a pattern[0m
[38;2;248;248;242m       with backreferences.[0m

[38;2;248;248;242m       When printing a match, the code prints  the  value  of  lc,  the  value  for  the[0m
[38;2;248;248;242m       file-type  or  the  last  matching specification with a `*', the value of rc, the[0m
[38;2;248;248;242m       string to display for the match itself, and then the value of ec if that  is  de‐[0m
[38;2;248;248;242m       fined or the values of lc, no, and rc if ec is not defined.[0m

[38;2;248;248;242m       The default values are ISO 6429 (ANSI) compliant and can be used on vt100 compat‐[0m
[38;2;248;248;242m       ible  terminals  such as xterms.  On monochrome terminals the default values will[0m
[38;2;248;248;242m       have no visible effect.  The colors function from the contribution can be used to[0m
[38;2;248;248;242m       get associative arrays containing the codes for ANSI terminals (see  the  section[0m
[38;2;248;248;242m       `Other  Functions'  in  zshcontrib(1)).   For  example, after loading colors, one[0m
[38;2;248;248;242m       could  use  `$color[red]'  to  get  the  code  for  foreground  color   red   and[0m
[38;2;248;248;242m       `$color[bg-green]' for the code for background color green.[0m

[38;2;248;248;242m       If the completion system invoked by compinit is used, these parameters should not[0m
[38;2;248;248;242m       be  set directly because the system controls them itself.  Instead, the list-col‐[0m
[38;2;248;248;242m       ors style should be used (see the section `Completion  System  Configuration'  in[0m
[38;2;248;248;242m       zshcompsys(1)).[0m

[38;2;248;248;242m   Scrolling in completion listings[0m
[38;2;248;248;242m       To  enable  scrolling through a completion list, the LISTPROMPT parameter must be[0m
[38;2;248;248;242m       set.  Its value will be used as the prompt; if it is the empty string, a  default[0m
[38;2;248;248;242m       prompt  will  be  used.  The value may contain escapes of the form `%x'.  It sup‐[0m
[38;2;248;248;242m       ports the escapes `%B', `%b', `%S', `%s', `%U', `%u', `%F', `%f', `%K', `%k'  and[0m
[38;2;248;248;242m       `%{...%}'  used  also  in  shell prompts as well as three pairs of additional se‐[0m
[38;2;248;248;242m       quences: a `%l' or `%L' is replaced by the number of the last line shown and  the[0m
[38;2;248;248;242m       total number of lines in the form `number/total'; a `%m' or `%M' is replaced with[0m
[38;2;248;248;242m       the  number  of the last match shown and the total number of matches; and `%p' or[0m
[38;2;248;248;242m       `%P' is replaced with `Top', `Bottom' or the position of the first line shown  in[0m
[38;2;248;248;242m       percent  of  the total number of lines, respectively.  In each of these cases the[0m
[38;2;248;248;242m       form with the uppercase letter will be replaced with a  string  of  fixed  width,[0m
[38;2;248;248;242m       padded to the right with spaces, while the lowercase form will not be padded.[0m

[38;2;248;248;242m       If  the parameter LISTPROMPT is set, the completion code will not ask if the list[0m
[38;2;248;248;242m       should be shown.  Instead it immediately starts displaying the list, stopping af‐[0m
[38;2;248;248;242m       ter the first screenful, showing the prompt at the bottom, waiting for a keypress[0m
[38;2;248;248;242m       after temporarily switching to the listscroll keymap.  Some of the zle  functions[0m
[38;2;248;248;242m       have a special meaning while scrolling lists:[0m

[38;2;248;248;242m       send-break[0m
[38;2;248;248;242m              stops listing discarding the key pressed[0m

[38;2;248;248;242m       accept-line, down-history, down-line-or-history[0m
[38;2;248;248;242m       down-line-or-search, vi-down-line-or-history[0m
[38;2;248;248;242m              scrolls forward one line[0m

[38;2;248;248;242m       complete-word, menu-complete, expand-or-complete[0m
[38;2;248;248;242m       expand-or-complete-prefix, menu-complete-or-expand[0m
[38;2;248;248;242m              scrolls forward one screenful[0m

[38;2;248;248;242m       accept-search[0m
[38;2;248;248;242m              stop listing but take no other action[0m

[38;2;248;248;242m       Every  other  character stops listing and immediately processes the key as usual.[0m
[38;2;248;248;242m       Any key that is not bound in the listscroll keymap or  that  is  bound  to  unde‐[0m
[38;2;248;248;242m       fined-key is looked up in the keymap currently selected.[0m

[38;2;248;248;242m       As  for  the  ZLS_COLORS and ZLS_COLOURS parameters, LISTPROMPT should not be set[0m
[38;2;248;248;242m       directly when using the shell function based  completion  system.   Instead,  the[0m
[38;2;248;248;242m       list-prompt style should be used.[0m

[38;2;248;248;242m   Menu selection[0m
[38;2;248;248;242m       The  zsh/complist  module  also  offers an alternative style of selecting matches[0m
[38;2;248;248;242m       from a list, called menu selection, which can be used if the shell is set  up  to[0m
[38;2;248;248;242m       return  to  the  last  prompt  after  showing  a  completion  list  (see  the AL‐[0m
[38;2;248;248;242m       WAYS_LAST_PROMPT option in zshoptions(1)).[0m

[38;2;248;248;242m       Menu selection can be invoked directly by the widget menu-select defined by  this[0m
[38;2;248;248;242m       module.   This  is  a standard ZLE widget that can be bound to a key in the usual[0m
[38;2;248;248;242m       way as described in zshzle(1).[0m

[38;2;248;248;242m       Alternatively, the parameter MENUSELECT can be set to an integer, which gives the[0m
[38;2;248;248;242m       minimum number of matches that must be present before menu selection is automati‐[0m
[38;2;248;248;242m       cally turned on.  This second method requires that menu  completion  be  started,[0m
[38;2;248;248;242m       either directly from a widget such as menu-complete, or due to one of the options[0m
[38;2;248;248;242m       MENU_COMPLETE  or  AUTO_MENU  being  set.   If  MENUSELECT is set, but is 0, 1 or[0m
[38;2;248;248;242m       empty, menu selection will always be started during an ambiguous menu completion.[0m

[38;2;248;248;242m       When using the completion system based on shell functions, the MENUSELECT parame‐[0m
[38;2;248;248;242m       ter should not be used (like the ZLS_COLORS and ZLS_COLOURS parameters  described[0m
[38;2;248;248;242m       above).  Instead, the menu style should be used with the select=... keyword.[0m

[38;2;248;248;242m       After  menu  selection  is started, the matches will be listed. If there are more[0m
[38;2;248;248;242m       matches than fit on the screen, only the first screenful is shown.   The  matches[0m
[38;2;248;248;242m       to  insert into the command line can be selected from this list.  In the list one[0m
[38;2;248;248;242m       match is highlighted using the value for ma from the  ZLS_COLORS  or  ZLS_COLOURS[0m
[38;2;248;248;242m       parameter.   The default value for this is `7' which forces the selected match to[0m
[38;2;248;248;242m       be highlighted using standout mode on a vt100-compatible  terminal.   If  neither[0m
[38;2;248;248;242m       ZLS_COLORS  nor ZLS_COLOURS is set, the same terminal control sequence as for the[0m
[38;2;248;248;242m       `%S' escape in prompts is used.[0m

[38;2;248;248;242m       If there are more matches than fit on the screen and the parameter MENUPROMPT  is[0m
[38;2;248;248;242m       set,  its value will be shown below the matches.  It supports the same escape se‐[0m
[38;2;248;248;242m       quences as LISTPROMPT, but the number of the match or line shown will be that  of[0m
[38;2;248;248;242m       the  one  where  the mark is placed.  If its value is the empty string, a default[0m
[38;2;248;248;242m       prompt will be used.[0m

[38;2;248;248;242m       The MENUSCROLL parameter can be used to specify how the list is scrolled.  If the[0m
[38;2;248;248;242m       parameter is unset, this is done line by line, if it is set to  `0'  (zero),  the[0m
[38;2;248;248;242m       list  will  scroll half the number of lines of the screen.  If the value is posi‐[0m
[38;2;248;248;242m       tive, it gives the number of lines to scroll and if it is negative, the list will[0m
[38;2;248;248;242m       be scrolled the number of lines of the screen minus the (absolute) value.[0m

[38;2;248;248;242m       As for the ZLS_COLORS, ZLS_COLOURS and LISTPROMPT parameters, neither  MENUPROMPT[0m
[38;2;248;248;242m       nor MENUSCROLL should be set directly when using the shell function based comple‐[0m
[38;2;248;248;242m       tion system.  Instead, the select-prompt and select-scroll styles should be used.[0m

[38;2;248;248;242m       The completion code sometimes decides not to show all of the matches in the list.[0m
[38;2;248;248;242m       These  hidden  matches are either matches for which the completion function which[0m
[38;2;248;248;242m       added them explicitly requested that they not appear in the list  (using  the  -n[0m
[38;2;248;248;242m       option  of  the  compadd  builtin  command) or they are matches which duplicate a[0m
[38;2;248;248;242m       string already in the list (because they differ only in things like  prefixes  or[0m
[38;2;248;248;242m       suffixes  that are not displayed).  In the list used for menu selection, however,[0m
[38;2;248;248;242m       even these matches are shown so that it is possible to select them.  To highlight[0m
[38;2;248;248;242m       such matches the hi and du capabilities in the ZLS_COLORS and ZLS_COLOURS parame‐[0m
[38;2;248;248;242m       ters are supported for hidden matches of the first and second kind, respectively.[0m

[38;2;248;248;242m       Selecting matches is done by moving the mark around using the zle movement  func‐[0m
[38;2;248;248;242m       tions.   When  not  all  matches can be shown on the screen at the same time, the[0m
[38;2;248;248;242m       list will scroll up and down when crossing the top or bottom line.  The following[0m
[38;2;248;248;242m       zle functions have special meaning during menu selection.  Note that the  follow‐[0m
[38;2;248;248;242m       ing  always perform the same task within the menu selection map and cannot be re‐[0m
[38;2;248;248;242m       placed by user defined widgets, nor can the set of functions be extended:[0m

[38;2;248;248;242m       accept-line, accept-search[0m
[38;2;248;248;242m              accept the current match and leave menu selection (but do  not  cause  the[0m
[38;2;248;248;242m              command line to be accepted)[0m

[38;2;248;248;242m       send-break[0m
[38;2;248;248;242m              leaves  menu  selection  and restores the previous contents of the command[0m
[38;2;248;248;242m              line[0m

[38;2;248;248;242m       redisplay, clear-screen[0m
[38;2;248;248;242m              execute their normal function without leaving menu selection[0m

[38;2;248;248;242m       accept-and-hold, accept-and-menu-complete[0m
[38;2;248;248;242m              accept the currently inserted match and continue selection allowing to se‐[0m
[38;2;248;248;242m              lect the next match to insert into the line[0m

[38;2;248;248;242m       accept-and-infer-next-history[0m
[38;2;248;248;242m              accepts the current match and then tries completion  with  menu  selection[0m
[38;2;248;248;242m              again;  in the case of files this allows one to select a directory and im‐[0m
[38;2;248;248;242m              mediately  attempt  to  complete  files in it;  if there are no matches, a[0m
[38;2;248;248;242m              message is shown and one can use undo to go back to completion on the pre‐[0m
[38;2;248;248;242m              vious level, every other key leaves menu selection  (including  the  other[0m
[38;2;248;248;242m              zle functions which are otherwise special during menu selection)[0m

[38;2;248;248;242m       undo   removes  matches  inserted  during  the menu selection by one of the three[0m
[38;2;248;248;242m              functions before[0m

[38;2;248;248;242m       down-history, down-line-or-history[0m
[38;2;248;248;242m       vi-down-line-or-history,  down-line-or-search[0m
[38;2;248;248;242m              moves the mark one line down[0m

[38;2;248;248;242m       up-history, up-line-or-history[0m
[38;2;248;248;242m       vi-up-line-or-history, up-line-or-search[0m
[38;2;248;248;242m              moves the mark one line up[0m

[38;2;248;248;242m       forward-char, vi-forward-char[0m
[38;2;248;248;242m              moves the mark one column right[0m

[38;2;248;248;242m       backward-char, vi-backward-char[0m
[38;2;248;248;242m              moves the mark one column left[0m

[38;2;248;248;242m       forward-word, vi-forward-word[0m
[38;2;248;248;242m       vi-forward-word-end, emacs-forward-word[0m
[38;2;248;248;242m              moves the mark one screenful down[0m

[38;2;248;248;242m       backward-word, vi-backward-word, emacs-backward-word[0m
[38;2;248;248;242m              moves the mark one screenful up[0m

[38;2;248;248;242m       vi-forward-blank-word, vi-forward-blank-word-end[0m
[38;2;248;248;242m              moves the mark to the first line of the next group of matches[0m

[38;2;248;248;242m       vi-backward-blank-word[0m
[38;2;248;248;242m              moves the mark to the last line of the previous group of matches[0m

[38;2;248;248;242m       beginning-of-history[0m
[38;2;248;248;242m              moves the mark to the first line[0m

[38;2;248;248;242m       end-of-history[0m
[38;2;248;248;242m              moves the mark to the last line[0m

[38;2;248;248;242m       beginning-of-buffer-or-history, beginning-of-line[0m
[38;2;248;248;242m       beginning-of-line-hist, vi-beginning-of-line[0m
[38;2;248;248;242m              moves the mark to the leftmost column[0m

[38;2;248;248;242m       end-of-buffer-or-history, end-of-line[0m
[38;2;248;248;242m       end-of-line-hist, vi-end-of-line[0m
[38;2;248;248;242m              moves the mark to the rightmost column[0m

[38;2;248;248;242m       complete-word, menu-complete, expand-or-complete[0m
[38;2;248;248;242m       expand-or-complete-prefix, menu-expand-or-complete[0m
[38;2;248;248;242m              moves the mark to the next match[0m

[38;2;248;248;242m       reverse-menu-complete[0m
[38;2;248;248;242m              moves the mark to the previous match[0m

[38;2;248;248;242m       vi-insert[0m
[38;2;248;248;242m              this toggles between normal and interactive mode; in interactive mode  the[0m
[38;2;248;248;242m              keys  bound  to self-insert and self-insert-unmeta insert into the command[0m
[38;2;248;248;242m              line as in normal editing mode but without leaving menu  selection;  after[0m
[38;2;248;248;242m              each  character  completion is tried again and the list changes to contain[0m
[38;2;248;248;242m              only the new matches; the completion widgets make the longest  unambiguous[0m
[38;2;248;248;242m              string  be  inserted in the command line and undo and backward-delete-char[0m
[38;2;248;248;242m              go back to the previous set of matches[0m

[38;2;248;248;242m       history-incremental-search-forward[0m
[38;2;248;248;242m       history-incremental-search-backward[0m
[38;2;248;248;242m              this starts incremental searches in the list of completions displayed;  in[0m
[38;2;248;248;242m              this  mode,  accept-line only leaves incremental search, going back to the[0m
[38;2;248;248;242m              normal menu selection mode[0m

[38;2;248;248;242m       All movement functions wrap around at the  edges;  any  other  zle  function  not[0m
[38;2;248;248;242m       listed  leaves menu selection and executes that function.  It is possible to make[0m
[38;2;248;248;242m       widgets in the above list do the same by using the form of the widget with a  `.'[0m
[38;2;248;248;242m       in  front.  For example, the widget `.accept-line' has the effect of leaving menu[0m
[38;2;248;248;242m       selection and accepting the entire command line.[0m

[38;2;248;248;242m       During this selection the widget uses the keymap menuselect.  Any key that is not[0m
[38;2;248;248;242m       defined in this keymap or that is bound to undefined-key  is  looked  up  in  the[0m
[38;2;248;248;242m       keymap  currently  selected.  This is used to ensure that the most important keys[0m
[38;2;248;248;242m       used during selection (namely the cursor keys, return, and TAB) have sensible de‐[0m
[38;2;248;248;242m       faults.  However, keys in the menuselect keymap can be  modified  directly  using[0m
[38;2;248;248;242m       the  bindkey builtin command (see zshmodules(1)). For example, to make the return[0m
[38;2;248;248;242m       key leave menu selection without accepting the match currently selected one could[0m
[38;2;248;248;242m       call[0m

[38;2;248;248;242m              bindkey -M menuselect '^M' send-break[0m

[38;2;248;248;242m       after loading the zsh/complist module.[0m

[38;2;248;248;242mTHE ZSH/COMPUTIL MODULE[0m
[38;2;248;248;242m       The zsh/computil module adds several builtin commands that are used  by  some  of[0m
[38;2;248;248;242m       the  completion  functions in the completion system based on shell functions (see[0m
[38;2;248;248;242m       zshcompsys(1) ).  Except for compquote  these  builtin  commands  are  very  spe‐[0m
[38;2;248;248;242m       cialised  and  thus  not  very interesting when writing your own completion func‐[0m
[38;2;248;248;242m       tions.  In summary, these builtin commands are:[0m

[38;2;248;248;242m       comparguments[0m
[38;2;248;248;242m              This is used by the _arguments function to do  the  argument  and  command[0m
[38;2;248;248;242m              line parsing.  Like compdescribe it has an option -i to do the parsing and[0m
[38;2;248;248;242m              initialize some internal state and various options to access the state in‐[0m
[38;2;248;248;242m              formation to decide what should be completed.[0m

[38;2;248;248;242m       compdescribe[0m
[38;2;248;248;242m              This  is  used  by  the  _describe  function to build the displays for the[0m
[38;2;248;248;242m              matches and to get the strings to add as matches with their  options.   On[0m
[38;2;248;248;242m              the first call one of the options -i or -I should be supplied as the first[0m
[38;2;248;248;242m              argument.   In  the  first  case, display strings without the descriptions[0m
[38;2;248;248;242m              will be generated, in the second case, the string  used  to  separate  the[0m
[38;2;248;248;242m              matches  from  their descriptions must be given as the second argument and[0m
[38;2;248;248;242m              the descriptions (if any) will be shown.  All other arguments are like the[0m
[38;2;248;248;242m              definition arguments to _describe itself.[0m

[38;2;248;248;242m              Once compdescribe has been called with either the -i or the -I option,  it[0m
[38;2;248;248;242m              can  be repeatedly called with the -g option and the names of four parame‐[0m
[38;2;248;248;242m              ters as its arguments.  This will  step  through  the  different  sets  of[0m
[38;2;248;248;242m              matches  and  store  the value of compstate[list] in the first scalar, the[0m
[38;2;248;248;242m              options for compadd in the second array, the matches in the  third  array,[0m
[38;2;248;248;242m              and  the  strings  to be displayed in the completion listing in the fourth[0m
[38;2;248;248;242m              array.  The arrays may then be directly given to compadd to  register  the[0m
[38;2;248;248;242m              matches with the completion code.[0m

[38;2;248;248;242m       compfiles[0m
[38;2;248;248;242m              Used  by  the  _path_files function to optimize complex recursive filename[0m
[38;2;248;248;242m              generation (globbing).  It does three things.  With the -p and -P  options[0m
[38;2;248;248;242m              it  builds  the  glob patterns to use, including the paths already handled[0m
[38;2;248;248;242m              and trying to optimize the patterns with respect to the prefix and  suffix[0m
[38;2;248;248;242m              from  the  line and the match specification currently used.  The -i option[0m
[38;2;248;248;242m              does the directory tests for the ignore-parents style and  the  -r  option[0m
[38;2;248;248;242m              tests  if  a  component for some of the matches are equal to the string on[0m
[38;2;248;248;242m              the line and removes all other matches if that is true.[0m

[38;2;248;248;242m       compgroups[0m
[38;2;248;248;242m              Used by the _tags function to implement the internals of  the  group-order[0m
[38;2;248;248;242m              style.   This  only  takes its arguments as names of completion groups and[0m
[38;2;248;248;242m              creates the groups for it (all six types: sorted and unsorted, both  with‐[0m
[38;2;248;248;242m              out  removing  duplicates,  with removing all duplicates and with removing[0m
[38;2;248;248;242m              consecutive duplicates).[0m

[38;2;248;248;242m       compquote [ -p ] names ...[0m
[38;2;248;248;242m              There may be reasons to write completion functions that have  to  add  the[0m
[38;2;248;248;242m              matches  using  the  -Q  option to compadd and perform quoting themselves.[0m
[38;2;248;248;242m              Instead of interpreting the first character of the all_quotes key  of  the[0m
[38;2;248;248;242m              compstate  special  association  and using the q flag for parameter expan‐[0m
[38;2;248;248;242m              sions, one can use this builtin command.  The arguments are the  names  of[0m
[38;2;248;248;242m              scalar  or  array parameters and the values of these parameters are quoted[0m
[38;2;248;248;242m              as needed for the innermost quoting level.  If the  -p  option  is  given,[0m
[38;2;248;248;242m              quoting  is done as if there is some prefix before the values of the para‐[0m
[38;2;248;248;242m              meters, so that a leading equal sign will not be quoted.[0m

[38;2;248;248;242m              The return status is non-zero in case of an error and zero otherwise.[0m

[38;2;248;248;242m       comptags[0m
[38;2;248;248;242m       comptry[0m
[38;2;248;248;242m              These implement the internals of the tags mechanism.[0m

[38;2;248;248;242m       compvalues[0m
[38;2;248;248;242m              Like comparguments, but for the _values function.[0m

[38;2;248;248;242mTHE ZSH/CURSES MODULE[0m
[38;2;248;248;242m       The zsh/curses module makes available one builtin command and various parameters.[0m

[38;2;248;248;242m   Builtin[0m
[38;2;248;248;242m       zcurses init[0m
[38;2;248;248;242m       zcurses end[0m
[38;2;248;248;242m       zcurses addwin targetwin nlines ncols begin_y begin_x [ parentwin ][0m
[38;2;248;248;242m       zcurses delwin targetwin[0m
[38;2;248;248;242m       zcurses refresh [ targetwin ... ][0m
[38;2;248;248;242m       zcurses touch targetwin ...[0m
[38;2;248;248;242m       zcurses move targetwin new_y new_x[0m
[38;2;248;248;242m       zcurses clear targetwin [ redraw | eol | bot ][0m
[38;2;248;248;242m       zcurses position targetwin array[0m
[38;2;248;248;242m       zcurses char targetwin character[0m
[38;2;248;248;242m       zcurses string targetwin string[0m
[38;2;248;248;242m       zcurses border targetwin border[0m
[38;2;248;248;242m       zcurses attr targetwin [ [+|-]attribute | fg_col/bg_col ] [...][0m
[38;2;248;248;242m       zcurses bg targetwin [ [+|-]attribute | fg_col/bg_col | @char ] [...][0m
[38;2;248;248;242m       zcurses scroll targetwin [ on | off | [+|-]lines ][0m
[38;2;248;248;242m       zcurses input targetwin [ param [ kparam [ mparam ] ] ][0m
[38;2;248;248;242m       zcurses mouse [ delay num | [+|-]motion ][0m
[38;2;248;248;242m       zcurses timeout targetwin intval[0m
[38;2;248;248;242m       zcurses querychar targetwin [ param ][0m
[38;2;248;248;242m       zcurses resize height width [ endwin | nosave | endwin_nosave ][0m
[38;2;248;248;242m              Manipulate curses windows.  All uses of this command should  be  bracketed[0m
[38;2;248;248;242m              by  `zcurses  init'  to initialise use of curses, and `zcurses end' to end[0m
[38;2;248;248;242m              it; omitting `zcurses end' can cause the terminal to  be  in  an  unwanted[0m
[38;2;248;248;242m              state.[0m

[38;2;248;248;242m              The  subcommand  addwin  creates  a  window  with  nlines  lines and ncols[0m
[38;2;248;248;242m              columns.  Its upper left corner will be placed at row begin_y  and  column[0m
[38;2;248;248;242m              begin_x  of the screen.  targetwin is a string and refers to the name of a[0m
[38;2;248;248;242m              window that is not currently assigned.  Note in particular the curses con‐[0m
[38;2;248;248;242m              vention that vertical values appear before horizontal values.[0m

[38;2;248;248;242m              If addwin is given an existing window as the final argument, the new  win‐[0m
[38;2;248;248;242m              dow is created as a subwindow of parentwin.  This differs from an ordinary[0m
[38;2;248;248;242m              new  window  in  that the memory of the window contents is shared with the[0m
[38;2;248;248;242m              parent's memory.  Subwindows must be deleted before  their  parent.   Note[0m
[38;2;248;248;242m              that  the  coordinates  of  subwindows are relative to the screen, not the[0m
[38;2;248;248;242m              parent, as with other windows.[0m

[38;2;248;248;242m              Use the subcommand delwin to delete a window created  with  addwin.   Note[0m
[38;2;248;248;242m              that  end  does  not  implicitly  delete windows, and that delwin does not[0m
[38;2;248;248;242m              erase the screen image of the window.[0m

[38;2;248;248;242m              The window corresponding to the full visible screen is called  stdscr;  it[0m
[38;2;248;248;242m              always exists after `zcurses init' and cannot be delete with delwin.[0m

[38;2;248;248;242m              The subcommand refresh will refresh window targetwin; this is necessary to[0m
[38;2;248;248;242m              make  any pending changes (such as characters you have prepared for output[0m
[38;2;248;248;242m              with char) visible on the screen.  refresh without an argument causes  the[0m
[38;2;248;248;242m              screen  to  be  cleared  and  redrawn.  If multiple windows are given, the[0m
[38;2;248;248;242m              screen is updated once at the end.[0m

[38;2;248;248;242m              The subcommand touch marks the targetwins listed as changed.  This is nec‐[0m
[38;2;248;248;242m              essary before refreshing windows if a window that was in front of  another[0m
[38;2;248;248;242m              window (which may be stdscr) is deleted.[0m

[38;2;248;248;242m              The  subcommand move moves the cursor position in targetwin to new coordi‐[0m
[38;2;248;248;242m              nates new_y and new_x.  Note that the subcommand string (but not the  sub‐[0m
[38;2;248;248;242m              command char) advances the cursor position over the characters added.[0m

[38;2;248;248;242m              The  subcommand  clear erases the contents of targetwin.  One (and no more[0m
[38;2;248;248;242m              than one) of three options may be specified.  With the option  redraw,  in[0m
[38;2;248;248;242m              addition the next refresh of targetwin will cause the screen to be cleared[0m
[38;2;248;248;242m              and  repainted.  With the option eol, targetwin is only cleared to the end[0m
[38;2;248;248;242m              of the current cursor line.  With the option bot, targetwin is cleared  to[0m
[38;2;248;248;242m              the end of the window, i.e everything to the right and below the cursor is[0m
[38;2;248;248;242m              cleared.[0m

[38;2;248;248;242m              The subcommand position writes various positions associated with targetwin[0m
[38;2;248;248;242m              into the array named array.  These are, in order:[0m
[38;2;248;248;242m              -      The  y  and x coordinates of the cursor relative to the top left of[0m
[38;2;248;248;242m                     targetwin[0m
[38;2;248;248;242m              -      The y and x coordinates of the top left of targetwin on the screen[0m
[38;2;248;248;242m              -      The size of targetwin in y and x dimensions.[0m

[38;2;248;248;242m              Outputting characters and strings are achieved by char and string  respec‐[0m
[38;2;248;248;242m              tively.[0m

[38;2;248;248;242m              To  draw a border around window targetwin, use border.  Note that the bor‐[0m
[38;2;248;248;242m              der is not subsequently handled specially:  in other words, the border  is[0m
[38;2;248;248;242m              simply a set of characters output at the edge of the window.  Hence it can[0m
[38;2;248;248;242m              be overwritten, can scroll off the window, etc.[0m

[38;2;248;248;242m              The  subcommand  attr  will set targetwin's attributes or foreground/back‐[0m
[38;2;248;248;242m              ground color pair for any successive  character  output.   Each  attribute[0m
[38;2;248;248;242m              given  on the line may be prepended by a + to set or a - to unset that at‐[0m
[38;2;248;248;242m              tribute; + is assumed if absent.   The  attributes  supported  are  blink,[0m
[38;2;248;248;242m              bold, dim, reverse, standout, and underline.[0m

[38;2;248;248;242m              Each  fg_col/bg_col  attribute (to be read as `fg_col on bg_col') sets the[0m
[38;2;248;248;242m              foreground and background color for character output.  The  color  default[0m
[38;2;248;248;242m              is sometimes available (in particular if the library is ncurses), specify‐[0m
[38;2;248;248;242m              ing  the  foreground  or background color with which the terminal started.[0m
[38;2;248;248;242m              The color pair default/default is always available. To use more than the 8[0m
[38;2;248;248;242m              named colors (red, green, etc.) construct the  fg_col/bg_col  pairs  where[0m
[38;2;248;248;242m              fg_col  and  bg_col  are decimal integers, e.g 128/200.  The maximum color[0m
[38;2;248;248;242m              value is 254 if the terminal supports 256 colors.[0m

[38;2;248;248;242m              bg overrides the color and other attributes of all characters in the  win‐[0m
[38;2;248;248;242m              dow.   Its usual use is to set the background initially, but it will over‐[0m
[38;2;248;248;242m              write the attributes of any characters at the time when it is called.   In[0m
[38;2;248;248;242m              addition to the arguments allowed with attr, an argument @char specifies a[0m
[38;2;248;248;242m              character  to  be  shown in otherwise blank areas of the window.  Owing to[0m
[38;2;248;248;242m              limitations of curses this cannot be a multibyte character (use  of  ASCII[0m
[38;2;248;248;242m              characters only is recommended).  As the specified set of attributes over‐[0m
[38;2;248;248;242m              ride  the  existing background, turning attributes off in the arguments is[0m
[38;2;248;248;242m              not useful, though this does not cause an error.[0m

[38;2;248;248;242m              The subcommand scroll can be used with on or off  to  enabled  or  disable[0m
[38;2;248;248;242m              scrolling  of a window when the cursor would otherwise move below the win‐[0m
[38;2;248;248;242m              dow due to typing or output.  It can also be used with a positive or nega‐[0m
[38;2;248;248;242m              tive integer to scroll the window up or down the  given  number  of  lines[0m
[38;2;248;248;242m              without  changing  the current cursor position (which therefore appears to[0m
[38;2;248;248;242m              move in the opposite direction relative to the  window).   In  the  second[0m
[38;2;248;248;242m              case,  if scrolling is off it is temporarily turned on to allow the window[0m
[38;2;248;248;242m              to be scrolled.[0m

[38;2;248;248;242m              The subcommand input reads a single  character  from  the  window  without[0m
[38;2;248;248;242m              echoing  it  back.   If param is supplied the character is assigned to the[0m
[38;2;248;248;242m              parameter param, else it is assigned to the parameter REPLY.[0m

[38;2;248;248;242m              If both param and kparam are supplied, the key is read in  `keypad'  mode.[0m
[38;2;248;248;242m              In  this mode special keys such as function keys and arrow keys return the[0m
[38;2;248;248;242m              name of the key in the parameter kparam.  The key names are the macros de‐[0m
[38;2;248;248;242m              fined in the curses.h or ncurses.h with the  prefix  `KEY_'  removed;  see[0m
[38;2;248;248;242m              also  the description of the parameter zcurses_keycodes below.  Other keys[0m
[38;2;248;248;242m              cause a value to be set in param as before.  On a successful  return  only[0m
[38;2;248;248;242m              one of param or kparam contains a non-empty string; the other is set to an[0m
[38;2;248;248;242m              empty string.[0m

[38;2;248;248;242m              If mparam is also supplied, input attempts to handle mouse input.  This is[0m
[38;2;248;248;242m              only available with the ncurses library; mouse handling can be detected by[0m
[38;2;248;248;242m              checking  for  the exit status of `zcurses mouse' with no arguments.  If a[0m
[38;2;248;248;242m              mouse button is clicked (or double- or triple-clicked, or pressed  or  re‐[0m
[38;2;248;248;242m              leased with a configurable delay from being clicked) then kparam is set to[0m
[38;2;248;248;242m              the  string MOUSE, and mparam is set to an array consisting of the follow‐[0m
[38;2;248;248;242m              ing elements:[0m
[38;2;248;248;242m              -      An identifier to discriminate different input devices; this is only[0m
[38;2;248;248;242m                     rarely useful.[0m
[38;2;248;248;242m              -      The x, y and z coordinates of the mouse click relative to the  full[0m
[38;2;248;248;242m                     screen,  as three elements in that order (i.e. the y coordinate is,[0m
[38;2;248;248;242m                     unusually, after the x  coordinate).   The  z  coordinate  is  only[0m
[38;2;248;248;242m                     available  for  a few unusual input devices and is otherwise set to[0m
[38;2;248;248;242m                     zero.[0m
[38;2;248;248;242m              -      Any events that occurred as separate items; usually there  will  be[0m
[38;2;248;248;242m                     just  one.   An  event consists of PRESSED, RELEASED, CLICKED, DOU‐[0m
[38;2;248;248;242m                     BLE_CLICKED or TRIPLE_CLICKED followed immediately (in the same el‐[0m
[38;2;248;248;242m                     ement) by the number of the button.[0m
[38;2;248;248;242m              -      If the shift key was pressed, the string SHIFT.[0m
[38;2;248;248;242m              -      If the control key was pressed, the string CTRL.[0m
[38;2;248;248;242m              -      If the alt key was pressed, the string ALT.[0m

[38;2;248;248;242m              Not all mouse events may be passed through to the  terminal  window;  most[0m
[38;2;248;248;242m              terminal  emulators  handle  some  mouse events themselves.  Note that the[0m
[38;2;248;248;242m              ncurses manual implies that using input both with and without  mouse  han‐[0m
[38;2;248;248;242m              dling may cause the mouse cursor to appear and disappear.[0m

[38;2;248;248;242m              The subcommand mouse can be used to configure the use of the mouse.  There[0m
[38;2;248;248;242m              is  no window argument; mouse options are global.  `zcurses mouse' with no[0m
[38;2;248;248;242m              arguments returns status 0 if mouse handling is possible, else  status  1.[0m
[38;2;248;248;242m              Otherwise,  the possible arguments (which may be combined on the same com‐[0m
[38;2;248;248;242m              mand line) are as follows.  delay num sets the maximum delay in  millisec‐[0m
[38;2;248;248;242m              onds  between  press  and  release events to be considered as a click; the[0m
[38;2;248;248;242m              value 0 disables click resolution, and the default is one sixth of a  sec‐[0m
[38;2;248;248;242m              ond.   motion  proceeded by an optional `+' (the default) or - turns on or[0m
[38;2;248;248;242m              off reporting of mouse motion in addition to clicks, presses and releases,[0m
[38;2;248;248;242m              which are always reported.  However, it appears reports for  mouse  motion[0m
[38;2;248;248;242m              are not currently implemented.[0m

[38;2;248;248;242m              The subcommand timeout specifies a timeout value for input from targetwin.[0m
[38;2;248;248;242m              If  intval is negative, `zcurses input' waits indefinitely for a character[0m
[38;2;248;248;242m              to be typed; this is the default.  If intval is zero, `zcurses input'  re‐[0m
[38;2;248;248;242m              turns  immediately; if there is typeahead it is returned, else no input is[0m
[38;2;248;248;242m              done and status 1 is returned.  If intval  is  positive,  `zcurses  input'[0m
[38;2;248;248;242m              waits  intval  milliseconds  for  input and if there is none at the end of[0m
[38;2;248;248;242m              that period returns status 1.[0m

[38;2;248;248;242m              The subcommand querychar queries the character at the current cursor posi‐[0m
[38;2;248;248;242m              tion.  The return values are stored in the array named param if  supplied,[0m
[38;2;248;248;242m              else in the array reply.  The first value is the character (which may be a[0m
[38;2;248;248;242m              multibyte  character if the system supports them); the second is the color[0m
[38;2;248;248;242m              pair in the usual fg_col/bg_col notation, or 0 if color is not  supported.[0m
[38;2;248;248;242m              Any  attributes  other than color that apply to the character, as set with[0m
[38;2;248;248;242m              the subcommand attr, appear as additional elements.[0m

[38;2;248;248;242m              The subcommand resize resizes stdscr and all windows to  given  dimensions[0m
[38;2;248;248;242m              (windows that stick out from the new dimensions are resized down). The un‐[0m
[38;2;248;248;242m              derlying  curses  extension (resize_term call) can be unavailable. To ver‐[0m
[38;2;248;248;242m              ify, zeroes can be used for height and width. If the result of the subcom‐[0m
[38;2;248;248;242m              mand is 0, resize_term is available (2 otherwise). Tests show that  resiz‐[0m
[38;2;248;248;242m              ing  can  be  normally accomplished by calling zcurses end and zcurses re‐[0m
[38;2;248;248;242m              fresh. The resize subcommand is provided for versatility. Multiple  system[0m
[38;2;248;248;242m              configurations  have  been checked and zcurses end and zcurses refresh are[0m
[38;2;248;248;242m              still needed for correct terminal state after resize. To invoke them  with[0m
[38;2;248;248;242m              resize,  use endwin argument.  Using nosave argument will cause new termi‐[0m
[38;2;248;248;242m              nal state to not be saved internally by zcurses. This is also provided for[0m
[38;2;248;248;242m              versatility and should normally be not needed.[0m

[38;2;248;248;242m   Parameters[0m
[38;2;248;248;242m       ZCURSES_COLORS[0m
[38;2;248;248;242m              Readonly integer.  The maximum number of  colors  the  terminal  supports.[0m
[38;2;248;248;242m              This value is initialised by the curses library and is not available until[0m
[38;2;248;248;242m              the first time zcurses init is run.[0m

[38;2;248;248;242m       ZCURSES_COLOR_PAIRS[0m
[38;2;248;248;242m              Readonly  integer.   The  maximum number of color pairs fg_col/bg_col that[0m
[38;2;248;248;242m              may be defined in `zcurses attr' commands; note this limit applies to  all[0m
[38;2;248;248;242m              color  pairs that have been used whether or not they are currently active.[0m
[38;2;248;248;242m              This value is initialised by the curses library and is not available until[0m
[38;2;248;248;242m              the first time zcurses init is run.[0m

[38;2;248;248;242m       zcurses_attrs[0m
[38;2;248;248;242m              Readonly array.  The attributes supported by zsh/curses; available as soon[0m
[38;2;248;248;242m              as the module is loaded.[0m

[38;2;248;248;242m       zcurses_colors[0m
[38;2;248;248;242m              Readonly array.  The colors supported by zsh/curses; available as soon  as[0m
[38;2;248;248;242m              the module is loaded.[0m

[38;2;248;248;242m       zcurses_keycodes[0m
[38;2;248;248;242m              Readonly  array.   The values that may be returned in the second parameter[0m
[38;2;248;248;242m              supplied to `zcurses input' in the order in which they are defined  inter‐[0m
[38;2;248;248;242m              nally  by  curses.   Not all function keys are listed, only F0; curses re‐[0m
[38;2;248;248;242m              serves space for F0 up to F63.[0m

[38;2;248;248;242m       zcurses_windows[0m
[38;2;248;248;242m              Readonly array.  The current list of windows, i.e. all windows  that  have[0m
[38;2;248;248;242m              been created with `zcurses addwin' and not removed with `zcurses delwin'.[0m

[38;2;248;248;242mTHE ZSH/DATETIME MODULE[0m
[38;2;248;248;242m       The zsh/datetime module makes available one builtin command:[0m

[38;2;248;248;242m       strftime [ -s scalar | -n ] format [ epochtime [ nanoseconds ] ][0m
[38;2;248;248;242m       strftime -r [ -q ] [ -s scalar | -n ] format timestring[0m
[38;2;248;248;242m              Output  the  date in the format specified.  With no epochtime, the current[0m
[38;2;248;248;242m              system date/time is used; optionally, epochtime may be used to specify the[0m
[38;2;248;248;242m              number of seconds since the epoch, and  nanoseconds  may  additionally  be[0m
[38;2;248;248;242m              used  to specify the number of nanoseconds past the second (otherwise that[0m
[38;2;248;248;242m              number is assumed to be 0).  See strftime(3) for details.  The zsh  exten‐[0m
[38;2;248;248;242m              sions described in the section EXPANSION OF PROMPT SEQUENCES in zshmisc(1)[0m
[38;2;248;248;242m              are also available.[0m

[38;2;248;248;242m              -n     Suppress printing a newline after the formatted string.[0m

[38;2;248;248;242m              -q     Run  quietly; suppress printing of all error messages described be‐[0m
[38;2;248;248;242m                     low.  Errors for invalid epochtime values are always printed.[0m

[38;2;248;248;242m              -r     With the option -r (reverse), use format to parse the input  string[0m
[38;2;248;248;242m                     timestring  and  output  the  number  of seconds since the epoch at[0m
[38;2;248;248;242m                     which the time occurred.  The parsing is implemented by the  system[0m
[38;2;248;248;242m                     function strptime; see strptime(3).  This means that zsh format ex‐[0m
[38;2;248;248;242m                     tensions are not available, but for reverse lookup they are not re‐[0m
[38;2;248;248;242m                     quired.[0m

[38;2;248;248;242m                     In  most implementations of strftime any timezone in the timestring[0m
[38;2;248;248;242m                     is ignored and the local timezone declared by  the  TZ  environment[0m
[38;2;248;248;242m                     variable is used; other parameters are set to zero if not present.[0m

[38;2;248;248;242m                     If  timestring  does  not match format the command returns status 1[0m
[38;2;248;248;242m                     and prints an error message.  If timestring matches format but  not[0m
[38;2;248;248;242m                     all characters in timestring were used, the conversion succeeds but[0m
[38;2;248;248;242m                     also prints an error message.[0m

[38;2;248;248;242m                     If  either of the system functions strptime or mktime is not avail‐[0m
[38;2;248;248;242m                     able, status 2 is returned and an error message is printed.[0m

[38;2;248;248;242m              -s scalar[0m
[38;2;248;248;242m                     Assign the date string (or epoch time in seconds if -r is given) to[0m
[38;2;248;248;242m                     scalar instead of printing it.[0m

[38;2;248;248;242m              Note that depending on the system's declared integral time type,  strftime[0m
[38;2;248;248;242m              may  produce  incorrect  results  for  epoch times greater than 2147483647[0m
[38;2;248;248;242m              which corresponds to 2038-01-19 03:14:07 +0000.[0m

[38;2;248;248;242m       The zsh/datetime module makes available several parameters; all are readonly:[0m

[38;2;248;248;242m       EPOCHREALTIME[0m
[38;2;248;248;242m              A floating point value representing the number of seconds since the epoch.[0m
[38;2;248;248;242m              The notional accuracy is to  nanoseconds  if  the  clock_gettime  call  is[0m
[38;2;248;248;242m              available and to microseconds otherwise, but in practice the range of dou‐[0m
[38;2;248;248;242m              ble precision floating point and shell scheduling latencies may be signif‐[0m
[38;2;248;248;242m              icant effects.[0m

[38;2;248;248;242m       EPOCHSECONDS[0m
[38;2;248;248;242m              An integer value representing the number of seconds since the epoch.[0m

[38;2;248;248;242m       epochtime[0m
[38;2;248;248;242m              An  array  value  containing  the number of seconds since the epoch in the[0m
[38;2;248;248;242m              first element and the remainder of the time since the epoch in nanoseconds[0m
[38;2;248;248;242m              in the second element.  To ensure the two elements are consistent the  ar‐[0m
[38;2;248;248;242m              ray  should be copied or otherwise referenced as a single substitution be‐[0m
[38;2;248;248;242m              fore the values are used.  The following idiom may be used:[0m

[38;2;248;248;242m                     for secs nsecs in $epochtime; do[0m
[38;2;248;248;242m                       ...[0m
[38;2;248;248;242m                     done[0m

[38;2;248;248;242mTHE ZSH/DB/GDBM MODULE[0m
[38;2;248;248;242m       The zsh/db/gdbm module is used to create "tied" associative arrays that interface[0m
[38;2;248;248;242m       to database files.  If the GDBM interface is not available, the builtins  defined[0m
[38;2;248;248;242m       by this module will report an error.  This module is also intended as a prototype[0m
[38;2;248;248;242m       for  creating  additional  database interfaces, so the ztie builtin may move to a[0m
[38;2;248;248;242m       more generic module in the future.[0m

[38;2;248;248;242m       The builtins in this module are:[0m

[38;2;248;248;242m       ztie -d db/gdbm -f filename [ -r ] arrayname[0m
[38;2;248;248;242m              Open the GDBM database identified by filename and, if  successful,  create[0m
[38;2;248;248;242m              the  associative  array  arrayname  linked to the file.  To create a local[0m
[38;2;248;248;242m              tied array, the parameter must first be declared, so commands  similar  to[0m
[38;2;248;248;242m              the following would be executed inside a function scope:[0m

[38;2;248;248;242m                     local -A sampledb[0m
[38;2;248;248;242m                     ztie -d db/gdbm -f sample.gdbm sampledb[0m

[38;2;248;248;242m              The -r option opens the database file for reading only, creating a parame‐[0m
[38;2;248;248;242m              ter  with  the readonly attribute.  Without this option, using `ztie' on a[0m
[38;2;248;248;242m              file for which the user does not have write permission is  an  error.   If[0m
[38;2;248;248;242m              writable, the database is opened synchronously so fields changed in array‐[0m
[38;2;248;248;242m              name are immediately written to filename.[0m

[38;2;248;248;242m              Changes  to  the file modes filename after it has been opened do not alter[0m
[38;2;248;248;242m              the state of arrayname, but `typeset -r arrayname' works as expected.[0m

[38;2;248;248;242m       zuntie [ -u ] arrayname ...[0m
[38;2;248;248;242m              Close the GDBM database associated with each arrayname and then unset  the[0m
[38;2;248;248;242m              parameter.  The -u option forces an unset of parameters made readonly with[0m
[38;2;248;248;242m              `ztie -r'.[0m

[38;2;248;248;242m              This happens automatically if the parameter is explicitly unset or its lo‐[0m
[38;2;248;248;242m              cal  scope (function) ends.  Note that a readonly parameter may not be ex‐[0m
[38;2;248;248;242m              plicitly unset, so the only way to unset a global parameter  created  with[0m
[38;2;248;248;242m              `ztie -r' is to use `zuntie -u'.[0m

[38;2;248;248;242m       zgdbmpath parametername[0m
[38;2;248;248;242m              Put path to database file assigned to parametername into REPLY scalar.[0m

[38;2;248;248;242m       zgdbm_tied[0m
[38;2;248;248;242m              Array holding names of all tied parameters.[0m

[38;2;248;248;242m       The  fields of an associative array tied to GDBM are neither cached nor otherwise[0m
[38;2;248;248;242m       stored in memory, they are read from or written to the database  on  each  refer‐[0m
[38;2;248;248;242m       ence.  Thus, for example, the values in a readonly array may be changed by a sec‐[0m
[38;2;248;248;242m       ond writer of the same database file.[0m

[38;2;248;248;242mTHE ZSH/DELTOCHAR MODULE[0m

[38;2;248;248;242m       delete-to-char[0m
[38;2;248;248;242m              Read a character from the keyboard, and delete from the cursor position up[0m
[38;2;248;248;242m              to  and  including the next (or, with repeat count n, the nth) instance of[0m
[38;2;248;248;242m              that character.  Negative repeat counts mean delete backwards.[0m

[38;2;248;248;242m       zap-to-char[0m
[38;2;248;248;242m              This behaves like delete-to-char, except that the final occurrence of  the[0m
[38;2;248;248;242m              character itself is not deleted.[0m

[38;2;248;248;242m       The purpose of the module is to serve as an example of how to write a module.[0m

[38;2;248;248;242mTHE ZSH/FILES MODULE[0m
[38;2;248;248;242m       The  zsh/files  module makes available some common commands for file manipulation[0m
[38;2;248;248;242m       as builtins; these commands are probably not needed for  many  normal  situations[0m
[38;2;248;248;242m       but  can  be  useful in emergency recovery situations with constrained resources.[0m
[38;2;248;248;242m       The commands do not implement all features now  required  by  relevant  standards[0m
[38;2;248;248;242m       committees.[0m

[38;2;248;248;242m       For  all commands, a variant beginning zf_ is also available and loaded automati‐[0m
[38;2;248;248;242m       cally.  Using the features capability of zmodload will let you  load  only  those[0m
[38;2;248;248;242m       names  you want.  Note that it's possible to load only the builtins with zsh-spe‐[0m
[38;2;248;248;242m       cific names using the following command:[0m

[38;2;248;248;242m              zmodload -m -F zsh/files b:zf_\*[0m

[38;2;248;248;242m       The commands loaded by default are:[0m

[38;2;248;248;242m       chgrp [ -hRs ] group filename ...[0m
[38;2;248;248;242m              Changes group of files specified.  This is  equivalent  to  chown  with  a[0m
[38;2;248;248;242m              user-spec argument of `:group'.[0m

[38;2;248;248;242m       chmod [ -Rs ] mode filename ...[0m
[38;2;248;248;242m              Changes mode of files specified.[0m

[38;2;248;248;242m              The specified mode must be in octal.[0m

[38;2;248;248;242m              The -R option causes chmod to recursively descend into directories, chang‐[0m
[38;2;248;248;242m              ing  the mode of all files in the directory after changing the mode of the[0m
[38;2;248;248;242m              directory itself.[0m

[38;2;248;248;242m              The -s option is a zsh extension to chmod functionality.  It enables para‐[0m
[38;2;248;248;242m              noid behaviour, intended to avoid security problems involving a chmod  be‐[0m
[38;2;248;248;242m              ing  tricked  into  affecting files other than the ones intended.  It will[0m
[38;2;248;248;242m              refuse to follow  symbolic  links,  so  that  (for  example)  ``chmod  600[0m
[38;2;248;248;242m              /tmp/foo/passwd'' can't accidentally chmod /etc/passwd if /tmp/foo happens[0m
[38;2;248;248;242m              to be a link to /etc.  It will also check where it is after leaving direc‐[0m
[38;2;248;248;242m              tories,  so  that  a recursive chmod of a deep directory tree can't end up[0m
[38;2;248;248;242m              recursively chmoding /usr as a result of directories being  moved  up  the[0m
[38;2;248;248;242m              tree.[0m

[38;2;248;248;242m       chown [ -hRs ] user-spec filename ...[0m
[38;2;248;248;242m              Changes ownership and group of files specified.[0m

[38;2;248;248;242m              The user-spec can be in four forms:[0m

[38;2;248;248;242m              user   change owner to user; do not change group[0m
[38;2;248;248;242m              user:: change owner to user; do not change group[0m
[38;2;248;248;242m              user:  change owner to user; change group to user's primary group[0m
[38;2;248;248;242m              user:group[0m
[38;2;248;248;242m                     change owner to user; change group to group[0m
[38;2;248;248;242m              :group do not change owner; change group to group[0m

[38;2;248;248;242m              In  each case, the `:' may instead be a `.'.  The rule is that if there is[0m
[38;2;248;248;242m              a `:' then the separator is `:', otherwise if there is a `.' then the sep‐[0m
[38;2;248;248;242m              arator is `.', otherwise there is no separator.[0m

[38;2;248;248;242m              Each of user and group may be either a username (or group name, as  appro‐[0m
[38;2;248;248;242m              priate)  or  a decimal user ID (group ID).  Interpretation as a name takes[0m
[38;2;248;248;242m              precedence, if there is an all-numeric username (or group name).[0m

[38;2;248;248;242m              If the target is a symbolic link, the -h option causes chown  to  set  the[0m
[38;2;248;248;242m              ownership of the link instead of its target.[0m

[38;2;248;248;242m              The -R option causes chown to recursively descend into directories, chang‐[0m
[38;2;248;248;242m              ing  the ownership of all files in the directory after changing the owner‐[0m
[38;2;248;248;242m              ship of the directory itself.[0m

[38;2;248;248;242m              The -s option is a zsh extension to chown functionality.  It enables para‐[0m
[38;2;248;248;242m              noid behaviour, intended to avoid security problems involving a chown  be‐[0m
[38;2;248;248;242m              ing  tricked  into  affecting files other than the ones intended.  It will[0m
[38;2;248;248;242m              refuse to follow symbolic links,  so  that  (for  example)  ``chown  luser[0m
[38;2;248;248;242m              /tmp/foo/passwd'' can't accidentally chown /etc/passwd if /tmp/foo happens[0m
[38;2;248;248;242m              to be a link to /etc.  It will also check where it is after leaving direc‐[0m
[38;2;248;248;242m              tories,  so  that  a recursive chown of a deep directory tree can't end up[0m
[38;2;248;248;242m              recursively chowning /usr as a result of directories being  moved  up  the[0m
[38;2;248;248;242m              tree.[0m

[38;2;248;248;242m       ln [ -dfhins ] filename dest[0m
[38;2;248;248;242m       ln [ -dfhins ] filename ... dir[0m
[38;2;248;248;242m              Creates hard (or, with -s, symbolic) links.  In the first form, the speci‐[0m
[38;2;248;248;242m              fied  destination is created, as a link to the specified filename.  In the[0m
[38;2;248;248;242m              second form, each of the filenames is taken in turn, and linked to a path‐[0m
[38;2;248;248;242m              name in the specified directory that has the same last pathname component.[0m

[38;2;248;248;242m              Normally, ln will not attempt to create hard links to  directories.   This[0m
[38;2;248;248;242m              check  can  be  overridden  using  the  -d option.  Typically only the su‐[0m
[38;2;248;248;242m              per-user can actually succeed in creating hard links to directories.  This[0m
[38;2;248;248;242m              does not apply to symbolic links in any case.[0m

[38;2;248;248;242m              By default, existing files cannot be replaced by  links.   The  -i  option[0m
[38;2;248;248;242m              causes  the user to be queried about replacing existing files.  The -f op‐[0m
[38;2;248;248;242m              tion causes existing files to be silently deleted, without  querying.   -f[0m
[38;2;248;248;242m              takes precedence.[0m

[38;2;248;248;242m              The  -h and -n options are identical and both exist for compatibility; ei‐[0m
[38;2;248;248;242m              ther one indicates that if the target is a symlink then it should  not  be[0m
[38;2;248;248;242m              dereferenced.   Typically  this is used in combination with -sf so that if[0m
[38;2;248;248;242m              an existing link points to a directory then it will be removed, instead of[0m
[38;2;248;248;242m              followed.  If this option is used with multiple filenames and  the  target[0m
[38;2;248;248;242m              is a symbolic link pointing to a directory then the result is an error.[0m

[38;2;248;248;242m       mkdir [ -p ] [ -m mode ] dir ...[0m
[38;2;248;248;242m              Creates  directories.  With the -p option, non-existing parent directories[0m
[38;2;248;248;242m              are first created if necessary, and there will be no complaint if the  di‐[0m
[38;2;248;248;242m              rectory already exists.  The -m option can be used to specify (in octal) a[0m
[38;2;248;248;242m              set  of  file  permissions for the created directories, otherwise mode 777[0m
[38;2;248;248;242m              modified by the current umask (see umask(2)) is used.[0m

[38;2;248;248;242m       mv [ -fi ] filename dest[0m
[38;2;248;248;242m       mv [ -fi ] filename ... dir[0m
[38;2;248;248;242m              Moves files.  In the first form, the specified filename is  moved  to  the[0m
[38;2;248;248;242m              specified destination.  In the second form, each of the filenames is taken[0m
[38;2;248;248;242m              in  turn,  and moved to a pathname in the specified directory that has the[0m
[38;2;248;248;242m              same last pathname component.[0m

[38;2;248;248;242m              By default, the user will be queried before replacing any  file  that  the[0m
[38;2;248;248;242m              user cannot write to, but writable files will be silently removed.  The -i[0m
[38;2;248;248;242m              option  causes  the user to be queried about replacing any existing files.[0m
[38;2;248;248;242m              The -f option causes any existing files to be  silently  deleted,  without[0m
[38;2;248;248;242m              querying.  -f takes precedence.[0m

[38;2;248;248;242m              Note that this mv will not move files across devices.  Historical versions[0m
[38;2;248;248;242m              of mv, when actual renaming is impossible, fall back on copying and remov‐[0m
[38;2;248;248;242m              ing files; if this behaviour is desired, use cp and rm manually.  This may[0m
[38;2;248;248;242m              change in a future version.[0m

[38;2;248;248;242m       rm [ -dfiRrs ] filename ...[0m
[38;2;248;248;242m              Removes files and directories specified.[0m

[38;2;248;248;242m              Normally,  rm  will  not  remove directories (except with the -R or -r op‐[0m
[38;2;248;248;242m              tions).  The -d option causes rm to try removing directories  with  unlink[0m
[38;2;248;248;242m              (see  unlink(2)),  the same method used for files.  Typically only the su‐[0m
[38;2;248;248;242m              per-user can actually succeed in unlinking directories in  this  way.   -d[0m
[38;2;248;248;242m              takes precedence over -R and -r.[0m

[38;2;248;248;242m              By  default,  the  user  will be queried before removing any file that the[0m
[38;2;248;248;242m              user cannot write to, but writable files will be silently removed.  The -i[0m
[38;2;248;248;242m              option causes the user to be queried about removing any files.  The -f op‐[0m
[38;2;248;248;242m              tion causes files to be silently deleted, without querying, and suppresses[0m
[38;2;248;248;242m              all error indications.  -f takes precedence.[0m

[38;2;248;248;242m              The -R and -r options cause rm to recursively  descend  into  directories,[0m
[38;2;248;248;242m              deleting all files in the directory before removing the directory with the[0m
[38;2;248;248;242m              rmdir system call (see rmdir(2)).[0m

[38;2;248;248;242m              The -s option is a zsh extension to rm functionality.  It enables paranoid[0m
[38;2;248;248;242m              behaviour, intended to avoid common security problems involving a root-run[0m
[38;2;248;248;242m              rm  being  tricked  into  removing files other than the ones intended.  It[0m
[38;2;248;248;242m              will  refuse  to  follow  symbolic  links,  so  that  (for  example)  ``rm[0m
[38;2;248;248;242m              /tmp/foo/passwd''  can't  accidentally remove /etc/passwd if /tmp/foo hap‐[0m
[38;2;248;248;242m              pens to be a link to /etc.  It will also check where it is  after  leaving[0m
[38;2;248;248;242m              directories,  so  that  a recursive removal of a deep directory tree can't[0m
[38;2;248;248;242m              end up recursively removing /usr as a result of directories being moved up[0m
[38;2;248;248;242m              the tree.[0m

[38;2;248;248;242m       rmdir dir ...[0m
[38;2;248;248;242m              Removes empty directories specified.[0m

[38;2;248;248;242m       sync   Calls the system call of the same name (see sync(2)), which flushes  dirty[0m
[38;2;248;248;242m              buffers  to  disk.   It might return before the I/O has actually been com‐[0m
[38;2;248;248;242m              pleted.[0m

[38;2;248;248;242m              An associative array that maps langinfo elements to their values.[0m

[38;2;248;248;242m              Your implementation may support a number of the following keys:[0m

[38;2;248;248;242m              CODESET, D_T_FMT, D_FMT, T_FMT, RADIXCHAR, THOUSEP, YESEXPR, NOEXPR,  CRN‐[0m

[38;2;248;248;242mTHE ZSH/MAPFILE MODULE[0m
[38;2;248;248;242m       The zsh/mapfile module provides one special associative array  parameter  of  the[0m
[38;2;248;248;242m       same name.[0m

[38;2;248;248;242m       mapfile[0m
[38;2;248;248;242m              This  associative  array  takes  as keys the names of files; the resulting[0m
[38;2;248;248;242m              value is the content of the file.  The value is treated identically to any[0m
[38;2;248;248;242m              other text coming from a parameter.  The value may also be assigned to, in[0m
[38;2;248;248;242m              which case the file in question is written (whether or not  it  originally[0m
[38;2;248;248;242m              existed);  or an element may be unset, which will delete the file in ques‐[0m
[38;2;248;248;242m              tion.  For example, `vared 'mapfile[myfile]'' works as  expected,  editing[0m
[38;2;248;248;242m              the file `myfile'.[0m

[38;2;248;248;242m              When  the array is accessed as a whole, the keys are the names of files in[0m
[38;2;248;248;242m              the current directory, and the values are empty (to save a  huge  overhead[0m
[38;2;248;248;242m              in  memory).   Thus ${(k)mapfile} has the same effect as the glob operator[0m
[38;2;248;248;242m              *(D), since files beginning with a dot are  not  special.   Care  must  be[0m
[38;2;248;248;242m              taken  with  expressions such as rm ${(k)mapfile}, which will delete every[0m
[38;2;248;248;242m              file in the current directory without the usual `rm *' test.[0m

[38;2;248;248;242m              The parameter mapfile may be made read-only; in that  case,  files  refer‐[0m
[38;2;248;248;242m              enced may not be written or deleted.[0m

[38;2;248;248;242m              A file may conveniently be read into an array as one line per element with[0m
[38;2;248;248;242m              the  form `array=("${(f@)mapfile[filename]}")'.  The double quotes and the[0m
[38;2;248;248;242m              `@' are necessary to prevent empty lines from being removed.  Note that if[0m
[38;2;248;248;242m              the file ends with a newline, the shell will split on the  final  newline,[0m
[38;2;248;248;242m              generating an additional empty field; this can be suppressed by using `ar‐[0m
[38;2;248;248;242m              ray=("${(f@)${mapfile[filename]%$'\n'}}")'.[0m

[38;2;248;248;242m   Limitations[0m
[38;2;248;248;242m       Although  reading  and  writing  of  the file in question is efficiently handled,[0m
[38;2;248;248;242m       zsh's internal memory management may be arbitrarily baroque; however, mapfile  is[0m
[38;2;248;248;242m       usually very much more efficient than anything involving a loop.  Note in partic‐[0m
[38;2;248;248;242m       ular  that the whole contents of the file will always reside physically in memory[0m
[38;2;248;248;242m       when accessed (possibly multiple times, due to  standard  parameter  substitution[0m
[38;2;248;248;242m       operations).   In  particular,  this  means  handling  of sufficiently long files[0m
[38;2;248;248;242m       (greater than the machine's swap space, or than the range of  the  pointer  type)[0m
[38;2;248;248;242m       will be incorrect.[0m

[38;2;248;248;242m       No  errors  are  printed  or  flagged for non-existent, unreadable, or unwritable[0m
[38;2;248;248;242m       files, as the parameter mechanism is too low in the shell execution hierarchy  to[0m
[38;2;248;248;242m       make this convenient.[0m

[38;2;248;248;242m       It  is  unfortunate that the mechanism for loading modules does not yet allow the[0m
[38;2;248;248;242m       user to specify the name of the shell parameter to be given  the  special  behav‐[0m
[38;2;248;248;242m       iour.[0m

[38;2;248;248;242mTHE ZSH/MATHFUNC MODULE[0m
[38;2;248;248;242m       The  zsh/mathfunc  module  provides  standard mathematical functions for use when[0m
[38;2;248;248;242m       evaluating mathematical formulae.  The syntax agrees with normal  C  and  FORTRAN[0m
[38;2;248;248;242m       conventions, for example,[0m

[38;2;248;248;242m              (( f = sin(0.3) ))[0m

[38;2;248;248;242m       assigns the sine of 0.3 to the parameter f.[0m

[38;2;248;248;242m       Most  functions  take floating point arguments and return a floating point value.[0m
[38;2;248;248;242m       However, any necessary conversions from or to integer type will be performed  au‐[0m
[38;2;248;248;242m       tomatically  by  the  shell.  Apart from atan with a second argument and the abs,[0m
[38;2;248;248;242m       int and float functions, all functions behave as noted in the manual page for the[0m
[38;2;248;248;242m       corresponding C function, except that any arguments out of range for the function[0m
[38;2;248;248;242m       in question will be detected by the shell and an error reported.[0m

[38;2;248;248;242m       The following functions take a single floating point argument: acos, acosh, asin,[0m
[38;2;248;248;242m       asinh, atan, atanh, cbrt, ceil, cos, cosh, erf, erfc, exp,  expm1,  fabs,  floor,[0m
[38;2;248;248;242m       gamma, j0, j1, lgamma, log, log10, log1p, log2, logb, sin, sinh, sqrt, tan, tanh,[0m
[38;2;248;248;242m       y0,  y1.   The atan function can optionally take a second argument, in which case[0m
[38;2;248;248;242m       it behaves like the C function atan2.  The ilogb function takes a single floating[0m
[38;2;248;248;242m       point argument, but returns an integer.[0m

[38;2;248;248;242m       The function signgam takes no arguments, and returns an integer, which is  the  C[0m
[38;2;248;248;242m       variable  of  the same name, as described in gamma(3).  Note that it is therefore[0m
[38;2;248;248;242m       only useful immediately after  a  call  to  gamma  or  lgamma.   Note  also  that[0m
[38;2;248;248;242m       `signgam()' and `signgam' are distinct expressions.[0m

[38;2;248;248;242m       The  functions min, max, and sum are defined not in this module but in the zmath‐[0m
[38;2;248;248;242m       func autoloadable function, described in the section `Mathematical Functions'  in[0m
[38;2;248;248;242m       zshcontrib(1).[0m

[38;2;248;248;242m       The following functions take two floating point arguments: copysign, fmod, hypot,[0m
[38;2;248;248;242m       nextafter.[0m

[38;2;248;248;242m       The  following  take  an integer first argument and a floating point second argu‐[0m
[38;2;248;248;242m       ment: jn, yn.[0m

[38;2;248;248;242m       The following take a floating point first argument and an  integer  second  argu‐[0m
[38;2;248;248;242m       ment: ldexp, scalb.[0m

[38;2;248;248;242m       The function abs does not convert the type of its single argument; it returns the[0m
[38;2;248;248;242m       absolute  value  of  either a floating point number or an integer.  The functions[0m
[38;2;248;248;242m       float and int convert their arguments into a floating point or integer value  (by[0m
[38;2;248;248;242m       truncation) respectively.[0m

[38;2;248;248;242m       Note that the C pow function is available in ordinary math evaluation as the `**'[0m
[38;2;248;248;242m       operator and is not provided here.[0m

[38;2;248;248;242m       The  function  rand48  is available if your system's mathematical library has the[0m
[38;2;248;248;242m       function erand48(3).  It returns a pseudo-random floating point number between  0[0m
[38;2;248;248;242m       and 1.  It takes a single string optional argument.[0m

[38;2;248;248;242m       If  the  argument  is not present, the random number seed is initialised by three[0m
[38;2;248;248;242m       calls to the rand(3) function --- this produces the same random  numbers  as  the[0m
[38;2;248;248;242m       next three values of $RANDOM.[0m

[38;2;248;248;242m       If  the  argument  is  present, it gives the name of a scalar parameter where the[0m
[38;2;248;248;242m       current random number seed will be stored.  On the first  call,  the  value  must[0m
[38;2;248;248;242m       contain  at  least  twelve hexadecimal digits (the remainder of the string is ig‐[0m
[38;2;248;248;242m       nored), or the seed will be initialised in the same  manner  as  for  a  call  to[0m
[38;2;248;248;242m       rand48  with  no  argument.  Subsequent calls to rand48(param) will then maintain[0m
[38;2;248;248;242m       the seed in the parameter param as a string of twelve hexadecimal digits, with no[0m
[38;2;248;248;242m       base signifier.  The random number sequences for different  parameters  are  com‐[0m
[38;2;248;248;242m       pletely  independent,  and are also independent from that used by calls to rand48[0m
[38;2;248;248;242m       with no argument.[0m

[38;2;248;248;242m       For example, consider[0m

[38;2;248;248;242m              print $(( rand48(seed) ))[0m
[38;2;248;248;242m              print $(( rand48() ))[0m
[38;2;248;248;242m              print $(( rand48(seed) ))[0m

[38;2;248;248;242m       Assuming $seed does not exist, it will be initialised by the first call.  In  the[0m
[38;2;248;248;242m       second  call, the default seed is initialised; note, however, that because of the[0m
[38;2;248;248;242m       properties of rand() there is a correlation between the seeds used  for  the  two[0m
[38;2;248;248;242m       initialisations,  so  for  more secure uses, you should generate your own 12-byte[0m
[38;2;248;248;242m       seed.  The third call returns to the same sequence of random numbers used in  the[0m
[38;2;248;248;242m       first call, unaffected by the intervening rand48().[0m

[38;2;248;248;242mTHE ZSH/NEARCOLOR MODULE[0m
[38;2;248;248;242m       The  zsh/nearcolor  module  replaces  colours  specified as hex triplets with the[0m
[38;2;248;248;242m       nearest colour in the 88 or 256 colour palettes that are widely used by  terminal[0m
[38;2;248;248;242m       emulators.   By  default,  24-bit  true  colour  escape  codes are generated when[0m
[38;2;248;248;242m       colours are specified using hex triplets.  These are not supported by all  termi‐[0m
[38;2;248;248;242m       nals.   The  purpose of this module is to make it easier to define colour prefer‐[0m
[38;2;248;248;242m       ences in a form that can work across a range of terminal emulators.[0m

[38;2;248;248;242m       Aside from the default colour, the ANSI standard for terminal escape  codes  pro‐[0m
[38;2;248;248;242m       vides for eight colours. The bright attribute brings this to sixteen. These basic[0m
[38;2;248;248;242m       colours are commonly used in terminal applications due to being widely supported.[0m
[38;2;248;248;242m       Expanded  88 and 256 colour palettes are also common and, while the first sixteen[0m
[38;2;248;248;242m       colours vary somewhat between terminals and configurations, these add a generally[0m
[38;2;248;248;242m       consistent and predictable set of colours.[0m

[38;2;248;248;242m       In order to use the zsh/nearcolor module, it only needs to be loaded. Thereafter,[0m
[38;2;248;248;242m       whenever a colour is specified using a hex triplet, it will be  compared  against[0m
[38;2;248;248;242m       each of the available colours and the closest will be selected. The first sixteen[0m
[38;2;248;248;242m       colours are never matched in this process due to being unpredictable.[0m

[38;2;248;248;242m       It isn't possible to reliably detect support for true colour in the terminal emu‐[0m
[38;2;248;248;242m       lator.  It  is therefore recommended to be selective in loading the zsh/nearcolor[0m
[38;2;248;248;242m       module. For example, the following checks the COLORTERM environment variable:[0m

[38;2;248;248;242m              [[ $COLORTERM = *(24bit|truecolor)* ]] || zmodload zsh/nearcolor[0m

[38;2;248;248;242m       Note that some terminals accept the true color escape codes but map  them  inter‐[0m
[38;2;248;248;242m       nally to a more limited palette in a similar manner to the zsh/nearcolor module.[0m

[38;2;248;248;242mTHE ZSH/NEWUSER MODULE[0m
[38;2;248;248;242m       The  zsh/newuser  module  is loaded at boot if it is available, the RCS option is[0m
[38;2;248;248;242m       set, and the PRIVILEGED option is not set (all three are true by default).   This[0m
[38;2;248;248;242m       takes  place  immediately  after  commands  in  the global zshenv file (typically[0m
[38;2;248;248;242m       /etc/zshenv), if any, have been executed.  If the module is not available  it  is[0m
[38;2;248;248;242m       silently ignored by the shell; the module may safely be removed from $MODULE_PATH[0m
[38;2;248;248;242m       by the administrator if it is not required.[0m

[38;2;248;248;242m       On  loading,  the  module  tests if any of the start-up files .zshenv, .zprofile,[0m
[38;2;248;248;242m       .zshrc or .zlogin exist in the directory given by the environment variable  ZDOT‐[0m
[38;2;248;248;242m       DIR,  or the user's home directory if that is not set.  The test is not performed[0m
[38;2;248;248;242m       and the module halts processing if the shell was in an emulation mode  (i.e.  had[0m
[38;2;248;248;242m       been invoked as some other shell than zsh).[0m

[38;2;248;248;242m       If  none  of  the  start-up  files were found, the module then looks for the file[0m
[38;2;248;248;242m       newuser first in a sitewide  directory,  usually  the  parent  directory  of  the[0m
[38;2;248;248;242m       site-functions  directory, and if that is not found the module searches in a ver‐[0m
[38;2;248;248;242m       sion-specific directory, usually the parent of the functions directory containing[0m
[38;2;248;248;242m       version-specific functions.  (These directories can be  configured  when  zsh  is[0m
[38;2;248;248;242m       built  using  the --enable-site-scriptdir=dir and --enable-scriptdir=dir flags to[0m
[38;2;248;248;242m       configure,   respectively;   the   defaults   are   prefix/share/zsh   and   pre‐[0m
[38;2;248;248;242m       fix/share/zsh/$ZSH_VERSION where the default prefix is /usr/local.)[0m

[38;2;248;248;242m       If the file newuser is found, it is then sourced in the same manner as a start-up[0m
[38;2;248;248;242m       file.   The  file  is  expected to contain code to install start-up files for the[0m
[38;2;248;248;242m       user, however any valid shell code will be executed.[0m

[38;2;248;248;242m       The zsh/newuser module is then unconditionally unloaded.[0m

[38;2;248;248;242m       Note that it is possible to achieve exactly the same effect  as  the  zsh/newuser[0m
[38;2;248;248;242m       module  by  adding  code  to  /etc/zshenv.  The module exists simply to allow the[0m
[38;2;248;248;242m       shell to make arrangements for new users without the  need  for  intervention  by[0m
[38;2;248;248;242m       package maintainers and system administrators.[0m

[38;2;248;248;242m       The  script  supplied  with the module invokes the shell function zsh-newuser-in‐[0m
[38;2;248;248;242m       stall.  This may be invoked directly by the user even if the  zsh/newuser  module[0m
[38;2;248;248;242m       is  disabled.   Note,  however,  that if the module is not installed the function[0m
[38;2;248;248;242m       will not be installed either.  The function is documented in  the  section  `User[0m
[38;2;248;248;242m       Configuration Functions' in zshcontrib(1).[0m

[38;2;248;248;242mTHE ZSH/PARAMETER MODULE[0m
[38;2;248;248;242m       The zsh/parameter module gives access to some of the internal hash tables used by[0m
[38;2;248;248;242m       the shell by defining some special parameters.[0m

[38;2;248;248;242m       options[0m
[38;2;248;248;242m              The  keys for this associative array are the names of the options that can[0m
[38;2;248;248;242m              be set and unset using the setopt and unsetopt builtins. The value of each[0m
[38;2;248;248;242m              key is either the string on if the option is currently set, or the  string[0m
[38;2;248;248;242m              off if the option is unset.  Setting a key to one of these strings is like[0m
[38;2;248;248;242m              setting or unsetting the option, respectively. Unsetting a key in this ar‐[0m
[38;2;248;248;242m              ray is like setting it to the value off.[0m

[38;2;248;248;242m       commands[0m
[38;2;248;248;242m              This  array gives access to the command hash table. The keys are the names[0m
[38;2;248;248;242m              of external commands, the values are the pathnames of the files that would[0m
[38;2;248;248;242m              be executed when the command would be invoked. Setting a key in this array[0m
[38;2;248;248;242m              defines a new entry in this table  in  the  same  way  as  with  the  hash[0m
[38;2;248;248;242m              builtin.  Unsetting  a key as in `unset "commands[foo]"' removes the entry[0m
[38;2;248;248;242m              for the given key from the command hash table.[0m

[38;2;248;248;242m       functions[0m
[38;2;248;248;242m              This associative array maps names of enabled functions  to  their  defini‐[0m
[38;2;248;248;242m              tions. Setting a key in it is like defining a function with the name given[0m
[38;2;248;248;242m              by  the  key  and the body given by the value. Unsetting a key removes the[0m
[38;2;248;248;242m              definition for the function named by the key.[0m

[38;2;248;248;242m       dis_functions[0m
[38;2;248;248;242m              Like functions but for disabled functions.[0m

[38;2;248;248;242m       functions_source[0m
[38;2;248;248;242m              This readonly associative array maps names of  enabled  functions  to  the[0m
[38;2;248;248;242m              name of the file containing the source of the function.[0m

[38;2;248;248;242m              For an autoloaded function that has already been loaded, or marked for au‐[0m
[38;2;248;248;242m              toload  with  an  absolute  path,  or  that has had its path resolved with[0m
[38;2;248;248;242m              `functions -r', this is the file found for autoloading, resolved to an ab‐[0m
[38;2;248;248;242m              solute path.[0m

[38;2;248;248;242m              For a function defined within the body of a script or sourced  file,  this[0m
[38;2;248;248;242m              is the name of that file.  In this case, this is the exact path originally[0m
[38;2;248;248;242m              used to that file, which may be a relative path.[0m

[38;2;248;248;242m              For  any other function, including any defined at an interactive prompt or[0m
[38;2;248;248;242m              an autoload function whose path has not yet been  resolved,  this  is  the[0m
[38;2;248;248;242m              empty  string.   However,  the hash element is reported as defined just so[0m
[38;2;248;248;242m              long as the function is present:  the keys to this hash are  the  same  as[0m
[38;2;248;248;242m              those to $functions.[0m

[38;2;248;248;242m       dis_functions_source[0m
[38;2;248;248;242m              Like functions_source but for disabled functions.[0m

[38;2;248;248;242m       builtins[0m
[38;2;248;248;242m              This  associative  array gives information about the builtin commands cur‐[0m
[38;2;248;248;242m              rently enabled. The keys are the names of the  builtin  commands  and  the[0m
[38;2;248;248;242m              values are either `undefined' for builtin commands that will automatically[0m
[38;2;248;248;242m              be  loaded from a module if invoked or `defined' for builtin commands that[0m
[38;2;248;248;242m              are already loaded.[0m

[38;2;248;248;242m       dis_builtins[0m
[38;2;248;248;242m              Like builtins but for disabled builtin commands.[0m

[38;2;248;248;242m       reswords[0m
[38;2;248;248;242m              This array contains the enabled reserved words.[0m

[38;2;248;248;242m       dis_reswords[0m
[38;2;248;248;242m              Like reswords but for disabled reserved words.[0m

[38;2;248;248;242m       patchars[0m
[38;2;248;248;242m              This array contains the enabled pattern characters.[0m

[38;2;248;248;242m       dis_patchars[0m
[38;2;248;248;242m              Like patchars but for disabled pattern characters.[0m

[38;2;248;248;242m       aliases[0m
[38;2;248;248;242m              This maps the names of the regular aliases currently enabled to their  ex‐[0m
[38;2;248;248;242m              pansions.[0m

[38;2;248;248;242m       dis_aliases[0m
[38;2;248;248;242m              Like aliases but for disabled regular aliases.[0m

[38;2;248;248;242m       galiases[0m
[38;2;248;248;242m              Like aliases, but for global aliases.[0m

[38;2;248;248;242m       dis_galiases[0m
[38;2;248;248;242m              Like galiases but for disabled global aliases.[0m

[38;2;248;248;242m       saliases[0m
[38;2;248;248;242m              Like raliases, but for suffix aliases.[0m

[38;2;248;248;242m       dis_saliases[0m
[38;2;248;248;242m              Like saliases but for disabled suffix aliases.[0m

[38;2;248;248;242m       parameters[0m
[38;2;248;248;242m              The  keys  in  this associative array are the names of the parameters cur‐[0m
[38;2;248;248;242m              rently defined. The values are strings describing the type of the  parame‐[0m
[38;2;248;248;242m              ter,  in  the  same  format used by the t parameter flag, see zshexpn(1) .[0m
[38;2;248;248;242m              Setting or unsetting keys in this array is not possible.[0m

[38;2;248;248;242m       modules[0m
[38;2;248;248;242m              An associative array giving information about modules. The  keys  are  the[0m
[38;2;248;248;242m              names  of the modules loaded, registered to be autoloaded, or aliased. The[0m
[38;2;248;248;242m              value says which state the named module is in and is one  of  the  strings[0m
[38;2;248;248;242m              `loaded', `autoloaded', or `alias:name', where name is the name the module[0m
[38;2;248;248;242m              is aliased to.[0m

[38;2;248;248;242m              Setting or unsetting keys in this array is not possible.[0m

[38;2;248;248;242m       dirstack[0m
[38;2;248;248;242m              A  normal array holding the elements of the directory stack. Note that the[0m
[38;2;248;248;242m              output of the dirs builtin command includes one more directory,  the  cur‐[0m
[38;2;248;248;242m              rent working directory.[0m

[38;2;248;248;242m       history[0m
[38;2;248;248;242m              This  associative  array  maps  history  event numbers to the full history[0m
[38;2;248;248;242m              lines.  Although it is presented as an associative array, the array of all[0m
[38;2;248;248;242m              values (${history[@]}) is guaranteed to be returned in order from most re‐[0m
[38;2;248;248;242m              cent to oldest history event, that is, by decreasing history event number.[0m

[38;2;248;248;242m       historywords[0m
[38;2;248;248;242m              A special array containing the words stored in the  history.   These  also[0m
[38;2;248;248;242m              appear in most to least recent order.[0m

[38;2;248;248;242m       jobdirs[0m
[38;2;248;248;242m              This  associative array maps job numbers to the directories from which the[0m
[38;2;248;248;242m              job was started (which may not be the current directory of the job).[0m

[38;2;248;248;242m              The keys of the associative arrays are  usually  valid  job  numbers,  and[0m
[38;2;248;248;242m              these are the values output with, for example, ${(k)jobdirs}.  Non-numeric[0m
[38;2;248;248;242m              job  references  may  be used when looking up a value; for example, ${job‐[0m
[38;2;248;248;242m              dirs[%+]} refers to the current job.[0m

[38;2;248;248;242m              See the jobs builtin for how job information is provided in a subshell.[0m

[38;2;248;248;242m       jobtexts[0m
[38;2;248;248;242m              This associative array maps job numbers to the texts of the command  lines[0m
[38;2;248;248;242m              that were used to start the jobs.[0m

[38;2;248;248;242m              Handling  of the keys of the associative array is as described for jobdirs[0m
[38;2;248;248;242m              above.[0m

[38;2;248;248;242m              See the jobs builtin for how job information is provided in a subshell.[0m

[38;2;248;248;242m       jobstates[0m
[38;2;248;248;242m              This associative array gives information about the states of the jobs cur‐[0m
[38;2;248;248;242m              rently known. The keys are the job numbers and the values are  strings  of[0m
[38;2;248;248;242m              the  form `job-state:mark:pid=state...'. The job-state gives the state the[0m
[38;2;248;248;242m              whole job is currently in, one of `running', `suspended', or  `done'.  The[0m
[38;2;248;248;242m              mark is `+' for the current job, `-' for the previous job and empty other‐[0m
[38;2;248;248;242m              wise.  This  is followed by one `:pid=state' for every process in the job.[0m
[38;2;248;248;242m              The pids are, of course, the process IDs and the state describes the state[0m
[38;2;248;248;242m              of that process.[0m

[38;2;248;248;242m              Handling of the keys of the associative array is as described for  jobdirs[0m
[38;2;248;248;242m              above.[0m

[38;2;248;248;242m              See the jobs builtin for how job information is provided in a subshell.[0m

[38;2;248;248;242m       nameddirs[0m
[38;2;248;248;242m              This  associative  array  maps the names of named directories to the path‐[0m
[38;2;248;248;242m              names they stand for.[0m

[38;2;248;248;242m       userdirs[0m
[38;2;248;248;242m              This associative array maps user names to the pathnames of their home  di‐[0m
[38;2;248;248;242m              rectories.[0m

[38;2;248;248;242m       usergroups[0m
[38;2;248;248;242m              This  associative  array  maps names of system groups of which the current[0m
[38;2;248;248;242m              user is a member to the corresponding group identifiers.  The contents are[0m
[38;2;248;248;242m              the same as the groups output by the id command.[0m

[38;2;248;248;242m       funcfiletrace[0m
[38;2;248;248;242m              This array contains the absolute line numbers and corresponding file names[0m
[38;2;248;248;242m              for the point where the current function, sourced file, or (if EVAL_LINENO[0m
[38;2;248;248;242m              is set) eval command was called.  The array is of the same length as func‐[0m
[38;2;248;248;242m              sourcetrace and functrace, but differs from funcsourcetrace  in  that  the[0m
[38;2;248;248;242m              line and file are the point of call, not the point of definition, and dif‐[0m
[38;2;248;248;242m              fers from functrace in that all values are absolute line numbers in files,[0m
[38;2;248;248;242m              rather than relative to the start of a function, if any.[0m

[38;2;248;248;242m       funcsourcetrace[0m
[38;2;248;248;242m              This  array  contains  the file names and line numbers of the points where[0m
[38;2;248;248;242m              the functions, sourced files, and (if EVAL_LINENO is  set)  eval  commands[0m
[38;2;248;248;242m              currently  being executed were defined.  The line number is the line where[0m
[38;2;248;248;242m              the `function name' or `name ()' started.  In the case  of  an  autoloaded[0m
[38;2;248;248;242m              function  the line number is reported as zero.  The format of each element[0m
[38;2;248;248;242m              is filename:lineno.[0m

[38;2;248;248;242m              For  functions autoloaded from a file in native zsh format, where only the[0m
[38;2;248;248;242m              body of the function occurs in the file, or for files that have been  exe‐[0m
[38;2;248;248;242m              cuted  by  the  source  or `.' builtins, the trace information is shown as[0m
[38;2;248;248;242m              filename:0, since the entire file is the definition.  The source file name[0m
[38;2;248;248;242m              is resolved to an absolute path when the function is loaded or the path to[0m
[38;2;248;248;242m              it otherwise resolved.[0m

[38;2;248;248;242m              Most users will be interested in the information in the funcfiletrace  ar‐[0m
[38;2;248;248;242m              ray instead.[0m

[38;2;248;248;242m       funcstack[0m
[38;2;248;248;242m              This  array  contains  the  names of the functions, sourced files, and (if[0m
[38;2;248;248;242m              EVAL_LINENO is set) eval commands. currently being executed. The first el‐[0m
[38;2;248;248;242m              ement is the name of the function using the parameter.[0m

[38;2;248;248;242m              The standard shell array zsh_eval_context can be  used  to  determine  the[0m
[38;2;248;248;242m              type  of shell construct being executed at each depth: note, however, that[0m
[38;2;248;248;242m              is in the opposite order, with the most recent item last, and it  is  more[0m
[38;2;248;248;242m              detailed, for example including an entry for toplevel, the main shell code[0m
[38;2;248;248;242m              being executed either interactively or from a script, which is not present[0m
[38;2;248;248;242m              in $funcstack.[0m

[38;2;248;248;242m       functrace[0m
[38;2;248;248;242m              This  array contains the names and line numbers of the callers correspond‐[0m
[38;2;248;248;242m              ing to the functions currently being executed.  The format of each element[0m
[38;2;248;248;242m              is name:lineno.  Callers are also shown for sourced files; the  caller  is[0m
[38;2;248;248;242m              the point where the source or `.' command was executed.[0m

[38;2;248;248;242mTHE ZSH/PCRE MODULE[0m
[38;2;248;248;242m       The zsh/pcre module makes some commands available as builtins:[0m

[38;2;248;248;242m       pcre_compile [ -aimxs ] PCRE[0m
[38;2;248;248;242m              Compiles a perl-compatible regular expression.[0m

[38;2;248;248;242m              Option -a will force the pattern to be anchored.  Option -i will compile a[0m
[38;2;248;248;242m              case-insensitive  pattern.   Option  -m will compile a multi-line pattern;[0m
[38;2;248;248;242m              that is, ^ and $ will match newlines within the pattern.  Option  -x  will[0m
[38;2;248;248;242m              compile  an  extended  pattern,  wherein whitespace and # comments are ig‐[0m
[38;2;248;248;242m              nored.  Option -s makes the dot metacharacter match  all  characters,  in‐[0m
[38;2;248;248;242m              cluding those that indicate newline.[0m

[38;2;248;248;242m       pcre_study[0m
[38;2;248;248;242m              Studies the previously-compiled PCRE which may result in faster matching.[0m

[38;2;248;248;242m       pcre_match [ -v var ] [ -a arr ] [ -n offset ] [ -b ] string[0m
[38;2;248;248;242m              Returns successfully if string matches the previously-compiled PCRE.[0m

[38;2;248;248;242m              Upon successful match, if the expression captures substrings within paren‐[0m
[38;2;248;248;242m              theses,  pcre_match  will  set the array match to those substrings, unless[0m
[38;2;248;248;242m              the -a option is given, in which case it will set the  array  arr.   Simi‐[0m
[38;2;248;248;242m              larly, the variable MATCH will be set to the entire matched portion of the[0m
[38;2;248;248;242m              string, unless the -v option is given, in which case the variable var will[0m
[38;2;248;248;242m              be  set.   No variables are altered if there is no successful match.  A -n[0m
[38;2;248;248;242m              option starts searching for a match  from  the  byte  offset  position  in[0m
[38;2;248;248;242m              string.   If  the -b option is given, the variable ZPCRE_OP will be set to[0m
[38;2;248;248;242m              an offset pair string, representing the byte offset positions of  the  en‐[0m
[38;2;248;248;242m              tire  matched  portion  within the string.  For example, a ZPCRE_OP set to[0m
[38;2;248;248;242m              "32 45" indicates that the matched portion began on  byte  offset  32  and[0m
[38;2;248;248;242m              ended  on  byte  offset 44.  Here, byte offset position 45 is the position[0m
[38;2;248;248;242m              directly after the matched portion.  Keep in mind that the  byte  position[0m
[38;2;248;248;242m              isn't necessarily the same as the character position when UTF-8 characters[0m
[38;2;248;248;242m              are  involved.  Consequently, the byte offset positions are only to be re‐[0m
[38;2;248;248;242m              lied on in the context of using them for subsequent  searches  on  string,[0m
[38;2;248;248;242m              using  an offset position as an argument to the -n option.  This is mostly[0m
[38;2;248;248;242m              used to implement the "find all non-overlapping matches" functionality.[0m

[38;2;248;248;242m              A simple example of "find all non-overlapping matches":[0m

[38;2;248;248;242m                     string="The following zip codes: 78884 90210 99513"[0m
[38;2;248;248;242m                     pcre_compile -m "\d{5}"[0m
[38;2;248;248;242m                     accum=()[0m
[38;2;248;248;242m                     pcre_match -b -- $string[0m
[38;2;248;248;242m                     while [[ $? -eq 0 ]] do[0m
[38;2;248;248;242m                         b=($=ZPCRE_OP)[0m
[38;2;248;248;242m                         accum+=$MATCH[0m
[38;2;248;248;242m                         pcre_match -b -n $b[2] -- $string[0m
[38;2;248;248;242m                     done[0m
[38;2;248;248;242m                     print -l $accum[0m

[38;2;248;248;242m       The zsh/pcre module makes available the following test condition:[0m

[38;2;248;248;242m       expr -pcre-match pcre[0m
[38;2;248;248;242m              Matches a string against a perl-compatible regular expression.[0m

[38;2;248;248;242m              For example,[0m

[38;2;248;248;242m                     [[ "$text" -pcre-match ^d+$ ]] &&[0m
[38;2;248;248;242m                     print text variable contains only "d's".[0m

[38;2;248;248;242m              If the REMATCH_PCRE option is  set,  the  =~  operator  is  equivalent  to[0m
[38;2;248;248;242m              -pcre-match,  and  the  NO_CASE_MATCH  option  may  be  used.   Note  that[0m
[38;2;248;248;242m              NO_CASE_MATCH never applies to the pcre_match builtin, instead use the  -i[0m
[38;2;248;248;242m              switch of pcre_compile.[0m

[38;2;248;248;242mTHE ZSH/PARAM/PRIVATE MODULE[0m
[38;2;248;248;242m       The  zsh/param/private module is used to create parameters whose scope is limited[0m
[38;2;248;248;242m       to the current function body, and not to other functions called  by  the  current[0m
[38;2;248;248;242m       function.[0m

[38;2;248;248;242m       This module provides a single autoloaded builtin:[0m

[38;2;248;248;242m       private [ {+|-}AHUahlmrtux ] [ {+|-}EFLRZi [ n ] ] [ name[=value] ... ][0m
[38;2;248;248;242m              The  private  builtin  accepts all the same options and arguments as local[0m
[38;2;248;248;242m              (zshbuiltins(1)) except for the `-T' option.  Tied parameters may  not  be[0m
[38;2;248;248;242m              made private.[0m

[38;2;248;248;242m              The  `-p' option is presently a no-op because the state of private parame‐[0m
[38;2;248;248;242m              ters cannot reliably be reloaded.  This also applies to  printing  private[0m
[38;2;248;248;242m              parameters with `typeset -p'.[0m

[38;2;248;248;242m              If  used  at  the  top level (outside a function scope), private creates a[0m
[38;2;248;248;242m              normal parameter in the same manner as  declare  or  typeset.   A  warning[0m
[38;2;248;248;242m              about  this is printed if WARN_CREATE_GLOBAL is set (zshoptions(1)).  Used[0m
[38;2;248;248;242m              inside a function scope, private creates a local parameter similar to  one[0m
[38;2;248;248;242m              declared with local, except having special properties noted below.[0m

[38;2;248;248;242m              Special  parameters  which expose or manipulate internal shell state, such[0m
[38;2;248;248;242m              as ARGC, argv, COLUMNS, LINES, UID, EUID, IFS,  PROMPT,  RANDOM,  SECONDS,[0m
[38;2;248;248;242m              etc.,  cannot  be  made private unless the `-h' option is used to hide the[0m
[38;2;248;248;242m              special meaning of the parameter.  This may change in the future.[0m

[38;2;248;248;242m       As with other typeset equivalents, private is both a builtin and a reserved word,[0m
[38;2;248;248;242m       so arrays may be assigned with parenthesized word  list  name=(value...)  syntax.[0m
[38;2;248;248;242m       However,  the reserved word `private' is not available until zsh/param/private is[0m
[38;2;248;248;242m       loaded, so care must be taken with order of execution and  parsing  for  function[0m
[38;2;248;248;242m       definitions  which use private.  To compensate for this, the module also adds the[0m
[38;2;248;248;242m       option `-P' to the `local' builtin to declare private parameters.[0m

[38;2;248;248;242m       For example, this construction fails if zsh/param/private has not yet been loaded[0m
[38;2;248;248;242m       when `bad_declaration' is defined:[0m
[38;2;248;248;242m              bad_declaration() {[0m
[38;2;248;248;242m                zmodload zsh/param/private[0m
[38;2;248;248;242m                private array=( one two three )[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       This construction works because local is already a keyword,  and  the  module  is[0m
[38;2;248;248;242m       loaded before the statement is executed:[0m
[38;2;248;248;242m              good_declaration() {[0m
[38;2;248;248;242m                zmodload zsh/param/private[0m
[38;2;248;248;242m                local -P array=( one two three )[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       The following is usable in scripts but may have trouble with autoload:[0m
[38;2;248;248;242m              zmodload zsh/param/private[0m
[38;2;248;248;242m              iffy_declaration() {[0m
[38;2;248;248;242m                private array=( one two three )[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       The  private  builtin may always be used with scalar assignments and for declara‐[0m
[38;2;248;248;242m       tions without assignments.[0m

[38;2;248;248;242m       Parameters declared with private have the following properties:[0m

[38;2;248;248;242m       •      Within the function body where it is declared, the parameter behaves as  a[0m
[38;2;248;248;242m              local, except as noted above for tied or special parameters.[0m

[38;2;248;248;242m       •      The  type  of  a parameter declared private cannot be changed in the scope[0m
[38;2;248;248;242m              where it was declared, even if the parameter is unset.  Thus an array can‐[0m
[38;2;248;248;242m              not be assigned to a private scalar, etc.[0m

[38;2;248;248;242m       •      Within any other function called by the declaring  function,  the  private[0m
[38;2;248;248;242m              parameter  does NOT hide other parameters of the same name, so for example[0m
[38;2;248;248;242m              a global parameter of the same name is visible and may be assigned or  un‐[0m
[38;2;248;248;242m              set.   This  includes calls to anonymous functions, although that may also[0m
[38;2;248;248;242m              change in the future.  However, the private name may not be  created  out‐[0m
[38;2;248;248;242m              side the local scope when it was not previously declared.[0m

[38;2;248;248;242m       •      An exported private remains in the environment of inner scopes but appears[0m
[38;2;248;248;242m              unset for the current shell in those scopes.  Generally, exporting private[0m
[38;2;248;248;242m              parameters should be avoided.[0m

[38;2;248;248;242m       Note  that  this  differs from the static scope defined by compiled languages de‐[0m
[38;2;248;248;242m       rived from C, in that the a new call to the same function creates  a  new  scope,[0m
[38;2;248;248;242m       i.e.,  the parameter is still associated with the call stack rather than with the[0m
[38;2;248;248;242m       function definition.  It differs from ksh `typeset -S' because the syntax used to[0m
[38;2;248;248;242m       define the function has no bearing on whether the parameter scope is respected.[0m

[38;2;248;248;242mTHE ZSH/REGEX MODULE[0m
[38;2;248;248;242m       The zsh/regex module makes available the following test condition:[0m

[38;2;248;248;242m       expr -regex-match regex[0m
[38;2;248;248;242m              Matches a string against a POSIX extended regular expression.  On success‐[0m
[38;2;248;248;242m              ful match, matched portion of the string will normally be  placed  in  the[0m
[38;2;248;248;242m              MATCH  variable.  If there are any capturing parentheses within the regex,[0m
[38;2;248;248;242m              then the match array variable will contain those.  If  the  match  is  not[0m
[38;2;248;248;242m              successful, then the variables will not be altered.[0m

[38;2;248;248;242m              For example,[0m

[38;2;248;248;242m                     [[ alphabetical -regex-match ^a([^a]+)a([^a]+)a ]] &&[0m
[38;2;248;248;242m                     print -l $MATCH X $match[0m

[38;2;248;248;242m              If the option REMATCH_PCRE is not set, then the =~ operator will automati‐[0m
[38;2;248;248;242m              cally  load  this module as needed and will invoke the -regex-match opera‐[0m
[38;2;248;248;242m              tor.[0m

[38;2;248;248;242m              If BASH_REMATCH is set, then the array BASH_REMATCH will be set instead of[0m
[38;2;248;248;242m              MATCH and match.[0m

[38;2;248;248;242m              Note that the zsh/regex module logic relies on the host system.  The  same[0m
[38;2;248;248;242m              expr and regex pair could produce different results on different platforms[0m
[38;2;248;248;242m              if a regex with non-standard syntax is given.[0m

[38;2;248;248;242m              For  example,  no  syntax  for  matching a word boundary is defined in the[0m
[38;2;248;248;242m              POSIX extended regular expression standard. GNU libc  and  BSD  libc  both[0m
[38;2;248;248;242m              provide such syntaxes as extensions (\b and [[:<:]]/[[:>:]] respectively),[0m
[38;2;248;248;242m              but  neither  of  these syntaxes is supported by both of these implementa‐[0m
[38;2;248;248;242m              tions.[0m

[38;2;248;248;242m              Refer to the regcomp(3) and re_format(7) manual pages on your  system  for[0m
[38;2;248;248;242m              locally-supported syntax.[0m

[38;2;248;248;242mTHE ZSH/SCHED MODULE[0m
[38;2;248;248;242m       The zsh/sched module makes available one builtin command and one parameter.[0m

[38;2;248;248;242m       sched [-o] [+]hh:mm[:ss] command ...[0m
[38;2;248;248;242m       sched [-o] [+]seconds command ...[0m
[38;2;248;248;242m       sched [ -item ][0m
[38;2;248;248;242m              Make  an entry in the scheduled list of commands to execute.  The time may[0m
[38;2;248;248;242m              be specified in either absolute or relative time,  and  either  as  hours,[0m
[38;2;248;248;242m              minutes  and  (optionally) seconds separated by a colon, or seconds alone.[0m
[38;2;248;248;242m              An  absolute  number  of  seconds  indicates  the  time  since  the  epoch[0m
[38;2;248;248;242m              (1970/01/01 00:00); this is useful in combination with the features in the[0m
[38;2;248;248;242m              zsh/datetime module, see the zsh/datetime module entry in zshmodules(1).[0m

[38;2;248;248;242m              With  no  arguments, prints the list of scheduled commands.  If the sched‐[0m
[38;2;248;248;242m              uled command has the -o flag set, this is shown at the start of  the  com‐[0m
[38;2;248;248;242m              mand.[0m

[38;2;248;248;242m              With the argument `-item', removes the given item from the list.  The num‐[0m
[38;2;248;248;242m              bering  of  the  list  is continuous and entries are in time order, so the[0m
[38;2;248;248;242m              numbering can change when entries are added or deleted.[0m

[38;2;248;248;242m              Commands are executed either immediately before a  prompt,  or  while  the[0m
[38;2;248;248;242m              shell's line editor is waiting for input.  In the latter case it is useful[0m
[38;2;248;248;242m              to  be  able to produce output that does not interfere with the line being[0m
[38;2;248;248;242m              edited.  Providing the option -o causes the shell  to  clear  the  command[0m
[38;2;248;248;242m              line  before the event and redraw it afterwards.  This should be used with[0m
[38;2;248;248;242m              any scheduled event that produces visible output to the  terminal;  it  is[0m
[38;2;248;248;242m              not  needed,  for  example, with output that updates a terminal emulator's[0m
[38;2;248;248;242m              title bar.[0m

[38;2;248;248;242m              To effect changes to the editor buffer when an  event  executes,  use  the[0m
[38;2;248;248;242m              `zle'  command with no arguments to test whether the editor is active, and[0m
[38;2;248;248;242m              if it is, then use `zle widget' to access the editor via the named widget.[0m

[38;2;248;248;242m              The sched builtin is not made available by default when the  shell  starts[0m
[38;2;248;248;242m              in a mode emulating another shell.  It can be made available with the com‐[0m
[38;2;248;248;242m              mand `zmodload -F zsh/sched b:sched'.[0m

[38;2;248;248;242m       zsh_scheduled_events[0m
[38;2;248;248;242m              A  readonly  array  corresponding  to  the  events  scheduled by the sched[0m
[38;2;248;248;242m              builtin.  The indices of the array correspond to the  numbers  shown  when[0m
[38;2;248;248;242m              sched is run with no arguments (provided that the KSH_ARRAYS option is not[0m
[38;2;248;248;242m              set).   The  value  of the array consists of the scheduled time in seconds[0m
[38;2;248;248;242m              since the epoch (see the section `The zsh/datetime Module' for  facilities[0m
[38;2;248;248;242m              for  using  this  number),  followed  by  a colon, followed by any options[0m
[38;2;248;248;242m              (which may be empty but will be preceded by a `-' otherwise), followed  by[0m
[38;2;248;248;242m              a colon, followed by the command to be executed.[0m

[38;2;248;248;242m              The  sched  builtin should be used for manipulating the events.  Note that[0m
[38;2;248;248;242m              this will have an immediate effect on the contents of the array,  so  that[0m
[38;2;248;248;242m              indices may become invalid.[0m

[38;2;248;248;242mTHE ZSH/NET/SOCKET MODULE[0m
[38;2;248;248;242m       The zsh/net/socket module makes available one builtin command:[0m

[38;2;248;248;242m       zsocket [ -altv ] [ -d fd ] [ args ][0m
[38;2;248;248;242m              zsocket  is  implemented  as  a builtin to allow full use of shell command[0m
[38;2;248;248;242m              line editing, file I/O, and job control mechanisms.[0m

[38;2;248;248;242m   Outbound Connections[0m
[38;2;248;248;242m       zsocket [ -v ] [ -d fd ] filename[0m
[38;2;248;248;242m              Open a new Unix domain connection to filename.  The shell parameter  REPLY[0m
[38;2;248;248;242m              will  be set to the file descriptor associated with that connection.  Cur‐[0m
[38;2;248;248;242m              rently, only stream connections are supported.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m              File descriptors can be closed with normal shell  syntax  when  no  longer[0m
[38;2;248;248;242m              needed, for example:[0m

[38;2;248;248;242m                     exec {REPLY}>&-[0m

[38;2;248;248;242m   Inbound Connections[0m
[38;2;248;248;242m       zsocket -l [ -v ] [ -d fd ] filename[0m
[38;2;248;248;242m              zsocket  -l will open a socket listening on filename.  The shell parameter[0m
[38;2;248;248;242m              REPLY will be set to the file descriptor associated  with  that  listener.[0m
[38;2;248;248;242m              The file descriptor remains open in subshells and forked external executa‐[0m
[38;2;248;248;242m              bles.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m       zsocket -a [ -tv ] [ -d targetfd ] listenfd[0m
[38;2;248;248;242m              zsocket  -a  will  accept  an incoming connection to the socket associated[0m
[38;2;248;248;242m              with listenfd.  The shell parameter REPLY will be set to the file descrip‐[0m
[38;2;248;248;242m              tor associated with the inbound connection.  The file  descriptor  remains[0m
[38;2;248;248;242m              open in subshells and forked external executables.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              If -t is specified, zsocket will return if no incoming connection is pend‐[0m
[38;2;248;248;242m              ing.  Otherwise it will wait for one.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242mTHE ZSH/STAT MODULE[0m
[38;2;248;248;242m       The zsh/stat module makes available one builtin command under two possible names:[0m

[38;2;248;248;242m       zstat [ -gnNolLtTrs ] [ -f fd ] [ -H hash ] [ -A array ] [ -F fmt ][0m
[38;2;248;248;242m             [ +element ] [ file ... ][0m
[38;2;248;248;242m       stat ...[0m
[38;2;248;248;242m              The  command  acts  as  a front end to the stat system call (see stat(2)).[0m
[38;2;248;248;242m              The same command is provided with two names; as the  name  stat  is  often[0m
[38;2;248;248;242m              used  by an external command it is recommended that only the zstat form of[0m
[38;2;248;248;242m              the command is used.  This can be arranged by loading the module with  the[0m
[38;2;248;248;242m              command `zmodload -F zsh/stat b:zstat'.[0m

[38;2;248;248;242m              If  the  stat call fails, the appropriate system error message printed and[0m
[38;2;248;248;242m              status 1 is returned.  The fields of struct stat  give  information  about[0m
[38;2;248;248;242m              the  files  provided  as  arguments  to the command.  In addition to those[0m
[38;2;248;248;242m              available from the stat call, an extra element `link' is provided.   These[0m
[38;2;248;248;242m              elements are:[0m

[38;2;248;248;242m              device The number of the device on which the file resides.[0m

[38;2;248;248;242m              inode  The unique number of the file on this device (`inode' number).[0m

[38;2;248;248;242m              mode   The  mode  of the file; that is, the file's type and access permis‐[0m
[38;2;248;248;242m                     sions.  With the -s option, this will be returned as a string  cor‐[0m
[38;2;248;248;242m                     responding to the first column in the display of the ls -l command.[0m

[38;2;248;248;242m              nlink  The number of hard links to the file.[0m

[38;2;248;248;242m              uid    The  user ID of the owner of the file.  With the -s option, this is[0m
[38;2;248;248;242m                     displayed as a user name.[0m

[38;2;248;248;242m              gid    The group ID of the file.  With the -s option, this is displayed as[0m
[38;2;248;248;242m                     a group name.[0m

[38;2;248;248;242m              rdev   The raw device number.  This is only useful for special devices.[0m

[38;2;248;248;242m              size   The size of the file in bytes.[0m

[38;2;248;248;242m              atime[0m
[38;2;248;248;242m              mtime[0m
[38;2;248;248;242m              ctime  The last access, modification and inode change times of  the  file,[0m
[38;2;248;248;242m                     respectively,  as  the  number of seconds since midnight GMT on 1st[0m
[38;2;248;248;242m                     January, 1970.  With the -s option, these are  printed  as  strings[0m
[38;2;248;248;242m                     for  the local time zone; the format can be altered with the -F op‐[0m
[38;2;248;248;242m                     tion, and with the -g option the times are in GMT.[0m

[38;2;248;248;242m              blksize[0m
[38;2;248;248;242m                     The number of bytes in one allocation block on the device on  which[0m
[38;2;248;248;242m                     the file resides.[0m

[38;2;248;248;242m              block  The number of disk blocks used by the file.[0m

[38;2;248;248;242m              link   If the file is a link and the -L option is in effect, this contains[0m
[38;2;248;248;242m                     the  name  of the file linked to, otherwise it is empty.  Note that[0m
[38;2;248;248;242m                     if this element is selected (``zstat +link'') then the -L option is[0m
[38;2;248;248;242m                     automatically used.[0m

[38;2;248;248;242m              A particular element may be selected by including its name preceded  by  a[0m
[38;2;248;248;242m              `+'  in  the option list; only one element is allowed.  The element may be[0m
[38;2;248;248;242m              shortened to any unique set of leading characters.   Otherwise,  all  ele‐[0m
[38;2;248;248;242m              ments will be shown for all files.[0m

[38;2;248;248;242m              Options:[0m

[38;2;248;248;242m              -A array[0m
[38;2;248;248;242m                     Instead  of  displaying the results on standard output, assign them[0m
[38;2;248;248;242m                     to an array, one struct stat element per  array  element  for  each[0m
[38;2;248;248;242m                     file  in  order.   In this case neither the name of the element nor[0m
[38;2;248;248;242m                     the name of the files appears in array unless the -t or -n  options[0m
[38;2;248;248;242m                     were given, respectively.  If -t is given, the element name appears[0m
[38;2;248;248;242m                     as  a  prefix to the appropriate array element; if -n is given, the[0m
[38;2;248;248;242m                     file name appears as a separate array  element  preceding  all  the[0m
[38;2;248;248;242m                     others.  Other formatting options are respected.[0m

[38;2;248;248;242m              -H hash[0m
[38;2;248;248;242m                     Similar to -A, but instead assign the values to hash.  The keys are[0m
[38;2;248;248;242m                     the  elements  listed above.  If the -n option is provided then the[0m
[38;2;248;248;242m                     name of the file is included in the hash with key name.[0m

[38;2;248;248;242m              -f fd  Use the file on file descriptor fd instead of named files; no  list[0m
[38;2;248;248;242m                     of file names is allowed in this case.[0m

[38;2;248;248;242m              -F fmt Supplies  a strftime (see strftime(3)) string for the formatting of[0m
[38;2;248;248;242m                     the time elements.  The format string supports all of the  zsh  ex‐[0m
[38;2;248;248;242m                     tensions  described in the section EXPANSION OF PROMPT SEQUENCES in[0m
[38;2;248;248;242m                     zshmisc(1).  In particular, -F %s.%N can be used to show timestamps[0m
[38;2;248;248;242m                     with nanosecond precision if supported by the system.  The  -s  op‐[0m
[38;2;248;248;242m                     tion is implied.[0m

[38;2;248;248;242m              -g     Show  the time elements in the GMT time zone.  The -s option is im‐[0m
[38;2;248;248;242m                     plied.[0m

[38;2;248;248;242m              -l     List the names of the type elements (to standard output or an array[0m
[38;2;248;248;242m                     as appropriate) and  return  immediately;  arguments,  and  options[0m
[38;2;248;248;242m                     other than -A, are ignored.[0m

[38;2;248;248;242m              -L     Perform an lstat (see lstat(2)) rather than a stat system call.  In[0m
[38;2;248;248;242m                     this case, if the file is a link, information about the link itself[0m
[38;2;248;248;242m                     rather  than  the target file is returned.  This option is required[0m
[38;2;248;248;242m                     to make the link element useful.  It's important to note that  this[0m
[38;2;248;248;242m                     is the exact opposite from ls(1), etc.[0m

[38;2;248;248;242m              -n     Always  show the names of files.  Usually these are only shown when[0m
[38;2;248;248;242m                     output is to standard output and there is more than one file in the[0m
[38;2;248;248;242m                     list.[0m

[38;2;248;248;242m              -N     Never show the names of files.[0m

[38;2;248;248;242m              -o     If a raw file mode is printed, show it in octal, which is more use‐[0m
[38;2;248;248;242m                     ful for human consumption than the default of decimal.   A  leading[0m
[38;2;248;248;242m                     zero  will be printed in this case.  Note that this does not affect[0m
[38;2;248;248;242m                     whether a raw or formatted file mode is shown, which is  controlled[0m
[38;2;248;248;242m                     by the -r and -s options, nor whether a mode is shown at all.[0m

[38;2;248;248;242m              -r     Print  raw  data (the default format) alongside string data (the -s[0m
[38;2;248;248;242m                     format); the string data appears in parentheses after the raw data.[0m

[38;2;248;248;242m              -s     Print mode, uid, gid and the three time elements as strings instead[0m
[38;2;248;248;242m                     of numbers.  In each case the format is like that of ls -l.[0m

[38;2;248;248;242m              -t     Always show the type names for the elements of struct  stat.   Usu‐[0m
[38;2;248;248;242m                     ally  these are only shown when output is to standard output and no[0m
[38;2;248;248;242m                     individual element has been selected.[0m

[38;2;248;248;242m              -T     Never show the type names of the struct stat elements.[0m

[38;2;248;248;242mTHE ZSH/SYSTEM MODULE[0m
[38;2;248;248;242m       The zsh/system module makes available various builtin commands and parameters.[0m

[38;2;248;248;242m   Builtins[0m
[38;2;248;248;242m       syserror [ -e errvar ] [ -p prefix ] [ errno | errname ][0m
[38;2;248;248;242m              This command prints out the error message associated with errno, a  system[0m
[38;2;248;248;242m              error number, followed by a newline to standard error.[0m

[38;2;248;248;242m              Instead  of  the  error number, a name errname, for example ENOENT, may be[0m
[38;2;248;248;242m              used.  The set of names is the same as the contents of the  array  errnos,[0m
[38;2;248;248;242m              see below.[0m

[38;2;248;248;242m              If  the  string  prefix is given, it is printed in front of the error mes‐[0m
[38;2;248;248;242m              sage, with no intervening space.[0m

[38;2;248;248;242m              If errvar is supplied, the entire message, without a newline, is  assigned[0m
[38;2;248;248;242m              to the parameter names errvar and nothing is output.[0m

[38;2;248;248;242m              A  return  status of 0 indicates the message was successfully printed (al‐[0m
[38;2;248;248;242m              though it may not be useful if the error number was out  of  the  system's[0m
[38;2;248;248;242m              range),  a  return status of 1 indicates an error in the parameters, and a[0m
[38;2;248;248;242m              return status of 2 indicates the error name was not recognised (no message[0m
[38;2;248;248;242m              is printed for this).[0m

[38;2;248;248;242m       sysopen [ -arw ] [ -m permissions ] [ -o options ][0m
[38;2;248;248;242m               -u fd file[0m
[38;2;248;248;242m              This command opens a file. The -r, -w and -a flags  indicate  whether  the[0m
[38;2;248;248;242m              file  should  be  opened for reading, writing and appending, respectively.[0m
[38;2;248;248;242m              The -m option allows the initial permissions to use when creating  a  file[0m
[38;2;248;248;242m              to  be specified in octal form.  The file descriptor is specified with -u.[0m
[38;2;248;248;242m              Either an explicit file descriptor in the range 0 to 9 can be specified or[0m
[38;2;248;248;242m              a variable name can be given to which the file descriptor number  will  be[0m
[38;2;248;248;242m              assigned.[0m

[38;2;248;248;242m              The  -o option allows various system specific options to be specified as a[0m
[38;2;248;248;242m              comma-separated list. The following is a list of  possible  options.  Note[0m
[38;2;248;248;242m              that, depending on the system, some may not be available.[0m
[38;2;248;248;242m              cloexec[0m
[38;2;248;248;242m                     mark  file  to be closed when other programs are executed (else the[0m
[38;2;248;248;242m                     file descriptor remains open in subshells and forked external  exe‐[0m
[38;2;248;248;242m                     cutables)[0m

[38;2;248;248;242m              create[0m
[38;2;248;248;242m              creat  create file if it does not exist[0m

[38;2;248;248;242m              excl   create file, error if it already exists[0m

[38;2;248;248;242m              noatime[0m
[38;2;248;248;242m                     suppress updating of the file atime[0m

[38;2;248;248;242m              nofollow[0m
[38;2;248;248;242m                     fail if file is a symbolic link[0m

[38;2;248;248;242m              nonblock[0m
[38;2;248;248;242m                     the file is opened in nonblocking mode[0m

[38;2;248;248;242m              sync   request that writes wait until data has been physically written[0m

[38;2;248;248;242m              truncate[0m
[38;2;248;248;242m              trunc  truncate file to size 0[0m

[38;2;248;248;242m              To close the file, use one of the following:[0m

[38;2;248;248;242m                     exec {fd}<&-[0m
[38;2;248;248;242m                     exec {fd}>&-[0m

[38;2;248;248;242m       sysread [ -c countvar ] [ -i infd ] [ -o outfd ][0m
[38;2;248;248;242m               [ -s bufsize ] [ -t timeout ] [ param ][0m
[38;2;248;248;242m              Perform a single system read from file descriptor infd, or zero if that is[0m
[38;2;248;248;242m              not  given.  The result of the read is stored in param or REPLY if that is[0m
[38;2;248;248;242m              not given.  If countvar is given, the number of bytes read is assigned  to[0m
[38;2;248;248;242m              the parameter named by countvar.[0m

[38;2;248;248;242m              The  maximum number of bytes read is bufsize or 8192 if that is not given,[0m
[38;2;248;248;242m              however the command returns as soon as any number of  bytes  was  success‐[0m
[38;2;248;248;242m              fully read.[0m

[38;2;248;248;242m              If  timeout is given, it specifies a timeout in seconds, which may be zero[0m
[38;2;248;248;242m              to poll the file descriptor.  This is handled by the poll system  call  if[0m
[38;2;248;248;242m              available, otherwise the select system call if available.[0m

[38;2;248;248;242m              If  outfd is given, an attempt is made to write all the bytes just read to[0m
[38;2;248;248;242m              the file descriptor outfd.  If this fails, because of a system error other[0m
[38;2;248;248;242m              than EINTR or because of an internal zsh error during  an  interrupt,  the[0m
[38;2;248;248;242m              bytes  read  but not written are stored in the parameter named by param if[0m
[38;2;248;248;242m              supplied (no default is used in this case), and the number of  bytes  read[0m
[38;2;248;248;242m              but  not  written  is stored in the parameter named by countvar if that is[0m
[38;2;248;248;242m              supplied.  If it was successful, countvar  contains  the  full  number  of[0m
[38;2;248;248;242m              bytes transferred, as usual, and param is not set.[0m

[38;2;248;248;242m              The  error  EINTR  (interrupted system call) is handled internally so that[0m
[38;2;248;248;242m              shell interrupts are transparent to the caller.  Any other error causes  a[0m
[38;2;248;248;242m              return.[0m

[38;2;248;248;242m              The possible return statuses are[0m
[38;2;248;248;242m              0      At  least  one byte of data was successfully read and, if appropri‐[0m
[38;2;248;248;242m                     ate, written.[0m

[38;2;248;248;242m              1      There was an error in the parameters to the command.  This  is  the[0m
[38;2;248;248;242m                     only error for which a message is printed to standard error.[0m

[38;2;248;248;242m              2      There  was  an  error on the read, or on polling the input file de‐[0m
[38;2;248;248;242m                     scriptor for a timeout.  The parameter ERRNO gives the error.[0m

[38;2;248;248;242m              3      Data were successfully read, but there was an error writing them to[0m
[38;2;248;248;242m                     outfd.  The parameter ERRNO gives the error.[0m

[38;2;248;248;242m              4      The attempt to read timed out.  Note this does  not  set  ERRNO  as[0m
[38;2;248;248;242m                     this is not a system error.[0m

[38;2;248;248;242m              5      No  system  error occurred, but zero bytes were read.  This usually[0m
[38;2;248;248;242m                     indicates end of file.  The parameters are  set  according  to  the[0m
[38;2;248;248;242m                     usual rules; no write to outfd is attempted.[0m

[38;2;248;248;242m       sysseek [ -u fd ] [ -w start|end|current ] offset[0m
[38;2;248;248;242m              The current file position at which future reads and writes will take place[0m
[38;2;248;248;242m              is  adjusted  to  the  specified byte offset. The offset is evaluated as a[0m
[38;2;248;248;242m              math expression. The -u option allows the file descriptor to be specified.[0m
[38;2;248;248;242m              By default the offset is specified relative to the start or the file  but,[0m
[38;2;248;248;242m              with  the  -w  option, it is possible to specify that the offset should be[0m
[38;2;248;248;242m              relative to the current position or the end of the file.[0m

[38;2;248;248;242m       syswrite [ -c countvar ] [ -o outfd ] data[0m
[38;2;248;248;242m              The data (a single string of bytes) are written  to  the  file  descriptor[0m
[38;2;248;248;242m              outfd,  or  1 if that is not given, using the write system call.  Multiple[0m
[38;2;248;248;242m              write operations may be used if the first does not write all the data.[0m

[38;2;248;248;242m              If countvar is given, the number of byte written is stored in the  parame‐[0m
[38;2;248;248;242m              ter named by countvar; this may not be the full length of data if an error[0m
[38;2;248;248;242m              occurred.[0m

[38;2;248;248;242m              The  error EINTR (interrupted system call) is handled internally by retry‐[0m
[38;2;248;248;242m              ing; otherwise an error causes the command to return.  For example, if the[0m
[38;2;248;248;242m              file descriptor is set to non-blocking output, an error  EAGAIN  (on  some[0m
[38;2;248;248;242m              systems, EWOULDBLOCK) may result in the command returning early.[0m

[38;2;248;248;242m              The  return  status may be 0 for success, 1 for an error in the parameters[0m
[38;2;248;248;242m              to the command, or 2 for an error  on  the  write;  no  error  message  is[0m
[38;2;248;248;242m              printed  in  the last case, but the parameter ERRNO will reflect the error[0m
[38;2;248;248;242m              that occurred.[0m

[38;2;248;248;242m       zsystem flock [ -t timeout ] [ -i interval ] [ -f var ] [-er] file[0m
[38;2;248;248;242m       zsystem flock -u fd_expr[0m
[38;2;248;248;242m              The builtin zsystem's subcommand flock performs advisory file locking (via[0m
[38;2;248;248;242m              the fcntl(2) system call) over the entire  contents  of  the  given  file.[0m
[38;2;248;248;242m              This  form of locking requires the processes accessing the file to cooper‐[0m
[38;2;248;248;242m              ate; its most obvious use is between two instances of the shell itself.[0m

[38;2;248;248;242m              In the first form the named file, which must already exist, is  locked  by[0m
[38;2;248;248;242m              opening  a file descriptor to the file and applying a lock to the file de‐[0m
[38;2;248;248;242m              scriptor.  The lock terminates when the shell  process  that  created  the[0m
[38;2;248;248;242m              lock  exits;  it is therefore often convenient to create file locks within[0m
[38;2;248;248;242m              subshells, since the lock is automatically released when the subshell  ex‐[0m
[38;2;248;248;242m              its.   Note  that  use  of the print builtin with the -u option will, as a[0m
[38;2;248;248;242m              side effect, release the lock, as will redirection  to  the  file  in  the[0m
[38;2;248;248;242m              shell  holding the lock.  To work around this use a subshell, e.g. `(print[0m
[38;2;248;248;242m              message) >> file'.  Status 0 is returned if the lock succeeds, else status[0m
[38;2;248;248;242m              1.[0m

[38;2;248;248;242m              In the second form the file descriptor given by the arithmetic  expression[0m
[38;2;248;248;242m              fd_expr  is  closed, releasing a lock.  The file descriptor can be queried[0m
[38;2;248;248;242m              by using the `-f var' form during the lock;  on  a  successful  lock,  the[0m
[38;2;248;248;242m              shell  variable  var  is set to the file descriptor used for locking.  The[0m
[38;2;248;248;242m              lock will be released if the file descriptor is closed by any other means,[0m
[38;2;248;248;242m              for example using `exec {var}>&-'; however, the form described  here  per‐[0m
[38;2;248;248;242m              forms a safety check that the file descriptor is in use for file locking.[0m

[38;2;248;248;242m              By  default the shell waits indefinitely for the lock to succeed.  The op‐[0m
[38;2;248;248;242m              tion -t timeout specifies a timeout for the lock  in  seconds;  fractional[0m
[38;2;248;248;242m              seconds  are  allowed.  During this period, the shell will attempt to lock[0m
[38;2;248;248;242m              the file every interval seconds if the -i interval option is given, other‐[0m
[38;2;248;248;242m              wise once a second.  (This interval is shortened before the  last  attempt[0m
[38;2;248;248;242m              if needed, so that the shell waits only until the timeout and not longer.)[0m
[38;2;248;248;242m              If the attempt times out, status 2 is returned.[0m

[38;2;248;248;242m              (Note: timeout is limited to 2^30-1 seconds (about 34 years), and interval[0m
[38;2;248;248;242m              to  0.999  *  LONG_MAX  microseconds (only about 35 minutes on 32-bit sys‐[0m
[38;2;248;248;242m              tems).)[0m

[38;2;248;248;242m              If the option -e is given, the file descriptor for the lock  is  preserved[0m
[38;2;248;248;242m              when the shell uses exec to start a new process; otherwise it is closed at[0m
[38;2;248;248;242m              that point and the lock released.[0m

[38;2;248;248;242m              If  the  option -r is given, the lock is only for reading, otherwise it is[0m
[38;2;248;248;242m              for reading and writing.  The file descriptor is opened accordingly.[0m

[38;2;248;248;242m       zsystem supports subcommand[0m
[38;2;248;248;242m              The builtin zsystem's subcommand supports tests whether a given subcommand[0m
[38;2;248;248;242m              is supported.  It returns status 0 if so,  else  status  1.   It  operates[0m
[38;2;248;248;242m              silently  unless  there was a syntax error (i.e. the wrong number of argu‐[0m
[38;2;248;248;242m              ments), in which case status 255 is returned.  Status 1 can  indicate  one[0m
[38;2;248;248;242m              of two things:  subcommand is known but not supported by the current oper‐[0m
[38;2;248;248;242m              ating  system,  or  subcommand  is  not known (possibly because this is an[0m
[38;2;248;248;242m              older version of the shell before it was implemented).[0m

[38;2;248;248;242m   Math Functions[0m
[38;2;248;248;242m       systell(fd)[0m
[38;2;248;248;242m              The systell math function returns the current file position for  the  file[0m
[38;2;248;248;242m              descriptor passed as an argument.[0m

[38;2;248;248;242m   Parameters[0m
[38;2;248;248;242m       errnos A  readonly array of the names of errors defined on the system.  These are[0m
[38;2;248;248;242m              typically macros defined in C by including the system header file errno.h.[0m
[38;2;248;248;242m              The index of each name (assuming the option KSH_ARRAYS  is  unset)  corre‐[0m
[38;2;248;248;242m              sponds to the error number.  Error numbers num before the last known error[0m
[38;2;248;248;242m              which have no name are given the name Enum in the array.[0m

[38;2;248;248;242m              Note  that  aliases for errors are not handled; only the canonical name is[0m
[38;2;248;248;242m              used.[0m

[38;2;248;248;242m       sysparams[0m
[38;2;248;248;242m              A readonly associative array.  The keys are:[0m

[38;2;248;248;242m              pid    Returns the process ID of the current process, even  in  subshells.[0m
[38;2;248;248;242m                     Compare $$, which returns the process ID of the main shell process.[0m

[38;2;248;248;242m              ppid   Returns  the  current  process  ID  of  the  parent  of the current[0m
[38;2;248;248;242m                     process, even in  subshells.   Compare  $PPID,  which  returns  the[0m
[38;2;248;248;242m                     process ID of the initial parent of the main shell process.[0m

[38;2;248;248;242m              procsubstpid[0m
[38;2;248;248;242m                     Returns the process ID of the last process started for process sub‐[0m
[38;2;248;248;242m                     stitution, i.e. the <(...) and >(...) expansions.[0m

[38;2;248;248;242mTHE ZSH/NET/TCP MODULE[0m
[38;2;248;248;242m       The zsh/net/tcp module makes available one builtin command:[0m

[38;2;248;248;242m       ztcp [ -acflLtv ] [ -d fd ] [ args ][0m
[38;2;248;248;242m              ztcp  is  implemented as a builtin to allow full use of shell command line[0m
[38;2;248;248;242m              editing, file I/O, and job control mechanisms.[0m

[38;2;248;248;242m              If ztcp is run with no options, it will output the contents of its session[0m
[38;2;248;248;242m              table.[0m

[38;2;248;248;242m              If it is run with only the option -L, it will output the contents  of  the[0m
[38;2;248;248;242m              session  table  in a format suitable for automatic parsing.  The option is[0m
[38;2;248;248;242m              ignored if given with a command to open or close a  session.   The  output[0m
[38;2;248;248;242m              consists of a set of lines, one per session, each containing the following[0m
[38;2;248;248;242m              elements separated by spaces:[0m

[38;2;248;248;242m              File descriptor[0m
[38;2;248;248;242m                     The  file descriptor in use for the connection.  For normal inbound[0m
[38;2;248;248;242m                     (I) and outbound (O) connections this may be read  and  written  by[0m
[38;2;248;248;242m                     the  usual shell mechanisms.  However, it should only be close with[0m
[38;2;248;248;242m                     `ztcp -c'.[0m

[38;2;248;248;242m              Connection type[0m
[38;2;248;248;242m                     A letter indicating how the session was created:[0m

[38;2;248;248;242m                     Z      A session created with the zftp command.[0m

[38;2;248;248;242m                     L      A connection opened for listening with `ztcp -l'.[0m

[38;2;248;248;242m                     I      An inbound connection accepted with `ztcp -a'.[0m

[38;2;248;248;242m                     O      An outbound connection created with `ztcp host ...'.[0m

[38;2;248;248;242m              The local host[0m
[38;2;248;248;242m                     This is usually set to an all-zero IP address as the address of the[0m
[38;2;248;248;242m                     localhost is irrelevant.[0m

[38;2;248;248;242m              The local port[0m
[38;2;248;248;242m                     This is likely to be zero unless the connection is for listening.[0m

[38;2;248;248;242m              The remote host[0m
[38;2;248;248;242m                     This is the fully qualified domain name of the peer, if  available,[0m
[38;2;248;248;242m                     else  an  IP  address.   It is an all-zero IP address for a session[0m
[38;2;248;248;242m                     opened for listening.[0m

[38;2;248;248;242m              The remote port[0m
[38;2;248;248;242m                     This is zero for a connection opened for listening.[0m

[38;2;248;248;242m   Outbound Connections[0m
[38;2;248;248;242m       ztcp [ -v ] [ -d fd ] host [ port ][0m
[38;2;248;248;242m              Open a new TCP connection to host.  If the port is omitted,  it  will  de‐[0m
[38;2;248;248;242m              fault  to  port 23.  The connection will be added to the session table and[0m
[38;2;248;248;242m              the shell parameter REPLY will be set to the  file  descriptor  associated[0m
[38;2;248;248;242m              with that connection.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m   Inbound Connections[0m
[38;2;248;248;242m       ztcp -l [ -v ] [ -d fd ] port[0m
[38;2;248;248;242m              ztcp  -l  will  open  a  socket listening on TCP port.  The socket will be[0m
[38;2;248;248;242m              added to the session table and the shell parameter REPLY will  be  set  to[0m
[38;2;248;248;242m              the file descriptor associated with that listener.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m       ztcp -a [ -tv ] [ -d targetfd ] listenfd[0m
[38;2;248;248;242m              ztcp  -a  will  accept  an incoming connection to the port associated with[0m
[38;2;248;248;242m              listenfd.  The connection will be added to the session table and the shell[0m
[38;2;248;248;242m              parameter REPLY will be set to the file descriptor associated with the in‐[0m
[38;2;248;248;242m              bound connection.[0m

[38;2;248;248;242m              If -d is specified, its argument will be taken as the target file descrip‐[0m
[38;2;248;248;242m              tor for the connection.[0m

[38;2;248;248;242m              If -t is specified, ztcp will return if no incoming connection is pending.[0m
[38;2;248;248;242m              Otherwise it will wait for one.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m   Closing Connections[0m
[38;2;248;248;242m       ztcp -cf [ -v ] [ fd ][0m
[38;2;248;248;242m       ztcp -c [ -v ] [ fd ][0m
[38;2;248;248;242m              ztcp -c will close the socket associated with fd.  The socket will be  re‐[0m
[38;2;248;248;242m              moved  from  the  session  table.  If fd is not specified, ztcp will close[0m
[38;2;248;248;242m              everything in the session table.[0m

[38;2;248;248;242m              Normally, sockets registered by zftp (see zshmodules(1) ) cannot be closed[0m
[38;2;248;248;242m              this way.  In order to force such a socket closed, use -f.[0m

[38;2;248;248;242m              In order to elicit more verbose output, use -v.[0m

[38;2;248;248;242m   Example[0m
[38;2;248;248;242m       Here is how to create a TCP connection between two instances of zsh.  We need  to[0m
[38;2;248;248;242m       pick an unassigned port; here we use the randomly chosen 5123.[0m

[38;2;248;248;242m       On host1,[0m
[38;2;248;248;242m              zmodload zsh/net/tcp[0m
[38;2;248;248;242m              ztcp -l 5123[0m
[38;2;248;248;242m              listenfd=$REPLY[0m
[38;2;248;248;242m              ztcp -a $listenfd[0m
[38;2;248;248;242m              fd=$REPLY[0m
[38;2;248;248;242m       The second from last command blocks until there is an incoming connection.[0m

[38;2;248;248;242m       Now create a connection from host2 (which may, of course, be the same machine):[0m
[38;2;248;248;242m              zmodload zsh/net/tcp[0m
[38;2;248;248;242m              ztcp host1 5123[0m
[38;2;248;248;242m              fd=$REPLY[0m

[38;2;248;248;242m       Now  on  each host, $fd contains a file descriptor for talking to the other.  For[0m
[38;2;248;248;242m       example, on host1:[0m
[38;2;248;248;242m              print This is a message >&$fd[0m
[38;2;248;248;242m       and on host2:[0m
[38;2;248;248;242m              read -r line <&$fd; print -r - $line[0m
[38;2;248;248;242m       prints `This is a message'.[0m

[38;2;248;248;242m       To tidy up, on host1:[0m
[38;2;248;248;242m              ztcp -c $listenfd[0m
[38;2;248;248;242m              ztcp -c $fd[0m
[38;2;248;248;242m       and on host2[0m
[38;2;248;248;242m              ztcp -c $fd[0m

[38;2;248;248;242mTHE ZSH/WATCH MODULE[0m
[38;2;248;248;242m       The zsh/watch module can be used to report when specific users  log  in  or  out.[0m
[38;2;248;248;242m       This is controlled via the following parameters.[0m

[38;2;248;248;242m       LOGCHECK[0m
[38;2;248;248;242m              The interval in seconds between checks for login/logout activity using the[0m
[38;2;248;248;242m              watch parameter.[0m

[38;2;248;248;242m       watch <S> <Z> (WATCH <S>)[0m
[38;2;248;248;242m              An array (colon-separated list) of login/logout events to report.[0m

[38;2;248;248;242m              If it contains the single word `all', then all login/logout events are re‐[0m
[38;2;248;248;242m              ported.   If  it contains the single word `notme', then all events are re‐[0m
[38;2;248;248;242m              ported as with `all' except $USERNAME.[0m

[38;2;248;248;242m              An entry in this list may consist of a username, an `@' followed by a  re‐[0m
[38;2;248;248;242m              mote  hostname, and a `%' followed by a line (tty).  Any of these may be a[0m
[38;2;248;248;242m              pattern (be sure to quote this during the assignment to watch so  that  it[0m
[38;2;248;248;242m              does  not  immediately  perform  file  generation); the setting of the EX‐[0m
[38;2;248;248;242m              TENDED_GLOB option is respected.  Any or all of these  components  may  be[0m
[38;2;248;248;242m              present  in  an  entry; if a login/logout event matches all of them, it is[0m
[38;2;248;248;242m              reported.[0m

[38;2;248;248;242m              For example, with the EXTENDED_GLOB option set, the following:[0m

[38;2;248;248;242m                     watch=('^(pws|barts)')[0m

[38;2;248;248;242m              causes reports for activity associated with any user  other  than  pws  or[0m
[38;2;248;248;242m              barts.[0m

[38;2;248;248;242m       WATCHFMT[0m
[38;2;248;248;242m              The format of login/logout reports if the watch parameter is set.  Default[0m
[38;2;248;248;242m              is `%n has %a %l from %m'.  Recognizes the following escape sequences:[0m

[38;2;248;248;242m              %n     The name of the user that logged in/out.[0m

[38;2;248;248;242m              %a     The observed action, i.e. "logged on" or "logged off".[0m

[38;2;248;248;242m              %l     The line (tty) the user is logged in on.[0m

[38;2;248;248;242m              %M     The full hostname of the remote host.[0m

[38;2;248;248;242m              %m     The hostname up to the first `.'.  If only the IP address is avail‐[0m
[38;2;248;248;242m                     able  or  the utmp field contains the name of an X-windows display,[0m
[38;2;248;248;242m                     the whole name is printed.[0m

[38;2;248;248;242m                     NOTE: The `%m' and `%M' escapes will work only if there is  a  host[0m
[38;2;248;248;242m                     name field in the utmp on your machine.  Otherwise they are treated[0m
[38;2;248;248;242m                     as ordinary strings.[0m

[38;2;248;248;242m              %F{color} (%f)[0m
[38;2;248;248;242m                     Start (stop) using a different foreground color.[0m

[38;2;248;248;242m              %K{color} (%k)[0m
[38;2;248;248;242m                     Start (stop) using a different background color.[0m

[38;2;248;248;242m              %S (%s)[0m
[38;2;248;248;242m                     Start (stop) standout mode.[0m

[38;2;248;248;242m              %U (%u)[0m
[38;2;248;248;242m                     Start (stop) underline mode.[0m

[38;2;248;248;242m              %B (%b)[0m
[38;2;248;248;242m                     Start (stop) boldface mode.[0m

[38;2;248;248;242m              %t[0m
[38;2;248;248;242m              %@     The time, in 12-hour, am/pm format.[0m

[38;2;248;248;242m              %T     The time, in 24-hour format.[0m

[38;2;248;248;242m              %w     The date in `day-dd' format.[0m

[38;2;248;248;242m              %W     The date in `mm/dd/yy' format.[0m

[38;2;248;248;242m              %D     The date in `yy-mm-dd' format.[0m

[38;2;248;248;242m              %D{string}[0m
[38;2;248;248;242m                     The  date formatted as string using the strftime function, with zsh[0m
[38;2;248;248;242m                     extensions as described by EXPANSION OF PROMPT  SEQUENCES  in  zsh‐[0m
[38;2;248;248;242m                     misc(1).[0m

[38;2;248;248;242m              %(x:true-text:false-text)[0m
[38;2;248;248;242m                     Specifies  a  ternary expression.  The character following the x is[0m
[38;2;248;248;242m                     arbitrary; the same character is used to separate the text for  the[0m
[38;2;248;248;242m                     "true" result from that for the "false" result.  Both the separator[0m
[38;2;248;248;242m                     and the right parenthesis may be escaped with a backslash.  Ternary[0m
[38;2;248;248;242m                     expressions may be nested.[0m

[38;2;248;248;242m                     The  test character x may be any one of `l', `n', `m' or `M', which[0m
[38;2;248;248;242m                     indicate a `true' result if the corresponding escape sequence would[0m
[38;2;248;248;242m                     return a non-empty value; or it  may  be  `a',  which  indicates  a[0m
[38;2;248;248;242m                     `true'  result  if the watched user has logged in, or `false' if he[0m
[38;2;248;248;242m                     has logged out.  Other characters  evaluate  to  neither  true  nor[0m
[38;2;248;248;242m                     false; the entire expression is omitted in this case.[0m

[38;2;248;248;242m                     If  the result is `true', then the true-text is formatted according[0m
[38;2;248;248;242m                     to the rules above and printed, and the false-text is skipped.   If[0m
[38;2;248;248;242m                     `false',  the  true-text is skipped and the false-text is formatted[0m
[38;2;248;248;242m                     and printed.  Either or both of the branches may be empty, but both[0m
[38;2;248;248;242m                     separators must be present in any case.[0m

[38;2;248;248;242m       Furthermore, the zsh/watch module makes available one builtin command:[0m

[38;2;248;248;242m       log    List all users currently logged in who are affected by the current setting[0m
[38;2;248;248;242m              of the watch parameter.[0m

[38;2;248;248;242mTHE ZSH/ZFTP MODULE[0m
[38;2;248;248;242m       The zsh/zftp module makes available one builtin command:[0m

[38;2;248;248;242m       zftp subcommand [ args ][0m
[38;2;248;248;242m              The zsh/zftp module is a client for FTP (file transfer protocol).   It  is[0m
[38;2;248;248;242m              implemented  as a builtin to allow full use of shell command line editing,[0m
[38;2;248;248;242m              file I/O, and job control mechanisms.  Often, users  will  access  it  via[0m
[38;2;248;248;242m              shell  functions  providing  a  more powerful interface; a set is provided[0m
[38;2;248;248;242m              with the zsh distribution and is described in zshzftpsys(1).  However, the[0m
[38;2;248;248;242m              zftp command is entirely usable in its own right.[0m

[38;2;248;248;242m              All commands consist of the command name zftp followed by the  name  of  a[0m
[38;2;248;248;242m              subcommand.  These are listed below.  The return status of each subcommand[0m
[38;2;248;248;242m              is  supposed  to  reflect  the success or failure of the remote operation.[0m
[38;2;248;248;242m              See a description of the variable ZFTP_VERBOSE for more information on how[0m
[38;2;248;248;242m              responses from the server may be printed.[0m

[38;2;248;248;242m   Subcommands[0m
[38;2;248;248;242m       open host[:port] [ user [ password [ account ] ] ][0m
[38;2;248;248;242m              Open a new FTP session to host, which may be the name  of  a  TCP/IP  con‐[0m
[38;2;248;248;242m              nected host or an IP number in the standard dot notation.  If the argument[0m
[38;2;248;248;242m              is  in  the  form host:port, open a connection to TCP port port instead of[0m
[38;2;248;248;242m              the standard FTP port 21.  This may be the name of a TCP service or a num‐[0m
[38;2;248;248;242m              ber:  see the description of ZFTP_PORT below for more information.[0m

[38;2;248;248;242m              If IPv6 addresses in colon format are used, the host should be  surrounded[0m
[38;2;248;248;242m              by  quoted  square  brackets  to distinguish it from the port, for example[0m
[38;2;248;248;242m              '[fe80::203:baff:fe02:8b56]'.  For consistency this is  allowed  with  all[0m
[38;2;248;248;242m              forms of host.[0m

[38;2;248;248;242m              Remaining  arguments  are passed to the login subcommand.  Note that if no[0m
[38;2;248;248;242m              arguments beyond host are supplied, open will not automatically  call  lo‐[0m
[38;2;248;248;242m              gin.   If  no  arguments at all are supplied, open will use the parameters[0m
[38;2;248;248;242m              set by the params subcommand.[0m

[38;2;248;248;242m              After a successful open, the shell variables ZFTP_HOST, ZFTP_PORT, ZFTP_IP[0m
[38;2;248;248;242m              and ZFTP_SYSTEM are available; see `Variables' below.[0m

[38;2;248;248;242m       login [ name [ password [ account ] ] ][0m
[38;2;248;248;242m       user [ name [ password [ account ] ] ][0m
[38;2;248;248;242m              Login the user name with parameters password and account.  Any of the  pa‐[0m
[38;2;248;248;242m              rameters  can  be  omitted, and will be read from standard input if needed[0m
[38;2;248;248;242m              (name is always needed).  If standard input is a terminal,  a  prompt  for[0m
[38;2;248;248;242m              each  one  will  be  printed  on  standard  error and password will not be[0m
[38;2;248;248;242m              echoed.  If any of the parameters are  not  used,  a  warning  message  is[0m
[38;2;248;248;242m              printed.[0m

[38;2;248;248;242m              After  a successful login, the shell variables ZFTP_USER, ZFTP_ACCOUNT and[0m
[38;2;248;248;242m              ZFTP_PWD are available; see `Variables' below.[0m

[38;2;248;248;242m              This command may be re-issued when a user is already logged  in,  and  the[0m
[38;2;248;248;242m              server will first be reinitialized for a new user.[0m

[38;2;248;248;242m       params [ host [ user [ password [ account ] ] ] ][0m
[38;2;248;248;242m       params -[0m
[38;2;248;248;242m              Store  the  given  parameters  for a later open command with no arguments.[0m
[38;2;248;248;242m              Only those given on the command line will be remembered.  If no  arguments[0m
[38;2;248;248;242m              are given, the parameters currently set are printed, although the password[0m
[38;2;248;248;242m              will  appear as a line of stars; the return status is one if no parameters[0m
[38;2;248;248;242m              were set, zero otherwise.[0m

[38;2;248;248;242m              Any of the parameters may be specified as a `?',  which  may  need  to  be[0m
[38;2;248;248;242m              quoted  to protect it from shell expansion.  In this case, the appropriate[0m
[38;2;248;248;242m              parameter will be read from stdin as with the login subcommand,  including[0m
[38;2;248;248;242m              special handling of password.  If the `?' is followed by a string, that is[0m
[38;2;248;248;242m              used  as  the prompt for reading the parameter instead of the default mes‐[0m
[38;2;248;248;242m              sage (any necessary punctuation and whitespace should be included  at  the[0m
[38;2;248;248;242m              end  of  the  prompt).   The  first  letter of the parameter (only) may be[0m
[38;2;248;248;242m              quoted with a `\'; hence an argument "\\$word" guarantees that the  string[0m
[38;2;248;248;242m              from  the  shell parameter $word will be treated literally, whether or not[0m
[38;2;248;248;242m              it begins with a `?'.[0m

[38;2;248;248;242m              If instead a single `-' is given, the existing  parameters,  if  any,  are[0m
[38;2;248;248;242m              deleted.   In  that case, calling open with no arguments will cause an er‐[0m
[38;2;248;248;242m              ror.[0m

[38;2;248;248;242m              The list of parameters is not deleted after a close, however  it  will  be[0m
[38;2;248;248;242m              deleted if the zsh/zftp module is unloaded.[0m

[38;2;248;248;242m              For example,[0m

[38;2;248;248;242m                     zftp params ftp.elsewhere.xx juser '?Password for juser: '[0m

[38;2;248;248;242m              will  store  the  host ftp.elsewhere.xx and the user juser and then prompt[0m
[38;2;248;248;242m              the user for the corresponding password with the given prompt.[0m

[38;2;248;248;242m       test   Test the connection; if the server has reported that  it  has  closed  the[0m
[38;2;248;248;242m              connection (maybe due to a timeout), return status 2; if no connection was[0m
[38;2;248;248;242m              open  anyway,  return status 1; else return status 0.  The test subcommand[0m
[38;2;248;248;242m              is silent, apart from messages printed by the $ZFTP_VERBOSE mechanism,  or[0m
[38;2;248;248;242m              error messages if the connection closes.  There is no network overhead for[0m
[38;2;248;248;242m              this test.[0m

[38;2;248;248;242m              The test is only supported on systems with either the select(2) or poll(2)[0m
[38;2;248;248;242m              system  calls;  otherwise  the  message  `not supported on this system' is[0m
[38;2;248;248;242m              printed instead.[0m

[38;2;248;248;242m              The test subcommand will automatically be called at the start of any other[0m
[38;2;248;248;242m              subcommand for the current session when a connection is open.[0m

[38;2;248;248;242m       cd directory[0m
[38;2;248;248;242m              Change the remote directory to directory.  Also alters the shell  variable[0m
[38;2;248;248;242m              ZFTP_PWD.[0m

[38;2;248;248;242m       cdup   Change the remote directory to the one higher in the directory tree.  Note[0m
[38;2;248;248;242m              that cd .. will also work correctly on non-UNIX systems.[0m

[38;2;248;248;242m       dir [ arg ... ][0m
[38;2;248;248;242m              Give a (verbose) listing of the remote directory.  The args are passed di‐[0m
[38;2;248;248;242m              rectly to the server. The command's behaviour is implementation dependent,[0m
[38;2;248;248;242m              but  a  UNIX  server  will typically interpret args as arguments to the ls[0m
[38;2;248;248;242m              command and with no arguments return the result of `ls -l'. The  directory[0m
[38;2;248;248;242m              is listed to standard output.[0m

[38;2;248;248;242m       ls [ arg ... ][0m
[38;2;248;248;242m              Give  a  (short) listing of the remote directory.  With no arg, produces a[0m
[38;2;248;248;242m              raw list of the files in the directory, one per line.   Otherwise,  up  to[0m
[38;2;248;248;242m              vagaries of the server implementation, behaves similar to dir.[0m

[38;2;248;248;242m       type [ type ][0m
[38;2;248;248;242m              Change  the  type  for  the transfer to type, or print the current type if[0m
[38;2;248;248;242m              type is absent.  The allowed values are `A' (ASCII), `I' (Image, i.e.  bi‐[0m
[38;2;248;248;242m              nary), or `B' (a synonym for `I').[0m

[38;2;248;248;242m              The  FTP default for a transfer is ASCII.  However, if zftp finds that the[0m
[38;2;248;248;242m              remote host is a UNIX machine  with  8-bit  byes,  it  will  automatically[0m
[38;2;248;248;242m              switch  to  using  binary  for  file transfers upon open.  This can subse‐[0m
[38;2;248;248;242m              quently be overridden.[0m

[38;2;248;248;242m              The transfer type is only passed to the remote host when a data connection[0m
[38;2;248;248;242m              is established; this command involves no network overhead.[0m

[38;2;248;248;242m       ascii  The same as type A.[0m

[38;2;248;248;242m       binary The same as type I.[0m

[38;2;248;248;242m       mode [ S | B ][0m
[38;2;248;248;242m              Set the mode type to stream (S) or block (B).  Stream mode is the default;[0m
[38;2;248;248;242m              block mode is not widely supported.[0m

[38;2;248;248;242m       remote file ...[0m
[38;2;248;248;242m       local [ file ... ][0m
[38;2;248;248;242m              Print the size and last modification time of the remote  or  local  files.[0m
[38;2;248;248;242m              If  there  is  more  than  one  item  on the list, the name of the file is[0m
[38;2;248;248;242m              printed first.  The first number is the file size, the second is the  last[0m
[38;2;248;248;242m              modification  time  of the file in the format CCYYMMDDhhmmSS consisting of[0m
[38;2;248;248;242m              year, month, date, hour, minutes and seconds in GMT.  Note that this  for‐[0m
[38;2;248;248;242m              mat,  including the length, is guaranteed, so that time strings can be di‐[0m
[38;2;248;248;242m              rectly compared via the [[ builtin's < and > operators, even if  they  are[0m
[38;2;248;248;242m              too long to be represented as integers.[0m

[38;2;248;248;242m              Not  all servers support the commands for retrieving this information.  In[0m
[38;2;248;248;242m              that case, the remote command will print nothing and return status 2, com‐[0m
[38;2;248;248;242m              pared with status 1 for a file not found.[0m

[38;2;248;248;242m              The local command (but not remote) may be used with no arguments, in which[0m
[38;2;248;248;242m              case the information comes from examining file descriptor zero.   This  is[0m
[38;2;248;248;242m              the same file as seen by a put command with no further redirection.[0m

[38;2;248;248;242m       get file ...[0m
[38;2;248;248;242m              Retrieve all files from the server, concatenating them and sending them to[0m
[38;2;248;248;242m              standard output.[0m

[38;2;248;248;242m       put file ...[0m
[38;2;248;248;242m              For each file, read a file from standard input and send that to the remote[0m
[38;2;248;248;242m              host with the given name.[0m

[38;2;248;248;242m       append file ...[0m
[38;2;248;248;242m              As  put, but if the remote file already exists, data is appended to it in‐[0m
[38;2;248;248;242m              stead of overwriting it.[0m

[38;2;248;248;242m       getat file point[0m
[38;2;248;248;242m       putat file point[0m
[38;2;248;248;242m       appendat file point[0m
[38;2;248;248;242m              Versions of get, put and append which will start the transfer at the given[0m
[38;2;248;248;242m              point in the remote file.  This is useful for appending to  an  incomplete[0m
[38;2;248;248;242m              local  file.  However, note that this ability is not universally supported[0m
[38;2;248;248;242m              by servers (and is not quite the behaviour specified by the standard).[0m

[38;2;248;248;242m       delete file ...[0m
[38;2;248;248;242m              Delete the list of files on the server.[0m

[38;2;248;248;242m       mkdir directory[0m
[38;2;248;248;242m              Create a new directory directory on the server.[0m

[38;2;248;248;242m       rmdir directory[0m
[38;2;248;248;242m              Delete the directory directory  on the server.[0m

[38;2;248;248;242m       rename old-name new-name[0m
[38;2;248;248;242m              Rename file old-name to new-name on the server.[0m

[38;2;248;248;242m       site arg ...[0m
[38;2;248;248;242m              Send a host-specific command to the server.  You will probably  only  need[0m
[38;2;248;248;242m              this if instructed by the server to use it.[0m

[38;2;248;248;242m       quote arg ...[0m
[38;2;248;248;242m              Send  the  raw FTP command sequence to the server.  You should be familiar[0m
[38;2;248;248;242m              with the FTP command set as defined in RFC959 before doing  this.   Useful[0m
[38;2;248;248;242m              commands may include STAT and HELP.  Note also the mechanism for returning[0m
[38;2;248;248;242m              messages  as  described for the variable ZFTP_VERBOSE below, in particular[0m
[38;2;248;248;242m              that all messages from the control connection are sent to standard error.[0m

[38;2;248;248;242m       close[0m
[38;2;248;248;242m       quit   Close the current data  connection.   This  unsets  the  shell  parameters[0m
[38;2;248;248;242m              ZFTP_HOST,   ZFTP_PORT,  ZFTP_IP,  ZFTP_SYSTEM,  ZFTP_USER,  ZFTP_ACCOUNT,[0m
[38;2;248;248;242m              ZFTP_PWD, ZFTP_TYPE and ZFTP_MODE.[0m

[38;2;248;248;242m       session [ sessname ][0m
[38;2;248;248;242m              Allows multiple FTP sessions to be used at once.  The name of the  session[0m
[38;2;248;248;242m              is  an  arbitrary string of characters; the default session is called `de‐[0m
[38;2;248;248;242m              fault'.  If this command is called without an argument, it will  list  all[0m
[38;2;248;248;242m              the  current  sessions; with an argument, it will either switch to the ex‐[0m
[38;2;248;248;242m              isting session called sessname, or create a new session of that name.[0m

[38;2;248;248;242m              Each session remembers the status of the connection, the  set  of  connec‐[0m
[38;2;248;248;242m              tion-specific  shell  parameters (the same set as are unset when a connec‐[0m
[38;2;248;248;242m              tion closes, as given in the description of close), and any  user  parame‐[0m
[38;2;248;248;242m              ters specified with the params subcommand.  Changing to a previous session[0m
[38;2;248;248;242m              restores  those  values; changing to a new session initialises them in the[0m
[38;2;248;248;242m              same way as if zftp had just been loaded.  The name of the current session[0m
[38;2;248;248;242m              is given by the parameter ZFTP_SESSION.[0m

[38;2;248;248;242m       rmsession [ sessname ][0m
[38;2;248;248;242m              Delete a session; if a name is not given, the current session is  deleted.[0m
[38;2;248;248;242m              If  the  current session is deleted, the earliest existing session becomes[0m
[38;2;248;248;242m              the new current session, otherwise the current session is not changed.  If[0m
[38;2;248;248;242m              the session being deleted is the only one, a new session called  `default'[0m
[38;2;248;248;242m              is  created  and becomes the current session; note that this is a new ses‐[0m
[38;2;248;248;242m              sion even if the session being deleted is also  called  `default'.  It  is[0m
[38;2;248;248;242m              recommended  that  sessions not be deleted while background commands which[0m
[38;2;248;248;242m              use zftp are still active.[0m

[38;2;248;248;242m   Parameters[0m
[38;2;248;248;242m       The following shell parameters are used by zftp.  Currently none of them are spe‐[0m
[38;2;248;248;242m       cial.[0m

[38;2;248;248;242m       ZFTP_TMOUT[0m
[38;2;248;248;242m              Integer.  The time in seconds to wait for a network operation to  complete[0m
[38;2;248;248;242m              before  returning an error.  If this is not set when the module is loaded,[0m
[38;2;248;248;242m              it will be given the default value 60.  A value of zero  turns  off  time‐[0m
[38;2;248;248;242m              outs.   If  a  timeout occurs on the control connection it will be closed.[0m
[38;2;248;248;242m              Use a larger value if this occurs too frequently.[0m

[38;2;248;248;242m       ZFTP_IP[0m
[38;2;248;248;242m              Readonly.  The IP address of the current connection in dot notation.[0m

[38;2;248;248;242m       ZFTP_HOST[0m
[38;2;248;248;242m              Readonly.  The hostname of the current remote server.   If  the  host  was[0m
[38;2;248;248;242m              opened  as  an  IP number, ZFTP_HOST contains that instead; this saves the[0m
[38;2;248;248;242m              overhead for a name lookup, as IP numbers are most commonly  used  when  a[0m
[38;2;248;248;242m              nameserver is unavailable.[0m

[38;2;248;248;242m       ZFTP_PORT[0m
[38;2;248;248;242m              Readonly.   The  number  of the remote TCP port to which the connection is[0m
[38;2;248;248;242m              open (even if the port was originally specified as a named service).  Usu‐[0m
[38;2;248;248;242m              ally this is the standard FTP port, 21.[0m

[38;2;248;248;242m              In the unlikely event that your system does not have the appropriate  con‐[0m
[38;2;248;248;242m              version  functions, this appears in network byte order.  If your system is[0m
[38;2;248;248;242m              little-endian, the port then consists of two swapped bytes and  the  stan‐[0m
[38;2;248;248;242m              dard port will be reported as 5376.  In that case, numeric ports passed to[0m
[38;2;248;248;242m              zftp open will also need to be in this format.[0m

[38;2;248;248;242m       ZFTP_SYSTEM[0m
[38;2;248;248;242m              Readonly.  The system type string returned by the server in response to an[0m
[38;2;248;248;242m              FTP  SYST  request.  The most interesting case is a string beginning "UNIX[0m
[38;2;248;248;242m              Type: L8", which ensures maximum compatibility with a local UNIX host.[0m

[38;2;248;248;242m       ZFTP_TYPE[0m
[38;2;248;248;242m              Readonly.  The type to be used for data transfers ,  either  `A'  or  `I'.[0m
[38;2;248;248;242m              Use the type subcommand to change this.[0m

[38;2;248;248;242m       ZFTP_USER[0m
[38;2;248;248;242m              Readonly.  The username currently logged in, if any.[0m

[38;2;248;248;242m       ZFTP_ACCOUNT[0m
[38;2;248;248;242m              Readonly.   The account name of the current user, if any.  Most servers do[0m
[38;2;248;248;242m              not require an account name.[0m

[38;2;248;248;242m       ZFTP_PWD[0m
[38;2;248;248;242m              Readonly.  The current directory on the server.[0m

[38;2;248;248;242m       ZFTP_CODE[0m
[38;2;248;248;242m              Readonly.  The three digit code of the last FTP reply from the server as a[0m
[38;2;248;248;242m              string.  This can still be read after the connection is closed, and is not[0m
[38;2;248;248;242m              changed when the current session changes.[0m

[38;2;248;248;242m       ZFTP_REPLY[0m
[38;2;248;248;242m              Readonly.  The last line of the last reply sent by the server.   This  can[0m
[38;2;248;248;242m              still  be read after the connection is closed, and is not changed when the[0m
[38;2;248;248;242m              current session changes.[0m

[38;2;248;248;242m       ZFTP_SESSION[0m
[38;2;248;248;242m              Readonly.  The name of the current FTP session; see the description of the[0m
[38;2;248;248;242m              session subcommand.[0m

[38;2;248;248;242m       ZFTP_PREFS[0m
[38;2;248;248;242m              A string of preferences for altering aspects of  zftp's  behaviour.   Each[0m
[38;2;248;248;242m              preference is a single character.  The following are defined:[0m

[38;2;248;248;242m              P      Passive:   attempt  to  make the remote server initiate data trans‐[0m
[38;2;248;248;242m                     fers.  This is slightly more efficient than sendport mode.  If  the[0m
[38;2;248;248;242m                     letter S occurs later in the string, zftp will use sendport mode if[0m
[38;2;248;248;242m                     passive mode is not available.[0m

[38;2;248;248;242m              S      Sendport:  initiate transfers by the FTP PORT command.  If this oc‐[0m
[38;2;248;248;242m                     curs  before  any  P  in the string, passive mode will never be at‐[0m
[38;2;248;248;242m                     tempted.[0m

[38;2;248;248;242m              D      Dumb:  use only the bare minimum of FTP  commands.   This  prevents[0m
[38;2;248;248;242m                     the  variables  ZFTP_SYSTEM  and  ZFTP_PWD from being set, and will[0m
[38;2;248;248;242m                     mean all  connections  default  to  ASCII  type.   It  may  prevent[0m
[38;2;248;248;242m                     ZFTP_SIZE  from  being set during a transfer if the server does not[0m
[38;2;248;248;242m                     send it anyway (many servers do).[0m

[38;2;248;248;242m              If ZFTP_PREFS is not set when zftp is loaded, it will be set to a  default[0m
[38;2;248;248;242m              of  `PS', i.e. use passive mode if available, otherwise fall back to send‐[0m
[38;2;248;248;242m              port mode.[0m

[38;2;248;248;242m       ZFTP_VERBOSE[0m
[38;2;248;248;242m              A string of digits between 0 and 5 inclusive, specifying  which  responses[0m
[38;2;248;248;242m              from  the  server  should be printed.  All responses go to standard error.[0m
[38;2;248;248;242m              If any of the numbers 1 to 5 appear in the string, raw responses from  the[0m
[38;2;248;248;242m              server with reply codes beginning with that digit will be printed to stan‐[0m
[38;2;248;248;242m              dard  error.   The first digit of the three digit reply code is defined by[0m
[38;2;248;248;242m              RFC959 to correspond to:[0m

[38;2;248;248;242m              1.     A positive preliminary reply.[0m

[38;2;248;248;242m              2.     A positive completion reply.[0m

[38;2;248;248;242m              3.     A positive intermediate reply.[0m

[38;2;248;248;242m              4.     A transient negative completion reply.[0m

[38;2;248;248;242m              5.     A permanent negative completion reply.[0m

[38;2;248;248;242m              It should be noted that, for  unknown  reasons,  the  reply  `Service  not[0m
[38;2;248;248;242m              available',  which  forces  termination  of a connection, is classified as[0m
[38;2;248;248;242m              421, i.e. `transient negative', an interesting interpretation of the  word[0m
[38;2;248;248;242m              `transient'.[0m

[38;2;248;248;242m              The  code 0 is special:  it indicates that all but the last line of multi‐[0m
[38;2;248;248;242m              line replies read from the server will be printed to standard error  in  a[0m
[38;2;248;248;242m              processed  format.   By convention, servers use this mechanism for sending[0m
[38;2;248;248;242m              information for the user to read.   The  appropriate  reply  code,  if  it[0m
[38;2;248;248;242m              matches the same response, takes priority.[0m

[38;2;248;248;242m              If  ZFTP_VERBOSE is not set when zftp is loaded, it will be set to the de‐[0m
[38;2;248;248;242m              fault value 450, i.e., messages destined for the user and all errors  will[0m
[38;2;248;248;242m              be  printed.  A null string is valid and specifies that no messages should[0m
[38;2;248;248;242m              be printed.[0m

[38;2;248;248;242m   Functions[0m
[38;2;248;248;242m       zftp_chpwd[0m
[38;2;248;248;242m              If this function is set by the user, it is called every time the directory[0m
[38;2;248;248;242m              changes on the server, including when a user is logged in, or when a  con‐[0m
[38;2;248;248;242m              nection  is  closed.  In the last case, $ZFTP_PWD will be unset; otherwise[0m
[38;2;248;248;242m              it will reflect the new directory.[0m

[38;2;248;248;242m       zftp_progress[0m
[38;2;248;248;242m              If this function is set by the user, it will be called during a  get,  put[0m
[38;2;248;248;242m              or  append  operation each time sufficient data has been received from the[0m
[38;2;248;248;242m              host.  During a get, the data is sent to standard output, so it  is  vital[0m
[38;2;248;248;242m              that  this function should write to standard error or directly to the ter‐[0m
[38;2;248;248;242m              minal, not to standard output.[0m

[38;2;248;248;242m              When it is called with a transfer in progress,  the  following  additional[0m
[38;2;248;248;242m              shell parameters are set:[0m

[38;2;248;248;242m              ZFTP_FILE[0m
[38;2;248;248;242m                     The name of the remote file being transferred from or to.[0m

[38;2;248;248;242m              ZFTP_TRANSFER[0m
[38;2;248;248;242m                     A G for a get operation and a P for a put operation.[0m

[38;2;248;248;242m              ZFTP_SIZE[0m
[38;2;248;248;242m                     The  total size of the complete file being transferred: the same as[0m
[38;2;248;248;242m                     the first value provided by the remote and local subcommands for  a[0m
[38;2;248;248;242m                     particular  file.  If the server cannot supply this value for a re‐[0m
[38;2;248;248;242m                     mote file being retrieved, it will not be set.  If input is from  a[0m
[38;2;248;248;242m                     pipe  the  value  may  be incorrect and correspond simply to a full[0m
[38;2;248;248;242m                     pipe buffer.[0m

[38;2;248;248;242m              ZFTP_COUNT[0m
[38;2;248;248;242m                     The amount of data so far transferred; a number  between  zero  and[0m
[38;2;248;248;242m                     $ZFTP_SIZE, if that is set.  This number is always available.[0m

[38;2;248;248;242m              The  function is initially called with ZFTP_TRANSFER set appropriately and[0m
[38;2;248;248;242m              ZFTP_COUNT set to zero.  After the transfer is finished, the function will[0m
[38;2;248;248;242m              be called one more time with ZFTP_TRANSFER set to GF or  PF,  in  case  it[0m
[38;2;248;248;242m              wishes to tidy up.  It is otherwise never called twice with the same value[0m
[38;2;248;248;242m              of ZFTP_COUNT.[0m

[38;2;248;248;242m              Sometimes  the  progress meter may cause disruption.  It is up to the user[0m
[38;2;248;248;242m              to decide whether the function should be defined  and  to  use  unfunction[0m
[38;2;248;248;242m              when necessary.[0m

[38;2;248;248;242m   Problems[0m
[38;2;248;248;242m       A  connection may not be opened in the left hand side of a pipe as this occurs in[0m
[38;2;248;248;242m       a subshell and the file information is not updated in the  main  shell.   In  the[0m
[38;2;248;248;242m       case  of type or mode changes or closing the connection in a subshell, the infor‐[0m
[38;2;248;248;242m       mation is returned but variables are not updated until the  next  call  to  zftp.[0m
[38;2;248;248;242m       Other  status  changes in subshells will not be reflected by changes to the vari‐[0m
[38;2;248;248;242m       ables (but should be otherwise harmless).[0m

[38;2;248;248;242m       Deleting sessions while a zftp command is active in the background can have unex‐[0m
[38;2;248;248;242m       pected effects, even if it does not use the session being deleted.  This  is  be‐[0m
[38;2;248;248;242m       cause  all  shell subprocesses share information on the state of all connections,[0m
[38;2;248;248;242m       and deleting a session changes the ordering of that information.[0m

[38;2;248;248;242m       On some operating systems, the control connection is not valid after a fork(), so[0m
[38;2;248;248;242m       that operations in subshells, on the left hand side of  a  pipeline,  or  in  the[0m
[38;2;248;248;242m       background  are not possible, as they should be.  This is presumably a bug in the[0m
[38;2;248;248;242m       operating system.[0m

[38;2;248;248;242mTHE ZSH/ZLE MODULE[0m
[38;2;248;248;242m       The zsh/zle module contains the Zsh Line Editor.  See zshzle(1).[0m

[38;2;248;248;242mTHE ZSH/ZLEPARAMETER MODULE[0m
[38;2;248;248;242m       The zsh/zleparameter module defines two special parameters that can  be  used  to[0m
[38;2;248;248;242m       access internal information of the Zsh Line Editor (see zshzle(1)).[0m

[38;2;248;248;242m       keymaps[0m
[38;2;248;248;242m              This array contains the names of the keymaps currently defined.[0m

[38;2;248;248;242m       widgets[0m
[38;2;248;248;242m              This associative array contains one entry per widget. The name of the wid‐[0m
[38;2;248;248;242m              get is the key and the value gives information about the widget. It is ei‐[0m
[38;2;248;248;242m              ther[0m
[38;2;248;248;242m                the string `builtin' for builtin widgets,[0m
[38;2;248;248;242m                a string of the form `user:name' for user-defined widgets,[0m
[38;2;248;248;242m                  where name is the name of the shell function implementing the widget,[0m
[38;2;248;248;242m                a string of the form `completion:type:name'[0m
[38;2;248;248;242m                  for completion widgets,[0m
[38;2;248;248;242m                or a null value if the widget is not yet fully defined.  In the penulti‐[0m
[38;2;248;248;242m              mate  case,  type  is the name of the builtin widget the completion widget[0m
[38;2;248;248;242m              imitates in its behavior and name is the name of the shell function imple‐[0m
[38;2;248;248;242m              menting the completion widget.[0m

[38;2;248;248;242mTHE ZSH/ZPROF MODULE[0m
[38;2;248;248;242m       When loaded, the zsh/zprof causes shell functions to be profiled.  The  profiling[0m
[38;2;248;248;242m       results  can  be  obtained  with the zprof builtin command made available by this[0m
[38;2;248;248;242m       module.  There is no way to turn profiling off other than unloading the module.[0m

[38;2;248;248;242m       zprof [ -c ][0m
[38;2;248;248;242m              Without the -c option, zprof lists profiling results to  standard  output.[0m
[38;2;248;248;242m              The format is comparable to that of commands like gprof.[0m

[38;2;248;248;242m              At  the  top  there is a summary listing all functions that were called at[0m
[38;2;248;248;242m              least once.  This summary is sorted in decreasing order of the  amount  of[0m
[38;2;248;248;242m              time  spent  in each.  The lines contain the number of the function in or‐[0m
[38;2;248;248;242m              der, which is used in other parts of the list  in  suffixes  of  the  form[0m
[38;2;248;248;242m              `[num]',  then  the  number of calls made to the function.  The next three[0m
[38;2;248;248;242m              columns list the time in milliseconds spent in the function  and  its  de‐[0m
[38;2;248;248;242m              scendants,  the average time in milliseconds spent in the function and its[0m
[38;2;248;248;242m              descendants per call and the percentage of time spent in all  shell  func‐[0m
[38;2;248;248;242m              tions  used  in  this  function  and its descendants.  The following three[0m
[38;2;248;248;242m              columns give the same information, but counting only the time spent in the[0m
[38;2;248;248;242m              function itself.  The final column shows the name of the function.[0m

[38;2;248;248;242m              After the summary, detailed information about every function that was  in‐[0m
[38;2;248;248;242m              voked is listed, sorted in decreasing order of the amount of time spent in[0m
[38;2;248;248;242m              each  function and its descendants.  Each of these entries consists of de‐[0m
[38;2;248;248;242m              scriptions for the functions that called the function described, the func‐[0m
[38;2;248;248;242m              tion itself, and the functions that were called from it.  The  description[0m
[38;2;248;248;242m              for  the  function itself has the same format as in the summary (and shows[0m
[38;2;248;248;242m              the same information).  The other lines don't show the number of the func‐[0m
[38;2;248;248;242m              tion at the beginning and have their function named indented  to  make  it[0m
[38;2;248;248;242m              easier  to distinguish the line showing the function described in the sec‐[0m
[38;2;248;248;242m              tion from the surrounding lines.[0m

[38;2;248;248;242m              The information shown in this case is almost the same as in  the  summary,[0m
[38;2;248;248;242m              but only refers to the call hierarchy being displayed.  For example, for a[0m
[38;2;248;248;242m              calling  function the column showing the total running time lists the time[0m
[38;2;248;248;242m              spent in the described function and its descendants  only  for  the  times[0m
[38;2;248;248;242m              when it was called from that particular calling function.  Likewise, for a[0m
[38;2;248;248;242m              called  function,  this  columns  lists the total time spent in the called[0m
[38;2;248;248;242m              function and its descendants only for the times when it  was  called  from[0m
[38;2;248;248;242m              the function described.[0m

[38;2;248;248;242m              Also  in  this  case, the column showing the number of calls to a function[0m
[38;2;248;248;242m              also shows a slash and then the total number of invocations  made  to  the[0m
[38;2;248;248;242m              called function.[0m

[38;2;248;248;242m              As long as the zsh/zprof module is loaded, profiling will be done and mul‐[0m
[38;2;248;248;242m              tiple  invocations  of  the  zprof builtin command will show the times and[0m
[38;2;248;248;242m              numbers of calls since the module was loaded.  With  the  -c  option,  the[0m
[38;2;248;248;242m              zprof  builtin  command will reset its internal counters and will not show[0m
[38;2;248;248;242m              the listing.[0m

[38;2;248;248;242mTHE ZSH/ZPTY MODULE[0m
[38;2;248;248;242m       The zsh/zpty module offers one builtin:[0m

[38;2;248;248;242m       zpty [ -e ] [ -b ] name [ arg ... ][0m
[38;2;248;248;242m              The arguments following name are concatenated with  spaces  between,  then[0m
[38;2;248;248;242m              executed as a command, as if passed to the eval builtin.  The command runs[0m
[38;2;248;248;242m              under  a  newly  assigned pseudo-terminal; this is useful for running com‐[0m
[38;2;248;248;242m              mands non-interactively which expect an interactive environment.  The name[0m
[38;2;248;248;242m              is not part of the command, but is used to refer to this command in  later[0m
[38;2;248;248;242m              calls to zpty.[0m

[38;2;248;248;242m              With the -e option, the pseudo-terminal is set up so that input characters[0m
[38;2;248;248;242m              are echoed.[0m

[38;2;248;248;242m              With  the -b option, input to and output from the pseudo-terminal are made[0m
[38;2;248;248;242m              non-blocking.[0m

[38;2;248;248;242m              The shell parameter REPLY is set to the file descriptor  assigned  to  the[0m
[38;2;248;248;242m              master  side of the pseudo-terminal.  This allows the terminal to be moni‐[0m
[38;2;248;248;242m              tored with ZLE descriptor handlers (see  zshzle(1))  or  manipulated  with[0m
[38;2;248;248;242m              sysread  and syswrite (see THE ZSH/SYSTEM MODULE in zshmodules(1)).  Warn‐[0m
[38;2;248;248;242m              ing: Use of sysread and syswrite is not recommended; use zpty -r and  zpty[0m
[38;2;248;248;242m              -w unless you know exactly what you are doing.[0m

[38;2;248;248;242m       zpty -d [ name ... ][0m
[38;2;248;248;242m              The second form, with the -d option, is used to delete commands previously[0m
[38;2;248;248;242m              started,  by  supplying  a  list of their names.  If no name is given, all[0m
[38;2;248;248;242m              commands are deleted.  Deleting a command causes the HUP signal to be sent[0m
[38;2;248;248;242m              to the corresponding process.[0m

[38;2;248;248;242m       zpty -w [ -n ] name [ string ... ][0m
[38;2;248;248;242m              The -w option can be used to send the to command name the given strings as[0m
[38;2;248;248;242m              input (separated by spaces).  If the -n option is not given, a newline  is[0m
[38;2;248;248;242m              added at the end.[0m

[38;2;248;248;242m              If  no string is provided, the standard input is copied to the pseudo-ter‐[0m
[38;2;248;248;242m              minal; this may stop before copying the full input if the  pseudo-terminal[0m
[38;2;248;248;242m              is  non-blocking.   The exact input is always copied: the -n option is not[0m
[38;2;248;248;242m              applied.[0m

[38;2;248;248;242m              Note that the command under the pseudo-terminal sees this input as  if  it[0m
[38;2;248;248;242m              were  typed,  so beware when sending special tty driver characters such as[0m
[38;2;248;248;242m              word-erase, line-kill, and end-of-file.[0m

[38;2;248;248;242m       zpty -r [ -mt ] name [ param [ pattern ] ][0m
[38;2;248;248;242m              The -r option can be used to read the output of the  command  name.   With[0m
[38;2;248;248;242m              only  a  name  argument, the output read is copied to the standard output.[0m
[38;2;248;248;242m              Unless the pseudo-terminal is non-blocking, copying  continues  until  the[0m
[38;2;248;248;242m              command  under  the pseudo-terminal exits; when non-blocking, only as much[0m
[38;2;248;248;242m              output as is immediately available is copied.  The return status  is  zero[0m
[38;2;248;248;242m              if any output is copied.[0m

[38;2;248;248;242m              When  also  given a param argument, at most one line is read and stored in[0m
[38;2;248;248;242m              the parameter named param.  Less than a full  line  may  be  read  if  the[0m
[38;2;248;248;242m              pseudo-terminal  is  non-blocking.   The return status is zero if at least[0m
[38;2;248;248;242m              one character is stored in param.[0m

[38;2;248;248;242m              If a pattern is given as well, output is read until the whole string  read[0m
[38;2;248;248;242m              matches  the pattern, even in the non-blocking case.  The return status is[0m
[38;2;248;248;242m              zero if the string read matches the pattern, or if the command has  exited[0m
[38;2;248;248;242m              but  at  least  one  character  could  still be read.  If the option -m is[0m
[38;2;248;248;242m              present, the return status is zero only if the  pattern  matches.   As  of[0m
[38;2;248;248;242m              this  writing,  a  maximum  of one megabyte of output can be consumed this[0m
[38;2;248;248;242m              way; if a full megabyte is read without matching the pattern,  the  return[0m
[38;2;248;248;242m              status is non-zero.[0m

[38;2;248;248;242m              In  all cases, the return status is non-zero if nothing could be read, and[0m
[38;2;248;248;242m              is 2 if this is because the command has finished.[0m

[38;2;248;248;242m              If the -r option is combined with the -t option, zpty tests whether output[0m
[38;2;248;248;242m              is available before trying to read.  If no output is available, zpty imme‐[0m
[38;2;248;248;242m              diately returns the status 1.  When used with a pattern, the behaviour  on[0m
[38;2;248;248;242m              a failed poll is similar to when the command has exited:  the return value[0m
[38;2;248;248;242m              is  zero if at least one character could still be read even if the pattern[0m
[38;2;248;248;242m              failed to match.[0m

[38;2;248;248;242m       zpty -t name[0m
[38;2;248;248;242m              The -t option without the -r option can be used to test whether  the  com‐[0m
[38;2;248;248;242m              mand  name  is  still running.  It returns a zero status if the command is[0m
[38;2;248;248;242m              running and a non-zero value otherwise.[0m

[38;2;248;248;242m       zpty [ -L ][0m
[38;2;248;248;242m              The last form, without any arguments, is used to list  the  commands  cur‐[0m
[38;2;248;248;242m              rently  defined.   If  the -L option is given, this is done in the form of[0m
[38;2;248;248;242m              calls to the zpty builtin.[0m

[38;2;248;248;242mTHE ZSH/ZSELECT MODULE[0m
[38;2;248;248;242m       The zsh/zselect module makes available one builtin command:[0m

[38;2;248;248;242m       zselect [ -rwe ] [ -t timeout ] [ -a array ] [ -A assoc ] [ fd ... ][0m
[38;2;248;248;242m              The zselect builtin is a front-end to  the  `select'  system  call,  which[0m
[38;2;248;248;242m              blocks  until a file descriptor is ready for reading or writing, or has an[0m
[38;2;248;248;242m              error condition, with an optional timeout.  If this is  not  available  on[0m
[38;2;248;248;242m              your  system,  the  command  prints  an error message and returns status 2[0m
[38;2;248;248;242m              (normal errors return status 1).  For more information, see your  system's[0m
[38;2;248;248;242m              documentation  for  select(3).  Note there is no connection with the shell[0m
[38;2;248;248;242m              builtin of the same name.[0m

[38;2;248;248;242m              Arguments and options may be intermingled in any order.  Non-option  argu‐[0m
[38;2;248;248;242m              ments  are  file descriptors, which must be decimal integers.  By default,[0m
[38;2;248;248;242m              file descriptors are to be tested for reading, i.e.  zselect  will  return[0m
[38;2;248;248;242m              when  data  is available to be read from the file descriptor, or more pre‐[0m
[38;2;248;248;242m              cisely, when a read operation from the file  descriptor  will  not  block.[0m
[38;2;248;248;242m              After  a  -r,  -w  and -e, the given file descriptors are to be tested for[0m
[38;2;248;248;242m              reading, writing, or error conditions.  These  options  and  an  arbitrary[0m
[38;2;248;248;242m              list of file descriptors may be given in any order.[0m

[38;2;248;248;242m              (The  presence of an `error condition' is not well defined in the documen‐[0m
[38;2;248;248;242m              tation for many implementations of the select system call.   According  to[0m
[38;2;248;248;242m              recent versions of the POSIX specification, it is really an exception con‐[0m
[38;2;248;248;242m              dition, of which the only standard example is out-of-band data received on[0m
[38;2;248;248;242m              a socket.  So zsh users are unlikely to find the -e option useful.)[0m

[38;2;248;248;242m              The  option  `-t  timeout'  specifies a timeout in hundredths of a second.[0m
[38;2;248;248;242m              This may be zero, in which case the file descriptors will simply be polled[0m
[38;2;248;248;242m              and zselect will return immediately.  It is possible to call zselect  with[0m
[38;2;248;248;242m              no  file descriptors and a non-zero timeout for use as a finer-grained re‐[0m
[38;2;248;248;242m              placement for `sleep'; note, however, the return status is always 1 for  a[0m
[38;2;248;248;242m              timeout.[0m

[38;2;248;248;242m              The  option  `-a array' indicates that array should be set to indicate the[0m
[38;2;248;248;242m              file descriptor(s) which are ready.  If the option is not given, the array[0m
[38;2;248;248;242m              reply will be used for this purpose.  The array will contain a string sim‐[0m
[38;2;248;248;242m              ilar to the arguments for zselect.  For example,[0m

[38;2;248;248;242m                     zselect -t 0 -r 0 -w 1[0m

[38;2;248;248;242m              might return immediately with status 0 and $reply containing `-r 0  -w  1'[0m
[38;2;248;248;242m              to show that both file descriptors are ready for the requested operations.[0m

[38;2;248;248;242m              The option `-A assoc' indicates that the associative array assoc should be[0m
[38;2;248;248;242m              set to indicate the file descriptor(s) which are ready.  This option over‐[0m
[38;2;248;248;242m              rides  the  option  -a, nor will reply be modified.  The keys of assoc are[0m
[38;2;248;248;242m              the file descriptors, and the corresponding values are any of the  charac‐[0m
[38;2;248;248;242m              ters `rwe' to indicate the condition.[0m

[38;2;248;248;242m              The  command returns status 0 if some file descriptors are ready for read‐[0m
[38;2;248;248;242m              ing.  If the operation timed out, or a timeout of 0 was given and no  file[0m
[38;2;248;248;242m              descriptors were ready, or there was an error, it returns status 1 and the[0m
[38;2;248;248;242m              array will not be set (nor modified in any way).  If there was an error in[0m
[38;2;248;248;242m              the select operation the appropriate error message is printed.[0m

[38;2;248;248;242mTHE ZSH/ZUTIL MODULE[0m
[38;2;248;248;242m       The zsh/zutil module only adds some builtins:[0m

[38;2;248;248;242m       zstyle [ -L [ metapattern [ style ] ] ][0m
[38;2;248;248;242m       zstyle [ -e | - | -- ] pattern style string ...[0m
[38;2;248;248;242m       zstyle -d [ pattern [ style ... ] ][0m
[38;2;248;248;242m       zstyle -g name [ pattern [ style ] ][0m
[38;2;248;248;242m       zstyle -{a|b|s} context style name [ sep ][0m
[38;2;248;248;242m       zstyle -{T|t} context style [ string ... ][0m
[38;2;248;248;242m       zstyle -m context style pattern[0m
[38;2;248;248;242m              This  builtin  command  is  used  to define and lookup styles.  Styles are[0m
[38;2;248;248;242m              pairs of names and values, where the  values  consist  of  any  number  of[0m
[38;2;248;248;242m              strings.   They  are  stored  together with patterns and lookup is done by[0m
[38;2;248;248;242m              giving a string, called the `context', which is matched against  the  pat‐[0m
[38;2;248;248;242m              terns.   The  definition stored for the most specific pattern that matches[0m
[38;2;248;248;242m              will be returned.[0m

[38;2;248;248;242m              A pattern is considered to be more specific than another  if  it  contains[0m
[38;2;248;248;242m              more  components  (substrings  separated by colons) or if the patterns for[0m
[38;2;248;248;242m              the components are more specific, where simple strings are  considered  to[0m
[38;2;248;248;242m              be  more  specific than patterns and complex patterns are considered to be[0m
[38;2;248;248;242m              more specific than the pattern `*'.  A `*' in the pattern will match  zero[0m
[38;2;248;248;242m              or  more  characters  in  the context; colons are not treated specially in[0m
[38;2;248;248;242m              this regard.  If two patterns are equally specific, the tie is  broken  in[0m
[38;2;248;248;242m              favour of the pattern that was defined first.[0m

[38;2;248;248;242m              Example[0m

[38;2;248;248;242m              For example, a fictional `weather' plugin might state in its documentation[0m
[38;2;248;248;242m              that   it   looks   up   the   preferred-precipitation   style  under  the[0m
[38;2;248;248;242m              `:weather:continent:day-of-the-week:phase-of-the-moon' context.  According[0m
[38;2;248;248;242m              to this, you might set the following in your zshrc:[0m

[38;2;248;248;242m                     zstyle ':weather:europe:*' preferred-precipitation rain[0m
[38;2;248;248;242m                     zstyle ':weather:*:Sunday:*' preferred-precipitation snow[0m

[38;2;248;248;242m              Then the plugin would run under the hood a command such as[0m

[38;2;248;248;242m                     zstyle -s ":weather:${continent}:${day_of_week}:${moon_phase}" preferred-precipitation REPLY[0m

[38;2;248;248;242m              in order to retrieve your preference into the scalar variable $REPLY.   On[0m
[38;2;248;248;242m              Sundays  $REPLY  would  be  set  to  `snow';  in Europe it would be set to[0m
[38;2;248;248;242m              `rain'; and on Sundays in Europe it would be set to `snow' again,  because[0m
[38;2;248;248;242m              the  patterns `:weather:europe:*' and `:weather:*:Sunday:*' both match the[0m
[38;2;248;248;242m              context argument to zstyle -s, are equally specific,  and  the  latter  is[0m
[38;2;248;248;242m              more specific (because it has more colon-separated components).[0m

[38;2;248;248;242m              Usage[0m

[38;2;248;248;242m              The forms that operate on patterns are the following.[0m

[38;2;248;248;242m              zstyle [ -L [ metapattern [ style ] ] ][0m
[38;2;248;248;242m                     Without  arguments,  lists  style definitions.  Styles are shown in[0m
[38;2;248;248;242m                     alphabetic order and patterns are shown in the  order  zstyle  will[0m
[38;2;248;248;242m                     test them.[0m

[38;2;248;248;242m                     If  the -L option is given, listing is done in the form of calls to[0m
[38;2;248;248;242m                     zstyle.  The optional first argument,  metapattern,  is  a  pattern[0m
[38;2;248;248;242m                     which  will  be matched against the string supplied as pattern when[0m
[38;2;248;248;242m                     the style was defined.  Note: this means, for example,  `zstyle  -L[0m
[38;2;248;248;242m                     ":completion:*"'  will  match any supplied pattern beginning `:com‐[0m
[38;2;248;248;242m                     pletion:', not just ":completion:*":  use ':completion:\*' to match[0m
[38;2;248;248;242m                     that.  The optional second argument limits the output to a specific[0m
[38;2;248;248;242m                     style (not a pattern).  -L is not compatible  with  any  other  op‐[0m
[38;2;248;248;242m                     tions.[0m

[38;2;248;248;242m              zstyle [ - | -- | -e ] pattern style string ...[0m
[38;2;248;248;242m                     Defines  the  given  style  for the pattern with the strings as the[0m
[38;2;248;248;242m                     value.  If the -e option is given, the strings will be concatenated[0m
[38;2;248;248;242m                     (separated by spaces) and the resulting string  will  be  evaluated[0m
[38;2;248;248;242m                     (in  the  same  way as it is done by the eval builtin command) when[0m
[38;2;248;248;242m                     the style is looked up.  In this case the parameter `reply' must be[0m
[38;2;248;248;242m                     assigned to set the strings returned after the evaluation.   Before[0m
[38;2;248;248;242m                     evaluating  the value, reply is unset, and if it is still unset af‐[0m
[38;2;248;248;242m                     ter the evaluation, the style is treated as if it were not set.[0m

[38;2;248;248;242m              zstyle -d [ pattern [ style ... ] ][0m
[38;2;248;248;242m                     Delete style definitions. Without  arguments  all  definitions  are[0m
[38;2;248;248;242m                     deleted,  with  a  pattern  all  definitions  for  that pattern are[0m
[38;2;248;248;242m                     deleted and if any styles are given, then  only  those  styles  are[0m
[38;2;248;248;242m                     deleted for the pattern.[0m

[38;2;248;248;242m              zstyle -g name [ pattern [ style ] ][0m
[38;2;248;248;242m                     Retrieve a style definition. The name is used as the name of an ar‐[0m
[38;2;248;248;242m                     ray in which the results are stored. Without any further arguments,[0m
[38;2;248;248;242m                     all  patterns  defined  are returned. With a pattern the styles de‐[0m
[38;2;248;248;242m                     fined for that pattern are returned and with both a pattern  and  a[0m
[38;2;248;248;242m                     style, the value strings of that combination is returned.[0m

[38;2;248;248;242m              The other forms can be used to look up or test styles for a given context.[0m

[38;2;248;248;242m              zstyle -s context style name [ sep ][0m
[38;2;248;248;242m                     The  parameter name is set to the value of the style interpreted as[0m
[38;2;248;248;242m                     a string.  If the value contains several strings they are  concate‐[0m
[38;2;248;248;242m                     nated with spaces (or with the sep string if that is given) between[0m
[38;2;248;248;242m                     them.[0m

[38;2;248;248;242m                     Return 0 if the style is set, 1 otherwise.[0m

[38;2;248;248;242m              zstyle -b context style name[0m
[38;2;248;248;242m                     The  value is stored in name as a boolean, i.e. as the string `yes'[0m
[38;2;248;248;242m                     if the value has only one string and that string is equal to one of[0m
[38;2;248;248;242m                     `yes', `true', `on', or `1'. If the value is any  other  string  or[0m
[38;2;248;248;242m                     has more than one string, the parameter is set to `no'.[0m

[38;2;248;248;242m                     Return 0 if name is set to `yes', 1 otherwise.[0m

[38;2;248;248;242m              zstyle -a context style name[0m
[38;2;248;248;242m                     The  value is stored in name as an array. If name is declared as an[0m
[38;2;248;248;242m                     associative array,  the first, third, etc. strings are used as  the[0m
[38;2;248;248;242m                     keys and the other strings are used as the values.[0m

[38;2;248;248;242m                     Return 0 if the style is set, 1 otherwise.[0m

[38;2;248;248;242m              zstyle -t context style [ string ... ][0m
[38;2;248;248;242m              zstyle -T context style [ string ... ][0m
[38;2;248;248;242m                     Test the value of a style, i.e. the -t option only returns a status[0m
[38;2;248;248;242m                     (sets  $?).   Without  any  string the return status is zero if the[0m
[38;2;248;248;242m                     style is defined for at least one matching pattern,  has  only  one[0m
[38;2;248;248;242m                     string  in  its  value,  and that is equal to one of `true', `yes',[0m
[38;2;248;248;242m                     `on' or `1'. If any strings are given the status  is  zero  if  and[0m
[38;2;248;248;242m                     only if at least one of the strings is equal to at least one of the[0m
[38;2;248;248;242m                     strings  in  the  value. If the style is defined but doesn't match,[0m
[38;2;248;248;242m                     the return status is 1. If the style is not defined, the status  is[0m
[38;2;248;248;242m                     2.[0m

[38;2;248;248;242m                     The -T option tests the values of the style like -t, but it returns[0m
[38;2;248;248;242m                     status  zero  (rather  than  2) if the style is not defined for any[0m
[38;2;248;248;242m                     matching pattern.[0m

[38;2;248;248;242m              zstyle -m context style pattern[0m
[38;2;248;248;242m                     Match a value. Returns status zero if the pattern matches at  least[0m
[38;2;248;248;242m                     one of the strings in the value.[0m

[38;2;248;248;242m       zformat -f param format spec ...[0m
[38;2;248;248;242m       zformat -F param format spec ...[0m
[38;2;248;248;242m       zformat -a array sep spec ...[0m
[38;2;248;248;242m              This builtin provides different forms of formatting. The first form is se‐[0m
[38;2;248;248;242m              lected with the -f option. In this case the format string will be modified[0m
[38;2;248;248;242m              by  replacing  sequences  starting  with a percent sign in it with strings[0m
[38;2;248;248;242m              from the specs.  Each spec should be of the form `char:string' which  will[0m
[38;2;248;248;242m              cause every appearance of the sequence `%char' in format to be replaced by[0m
[38;2;248;248;242m              the  string.  The `%' sequence may also contain optional minimum and maxi‐[0m
[38;2;248;248;242m              mum field width specifications between the `%' and the `char' in the  form[0m
[38;2;248;248;242m              `%min.maxc',  i.e. the minimum field width is given first and if the maxi‐[0m
[38;2;248;248;242m              mum field width is used, it has to be preceded by  a  dot.   Specifying  a[0m
[38;2;248;248;242m              minimum field width makes the result be padded with spaces to the right if[0m
[38;2;248;248;242m              the  string  is shorter than the requested width.  Padding to the left can[0m
[38;2;248;248;242m              be achieved by giving a negative minimum field width.  If a maximum  field[0m
[38;2;248;248;242m              width  is  specified, the string will be truncated after that many charac‐[0m
[38;2;248;248;242m              ters.  After all `%' sequences for the given specs  have  been  processed,[0m
[38;2;248;248;242m              the resulting string is stored in the parameter param.[0m

[38;2;248;248;242m              The  %-escapes  also  understand  ternary  expressions in the form used by[0m
[38;2;248;248;242m              prompts.  The % is followed by a `(' and then an ordinary format specifier[0m
[38;2;248;248;242m              character as described above.  There may be a set of digits either  before[0m
[38;2;248;248;242m              or  after  the  `(';  these specify a test number, which defaults to zero.[0m
[38;2;248;248;242m              Negative numbers are also allowed.  An arbitrary delimiter character  fol‐[0m
[38;2;248;248;242m              lows  the  format  specifier, which is followed by a piece of `true' text,[0m
[38;2;248;248;242m              the delimiter character again, a piece of  `false'  text,  and  a  closing[0m
[38;2;248;248;242m              parenthesis.  The complete expression (without the digits) thus looks like[0m
[38;2;248;248;242m              `%(X.text1.text2)', except that the `.' character is arbitrary.  The value[0m
[38;2;248;248;242m              given for the format specifier in the char:string expressions is evaluated[0m
[38;2;248;248;242m              as  a mathematical expression, and compared with the test number.  If they[0m
[38;2;248;248;242m              are the same, text1 is output, else text2 is output.  A parenthesis may be[0m
[38;2;248;248;242m              escaped in text2 as %).  Either of text1 or text2 may contain nested %-es‐[0m
[38;2;248;248;242m              capes.[0m

[38;2;248;248;242m              For example:[0m

[38;2;248;248;242m                     zformat -f REPLY "The answer is '%3(c.yes.no)'." c:3[0m

[38;2;248;248;242m              outputs "The answer is 'yes'." to REPLY since the  value  for  the  format[0m
[38;2;248;248;242m              specifier  c is 3, agreeing with the digit argument to the ternary expres‐[0m
[38;2;248;248;242m              sion.[0m

[38;2;248;248;242m              With -F instead of -f, ternary expressions choose between  the  `true'  or[0m
[38;2;248;248;242m              `false'  text  on the basis of whether the format specifier is present and[0m
[38;2;248;248;242m              non-empty.  A test number indicates a minimum width for the value given in[0m
[38;2;248;248;242m              the format specifier. Negative numbers reverse this, so the  test  is  for[0m
[38;2;248;248;242m              whether the value exceeds a maximum width.[0m

[38;2;248;248;242m              The  form,  using  the -a option, can be used for aligning strings.  Here,[0m
[38;2;248;248;242m              the specs are of the form `left:right' where `left' and `right' are  arbi‐[0m
[38;2;248;248;242m              trary  strings.  These strings are modified by replacing the colons by the[0m
[38;2;248;248;242m              sep string and padding the left strings with spaces to the right  so  that[0m
[38;2;248;248;242m              the sep strings in the result (and hence the right strings after them) are[0m
[38;2;248;248;242m              all  aligned  if  the  strings  are printed below each other.  All strings[0m
[38;2;248;248;242m              without a colon are left unchanged and all strings  with  an  empty  right[0m
[38;2;248;248;242m              string  have the trailing colon removed.  In both cases the lengths of the[0m
[38;2;248;248;242m              strings are not used to determine how the other strings are to be aligned.[0m
[38;2;248;248;242m              A colon in the left string can be escaped with a backslash.  The resulting[0m
[38;2;248;248;242m              strings are stored in the array.[0m

[38;2;248;248;242m       zregexparse[0m
[38;2;248;248;242m              This implements some internals of the _regex_arguments function.[0m

[38;2;248;248;242m       zparseopts [ -D -E -F -K -M ] [ -a array ] [ -A assoc ] [ - ] spec ...[0m
[38;2;248;248;242m              This builtin simplifies the parsing of options in  positional  parameters,[0m
[38;2;248;248;242m              i.e. the set of arguments given by $*.  Each spec describes one option and[0m
[38;2;248;248;242m              must be of the form `opt[=array]'.  If an option described by opt is found[0m
[38;2;248;248;242m              in  the  positional  parameters it is copied into the array specified with[0m
[38;2;248;248;242m              the -a option; if the optional `=array' is given,  it  is  instead  copied[0m
[38;2;248;248;242m              into  that  array, which should be declared as a normal array and never as[0m
[38;2;248;248;242m              an associative array.[0m

[38;2;248;248;242m              Note that it is an error to give any spec without an `=array'  unless  one[0m
[38;2;248;248;242m              of the -a or -A options is used.[0m

[38;2;248;248;242m              Unless  the  -E  option  is  given, parsing stops at the first string that[0m
[38;2;248;248;242m              isn't described by one of the specs.  Even with -E, parsing  always  stops[0m
[38;2;248;248;242m              at a positional parameter equal to `-' or `--'. See also -F.[0m

[38;2;248;248;242m              The  opt  description  must  be  one of the following.  Any of the special[0m
[38;2;248;248;242m              characters can appear in the option name provided  it  is  preceded  by  a[0m
[38;2;248;248;242m              backslash.[0m

[38;2;248;248;242m              name[0m
[38;2;248;248;242m              name+  The  name  is  the  name of the option without the leading `-'.  To[0m
[38;2;248;248;242m                     specify a GNU-style long option, one of the usual two  leading  `-'[0m
[38;2;248;248;242m                     must  be included in name; for example, a `--file' option is repre‐[0m
[38;2;248;248;242m                     sented by a name of `-file'.[0m

[38;2;248;248;242m                     If a `+' appears after name, the option is appended to  array  each[0m
[38;2;248;248;242m                     time it is found in the positional parameters; without the `+' only[0m
[38;2;248;248;242m                     the last occurrence of the option is preserved.[0m

[38;2;248;248;242m                     If  one  of  these  forms is used, the option takes no argument, so[0m
[38;2;248;248;242m                     parsing stops if the next positional parameter does not also  begin[0m
[38;2;248;248;242m                     with `-' (unless the -E option is used).[0m

[38;2;248;248;242m              name:[0m
[38;2;248;248;242m              name:-[0m
[38;2;248;248;242m              name:: If  one or two colons are given, the option takes an argument; with[0m
[38;2;248;248;242m                     one colon, the argument is mandatory and with two colons it is  op‐[0m
[38;2;248;248;242m                     tional.  The argument is appended to the array after the option it‐[0m
[38;2;248;248;242m                     self.[0m

[38;2;248;248;242m                     An  optional argument is put into the same array element as the op‐[0m
[38;2;248;248;242m                     tion name (note that this makes empty strings as  arguments  indis‐[0m
[38;2;248;248;242m                     tinguishable).  A mandatory argument is added as a separate element[0m
[38;2;248;248;242m                     unless  the  `:-'  form  is used, in which case the argument is put[0m
[38;2;248;248;242m                     into the same element.[0m

[38;2;248;248;242m                     A `+' as described above may appear between the name and the  first[0m
[38;2;248;248;242m                     colon.[0m

[38;2;248;248;242m              In  all  cases,  option-arguments must appear either immediately following[0m
[38;2;248;248;242m              the option in the same positional parameter or in the next  one.  Even  an[0m
[38;2;248;248;242m              optional  argument may appear in the next parameter, unless it begins with[0m
[38;2;248;248;242m              a `-'.  There is no special handling of `='  as  with  GNU-style  argument[0m
[38;2;248;248;242m              parsers;  given  the spec `-foo:', the positional parameter `--foo=bar' is[0m
[38;2;248;248;242m              parsed as `--foo' with an argument of `=bar'.[0m

[38;2;248;248;242m              When the names of two options that take no arguments overlap, the  longest[0m
[38;2;248;248;242m              one  wins,  so  that parsing for the specs `-foo -foobar' (for example) is[0m
[38;2;248;248;242m              unambiguous. However, due to the aforementioned handling  of  option-argu‐[0m
[38;2;248;248;242m              ments,  ambiguities  may arise when at least one overlapping spec takes an[0m
[38;2;248;248;242m              argument, as in `-foo: -foobar'. In that  case,  the  last  matching  spec[0m
[38;2;248;248;242m              wins.[0m

[38;2;248;248;242m              The  options  of zparseopts itself cannot be stacked because, for example,[0m
[38;2;248;248;242m              the stack `-DEK' is indistinguishable from a spec for the  GNU-style  long[0m
[38;2;248;248;242m              option `--DEK'.  The options of zparseopts itself are:[0m

[38;2;248;248;242m              -a array[0m
[38;2;248;248;242m                     As  described above, this names the default array in which to store[0m
[38;2;248;248;242m                     the recognised options.[0m

[38;2;248;248;242m              -A assoc[0m
[38;2;248;248;242m                     If this is given, the options and their values are also put into an[0m
[38;2;248;248;242m                     associative array with the option names as keys and  the  arguments[0m
[38;2;248;248;242m                     (if any) as the values.[0m

[38;2;248;248;242m              -D     If this option is given, all options found are removed from the po‐[0m
[38;2;248;248;242m                     sitional  parameters  of the calling shell or shell function, up to[0m
[38;2;248;248;242m                     but not including any not described by the  specs.   If  the  first[0m
[38;2;248;248;242m                     such parameter is `-' or `--', it is removed as well.  This is sim‐[0m
[38;2;248;248;242m                     ilar to using the shift builtin.[0m

[38;2;248;248;242m              -E     This changes the parsing rules to not stop at the first string that[0m
[38;2;248;248;242m                     isn't described by one of the specs.  It can be used to test for or[0m
[38;2;248;248;242m                     (if used together with -D) extract options and their arguments, ig‐[0m
[38;2;248;248;242m                     noring  all  other  options  and arguments that may be in the posi‐[0m
[38;2;248;248;242m                     tional parameters.  As indicated above, parsing still stops at  the[0m
[38;2;248;248;242m                     first  `-'  or  `--' not described by a spec, but it is not removed[0m
[38;2;248;248;242m                     when used with -D.[0m

[38;2;248;248;242m              -F     If this option is given, zparseopts immediately stops at the  first[0m
[38;2;248;248;242m                     option-like  parameter not described by one of the specs, prints an[0m
[38;2;248;248;242m                     error message, and returns status 1.  Removal (-D)  and  extraction[0m
[38;2;248;248;242m                     (-E)  are  not  performed, and option arrays are not updated.  This[0m
[38;2;248;248;242m                     provides basic validation for the given options.[0m

[38;2;248;248;242m                     Note that the appearance in the positional parameters of an  option[0m
[38;2;248;248;242m                     without  its required argument always aborts parsing and returns an[0m
[38;2;248;248;242m                     error as described above regardless of whether this option is used.[0m

[38;2;248;248;242m              -K     With this option, the arrays specified with the -a option and  with[0m
[38;2;248;248;242m                     the  `=array'  forms  are kept unchanged when none of the specs for[0m
[38;2;248;248;242m                     them is used.  Otherwise the entire array is replaced when  any  of[0m
[38;2;248;248;242m                     the specs is used.  Individual elements of associative arrays spec‐[0m
[38;2;248;248;242m                     ified  with the -A option are preserved by -K.  This allows assign‐[0m
[38;2;248;248;242m                     ment of default values to arrays before calling zparseopts.[0m

[38;2;248;248;242m              -M     This changes the assignment rules to implement a map among  equiva‐[0m
[38;2;248;248;242m                     lent  option names.  If any spec uses the `=array' form, the string[0m
[38;2;248;248;242m                     array is interpreted as the name of another spec, which is used  to[0m
[38;2;248;248;242m                     choose  where  to store the values.  If no other spec is found, the[0m
[38;2;248;248;242m                     values are stored as usual.  This changes only the way  the  values[0m
[38;2;248;248;242m                     are  stored,  not  the  way  $* is parsed, so results may be unpre‐[0m
[38;2;248;248;242m                     dictable if the `name+' specifier is used inconsistently.[0m

[38;2;248;248;242m              For example,[0m

[38;2;248;248;242m                     set -- -a -bx -c y -cz baz -cend[0m
[38;2;248;248;242m                     zparseopts a=foo b:=bar c+:=bar[0m

[38;2;248;248;242m              will have the effect of[0m

[38;2;248;248;242m                     foo=(-a)[0m
[38;2;248;248;242m                     bar=(-b x -c y -c z)[0m

[38;2;248;248;242m              The arguments from `baz' on will not be used.[0m

[38;2;248;248;242m              As an example for the -E option, consider:[0m

[38;2;248;248;242m                     set -- -a x -b y -c z arg1 arg2[0m
[38;2;248;248;242m                     zparseopts -E -D b:=bar[0m

[38;2;248;248;242m              will have the effect of[0m

[38;2;248;248;242m                     bar=(-b y)[0m
[38;2;248;248;242m                     set -- -a x -c z arg1 arg2[0m

[38;2;248;248;242m              I.e., the option -b and its arguments are taken from the positional  para‐[0m
[38;2;248;248;242m              meters and put into the array bar.[0m

[38;2;248;248;242m              The -M option can be used like this:[0m

[38;2;248;248;242m                     set -- -a -bx -c y -cz baz -cend[0m
[38;2;248;248;242m                     zparseopts -A bar -M a=foo b+: c:=b[0m

[38;2;248;248;242m              to have the effect of[0m

[38;2;248;248;242m                     foo=(-a)[0m
[38;2;248;248;242m                     bar=(-a '' -b xyz)[0m
