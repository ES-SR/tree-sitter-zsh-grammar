[38;2;248;248;242mZSHZLE(1)                        General Commands Manual                       ZSHZLE(1)[0m

[38;2;248;248;242mNAME[0m
[38;2;248;248;242m       zshzle - zsh command line editor[0m

[38;2;248;248;242mDESCRIPTION[0m
[38;2;248;248;242m       If  the  ZLE option is set (which it is by default in interactive shells) and the[0m
[38;2;248;248;242m       shell input is attached to the terminal, the user is able to edit command lines.[0m

[38;2;248;248;242m       There are two display modes.  The first, multiline mode, is the default.  It only[0m
[38;2;248;248;242m       works if the TERM parameter is set to a valid terminal type  that  can  move  the[0m
[38;2;248;248;242m       cursor up.  The second, single line mode, is used if TERM is invalid or incapable[0m
[38;2;248;248;242m       of  moving  the cursor up, or if the SINGLE_LINE_ZLE option is set.  This mode is[0m
[38;2;248;248;242m       similar to ksh, and uses no termcap sequences.  If TERM is "emacs", the  ZLE  op‐[0m
[38;2;248;248;242m       tion will be unset by default.[0m

[38;2;248;248;242m       The parameters BAUD, COLUMNS, and LINES are also used by the line editor. See Pa‐[0m
[38;2;248;248;242m       rameters Used By The Shell in zshparam(1).[0m

[38;2;248;248;242m       The  parameter zle_highlight is also used by the line editor; see Character High‐[0m
[38;2;248;248;242m       lighting below.  Highlighting of special characters and the  region  between  the[0m
[38;2;248;248;242m       cursor  and  the  mark  (as  set  with  set-mark-command in Emacs mode, or by vi‐[0m
[38;2;248;248;242m       sual-mode in Vi mode) is enabled by default; consult this reference for more  in‐[0m
[38;2;248;248;242m       formation.   Irascible  conservatives will wish to know that all highlighting may[0m
[38;2;248;248;242m       be disabled by the following setting:[0m

[38;2;248;248;242m              zle_highlight=(none)[0m

[38;2;248;248;242m       In many places, references are made to the numeric argument.  This can by default[0m
[38;2;248;248;242m       be entered in emacs mode by holding the alt key and typing a number, or  pressing[0m
[38;2;248;248;242m       escape  before each digit, and in vi command mode by typing the number before en‐[0m
[38;2;248;248;242m       tering a command.  Generally the numeric argument causes the next command entered[0m
[38;2;248;248;242m       to be repeated the specified number of times, unless otherwise noted below;  this[0m
[38;2;248;248;242m       is implemented by the digit-argument widget. See also the Arguments subsection of[0m
[38;2;248;248;242m       the Widgets section for some other ways the numeric argument can be modified.[0m

[38;2;248;248;242mKEYMAPS[0m
[38;2;248;248;242m       A  keymap  in  ZLE  contains a set of bindings between key sequences and ZLE com‐[0m
[38;2;248;248;242m       mands.  The empty key sequence cannot be bound.[0m

[38;2;248;248;242m       There can be any number of keymaps at any time, and each keymap has one  or  more[0m
[38;2;248;248;242m       names.   If  all  of a keymap's names are deleted, it disappears.  bindkey can be[0m
[38;2;248;248;242m       used to manipulate keymap names.[0m

[38;2;248;248;242m       Initially, there are eight keymaps:[0m

[38;2;248;248;242m       emacs  EMACS emulation[0m
[38;2;248;248;242m       viins  vi emulation - insert mode[0m
[38;2;248;248;242m       vicmd  vi emulation - command mode[0m
[38;2;248;248;242m       viopp  vi emulation - operator pending[0m
[38;2;248;248;242m       visual vi emulation - selection active[0m
[38;2;248;248;242m       isearch[0m
[38;2;248;248;242m              incremental search mode[0m
[38;2;248;248;242m       command[0m
[38;2;248;248;242m              read a command name[0m
[38;2;248;248;242m       .safe  fallback keymap[0m

[38;2;248;248;242m       The `.safe' keymap is special.  It can never be altered, and the name  can  never[0m
[38;2;248;248;242m       be  removed.  However, it can be linked to other names, which can be removed.  In[0m
[38;2;248;248;242m       the future other special keymaps may be added; users should avoid using names be‐[0m
[38;2;248;248;242m       ginning with `.' for their own keymaps.[0m

[38;2;248;248;242m       In addition to these names, either `emacs' or `viins' is also linked to the  name[0m
[38;2;248;248;242m       `main'.   If one of the VISUAL or EDITOR environment variables contain the string[0m
[38;2;248;248;242m       `vi' when the shell starts up then it will  be  `viins',  otherwise  it  will  be[0m
[38;2;248;248;242m       `emacs'.   bindkey's  -e and -v options provide a convenient way to override this[0m
[38;2;248;248;242m       default choice.[0m

[38;2;248;248;242m       When the editor starts up, it will select the  `main'  keymap.   If  that  keymap[0m
[38;2;248;248;242m       doesn't exist, it will use `.safe' instead.[0m

[38;2;248;248;242m       In  the  `.safe'  keymap,  each single key is bound to self-insert, except for ^J[0m
[38;2;248;248;242m       (line feed) and ^M (return) which are bound to accept-line.  This is deliberately[0m
[38;2;248;248;242m       not pleasant to use; if you are using it, it means you deleted the  main  keymap,[0m
[38;2;248;248;242m       and you should put it back.[0m

[38;2;248;248;242m   Reading Commands[0m
[38;2;248;248;242m       When  ZLE  is reading a command from the terminal, it may read a sequence that is[0m
[38;2;248;248;242m       bound to some command and is also a prefix of a longer  bound  string.   In  this[0m
[38;2;248;248;242m       case ZLE will wait a certain time to see if more characters are typed, and if not[0m
[38;2;248;248;242m       (or  they don't match any longer string) it will execute the binding.  This time‐[0m
[38;2;248;248;242m       out is defined by the KEYTIMEOUT parameter; its default is 0.4 sec.  There is  no[0m
[38;2;248;248;242m       timeout if the prefix string is not itself bound to a command.[0m

[38;2;248;248;242m       The  key  timeout  is also applied when ZLE is reading the bytes from a multibyte[0m
[38;2;248;248;242m       character string when it is in the appropriate mode.   (This  requires  that  the[0m
[38;2;248;248;242m       shell  was  compiled  with  multibyte mode enabled; typically also the locale has[0m
[38;2;248;248;242m       characters with the UTF-8 encoding, although any multibyte encoding known to  the[0m
[38;2;248;248;242m       operating  system  is supported.)  If the second or a subsequent byte is not read[0m
[38;2;248;248;242m       within the timeout period, the shell acts as if ? were typed and resets the input[0m
[38;2;248;248;242m       state.[0m

[38;2;248;248;242m       As well as ZLE commands, key sequences can be bound to other  strings,  by  using[0m
[38;2;248;248;242m       `bindkey  -s'.   When  such  a sequence is read, the replacement string is pushed[0m
[38;2;248;248;242m       back as input, and the command reading process starts again using these fake key‐[0m
[38;2;248;248;242m       strokes.  This input can itself invoke further replacement strings, but in  order[0m
[38;2;248;248;242m       to detect loops the process will be stopped if there are twenty such replacements[0m
[38;2;248;248;242m       without a real command being read.[0m

[38;2;248;248;242m       A  key  sequence  typed  by the user can be turned into a command name for use in[0m
[38;2;248;248;242m       user-defined widgets with the read-command widget, described  in  the  subsection[0m
[38;2;248;248;242m       `Miscellaneous' of the section `Standard Widgets' below.[0m

[38;2;248;248;242m   Local Keymaps[0m
[38;2;248;248;242m       While for normal editing a single keymap is used exclusively, in many modes a lo‐[0m
[38;2;248;248;242m       cal  keymap allows for some keys to be customised. For example, in an incremental[0m
[38;2;248;248;242m       search mode, a binding in the isearch keymap will override a binding in the  main[0m
[38;2;248;248;242m       keymap but all keys that are not overridden can still be used.[0m

[38;2;248;248;242m       If  a  key  sequence is defined in a local keymap, it will hide a key sequence in[0m
[38;2;248;248;242m       the global keymap that is a prefix of that sequence. An example  of  this  occurs[0m
[38;2;248;248;242m       with the binding of iw in viopp as this hides the binding of i in vicmd. However,[0m
[38;2;248;248;242m       a  longer sequence in the global keymap that shares the same prefix can still ap‐[0m
[38;2;248;248;242m       ply so for example the binding of ^Xa in the global keymap will be unaffected  by[0m
[38;2;248;248;242m       the binding of ^Xb in the local keymap.[0m

[38;2;248;248;242mZLE BUILTINS[0m
[38;2;248;248;242m       The  ZLE  module contains three related builtin commands. The bindkey command ma‐[0m
[38;2;248;248;242m       nipulates keymaps and key bindings; the vared command invokes ZLE on the value of[0m
[38;2;248;248;242m       a shell parameter; and the zle command manipulates  editing  widgets  and  allows[0m
[38;2;248;248;242m       command line access to ZLE commands from within shell functions.[0m

[38;2;248;248;242m       bindkey [ options ] -l [ -L ] [ keymap ... ][0m
[38;2;248;248;242m       bindkey [ options ] -d[0m
[38;2;248;248;242m       bindkey [ options ] -D keymap ...[0m
[38;2;248;248;242m       bindkey [ options ] -A old-keymap new-keymap[0m
[38;2;248;248;242m       bindkey [ options ] -N new-keymap [ old-keymap ][0m
[38;2;248;248;242m       bindkey [ options ] -m[0m
[38;2;248;248;242m       bindkey [ options ] -r in-string ...[0m
[38;2;248;248;242m       bindkey [ options ] -s in-string out-string ...[0m
[38;2;248;248;242m       bindkey [ options ] in-string command ...[0m
[38;2;248;248;242m       bindkey [ options ] [ in-string ][0m
[38;2;248;248;242m              bindkey's  options  can be divided into three categories: keymap selection[0m
[38;2;248;248;242m              for the current command, operation selection, and others.  The keymap  se‐[0m
[38;2;248;248;242m              lection options are:[0m

[38;2;248;248;242m              -e     Selects  keymap  `emacs' for any operations by the current command,[0m
[38;2;248;248;242m                     and also links `emacs' to `main' so that it is selected by  default[0m
[38;2;248;248;242m                     the next time the editor starts.[0m

[38;2;248;248;242m              -v     Selects  keymap  `viins' for any operations by the current command,[0m
[38;2;248;248;242m                     and also links `viins' to `main' so that it is selected by  default[0m
[38;2;248;248;242m                     the next time the editor starts.[0m

[38;2;248;248;242m              -a     Selects keymap `vicmd' for any operations by the current command.[0m

[38;2;248;248;242m              -M keymap[0m
[38;2;248;248;242m                     The  keymap specifies a keymap name that is selected for any opera‐[0m
[38;2;248;248;242m                     tions by the current command.[0m

[38;2;248;248;242m              If a keymap selection is required and none of the options above are  used,[0m
[38;2;248;248;242m              the  `main'  keymap is used.  Some operations do not permit a keymap to be[0m
[38;2;248;248;242m              selected, namely:[0m

[38;2;248;248;242m              -l     List all existing keymap names; if any arguments  are  given,  list[0m
[38;2;248;248;242m                     just those keymaps.[0m

[38;2;248;248;242m                     If the -L option is also used, list in the form of bindkey commands[0m
[38;2;248;248;242m                     to  create  or  link  the  keymaps.  `bindkey -lL main' shows which[0m
[38;2;248;248;242m                     keymap is linked to `main', if any, and hence if the standard emacs[0m
[38;2;248;248;242m                     or vi emulation is in effect.  This option does not show the  .safe[0m
[38;2;248;248;242m                     keymap  because it cannot be created in that fashion; however, nei‐[0m
[38;2;248;248;242m                     ther is `bindkey -lL .safe' reported as an error, it simply outputs[0m
[38;2;248;248;242m                     nothing.[0m

[38;2;248;248;242m              -d     Delete all existing keymaps and reset to the default state.[0m

[38;2;248;248;242m              -D keymap ...[0m
[38;2;248;248;242m                     Delete the named keymaps.[0m

[38;2;248;248;242m              -A old-keymap new-keymap[0m
[38;2;248;248;242m                     Make the new-keymap name an alias  for  old-keymap,  so  that  both[0m
[38;2;248;248;242m                     names  refer to the same keymap.  The names have equal standing; if[0m
[38;2;248;248;242m                     either is deleted, the other remains.  If there is already a keymap[0m
[38;2;248;248;242m                     with the new-keymap name, it is deleted.[0m

[38;2;248;248;242m              -N new-keymap [ old-keymap ][0m
[38;2;248;248;242m                     Create a new keymap, named new-keymap.  If  a  keymap  already  has[0m
[38;2;248;248;242m                     that  name, it is deleted.  If an old-keymap name is given, the new[0m
[38;2;248;248;242m                     keymap is initialized to be a duplicate of it,  otherwise  the  new[0m
[38;2;248;248;242m                     keymap will be empty.[0m

[38;2;248;248;242m              To use a newly created keymap, it should be linked to main.  Hence the se‐[0m
[38;2;248;248;242m              quence of commands to create and use a new keymap `mymap' initialized from[0m
[38;2;248;248;242m              the emacs keymap (which remains unchanged) is:[0m

[38;2;248;248;242m                     bindkey -N mymap emacs[0m
[38;2;248;248;242m                     bindkey -A mymap main[0m

[38;2;248;248;242m              Note that while `bindkey -A newmap main' will work when newmap is emacs or[0m
[38;2;248;248;242m              viins,  it will not work for vicmd, as switching from vi insert to command[0m
[38;2;248;248;242m              mode becomes impossible.[0m

[38;2;248;248;242m              The following operations act on the `main' keymap if no  keymap  selection[0m
[38;2;248;248;242m              option was given:[0m

[38;2;248;248;242m              -m     Add  the  built-in set of meta-key bindings to the selected keymap.[0m
[38;2;248;248;242m                     Only keys that are unbound or bound to self-insert are affected.[0m

[38;2;248;248;242m              -r in-string ...[0m
[38;2;248;248;242m                     Unbind the specified in-strings in the selected  keymap.   This  is[0m
[38;2;248;248;242m                     exactly equivalent to binding the strings to undefined-key.[0m

[38;2;248;248;242m                     When -R is also used, interpret the in-strings as ranges.[0m

[38;2;248;248;242m                     When -p is also used, the in-strings specify prefixes.  Any binding[0m
[38;2;248;248;242m                     that has the given in-string as a prefix, not including the binding[0m
[38;2;248;248;242m                     for the in-string itself, if any, will be removed.  For example,[0m

[38;2;248;248;242m                            bindkey -rpM viins '^['[0m

[38;2;248;248;242m                     will  remove all bindings in the vi-insert keymap beginning with an[0m
[38;2;248;248;242m                     escape character (probably cursor keys), but leave the binding  for[0m
[38;2;248;248;242m                     the escape character itself (probably vi-cmd-mode).  This is incom‐[0m
[38;2;248;248;242m                     patible with the option -R.[0m

[38;2;248;248;242m              -s in-string out-string ...[0m
[38;2;248;248;242m                     Bind  each  in-string to each out-string.  When in-string is typed,[0m
[38;2;248;248;242m                     out-string will be pushed back and treated as input to the line ed‐[0m
[38;2;248;248;242m                     itor.  When -R is also used, interpret the in-strings as ranges.[0m

[38;2;248;248;242m                     Note that both in-string and out-string are  subject  to  the  same[0m
[38;2;248;248;242m                     form of interpretation, as described below.[0m

[38;2;248;248;242m              in-string command ...[0m
[38;2;248;248;242m                     Bind  each  in-string  to each command.  When -R is used, interpret[0m
[38;2;248;248;242m                     the in-strings as ranges.[0m

[38;2;248;248;242m              [ in-string ][0m
[38;2;248;248;242m                     List key bindings.  If an in-string is specified,  the  binding  of[0m
[38;2;248;248;242m                     that  string  in  the selected keymap is displayed.  Otherwise, all[0m
[38;2;248;248;242m                     key bindings in the selected keymap are displayed.  (As  a  special[0m
[38;2;248;248;242m                     case,  if the -e or -v option is used alone, the keymap is not dis‐[0m
[38;2;248;248;242m                     played - the implicit linking of keymaps is  the  only  thing  that[0m
[38;2;248;248;242m                     happens.)[0m

[38;2;248;248;242m                     When  the  option  -p  is used, the in-string must be present.  The[0m
[38;2;248;248;242m                     listing shows all bindings which have the given key sequence  as  a[0m
[38;2;248;248;242m                     prefix, not including any bindings for the key sequence itself.[0m

[38;2;248;248;242m                     When the -L option is used, the list is in the form of bindkey com‐[0m
[38;2;248;248;242m                     mands to create the key bindings.[0m

[38;2;248;248;242m              When  the  -R option is used as noted above, a valid range consists of two[0m
[38;2;248;248;242m              characters, with an optional `-' between them.  All characters between the[0m
[38;2;248;248;242m              two specified, inclusive, are bound as specified.[0m

[38;2;248;248;242m              For either in-string or out-string, the  following  escape  sequences  are[0m
[38;2;248;248;242m              recognised:[0m

[38;2;248;248;242m              \a     bell character[0m
[38;2;248;248;242m              \b     backspace[0m
[38;2;248;248;242m              \e, \E escape[0m
[38;2;248;248;242m              \f     form feed[0m
[38;2;248;248;242m              \n     linefeed (newline)[0m
[38;2;248;248;242m              \r     carriage return[0m
[38;2;248;248;242m              \t     horizontal tab[0m
[38;2;248;248;242m              \v     vertical tab[0m
[38;2;248;248;242m              \NNN   character code in octal[0m
[38;2;248;248;242m              \xNN   character code in hexadecimal[0m
[38;2;248;248;242m              \uNNNN unicode character code in hexadecimal[0m
[38;2;248;248;242m              \UNNNNNNNN[0m
[38;2;248;248;242m                     unicode character code in hexadecimal[0m
[38;2;248;248;242m              \M[-]X character with meta bit set[0m
[38;2;248;248;242m              \C[-]X control character[0m
[38;2;248;248;242m              ^X     control character[0m

[38;2;248;248;242m              In  all other cases, `\' escapes the following character.  Delete is writ‐[0m
[38;2;248;248;242m              ten as `^?'.  Note that `\M^?' and `^\M?' are not the same, and that  (un‐[0m
[38;2;248;248;242m              like emacs), the bindings `\M-X' and `\eX' are entirely distinct, although[0m
[38;2;248;248;242m              they are initialized to the same bindings by `bindkey -m'.[0m

[38;2;248;248;242m       vared [ -Aacghe ] [ -p prompt ] [ -r rprompt ][0m
[38;2;248;248;242m             [ -M main-keymap ] [ -m vicmd-keymap ][0m
[38;2;248;248;242m             [ -i init-widget ] [ -f finish-widget ][0m
[38;2;248;248;242m             [ -t tty ] name[0m
[38;2;248;248;242m              The  value  of  the parameter name is loaded into the edit buffer, and the[0m
[38;2;248;248;242m              line editor is invoked.  When the editor exits, name is set to the  string[0m
[38;2;248;248;242m              value returned by the editor.  When the -c flag is given, the parameter is[0m
[38;2;248;248;242m              created  if it doesn't already exist.  The -a flag may be given with -c to[0m
[38;2;248;248;242m              create an array parameter, or the -A flag to create an associative  array.[0m
[38;2;248;248;242m              If  the  type  of an existing parameter does not match the type to be cre‐[0m
[38;2;248;248;242m              ated, the parameter is unset and recreated.  The -g flag may be  given  to[0m
[38;2;248;248;242m              suppress warnings from the WARN_CREATE_GLOBAL and WARN_NESTED_VAR options.[0m

[38;2;248;248;242m              If  an  array  or array slice is being edited, separator characters as de‐[0m
[38;2;248;248;242m              fined in $IFS will be shown quoted with a backslash, as  will  backslashes[0m
[38;2;248;248;242m              themselves.   Conversely,  when  the edited text is split into an array, a[0m
[38;2;248;248;242m              backslash quotes an immediately following  separator  character  or  back‐[0m
[38;2;248;248;242m              slash;  no  other  special  handling  of  backslashes,  or any handling of[0m
[38;2;248;248;242m              quotes, is performed.[0m

[38;2;248;248;242m              Individual elements of existing array or associative array parameters  may[0m
[38;2;248;248;242m              be edited by using subscript syntax on name.  New elements are created au‐[0m
[38;2;248;248;242m              tomatically, even without -c.[0m

[38;2;248;248;242m              If  the -p flag is given, the following string will be taken as the prompt[0m
[38;2;248;248;242m              to display at the left.  If the -r flag is  given,  the  following  string[0m
[38;2;248;248;242m              gives  the  prompt  to display at the right.  If the -h flag is specified,[0m
[38;2;248;248;242m              the history can be accessed from ZLE. If the -e flag is given,  typing  ^D[0m
[38;2;248;248;242m              (Control-D)  on  an  empty  line  causes  vared to exit immediately with a[0m
[38;2;248;248;242m              non-zero return value.[0m

[38;2;248;248;242m              The -M option gives a keymap to link to the main  keymap  during  editing,[0m
[38;2;248;248;242m              and  the -m option gives a keymap to link to the vicmd keymap during edit‐[0m
[38;2;248;248;242m              ing.  For vi-style editing, this allows a pair of keymaps to override  vi‐[0m
[38;2;248;248;242m              ins  and  vicmd.   For emacs-style editing, only -M is normally needed but[0m
[38;2;248;248;242m              the -m option may still be used.  On exit, the previous  keymaps  will  be[0m
[38;2;248;248;242m              restored.[0m

[38;2;248;248;242m              Vared  calls  the usual `zle-line-init' and `zle-line-finish' hooks before[0m
[38;2;248;248;242m              and after it takes control. Using the -i and -f options, it is possible to[0m
[38;2;248;248;242m              replace these with other custom widgets.[0m

[38;2;248;248;242m              If `-t tty' is given, tty is the name of a terminal device to be used  in‐[0m
[38;2;248;248;242m              stead of the default /dev/tty.  If tty does not refer to a terminal an er‐[0m
[38;2;248;248;242m              ror is reported.[0m

[38;2;248;248;242m       zle[0m
[38;2;248;248;242m       zle -l [ -L | -a ] [ string ... ][0m
[38;2;248;248;242m       zle -D widget ...[0m
[38;2;248;248;242m       zle -A old-widget new-widget[0m
[38;2;248;248;242m       zle -N widget [ function ][0m
[38;2;248;248;242m       zle -f flag [ flag... ][0m
[38;2;248;248;242m       zle -C widget completion-widget function[0m
[38;2;248;248;242m       zle -R [ -c ] [ display-string ] [ string ... ][0m
[38;2;248;248;242m       zle -M string[0m
[38;2;248;248;242m       zle -U string[0m
[38;2;248;248;242m       zle -K keymap[0m
[38;2;248;248;242m       zle -F [ -L | -w ] [ fd [ handler ] ][0m
[38;2;248;248;242m       zle -I[0m
[38;2;248;248;242m       zle -T [ tc function | -r tc | -L ][0m
[38;2;248;248;242m       zle widget [ -n num ] [ -f flag ] [ -Nw ] [ -K keymap ] args ...[0m
[38;2;248;248;242m              The zle builtin performs a number of different actions concerning ZLE.[0m

[38;2;248;248;242m              With  no options and no arguments, only the return status will be set.  It[0m
[38;2;248;248;242m              is zero if ZLE is currently active and widgets could be invoked using this[0m
[38;2;248;248;242m              builtin command and non-zero otherwise.  Note that even if non-zero status[0m
[38;2;248;248;242m              is returned, zle may still be active as part  of  the  completion  system;[0m
[38;2;248;248;242m              this does not allow direct calls to ZLE widgets.[0m

[38;2;248;248;242m              Otherwise, which operation it performs depends on its options:[0m

[38;2;248;248;242m              -l [ -L | -a ] [ string ][0m
[38;2;248;248;242m                     List  all existing user-defined widgets.  If the -L option is used,[0m
[38;2;248;248;242m                     list in the form of zle commands to create the widgets.[0m

[38;2;248;248;242m                     When combined with the -a option, all widget names are listed,  in‐[0m
[38;2;248;248;242m                     cluding the builtin ones. In this case the -L option is ignored.[0m

[38;2;248;248;242m                     If  at  least  one  string is given, and -a is present or -L is not[0m
[38;2;248;248;242m                     used, nothing will be printed.  The return status will be  zero  if[0m
[38;2;248;248;242m                     all  strings are names of existing widgets and non-zero if at least[0m
[38;2;248;248;242m                     one string is not a name of  a  defined  widget.   If  -a  is  also[0m
[38;2;248;248;242m                     present,  all  widget  names  are used for the comparison including[0m
[38;2;248;248;242m                     builtin widgets, else only user-defined widgets are used.[0m

[38;2;248;248;242m                     If at least one string is  present  and  the  -L  option  is  used,[0m
[38;2;248;248;242m                     user-defined  widgets matching any string are listed in the form of[0m
[38;2;248;248;242m                     zle commands to create the widgets.[0m

[38;2;248;248;242m              -D widget ...[0m
[38;2;248;248;242m                     Delete the named widgets.[0m

[38;2;248;248;242m              -A old-widget new-widget[0m
[38;2;248;248;242m                     Make the new-widget name an alias  for  old-widget,  so  that  both[0m
[38;2;248;248;242m                     names  refer to the same widget.  The names have equal standing; if[0m
[38;2;248;248;242m                     either is deleted, the other remains.  If there is already a widget[0m
[38;2;248;248;242m                     with the new-widget name, it is deleted.[0m

[38;2;248;248;242m              -N widget [ function ][0m
[38;2;248;248;242m                     Create a user-defined widget.  If there is already  a  widget  with[0m
[38;2;248;248;242m                     the  specified name, it is overwritten.  When the new widget is in‐[0m
[38;2;248;248;242m                     voked from within the  editor,  the  specified  shell  function  is[0m
[38;2;248;248;242m                     called.   If no function name is specified, it defaults to the same[0m
[38;2;248;248;242m                     name as the widget.  For further information, see the section `Wid‐[0m
[38;2;248;248;242m                     gets' below.[0m

[38;2;248;248;242m              -f flag [ flag... ][0m
[38;2;248;248;242m                     Set various flags on the running widget.  Possible values for  flag[0m
[38;2;248;248;242m                     are:[0m

[38;2;248;248;242m                     yank  for  indicating  that  the  widget  has  yanked text into the[0m
[38;2;248;248;242m                     buffer.  If the widget is wrapping an existing internal widget,  no[0m
[38;2;248;248;242m                     further  action is necessary, but if it has inserted the text manu‐[0m
[38;2;248;248;242m                     ally, then it should also take care to set YANK_START and  YANK_END[0m
[38;2;248;248;242m                     correctly.   yankbefore  does  the same but is used when the yanked[0m
[38;2;248;248;242m                     text appears after the cursor.[0m

[38;2;248;248;242m                     kill for indicating that text has been killed into  the  cutbuffer.[0m
[38;2;248;248;242m                     When  repeatedly  invoking  a  kill widget, text is appended to the[0m
[38;2;248;248;242m                     cutbuffer instead of replacing it, but when wrapping such  widgets,[0m
[38;2;248;248;242m                     it is necessary to call `zle -f kill' to retain this effect.[0m

[38;2;248;248;242m                     vichange for indicating that the widget represents a vi change that[0m
[38;2;248;248;242m                     can be repeated as a whole with `vi-repeat-change'. The flag should[0m
[38;2;248;248;242m                     be set early in the function before inspecting the value of NUMERIC[0m
[38;2;248;248;242m                     or  invoking other widgets. This has no effect for a widget invoked[0m
[38;2;248;248;242m                     from insert mode. If insert mode is active  when  the  widget  fin‐[0m
[38;2;248;248;242m                     ishes, the change extends until next returning to command mode.[0m

[38;2;248;248;242m              -C widget completion-widget function[0m
[38;2;248;248;242m                     Create  a  user-defined completion widget named widget. The comple‐[0m
[38;2;248;248;242m                     tion widget will behave like the built-in  completion-widget  whose[0m
[38;2;248;248;242m                     name  is  given  as completion-widget. To generate the completions,[0m
[38;2;248;248;242m                     the shell function function will be called.  For  further  informa‐[0m
[38;2;248;248;242m                     tion, see zshcompwid(1).[0m

[38;2;248;248;242m              -R [ -c ] [ display-string ] [ string ... ][0m
[38;2;248;248;242m                     Redisplay  the  command line.  If a display-string is given and not[0m
[38;2;248;248;242m                     empty, this is shown in the status line (immediately below the line[0m
[38;2;248;248;242m                     being edited).[0m

[38;2;248;248;242m                     If the optional strings are given they are listed below the  prompt[0m
[38;2;248;248;242m                     in  the same way as completion lists are printed. If no strings are[0m
[38;2;248;248;242m                     given but the -c option is used such a list is cleared.[0m

[38;2;248;248;242m                     Note that immediately after returning  from  running  widgets,  the[0m
[38;2;248;248;242m                     command  line will be redisplayed and the strings displayed will be[0m
[38;2;248;248;242m                     erased.  Therefore, this option is only useful for widgets that  do[0m
[38;2;248;248;242m                     not exit immediately after using it.[0m

[38;2;248;248;242m                     This  command can safely be called outside user defined widgets; if[0m
[38;2;248;248;242m                     zle is active, the display will be refreshed, while if zle  is  not[0m
[38;2;248;248;242m                     active, the command has no effect.  In this case there will usually[0m
[38;2;248;248;242m                     be no other arguments.[0m

[38;2;248;248;242m                     The status is zero if zle was active, else one.[0m

[38;2;248;248;242m              -M string[0m
[38;2;248;248;242m                     As  with the -R option, the string will be displayed below the com‐[0m
[38;2;248;248;242m                     mand line; unlike the -R option, the string will not  be  put  into[0m
[38;2;248;248;242m                     the  status  line  but  will  instead be printed normally below the[0m
[38;2;248;248;242m                     prompt.  This means that the string will still be  displayed  after[0m
[38;2;248;248;242m                     the  widget  returns  (until  it  is overwritten by subsequent com‐[0m
[38;2;248;248;242m                     mands).[0m

[38;2;248;248;242m              -U string[0m
[38;2;248;248;242m                     This pushes the characters in the string onto the  input  stack  of[0m
[38;2;248;248;242m                     ZLE.   After the widget currently executed finishes ZLE will behave[0m
[38;2;248;248;242m                     as if the characters in the string were typed by the user.[0m

[38;2;248;248;242m                     As ZLE uses a stack, if this option is  used  repeatedly  the  last[0m
[38;2;248;248;242m                     string pushed onto the stack will be processed first.  However, the[0m
[38;2;248;248;242m                     characters  in  each string will be processed in the order in which[0m
[38;2;248;248;242m                     they appear in the string.[0m

[38;2;248;248;242m              -K keymap[0m
[38;2;248;248;242m                     Selects the keymap named keymap.  An error  message  will  be  dis‐[0m
[38;2;248;248;242m                     played if there is no such keymap.[0m

[38;2;248;248;242m                     This  keymap selection affects the interpretation of following key‐[0m
[38;2;248;248;242m                     strokes within this invocation of ZLE.   Any  following  invocation[0m
[38;2;248;248;242m                     (e.g.,  the  next command line) will start as usual with the `main'[0m
[38;2;248;248;242m                     keymap selected.[0m

[38;2;248;248;242m              -F [ -L | -w ] [ fd [ handler ] ][0m
[38;2;248;248;242m                     Only available if your system supports one of the  `poll'  or  `se‐[0m
[38;2;248;248;242m                     lect' system calls; most modern systems do.[0m

[38;2;248;248;242m                     Installs  handler  (the  name  of a shell function) to handle input[0m
[38;2;248;248;242m                     from file descriptor fd.  Installing a handler for an fd  which  is[0m
[38;2;248;248;242m                     already  handled  causes  the existing handler to be replaced.  Any[0m
[38;2;248;248;242m                     number of handlers for any number of readable file descriptors  may[0m
[38;2;248;248;242m                     be installed.  Note that zle makes no attempt to check whether this[0m
[38;2;248;248;242m                     fd is actually readable when installing the handler.  The user must[0m
[38;2;248;248;242m                     make  their  own arrangements for handling the file descriptor when[0m
[38;2;248;248;242m                     zle is not active.[0m

[38;2;248;248;242m                     When zle is attempting to read data, it will examine both the  ter‐[0m
[38;2;248;248;242m                     minal and the list of handled fd's.  If data becomes available on a[0m
[38;2;248;248;242m                     handled  fd, zle calls handler with the fd which is ready for read‐[0m
[38;2;248;248;242m                     ing as the first argument.  Under normal circumstances this is  the[0m
[38;2;248;248;242m                     only argument, but if an error was detected, a second argument pro‐[0m
[38;2;248;248;242m                     vides  details: `hup' for a disconnect, `nval' for a closed or oth‐[0m
[38;2;248;248;242m                     erwise invalid descriptor, or `err' for any other condition.   Sys‐[0m
[38;2;248;248;242m                     tems that support only the `select' system call always use `err'.[0m

[38;2;248;248;242m                     If  the option -w is also given, the handler is instead a line edi‐[0m
[38;2;248;248;242m                     tor widget, typically a shell function made  into  a  widget  using[0m
[38;2;248;248;242m                     `zle  -N'.   In that case handler can use all the facilities of zle[0m
[38;2;248;248;242m                     to update the current editing line.  Note, however,  that  as  han‐[0m
[38;2;248;248;242m                     dling fd takes place at a low level changes to the display will not[0m
[38;2;248;248;242m                     automatically  appear; the widget should call `zle -R' to force re‐[0m
[38;2;248;248;242m                     display.  As of this writing, widget handlers only support a single[0m
[38;2;248;248;242m                     argument and thus are never passed a string  for  error  state,  so[0m
[38;2;248;248;242m                     widgets must be prepared to test the descriptor themselves.[0m

[38;2;248;248;242m                     If  either  type  of  handler  produces  output to the terminal, it[0m
[38;2;248;248;242m                     should call `zle -I' before doing so (see below).  Handlers  should[0m
[38;2;248;248;242m                     not attempt to read from the terminal.[0m

[38;2;248;248;242m                     If  no handler is given, but an fd is present, any handler for that[0m
[38;2;248;248;242m                     fd is removed.  If there is none, an error message is  printed  and[0m
[38;2;248;248;242m                     status 1 is returned.[0m

[38;2;248;248;242m                     If  no arguments are given, or the -L option is supplied, a list of[0m
[38;2;248;248;242m                     handlers is printed in a form which can be stored for later  execu‐[0m
[38;2;248;248;242m                     tion.[0m

[38;2;248;248;242m                     An  fd  (but not a handler) may optionally be given with the -L op‐[0m
[38;2;248;248;242m                     tion; in this case, the function will list the handler if any, else[0m
[38;2;248;248;242m                     silently return status 1.[0m

[38;2;248;248;242m                     Note that this feature should be used with care.  Activity  on  one[0m
[38;2;248;248;242m                     of the fd's which is not properly handled can cause the terminal to[0m
[38;2;248;248;242m                     become  unusable.  Removing an fd handler from within a signal trap[0m
[38;2;248;248;242m                     may cause unpredictable behavior.[0m

[38;2;248;248;242m                     Here is a simple example of using this feature.  A connection to  a[0m
[38;2;248;248;242m                     remote TCP port is created using the ztcp command; see the descrip‐[0m
[38;2;248;248;242m                     tion of the zsh/net/tcp module in zshmodules(1).  Then a handler is[0m
[38;2;248;248;242m                     installed  which  simply  prints out any data which arrives on this[0m
[38;2;248;248;242m                     connection.  Note that `select' will indicate  that  the  file  de‐[0m
[38;2;248;248;242m                     scriptor  needs  handling if the remote side has closed the connec‐[0m
[38;2;248;248;242m                     tion; we handle that by testing for a failed read.[0m

[38;2;248;248;242m                            if ztcp pwspc 2811; then[0m
[38;2;248;248;242m                              tcpfd=$REPLY[0m
[38;2;248;248;242m                              handler() {[0m
[38;2;248;248;242m                                zle -I[0m
[38;2;248;248;242m                                local line[0m
[38;2;248;248;242m                                if ! read -r line <&$1; then[0m
[38;2;248;248;242m                                  # select marks this fd if we reach EOF,[0m
[38;2;248;248;242m                                  # so handle this specially.[0m
[38;2;248;248;242m                                  print "[Read on fd $1 failed, removing.]" >&2[0m
[38;2;248;248;242m                                  zle -F $1[0m
[38;2;248;248;242m                                  return 1[0m
[38;2;248;248;242m                                fi[0m
[38;2;248;248;242m                                print -r - $line[0m
[38;2;248;248;242m                              }[0m
[38;2;248;248;242m                              zle -F $tcpfd handler[0m
[38;2;248;248;242m                            fi[0m

[38;2;248;248;242m              -I     Unusually, this option is most useful outside ordinary widget func‐[0m
[38;2;248;248;242m                     tions, though it may be used within if normal output to the  termi‐[0m
[38;2;248;248;242m                     nal  is required.  It invalidates the current zle display in prepa‐[0m
[38;2;248;248;242m                     ration for output; typically this will be from a trap function.  It[0m
[38;2;248;248;242m                     has no effect if zle is not active.  When a trap exits,  the  shell[0m
[38;2;248;248;242m                     checks  to  see if the display needs restoring, hence the following[0m
[38;2;248;248;242m                     will print output in such a way as not to disturb  the  line  being[0m
[38;2;248;248;242m                     edited:[0m

[38;2;248;248;242m                            TRAPUSR1() {[0m
[38;2;248;248;242m                              # Invalidate zle display[0m
[38;2;248;248;242m                              [[ -o zle ]] && zle -I[0m
[38;2;248;248;242m                              # Show output[0m
[38;2;248;248;242m                              print Hello[0m
[38;2;248;248;242m                            }[0m

[38;2;248;248;242m                     In  general,  the trap function may need to test whether zle is ac‐[0m
[38;2;248;248;242m                     tive before using this method (as shown in the example), since  the[0m
[38;2;248;248;242m                     zsh/zle  module  may  not even be loaded; if it is not, the command[0m
[38;2;248;248;242m                     can be skipped.[0m

[38;2;248;248;242m                     It is possible to call `zle -I' several times before control is re‐[0m
[38;2;248;248;242m                     turned to the editor; the display  will  only  be  invalidated  the[0m
[38;2;248;248;242m                     first time to minimise disruption.[0m

[38;2;248;248;242m                     Note  that  there are normally better ways of manipulating the dis‐[0m
[38;2;248;248;242m                     play from within zle widgets; see, for example, `zle -R' above.[0m

[38;2;248;248;242m                     The returned status is zero if zle  was  invalidated,  even  though[0m
[38;2;248;248;242m                     this  may  have  been by a previous call to `zle -I' or by a system[0m
[38;2;248;248;242m                     notification.  To test if a zle widget may be called at this point,[0m
[38;2;248;248;242m                     execute zle with no arguments and examine the return status.[0m

[38;2;248;248;242m              -T     This is used to add, list or remove internal transformations on the[0m
[38;2;248;248;242m                     processing performed by the line editor.  It is typically used only[0m
[38;2;248;248;242m                     for debugging or testing and is therefore of little interest to the[0m
[38;2;248;248;242m                     general user.[0m

[38;2;248;248;242m                     `zle -T transformation func' specifies that the  given  transforma‐[0m
[38;2;248;248;242m                     tion (see below) is effected by shell function func.[0m

[38;2;248;248;242m                     `zle -Tr transformation' removes the given transformation if it was[0m
[38;2;248;248;242m                     present (it is not an error if none was).[0m

[38;2;248;248;242m                     `zle  -TL' can be used to list all transformations currently in op‐[0m
[38;2;248;248;242m                     eration.[0m

[38;2;248;248;242m                     Currently the only transformation is tc.  This is used  instead  of[0m
[38;2;248;248;242m                     outputting  termcap codes to the terminal.  When the transformation[0m
[38;2;248;248;242m                     is in operation the shell function is passed the termcap code  that[0m
[38;2;248;248;242m                     would  be output as its first argument; if the operation required a[0m
[38;2;248;248;242m                     numeric argument, that is passed as a second argument.   The  func‐[0m
[38;2;248;248;242m                     tion should set the shell variable REPLY to the transformed termcap[0m
[38;2;248;248;242m                     code.  Typically this is used to produce some simply formatted ver‐[0m
[38;2;248;248;242m                     sion  of  the  code and optional argument for debugging or testing.[0m
[38;2;248;248;242m                     Note that this transformation is not applied to other  non-printing[0m
[38;2;248;248;242m                     characters such as carriage returns and newlines.[0m

[38;2;248;248;242m              widget [ -n num ] [ -f flag ] [ -Nw ] [ -K keymap ] args ...[0m
[38;2;248;248;242m                     Invoke the specified widget.  This can only be done when ZLE is ac‐[0m
[38;2;248;248;242m                     tive; normally this will be within a user-defined widget.[0m

[38;2;248;248;242m                     With  the  options  -n and -N, the current numeric argument will be[0m
[38;2;248;248;242m                     saved and then restored after the call to widget; `-n num' sets the[0m
[38;2;248;248;242m                     numeric argument temporarily to num, while `-N' sets it to the  de‐[0m
[38;2;248;248;242m                     fault, i.e. as if there were none.[0m

[38;2;248;248;242m                     With  the option -K, keymap will be used as the current keymap dur‐[0m
[38;2;248;248;242m                     ing the execution of the widget.  The previous keymap will  be  re‐[0m
[38;2;248;248;242m                     stored when the widget exits.[0m

[38;2;248;248;242m                     Normally, calling a widget in this way does not set the special pa‐[0m
[38;2;248;248;242m                     rameter  WIDGET and related parameters, so that the environment ap‐[0m
[38;2;248;248;242m                     pears as if the top-level widget called by the user were still  ac‐[0m
[38;2;248;248;242m                     tive.  With the option -w, WIDGET and related parameters are set to[0m
[38;2;248;248;242m                     reflect the widget being executed by the zle call.[0m

[38;2;248;248;242m                     Normally, when widget returns the special parameter LASTWIDGET will[0m
[38;2;248;248;242m                     point  to  it.   This can be inhibited by passing the option -f no‐[0m
[38;2;248;248;242m                     last.[0m

[38;2;248;248;242m                     Any further arguments will be passed to the widget;  note  that  as[0m
[38;2;248;248;242m                     standard  argument handling is performed, any general argument list[0m
[38;2;248;248;242m                     should be preceded by --.  If it is a  shell  function,  these  are[0m
[38;2;248;248;242m                     passed  down as positional parameters; for builtin widgets it is up[0m
[38;2;248;248;242m                     to the widget in question what it does with them.  Currently  argu‐[0m
[38;2;248;248;242m                     ments are only handled by the incremental-search commands, the his‐[0m
[38;2;248;248;242m                     tory-search-forward  and  -backward and the corresponding functions[0m
[38;2;248;248;242m                     prefixed by vi-, and by universal-argument.  No error is flagged if[0m
[38;2;248;248;242m                     the command does not use the arguments, or only uses some of them.[0m

[38;2;248;248;242m                     The return status reflects the success or failure of the  operation[0m
[38;2;248;248;242m                     carried  out  by  the widget, or if it is a user-defined widget the[0m
[38;2;248;248;242m                     return status of the shell function.[0m

[38;2;248;248;242m                     A non-zero return status causes the shell to beep when  the  widget[0m
[38;2;248;248;242m                     exits,  unless  the BEEP options was unset or the widget was called[0m
[38;2;248;248;242m                     via the zle command.  Thus if a user defined widget requires an im‐[0m
[38;2;248;248;242m                     mediate beep, it should call the beep widget directly.[0m

[38;2;248;248;242mZLE WIDGETS[0m
[38;2;248;248;242m       All actions in the editor are performed by `widgets'.  A widget's job  is  simply[0m
[38;2;248;248;242m       to perform some small action.  The ZLE commands that key sequences in keymaps are[0m
[38;2;248;248;242m       bound to are in fact widgets.  Widgets can be user-defined or built in.[0m

[38;2;248;248;242m       The  standard widgets built into ZLE are listed in the section `Standard Widgets'[0m
[38;2;248;248;242m       below.  Other built-in widgets can be  defined  by  other  modules  (see  zshmod‐[0m
[38;2;248;248;242m       ules(1)).  Each built-in widget has two names: its normal canonical name, and the[0m
[38;2;248;248;242m       same  name  preceded by a `.'.  The `.' name is special: it can't be rebound to a[0m
[38;2;248;248;242m       different widget.  This makes the widget available even when its usual  name  has[0m
[38;2;248;248;242m       been redefined.[0m

[38;2;248;248;242m       User-defined  widgets  are defined using `zle -N', and implemented as shell func‐[0m
[38;2;248;248;242m       tions.  When the widget is executed, the corresponding  shell  function  is  exe‐[0m
[38;2;248;248;242m       cuted,  and  can  perform  editing  (or  other)  actions.  It is recommended that[0m
[38;2;248;248;242m       user-defined widgets should not have names starting with `.'.[0m

[38;2;248;248;242mUSER-DEFINED WIDGETS[0m
[38;2;248;248;242m       User-defined widgets, being implemented as shell functions, can execute any  nor‐[0m
[38;2;248;248;242m       mal shell command.  They can also run other widgets (whether built-in or user-de‐[0m
[38;2;248;248;242m       fined) using the zle builtin command. The standard input of the function is redi‐[0m
[38;2;248;248;242m       rected  from /dev/null to prevent external commands from unintentionally blocking[0m
[38;2;248;248;242m       ZLE by reading from the terminal, but read -k or read -q  can  be  used  to  read[0m
[38;2;248;248;242m       characters.   Finally,  they  can examine and edit the ZLE buffer being edited by[0m
[38;2;248;248;242m       reading and setting the special parameters described below.[0m

[38;2;248;248;242m       These special parameters are always available in widget functions, but are not in[0m
[38;2;248;248;242m       any way special outside ZLE.  If they have some normal value  outside  ZLE,  that[0m
[38;2;248;248;242m       value  is  temporarily inaccessible, but will return when the widget function ex‐[0m
[38;2;248;248;242m       its.  These special parameters in fact have local scope, like parameters  created[0m
[38;2;248;248;242m       in a function using local.[0m

[38;2;248;248;242m       Inside  completion widgets and traps called while ZLE is active, these parameters[0m
[38;2;248;248;242m       are available read-only.[0m

[38;2;248;248;242m       Note that the parameters appear as local to any ZLE widget in which they  appear.[0m
[38;2;248;248;242m       Hence  if  it  is  desired to override them this needs to be done within a nested[0m
[38;2;248;248;242m       function:[0m

[38;2;248;248;242m              widget-function() {[0m
[38;2;248;248;242m                # $WIDGET here refers to the special variable[0m
[38;2;248;248;242m                # that is local inside widget-function[0m
[38;2;248;248;242m                () {[0m
[38;2;248;248;242m                   # This anonymous nested function allows WIDGET[0m
[38;2;248;248;242m                   # to be used as a local variable.  The -h[0m
[38;2;248;248;242m                   # removes the special status of the variable.[0m
[38;2;248;248;242m                   local -h WIDGET[0m
[38;2;248;248;242m                }[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       BUFFER (scalar)[0m
[38;2;248;248;242m              The entire contents of the edit buffer.  If it is written to,  the  cursor[0m
[38;2;248;248;242m              remains at the same offset, unless that would put it outside the buffer.[0m

[38;2;248;248;242m       BUFFERLINES (integer)[0m
[38;2;248;248;242m              The  number of screen lines needed for the edit buffer currently displayed[0m
[38;2;248;248;242m              on screen (i.e. without any changes to the preceding parameters done after[0m
[38;2;248;248;242m              the last redisplay); read-only.[0m

[38;2;248;248;242m       CONTEXT (scalar)[0m
[38;2;248;248;242m              The context in which zle was called to read a line; read-only.  One of the[0m
[38;2;248;248;242m              values:[0m

[38;2;248;248;242m              start  The start of a command line (at prompt PS1).[0m

[38;2;248;248;242m              cont   A continuation to a command line (at prompt PS2).[0m

[38;2;248;248;242m              select In a select loop (at prompt PS3).[0m

[38;2;248;248;242m              vared  Editing a variable in vared.[0m

[38;2;248;248;242m       CURSOR (integer)[0m
[38;2;248;248;242m              The offset of the cursor, within the edit buffer.  This is in the range  0[0m
[38;2;248;248;242m              to  $#BUFFER,  and  is by definition equal to $#LBUFFER.  Attempts to move[0m
[38;2;248;248;242m              the cursor outside the buffer will result in the cursor being moved to the[0m
[38;2;248;248;242m              appropriate end of the buffer.[0m

[38;2;248;248;242m       CUTBUFFER (scalar)[0m
[38;2;248;248;242m              The last item cut using one of the `kill-' commands; the string which  the[0m
[38;2;248;248;242m              next yank would insert in the line.  Later entries in the kill ring are in[0m
[38;2;248;248;242m              the  array  killring.   Note  that  the  command  `zle copy-region-as-kill[0m
[38;2;248;248;242m              string' can be used to set the text of the cut buffer from a  shell  func‐[0m
[38;2;248;248;242m              tion  and  cycle  the  kill  ring in the same way as interactively killing[0m
[38;2;248;248;242m              text.[0m

[38;2;248;248;242m       HISTNO (integer)[0m
[38;2;248;248;242m              The current history number.  Setting this has the same effect as moving up[0m
[38;2;248;248;242m              or down in the history to the corresponding history line.  An  attempt  to[0m
[38;2;248;248;242m              set  it is ignored if the line is not stored in the history.  Note this is[0m
[38;2;248;248;242m              not the same as the parameter HISTCMD, which always gives  the  number  of[0m
[38;2;248;248;242m              the  history  line being added to the main shell's history.  HISTNO refers[0m
[38;2;248;248;242m              to the line being retrieved within zle.[0m

[38;2;248;248;242m       ISEARCHMATCH_ACTIVE (integer)[0m
[38;2;248;248;242m       ISEARCHMATCH_START (integer)[0m
[38;2;248;248;242m       ISEARCHMATCH_END (integer)[0m
[38;2;248;248;242m              ISEARCHMATCH_ACTIVE indicates whether a part of the  BUFFER  is  currently[0m
[38;2;248;248;242m              matched  by an incremental search pattern. ISEARCHMATCH_START and ISEARCH‐[0m
[38;2;248;248;242m              MATCH_END give the location of the matched part and are in the same  units[0m
[38;2;248;248;242m              as  CURSOR.  They  are  only valid for reading when ISEARCHMATCH_ACTIVE is[0m
[38;2;248;248;242m              non-zero.[0m

[38;2;248;248;242m              All parameters are read-only.[0m

[38;2;248;248;242m       KEYMAP (scalar)[0m
[38;2;248;248;242m              The name of the currently selected keymap; read-only.[0m

[38;2;248;248;242m       KEYS (scalar)[0m
[38;2;248;248;242m              The keys typed to invoke this widget, as a literal string; read-only.[0m

[38;2;248;248;242m       KEYS_QUEUED_COUNT (integer)[0m
[38;2;248;248;242m              The number of bytes pushed back to the input queue and therefore available[0m
[38;2;248;248;242m              for reading immediately before any I/O is done; read-only.  See also PEND‐[0m
[38;2;248;248;242m              ING; the two values are distinct.[0m

[38;2;248;248;242m       killring (array)[0m
[38;2;248;248;242m              The array of previously killed items, with the most recently killed first.[0m
[38;2;248;248;242m              This gives the items that would be retrieved by a yank-pop in the same or‐[0m
[38;2;248;248;242m              der.  Note, however, that the most recently killed item is in  $CUTBUFFER;[0m
[38;2;248;248;242m              $killring shows the array of previous entries.[0m

[38;2;248;248;242m              The  default  size  for  the kill ring is eight, however the length may be[0m
[38;2;248;248;242m              changed by normal array operations.  Any empty string in the kill ring  is[0m
[38;2;248;248;242m              ignored  by  the yank-pop command, hence the size of the array effectively[0m
[38;2;248;248;242m              sets the maximum length of the kill ring, while  the  number  of  non-zero[0m
[38;2;248;248;242m              strings  gives the current length, both as seen by the user at the command[0m
[38;2;248;248;242m              line.[0m

[38;2;248;248;242m       LASTABORTEDSEARCH (scalar)[0m
[38;2;248;248;242m              The last search string used by an interactive search that was  aborted  by[0m
[38;2;248;248;242m              the user (status 3 returned by the search widget).[0m

[38;2;248;248;242m       LASTSEARCH (scalar)[0m
[38;2;248;248;242m              The  last search string used by an interactive search; read-only.  This is[0m
[38;2;248;248;242m              set even if the search failed (status 0, 1 or 2  returned  by  the  search[0m
[38;2;248;248;242m              widget), but not if it was aborted by the user.[0m

[38;2;248;248;242m       LASTWIDGET (scalar)[0m
[38;2;248;248;242m              The name of the last widget that was executed; read-only.[0m

[38;2;248;248;242m       LBUFFER (scalar)[0m
[38;2;248;248;242m              The  part  of the buffer that lies to the left of the cursor position.  If[0m
[38;2;248;248;242m              it is assigned to, only that part of the buffer is replaced, and the  cur‐[0m
[38;2;248;248;242m              sor remains between the new $LBUFFER and the old $RBUFFER.[0m

[38;2;248;248;242m       MARK (integer)[0m
[38;2;248;248;242m              Like  CURSOR,  but  for  the  mark. With vi-mode operators that wait for a[0m
[38;2;248;248;242m              movement command to select a region of text, setting MARK allows  the  se‐[0m
[38;2;248;248;242m              lection to extend in both directions from the initial cursor position.[0m

[38;2;248;248;242m       NUMERIC (integer)[0m
[38;2;248;248;242m              The  numeric argument. If no numeric argument was given, this parameter is[0m
[38;2;248;248;242m              unset. When this is set inside a widget function, builtin  widgets  called[0m
[38;2;248;248;242m              with  the  zle builtin command will use the value assigned. If it is unset[0m
[38;2;248;248;242m              inside a widget function, builtin widgets called behave as if  no  numeric[0m
[38;2;248;248;242m              argument was given.[0m

[38;2;248;248;242m       PENDING (integer)[0m
[38;2;248;248;242m              The number of bytes pending for input, i.e. the number of bytes which have[0m
[38;2;248;248;242m              already been typed and can immediately be read. On systems where the shell[0m
[38;2;248;248;242m              is  not  able  to  get this information, this parameter will always have a[0m
[38;2;248;248;242m              value of zero.  Read-only.  See also KEYS_QUEUED_COUNT; the two values are[0m
[38;2;248;248;242m              distinct.[0m

[38;2;248;248;242m       PREBUFFER (scalar)[0m
[38;2;248;248;242m              In a multi-line input at the secondary prompt,  this  read-only  parameter[0m
[38;2;248;248;242m              contains  the contents of the lines before the one the cursor is currently[0m
[38;2;248;248;242m              in.[0m

[38;2;248;248;242m       PREDISPLAY (scalar)[0m
[38;2;248;248;242m              Text to be displayed before the start of the editable text  buffer.   This[0m
[38;2;248;248;242m              does not have to be a complete line; to display a complete line, a newline[0m
[38;2;248;248;242m              must  be  appended  explicitly.   The text is reset on each new invocation[0m
[38;2;248;248;242m              (but not recursive invocation) of zle.[0m

[38;2;248;248;242m       POSTDISPLAY (scalar)[0m
[38;2;248;248;242m              Text to be displayed after the end of the editable text buffer.  This does[0m
[38;2;248;248;242m              not have to be a complete line; to display a complete line, a newline must[0m
[38;2;248;248;242m              be prepended explicitly.  The text is reset on each  new  invocation  (but[0m
[38;2;248;248;242m              not recursive invocation) of zle.[0m

[38;2;248;248;242m       RBUFFER (scalar)[0m
[38;2;248;248;242m              The  part of the buffer that lies to the right of the cursor position.  If[0m
[38;2;248;248;242m              it is assigned to, only that part of the buffer is replaced, and the  cur‐[0m
[38;2;248;248;242m              sor remains between the old $LBUFFER and the new $RBUFFER.[0m

[38;2;248;248;242m       REGION_ACTIVE (integer)[0m
[38;2;248;248;242m              Indicates if the region is currently active.  It can be assigned 0 or 1 to[0m
[38;2;248;248;242m              deactivate  and  activate  the region respectively. A value of 2 activates[0m
[38;2;248;248;242m              the region in line-wise mode with the highlighted text extending for whole[0m
[38;2;248;248;242m              lines only; see Character Highlighting below.[0m

[38;2;248;248;242m       region_highlight (array)[0m
[38;2;248;248;242m              Each element of this array may be set to a  string  that  describes  high‐[0m
[38;2;248;248;242m              lighting for an arbitrary region of the command line that will take effect[0m
[38;2;248;248;242m              the  next  time  the  command  line  is  redisplayed.  Highlighting of the[0m
[38;2;248;248;242m              non-editable parts of the command line in PREDISPLAY and  POSTDISPLAY  are[0m
[38;2;248;248;242m              possible, but note that the P flag is needed for character indexing to in‐[0m
[38;2;248;248;242m              clude PREDISPLAY.[0m

[38;2;248;248;242m              Each string consists of the following whitespace-separated parts:[0m

[38;2;248;248;242m              •      Optionally,  a  `P'  to  signify that the start and end offset that[0m
[38;2;248;248;242m                     follow include any string set by the PREDISPLAY special  parameter;[0m
[38;2;248;248;242m                     this  is  needed  if  the  predisplay  string itself is to be high‐[0m
[38;2;248;248;242m                     lighted.  Whitespace between the `P' and the start  offset  is  op‐[0m
[38;2;248;248;242m                     tional.[0m

[38;2;248;248;242m              •      A start offset in the same units as CURSOR.[0m

[38;2;248;248;242m              •      An end offset in the same units as CURSOR.[0m

[38;2;248;248;242m              •      A  highlight  specification in the same format as used for contexts[0m
[38;2;248;248;242m                     in the parameter zle_highlight, see the  section  `Character  High‐[0m
[38;2;248;248;242m                     lighting' below; for example, standout or fg=red,bold.[0m

[38;2;248;248;242m              •      Optionally,  a string of the form `memo=token'.  The token consists[0m
[38;2;248;248;242m                     of everything between the `=' and the next whitespace, comma,  NUL,[0m
[38;2;248;248;242m                     or  the end of the string.  The token is preserved verbatim but not[0m
[38;2;248;248;242m                     parsed in any way.[0m

[38;2;248;248;242m                     Plugins may use this to identify array elements  they  have  added:[0m
[38;2;248;248;242m                     for  example,  a  plugin might set token to its (the plugin's) name[0m
[38;2;248;248;242m                     and then use  `region_highlight=(  ${region_highlight:#*memo=token}[0m
[38;2;248;248;242m                     )' in order to remove array elements it have added.[0m

[38;2;248;248;242m                     (This example uses the `${name:#pattern}' array-grepping syntax de‐[0m
[38;2;248;248;242m                     scribed in the section `Parameter Expansion' in zshexpn(1).)[0m

[38;2;248;248;242m              For example,[0m

[38;2;248;248;242m                     region_highlight=("P0 20 bold memo=foobar")[0m

[38;2;248;248;242m              specifies  that the first twenty characters of the text including any pre‐[0m
[38;2;248;248;242m              display string should be highlighted in bold.[0m

[38;2;248;248;242m              Note that the effect of region_highlight is not saved  and  disappears  as[0m
[38;2;248;248;242m              soon as the line is accepted.[0m

[38;2;248;248;242m              Note  that zsh 5.8 and older do not support the `memo=token' field and may[0m
[38;2;248;248;242m              misparse the third (highlight specification) field when a memo is given.[0m

[38;2;248;248;242m              The final highlighting on the command line depends  on  both  region_high‐[0m
[38;2;248;248;242m              light  and zle_highlight; see the section CHARACTER HIGHLIGHTING below for[0m
[38;2;248;248;242m              details.[0m

[38;2;248;248;242m       registers (associative array)[0m
[38;2;248;248;242m              The contents of each of the vi register buffers. These are  typically  set[0m
[38;2;248;248;242m              using vi-set-buffer followed by a delete, change or yank command.[0m

[38;2;248;248;242m       SUFFIX_ACTIVE (integer)[0m
[38;2;248;248;242m       SUFFIX_START (integer)[0m
[38;2;248;248;242m       SUFFIX_END (integer)[0m
[38;2;248;248;242m              SUFFIX_ACTIVE  indicates  whether  an  auto-removable completion suffix is[0m
[38;2;248;248;242m              currently active. SUFFIX_START and SUFFIX_END give  the  location  of  the[0m
[38;2;248;248;242m              suffix  and are in the same units as CURSOR. They are only valid for read‐[0m
[38;2;248;248;242m              ing when SUFFIX_ACTIVE is non-zero.[0m

[38;2;248;248;242m              All parameters are read-only.[0m

[38;2;248;248;242m       UNDO_CHANGE_NO (integer)[0m
[38;2;248;248;242m              A number representing the state of the undo history.  The only use of this[0m
[38;2;248;248;242m              is passing as an argument to the undo widget in order to undo back to  the[0m
[38;2;248;248;242m              recorded point.  Read-only.[0m

[38;2;248;248;242m       UNDO_LIMIT_NO (integer)[0m
[38;2;248;248;242m              A  number corresponding to an existing change in the undo history; compare[0m
[38;2;248;248;242m              UNDO_CHANGE_NO.  If this is set to a value greater  than  zero,  the  undo[0m
[38;2;248;248;242m              command  will not allow the line to be undone beyond the given change num‐[0m
[38;2;248;248;242m              ber.  It is still possible to use `zle undo change' in a  widget  to  undo[0m
[38;2;248;248;242m              beyond  that  point;  in that case, it will not be possible to undo at all[0m
[38;2;248;248;242m              until UNDO_LIMIT_NO is reduced.  Set to 0 to disable the limit.[0m

[38;2;248;248;242m              A typical use of this variable in a widget function is  as  follows  (note[0m
[38;2;248;248;242m              the additional function scope is required):[0m

[38;2;248;248;242m                     () {[0m
[38;2;248;248;242m                       local UNDO_LIMIT_NO=$UNDO_CHANGE_NO[0m
[38;2;248;248;242m                       # Perform some form of recursive edit.[0m
[38;2;248;248;242m                     }[0m

[38;2;248;248;242m       WIDGET (scalar)[0m
[38;2;248;248;242m              The name of the widget currently being executed; read-only.[0m

[38;2;248;248;242m       WIDGETFUNC (scalar)[0m
[38;2;248;248;242m              The  name  of the shell function that implements a widget defined with ei‐[0m
[38;2;248;248;242m              ther zle -N or zle -C.  In the former case, this is the second argument to[0m
[38;2;248;248;242m              the zle -N command that defined the widget, or the first argument if there[0m
[38;2;248;248;242m              was no second argument.  In the latter case this is the third argument  to[0m
[38;2;248;248;242m              the zle -C command that defined the widget.  Read-only.[0m

[38;2;248;248;242m       WIDGETSTYLE (scalar)[0m
[38;2;248;248;242m              Describes  the implementation behind the completion widget currently being[0m
[38;2;248;248;242m              executed; the second argument that followed zle -C when the widget was de‐[0m
[38;2;248;248;242m              fined.  This is the name of a builtin completion widget.  For widgets  de‐[0m
[38;2;248;248;242m              fined with zle -N this is set to the empty string.  Read-only.[0m

[38;2;248;248;242m       YANK_ACTIVE (integer)[0m
[38;2;248;248;242m       YANK_START (integer)[0m
[38;2;248;248;242m       YANK_END (integer)[0m
[38;2;248;248;242m              YANK_ACTIVE  indicates whether text has just been yanked (pasted) into the[0m
[38;2;248;248;242m              buffer.  YANK_START and YANK_END give the location of the pasted text  and[0m
[38;2;248;248;242m              are  in  the  same  units as CURSOR.  They are only valid for reading when[0m
[38;2;248;248;242m              YANK_ACTIVE is non-zero.  They can also be assigned by widgets that insert[0m
[38;2;248;248;242m              text in a yank-like fashion, for example wrappers of bracketed-paste.  See[0m
[38;2;248;248;242m              also zle -f.[0m

[38;2;248;248;242m              YANK_ACTIVE is read-only.[0m

[38;2;248;248;242m       ZLE_RECURSIVE (integer)[0m
[38;2;248;248;242m              Usually zero, but  incremented  inside  any  instance  of  recursive-edit.[0m
[38;2;248;248;242m              Hence indicates the current recursion level.[0m

[38;2;248;248;242m              ZLE_RECURSIVE is read-only.[0m

[38;2;248;248;242m       ZLE_STATE (scalar)[0m
[38;2;248;248;242m              Contains  a  set  of  space-separated  words that describe the current zle[0m
[38;2;248;248;242m              state.[0m

[38;2;248;248;242m              Currently, the states shown are the  insert  mode  as  set  by  the  over‐[0m
[38;2;248;248;242m              write-mode  or  vi-replace widgets and whether history commands will visit[0m
[38;2;248;248;242m              imported entries as  controlled  by  the  set-local-history  widget.   The[0m
[38;2;248;248;242m              string  contains `insert' if characters to be inserted on the command line[0m
[38;2;248;248;242m              move existing characters to the right or `overwrite' if characters  to  be[0m
[38;2;248;248;242m              inserted overwrite existing characters. It contains `localhistory' if only[0m
[38;2;248;248;242m              local history commands will be visited or `globalhistory' if imported his‐[0m
[38;2;248;248;242m              tory commands will also be visited.[0m

[38;2;248;248;242m              The  substrings  are  sorted  in alphabetical order so that if you want to[0m
[38;2;248;248;242m              test for two specific substrings in a future-proof way, you can  do  match[0m
[38;2;248;248;242m              by doing:[0m

[38;2;248;248;242m   Special Widgets[0m
[38;2;248;248;242m       There  are a few user-defined widgets which are special to the shell.  If they do[0m

[38;2;248;248;242m              Push the buffer onto the buffer stack, and execute the  command  `run-help[0m
[38;2;248;248;242m              cmd',  where  cmd is the current command.  run-help is normally aliased to[0m
[38;2;248;248;242m              man.[0m

[38;2;248;248;242m       vi-set-buffer (unbound) (") (unbound)[0m
[38;2;248;248;242m              Specify a buffer to be used  in  the  following  command.   There  are  37[0m
[38;2;248;248;242m              buffers that can be specified: the 26 `named' buffers "a to "z, the `yank'[0m
[38;2;248;248;242m              buffer  "0, the nine `queued' buffers "1 to "9 and the `black hole' buffer[0m
[38;2;248;248;242m              "_.  The named buffers can also be specified as "A to "Z.[0m

[38;2;248;248;242m              When a buffer is specified for a cut, change or  yank  command,  the  text[0m
[38;2;248;248;242m              concerned  replaces  the  previous  contents of the specified buffer. If a[0m
[38;2;248;248;242m              named buffer is specified using a capital, the newly cut text is  appended[0m
[38;2;248;248;242m              to the buffer instead of overwriting it. When using the "_ buffer, nothing[0m
[38;2;248;248;242m              happens.  This  can  be  useful  for  deleting  text without affecting any[0m
[38;2;248;248;242m              buffers.[0m

[38;2;248;248;242m              If no buffer is specified for a cut or change command, "1 is used, and the[0m
[38;2;248;248;242m              contents of "1 to "8 are each shifted along one buffer; the contents of "9[0m
[38;2;248;248;242m              is lost. If no buffer is specified for a yank command,  "0  is  used.  Fi‐[0m
[38;2;248;248;242m              nally, a paste command without a specified buffer will paste the text from[0m
[38;2;248;248;242m              the most recent command regardless of any buffer that might have been used[0m
[38;2;248;248;242m              with that command.[0m

[38;2;248;248;242m              When  called from a widget function by the zle command, the buffer can op‐[0m
[38;2;248;248;242m              tionally be specified with an argument. For example,[0m

[38;2;248;248;242m                     zle vi-set-buffer A[0m

[38;2;248;248;242m       vi-set-mark (unbound) (m) (unbound)[0m
[38;2;248;248;242m              Set the specified mark at the cursor position.[0m

[38;2;248;248;242m       set-mark-command (^@) (unbound) (unbound)[0m
[38;2;248;248;242m              Set the mark at the cursor position.  If called with  a  negative  numeric[0m
[38;2;248;248;242m              argument,  do  not set the mark but deactivate the region so that it is no[0m
[38;2;248;248;242m              longer highlighted (it is still usable for other purposes).  Otherwise the[0m
[38;2;248;248;242m              region is marked as active.[0m

[38;2;248;248;242m       spell-word (ESC-$ ESC-S ESC-s) (unbound) (unbound)[0m
[38;2;248;248;242m              Attempt spelling correction on the current word.[0m

[38;2;248;248;242m       split-undo[0m
[38;2;248;248;242m              Breaks the undo sequence at the current change.  This is useful in vi mode[0m
[38;2;248;248;242m              as changes made in insert mode are coalesced  on  entering  command  mode.[0m
[38;2;248;248;242m              Similarly,  undo  will  normally  revert  as one all the changes made by a[0m
[38;2;248;248;242m              user-defined widget.[0m

[38;2;248;248;242m       undefined-key[0m
[38;2;248;248;242m              This command is executed when a key sequence that is not bound to any com‐[0m
[38;2;248;248;242m              mand is typed.  By default it beeps.[0m

[38;2;248;248;242m       undo (^_ ^Xu ^X^U) (u) (unbound)[0m
[38;2;248;248;242m              Incrementally undo  the  last  text  modification.   When  called  from  a[0m
[38;2;248;248;242m              user-defined  widget,  takes  an  optional  argument indicating a previous[0m
[38;2;248;248;242m              state of the undo history as returned by the UNDO_CHANGE_NO variable; mod‐[0m
[38;2;248;248;242m              ifications are undone until that state is reached, subject  to  any  limit[0m
[38;2;248;248;242m              imposed by the UNDO_LIMIT_NO variable.[0m

[38;2;248;248;242m              Note that when invoked from vi command mode, the full prior change made in[0m
[38;2;248;248;242m              insert  mode is reverted, the changes having been merged when command mode[0m
[38;2;248;248;242m              was selected.[0m

[38;2;248;248;242m       redo (unbound) (^R) (unbound)[0m
[38;2;248;248;242m              Incrementally redo undone text modifications.[0m

[38;2;248;248;242m       vi-undo-change (unbound) (unbound) (unbound)[0m
[38;2;248;248;242m              Undo the last text modification.  If repeated, redo the modification.[0m

[38;2;248;248;242m       visual-mode (unbound) (v) (unbound)[0m
[38;2;248;248;242m              Toggle vim-style visual selection mode. If line-wise visual mode  is  cur‐[0m
[38;2;248;248;242m              rently enabled then it is changed to being character-wise. If used follow‐[0m
[38;2;248;248;242m              ing  an  operator, it forces the subsequent movement command to be treated[0m
[38;2;248;248;242m              as a character-wise movement.[0m

[38;2;248;248;242m       visual-line-mode (unbound) (V) (unbound)[0m
[38;2;248;248;242m              Toggle vim-style line-wise visual selection mode. If character-wise visual[0m
[38;2;248;248;242m              mode is currently enabled then it is changed to being line-wise.  If  used[0m
[38;2;248;248;242m              following  an  operator,  it  forces the subsequent movement command to be[0m
[38;2;248;248;242m              treated as a line-wise movement.[0m

[38;2;248;248;242m       what-cursor-position (^X=) (ga) (unbound)[0m
[38;2;248;248;242m              Print the character under the cursor, its code as an  octal,  decimal  and[0m
[38;2;248;248;242m              hexadecimal  number, the current cursor position within the buffer and the[0m
[38;2;248;248;242m              column of the cursor in the current line.[0m

[38;2;248;248;242m       where-is[0m
[38;2;248;248;242m              Read the name of an editor command and print the listing of key  sequences[0m
[38;2;248;248;242m              that  invoke the specified command.  A restricted set of editing functions[0m
[38;2;248;248;242m              is available in the mini-buffer.  Keys are looked up in the  special  com‐[0m
[38;2;248;248;242m              mand keymap, and if not found there in the main keymap.[0m

[38;2;248;248;242m       which-command (ESC-?) (unbound) (unbound)[0m
[38;2;248;248;242m              Push the buffer onto the buffer stack, and execute the command `which-com‐[0m
[38;2;248;248;242m              mand  cmd'.  where  cmd is the current command.  which-command is normally[0m
[38;2;248;248;242m              aliased to whence.[0m

[38;2;248;248;242m       vi-digit-or-beginning-of-line (unbound) (0) (unbound)[0m
[38;2;248;248;242m              If the last command executed was a digit as part of an argument,  continue[0m
[38;2;248;248;242m              the argument.  Otherwise, execute vi-beginning-of-line.[0m

[38;2;248;248;242m   Text Objects[0m
[38;2;248;248;242m       Text objects are commands that can be used to select a block of text according to[0m
[38;2;248;248;242m       some criteria. They are a feature of the vim text editor and so are primarily in‐[0m
[38;2;248;248;242m       tended for use with vi operators or from visual selection mode. However, they can[0m
[38;2;248;248;242m       also be used from vi-insert or emacs mode. Key bindings listed below apply to the[0m
[38;2;248;248;242m       viopp and visual keymaps.[0m

[38;2;248;248;242m       select-a-blank-word (aW)[0m
[38;2;248;248;242m              Select  a word including adjacent blanks, where a word is defined as a se‐[0m
[38;2;248;248;242m              ries of non-blank characters. With a numeric argument, multiple words will[0m
[38;2;248;248;242m              be selected.[0m

[38;2;248;248;242m       select-a-shell-word (aa)[0m
[38;2;248;248;242m              Select the current command argument applying the normal rules for quoting.[0m

[38;2;248;248;242m       select-a-word (aw)[0m
[38;2;248;248;242m              Select a word including adjacent blanks, using the  normal  vi-style  word[0m
[38;2;248;248;242m              definition. With a numeric argument, multiple words will be selected.[0m

[38;2;248;248;242m       select-in-blank-word (iW)[0m
[38;2;248;248;242m              Select  a  word,  where a word is defined as a series of non-blank charac‐[0m
[38;2;248;248;242m              ters. With a numeric argument, multiple words will be selected.[0m

[38;2;248;248;242m       select-in-shell-word (ia)[0m
[38;2;248;248;242m              Select the current command argument applying the normal rules for quoting.[0m
[38;2;248;248;242m              If the argument begins and ends with matching quote characters, these  are[0m
[38;2;248;248;242m              not included in the selection.[0m

[38;2;248;248;242m       select-in-word (iw)[0m
[38;2;248;248;242m              Select  a  word, using the normal vi-style word definition. With a numeric[0m
[38;2;248;248;242m              argument, multiple words will be selected.[0m

[38;2;248;248;242mCHARACTER HIGHLIGHTING[0m
[38;2;248;248;242m       The line editor has the ability to highlight characters or regions  of  the  line[0m
[38;2;248;248;242m       that  have  a particular significance.  This is controlled by the array parameter[0m
[38;2;248;248;242m       zle_highlight, if it has been set by the user.[0m

[38;2;248;248;242m       If the parameter contains the single entry none all highlighting is  turned  off.[0m
[38;2;248;248;242m       Note the parameter is still expected to be an array.[0m

[38;2;248;248;242m       Otherwise  each  entry of the array should consist of a word indicating a context[0m
[38;2;248;248;242m       for highlighting, then a colon, then a comma-separated list of the types of high‐[0m
[38;2;248;248;242m       lighting to apply in that context.[0m

[38;2;248;248;242m       The contexts available for highlighting are the following:[0m

[38;2;248;248;242m       default[0m
[38;2;248;248;242m              Any text within the command line not affected by any  other  highlighting.[0m
[38;2;248;248;242m              Text outside the editable area of the command line is not affected.[0m

[38;2;248;248;242m       isearch[0m
[38;2;248;248;242m              When  one of the incremental history search widgets is active, the area of[0m
[38;2;248;248;242m              the command line matched by the search string or pattern.[0m

[38;2;248;248;242m       region The currently selected text. In emacs terminology, this is referred to  as[0m
[38;2;248;248;242m              the  region  and is bounded by the cursor (point) and the mark. The region[0m
[38;2;248;248;242m              is only highlighted if it is active, which is the case after the  mark  is[0m
[38;2;248;248;242m              modified  with  set-mark-command  or  exchange-point-and-mark.   Note that[0m
[38;2;248;248;242m              whether or not the region is active has no effect on its use within  emacs[0m
[38;2;248;248;242m              style widgets, it simply determines whether it is highlighted. In vi mode,[0m
[38;2;248;248;242m              the region corresponds to selected text in visual mode.[0m

[38;2;248;248;242m       special[0m
[38;2;248;248;242m              Individual characters that have no direct printable representation but are[0m
[38;2;248;248;242m              shown  in  a  special manner by the line editor.  These characters are de‐[0m
[38;2;248;248;242m              scribed below.[0m

[38;2;248;248;242m       suffix This context is used in completion for characters that are marked as  suf‐[0m
[38;2;248;248;242m              fixes  that will be removed if the completion ends at that point, the most[0m
[38;2;248;248;242m              obvious example being a slash (/) after a directory name.  Note that  suf‐[0m
[38;2;248;248;242m              fix removal is configurable; the circumstances under which the suffix will[0m
[38;2;248;248;242m              be removed may differ for different completions.[0m

[38;2;248;248;242m       paste  Following a command to paste text, the characters that were inserted.[0m

[38;2;248;248;242m       When region_highlight is set, the contexts that describe a region -- isearch, re‐[0m
[38;2;248;248;242m       gion,  suffix,  and paste -- are applied first, then region_highlight is applied,[0m
[38;2;248;248;242m       then the remaining zle_highlight contexts are applied.  If a particular character[0m
[38;2;248;248;242m       is affected by multiple specifications, the last specification wins.[0m

[38;2;248;248;242m       zle_highlight may contain additional fields  for  controlling  how  terminal  se‐[0m
[38;2;248;248;242m       quences  to  change  colours  are output.  Each of the following is followed by a[0m
[38;2;248;248;242m       colon and a string in the same form as for key bindings.  This will not be neces‐[0m
[38;2;248;248;242m       sary for the vast majority of terminals as the defaults shown in parentheses  are[0m
[38;2;248;248;242m       widely used.[0m

[38;2;248;248;242m       fg_start_code (\e[3)[0m
[38;2;248;248;242m              The  start of the escape sequence for the foreground colour.  This is fol‐[0m
[38;2;248;248;242m              lowed by one to three ASCII digits representing the colour.  Only used for[0m
[38;2;248;248;242m              palette colors, i.e. not 24-bit colors specified via a color triplet.[0m

[38;2;248;248;242m       fg_default_code (9)[0m
[38;2;248;248;242m              The number to use instead of the colour to reset  the  default  foreground[0m
[38;2;248;248;242m              colour.[0m

[38;2;248;248;242m       fg_end_code (m)[0m
[38;2;248;248;242m              The end of the escape sequence for the foreground colour.[0m

[38;2;248;248;242m       bg_start_code (\e[4)[0m
[38;2;248;248;242m              The  start  of  the  escape  sequence  for  the  background  colour.   See[0m
[38;2;248;248;242m              fg_start_code above.[0m

[38;2;248;248;242m       bg_default_code (9)[0m
[38;2;248;248;242m              The number to use instead of the colour to reset  the  default  background[0m
[38;2;248;248;242m              colour.[0m

[38;2;248;248;242m       bg_end_code (m)[0m
[38;2;248;248;242m              The end of the escape sequence for the background colour.[0m

[38;2;248;248;242m       The  available  types of highlighting are the following.  Note that not all types[0m
[38;2;248;248;242m       of highlighting are available on all terminals:[0m

[38;2;248;248;242m       none   No highlighting is applied to the given context.  It  is  not  useful  for[0m
[38;2;248;248;242m              this  to appear with other types of highlighting; it is used to override a[0m
[38;2;248;248;242m              default.[0m

[38;2;248;248;242m       fg=colour[0m
[38;2;248;248;242m              The foreground colour should be set to colour, a decimal integer, the name[0m
[38;2;248;248;242m              of one of the eight most widely-supported colours or as a `#' followed  by[0m
[38;2;248;248;242m              an RGB triplet in hexadecimal format.[0m

[38;2;248;248;242m              Not  all terminals support this and, of those that do, not all provide fa‐[0m
[38;2;248;248;242m              cilities to test the support, hence the user should decide  based  on  the[0m
[38;2;248;248;242m              terminal type.  Most terminals support the colours black, red, green, yel‐[0m
[38;2;248;248;242m              low,  blue,  magenta,  cyan and white, which can be set by name.  In addi‐[0m
[38;2;248;248;242m              tion. default may be used to set the terminal's default foreground colour.[0m
[38;2;248;248;242m              Abbreviations are allowed; b or bl selects black.  Some terminals may gen‐[0m
[38;2;248;248;242m              erate additional colours if the bold attribute is also present.[0m

[38;2;248;248;242m              On recent terminals and on systems with an  up-to-date  terminal  database[0m
[38;2;248;248;242m              the  number of colours supported may be tested by the command `echotc Co';[0m
[38;2;248;248;242m              if this succeeds, it indicates a limit on the number of colours which will[0m
[38;2;248;248;242m              be enforced by the line editor.  The number of colours is in any case lim‐[0m
[38;2;248;248;242m              ited to 256 (i.e. the range 0 to 255).[0m

[38;2;248;248;242m              Some modern terminal emulators have support for  24-bit  true  colour  (16[0m
[38;2;248;248;242m              million  colours).  In this case, the hex triplet format can be used. This[0m
[38;2;248;248;242m              consists of a `#' followed by either a three or six digit hexadecimal num‐[0m
[38;2;248;248;242m              ber describing the red, green and  blue  components  of  the  colour.  Hex[0m
[38;2;248;248;242m              triplets  can  also  be  used  with  88  and  256 colour terminals via the[0m
[38;2;248;248;242m              zsh/nearcolor module (see zshmodules(1)).[0m

[38;2;248;248;242m              Colour is also known as color.[0m

[38;2;248;248;242m       bg=colour[0m
[38;2;248;248;242m              The background colour should be set to colour.  This  works  similarly  to[0m
[38;2;248;248;242m              the  foreground  colour,  except the background is not usually affected by[0m
[38;2;248;248;242m              the bold attribute.[0m

[38;2;248;248;242m       bold   The characters in the given context are shown in a  bold  font.   Not  all[0m
[38;2;248;248;242m              terminals distinguish bold fonts.[0m

[38;2;248;248;242m       standout[0m
[38;2;248;248;242m              The  characters  in the given context are shown in the terminal's standout[0m
[38;2;248;248;242m              mode.  The actual effect is specific to the terminal; on many terminals it[0m
[38;2;248;248;242m              is inverse video.  On some such terminals, where the cursor does not blink[0m
[38;2;248;248;242m              it appears with standout mode negated, making it less than clear where the[0m
[38;2;248;248;242m              cursor actually is.  On such terminals one of the  other  effects  may  be[0m
[38;2;248;248;242m              preferable for highlighting the region and matched search string.[0m

[38;2;248;248;242m       underline[0m
[38;2;248;248;242m              The  characters in the given context are shown underlined.  Some terminals[0m
[38;2;248;248;242m              show the foreground in a different colour instead; in this case whitespace[0m
[38;2;248;248;242m              will not be highlighted.[0m

[38;2;248;248;242m       The characters described above as `special' are as follows.  The  formatting  de‐[0m
[38;2;248;248;242m       scribed here is used irrespective of whether the characters are highlighted:[0m

[38;2;248;248;242m       ASCII control characters[0m
[38;2;248;248;242m              Control  characters  in  the  ASCII range are shown as `^' followed by the[0m
[38;2;248;248;242m              base character.[0m

[38;2;248;248;242m       Unprintable multibyte characters[0m
[38;2;248;248;242m              This item applies to control characters not in the ASCII range, plus other[0m
[38;2;248;248;242m              characters as follows.  If the MULTIBYTE option is  in  effect,  multibyte[0m
[38;2;248;248;242m              characters not in the ASCII character set that are reported as having zero[0m
[38;2;248;248;242m              width  are treated as combining characters when the option COMBINING_CHARS[0m
[38;2;248;248;242m              is on.  If the option is off, or if a character appears where a  combining[0m
[38;2;248;248;242m              character is not valid, the character is treated as unprintable.[0m

[38;2;248;248;242m              Unprintable multibyte characters are shown as a hexadecimal number between[0m
[38;2;248;248;242m              angle brackets.  The number is the code point of the character in the wide[0m
[38;2;248;248;242m              character  set; this may or may not be Unicode, depending on the operating[0m
[38;2;248;248;242m              system.[0m

[38;2;248;248;242m       Invalid multibyte characters[0m
[38;2;248;248;242m              If the MULTIBYTE option is in effect, any sequence of one  or  more  bytes[0m
[38;2;248;248;242m              that  does  not  form  a  valid  character in the current character set is[0m
[38;2;248;248;242m              treated as a series of bytes each shown as a special character.  This case[0m
[38;2;248;248;242m              can be distinguished from other unprintable characters as  the  bytes  are[0m
[38;2;248;248;242m              represented  as two hexadecimal digits between angle brackets, as distinct[0m
[38;2;248;248;242m              from the four or eight digits that are  used  for  unprintable  characters[0m
[38;2;248;248;242m              that are nonetheless valid in the current character set.[0m

[38;2;248;248;242m              Not  all systems support this: for it to work, the system's representation[0m
[38;2;248;248;242m              of wide characters must be code values from the Universal  Character  Set,[0m
[38;2;248;248;242m              as defined by IS0 10646 (also known as Unicode).[0m

[38;2;248;248;242m       Wrapped double-width characters[0m
[38;2;248;248;242m              When a double-width character appears in the final column of a line, it is[0m
[38;2;248;248;242m              instead shown on the next line. The empty space left in the original posi‐[0m
[38;2;248;248;242m              tion is highlighted as a special character.[0m

[38;2;248;248;242m       If  zle_highlight is not set or no value applies to a particular context, the de‐[0m
[38;2;248;248;242m       faults applied are equivalent to[0m

[38;2;248;248;242m              zle_highlight=(region:standout special:standout[0m
[38;2;248;248;242m              suffix:bold isearch:underline paste:standout)[0m

[38;2;248;248;242m       i.e. both the region and special characters are shown in standout mode.[0m

[38;2;248;248;242m       Within widgets, arbitrary regions may be highlighted by setting the special array[0m
[38;2;248;248;242m       parameter region_highlight; see above.[0m
