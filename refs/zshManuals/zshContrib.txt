[38;2;248;248;242mNAME[0m
[38;2;248;248;242m       zshcontrib - user contributions to zsh[0m

[38;2;248;248;242mDESCRIPTION[0m
[38;2;248;248;242m       The  Zsh  source  distribution includes a number of items contributed by the user[0m
[38;2;248;248;242m       community.  These are not inherently a part of the shell, and  some  may  not  be[0m
[38;2;248;248;242m       available  in  every  zsh  installation.  The most significant of these are docu‐[0m
[38;2;248;248;242m       mented here.  For documentation on other contributed items such  as  shell  func‐[0m
[38;2;248;248;242m       tions, look for comments in the function source files.[0m

[38;2;248;248;242mUTILITIES[0m
[38;2;248;248;242m   Accessing On-Line Help[0m
[38;2;248;248;242m       The  key  sequence  ESC h is normally bound by ZLE to execute the run-help widget[0m
[38;2;248;248;242m       (see zshzle(1)).  This invokes the run-help command with the  command  word  from[0m
[38;2;248;248;242m       the current input line as its argument.  By default, run-help is an alias for the[0m
[38;2;248;248;242m       man  command,  so  this often fails when the command word is a shell builtin or a[0m
[38;2;248;248;242m       user-defined function.  By redefining the run-help alias,  one  can  improve  the[0m
[38;2;248;248;242m       on-line help provided by the shell.[0m

[38;2;248;248;242m       The helpfiles utility, found in the Util directory of the distribution, is a Perl[0m
[38;2;248;248;242m       program  that  can  be  used to process the zsh manual to produce a separate help[0m
[38;2;248;248;242m       file for each shell builtin and for many other shell features as well.   The  au‐[0m
[38;2;248;248;242m       toloadable  run-help  function, found in Functions/Misc, searches for these help‐[0m
[38;2;248;248;242m       files and performs several other tests to produce the most complete help possible[0m
[38;2;248;248;242m       for the command.[0m

[38;2;248;248;242m       Help files are installed by  default  to  a  subdirectory  of  /usr/share/zsh  or[0m
[38;2;248;248;242m       /usr/local/share/zsh.[0m

[38;2;248;248;242m       To  create your own help files with helpfiles, choose or create a directory where[0m
[38;2;248;248;242m       the individual command help files will reside.  For  example,  you  might  choose[0m
[38;2;248;248;242m       ~/zsh_help.   If  you  unpacked  the zsh distribution in your home directory, you[0m
[38;2;248;248;242m       would use the commands:[0m

[38;2;248;248;242m              mkdir ~/zsh_help[0m
[38;2;248;248;242m              perl ~/zsh-5.9/Util/helpfiles ~/zsh_help[0m

[38;2;248;248;242m       The HELPDIR parameter tells run-help where to look for the help files.  When  un‐[0m
[38;2;248;248;242m       set,  it  uses the default installation path.  To use your own set of help files,[0m
[38;2;248;248;242m       set this to the appropriate path in one of your startup files:[0m

[38;2;248;248;242m              HELPDIR=~/zsh_help[0m

[38;2;248;248;242m       To use the run-help function, you need to add lines something like the  following[0m
[38;2;248;248;242m       to your .zshrc or equivalent startup file:[0m

[38;2;248;248;242m              unalias run-help[0m
[38;2;248;248;242m              autoload run-help[0m

[38;2;248;248;242m       Note  that in order for `autoload run-help' to work, the run-help file must be in[0m
[38;2;248;248;242m       one of the directories named in your fpath array (see zshparam(1)).  This  should[0m
[38;2;248;248;242m       already  be  the case if you have a standard zsh installation; if it is not, copy[0m
[38;2;248;248;242m       Functions/Misc/run-help to an appropriate directory.[0m

[38;2;248;248;242m   Recompiling Functions[0m
[38;2;248;248;242m       If you frequently edit your zsh functions, or periodically update  your  zsh  in‐[0m
[38;2;248;248;242m       stallation  to  track the latest developments, you may find that function digests[0m
[38;2;248;248;242m       compiled with the zcompile builtin are frequently out of date with respect to the[0m
[38;2;248;248;242m       function source files.  This is not usually a problem, because zsh  always  looks[0m
[38;2;248;248;242m       for  the  newest  file  when  loading  a  function, but it may cause slower shell[0m
[38;2;248;248;242m       startup and function loading.  Also, if a digest file is explicitly  used  as  an[0m
[38;2;248;248;242m       element of fpath, zsh won't check whether any of its source files has changed.[0m

[38;2;248;248;242m       The  zrecompile  autoloadable  function,  found in Functions/Misc, can be used to[0m
[38;2;248;248;242m       keep function digests up to date.[0m

[38;2;248;248;242m       zrecompile [ -qt ] [ name ... ][0m
[38;2;248;248;242m       zrecompile [ -qt ] -p arg ... [ -- arg ... ][0m
[38;2;248;248;242m              This tries to find *.zwc files and automatically  re-compile  them  if  at[0m
[38;2;248;248;242m              least  one  of  the  original files is newer than the compiled file.  This[0m
[38;2;248;248;242m              works only if the names stored in the compiled files are full paths or are[0m
[38;2;248;248;242m              relative to the directory that contains the .zwc file.[0m

[38;2;248;248;242m              In the first form, each name is the name of a compiled file or a directory[0m
[38;2;248;248;242m              containing *.zwc files that should be checked.  If no arguments are given,[0m
[38;2;248;248;242m              the directories and *.zwc files in fpath are used.[0m

[38;2;248;248;242m              When -t is given, no compilation is performed, but a return status of zero[0m
[38;2;248;248;242m              (true) is set if there are files that need to be re-compiled and  non-zero[0m
[38;2;248;248;242m              (false)  otherwise.  The -q option quiets the chatty output that describes[0m
[38;2;248;248;242m              what zrecompile is doing.[0m

[38;2;248;248;242m              Without the -t option, the return status is zero if all files that  needed[0m
[38;2;248;248;242m              re-compilation  could be compiled and non-zero if compilation for at least[0m
[38;2;248;248;242m              one of the files failed.[0m

[38;2;248;248;242m              If the -p option is given, the args are interpreted as one or more sets of[0m
[38;2;248;248;242m              arguments for zcompile, separated by `--'.  For example:[0m

[38;2;248;248;242m                     zrecompile -p \[0m
[38;2;248;248;242m                                -R ~/.zshrc -- \[0m
[38;2;248;248;242m                                -M ~/.zcompdump -- \[0m
[38;2;248;248;242m                                ~/zsh/comp.zwc ~/zsh/Completion/*/_*[0m

[38;2;248;248;242m              This compiles ~/.zshrc into ~/.zshrc.zwc if that doesn't exist or if it is[0m
[38;2;248;248;242m              older than ~/.zshrc. The compiled file will be marked for reading  instead[0m
[38;2;248;248;242m              of  mapping.  The  same is done for ~/.zcompdump and ~/.zcompdump.zwc, but[0m
[38;2;248;248;242m              this compiled file is marked for mapping. The  last  line  re-creates  the[0m
[38;2;248;248;242m              file  ~/zsh/comp.zwc  if  any  of  the files matching the given pattern is[0m
[38;2;248;248;242m              newer than it.[0m

[38;2;248;248;242m              Without the -p option, zrecompile does not create function digests that do[0m
[38;2;248;248;242m              not already exist, nor does it add new functions to the digest.[0m

[38;2;248;248;242m       The following shell loop is an example of a method for creating function  digests[0m
[38;2;248;248;242m       for  all  functions in your fpath, assuming that you have write permission to the[0m
[38;2;248;248;242m       directories:[0m

[38;2;248;248;242m              for ((i=1; i <= $#fpath; ++i)); do[0m
[38;2;248;248;242m                dir=$fpath[i][0m
[38;2;248;248;242m                zwc=${dir:t}.zwc[0m
[38;2;248;248;242m                if [[ $dir == (.|..) || $dir == (.|..)/* ]]; then[0m
[38;2;248;248;242m                  continue[0m
[38;2;248;248;242m                fi[0m
[38;2;248;248;242m                files=($dir/*(N-.))[0m
[38;2;248;248;242m                if [[ -w $dir:h && -n $files ]]; then[0m
[38;2;248;248;242m                  files=(${${(M)files%/*/*}#/})[0m
[38;2;248;248;242m                  if ( cd $dir:h &&[0m
[38;2;248;248;242m                       zrecompile -p -U -z $zwc $files ); then[0m
[38;2;248;248;242m                    fpath[i]=$fpath[i].zwc[0m
[38;2;248;248;242m                  fi[0m
[38;2;248;248;242m                fi[0m
[38;2;248;248;242m              done[0m

[38;2;248;248;242m       The -U and -z options are appropriate for functions in the default zsh  installa‐[0m
[38;2;248;248;242m       tion  fpath; you may need to use different options for your personal function di‐[0m
[38;2;248;248;242m       rectories.[0m

[38;2;248;248;242m       Once the digests have been created and your fpath modified to refer to them,  you[0m
[38;2;248;248;242m       can keep them up to date by running zrecompile with no arguments.[0m

[38;2;248;248;242m   Keyboard Definition[0m
[38;2;248;248;242m       The  large number of possible combinations of keyboards, workstations, terminals,[0m
[38;2;248;248;242m       emulators, and window systems makes it impossible for zsh to  have  built-in  key[0m
[38;2;248;248;242m       bindings  for  every  situation.   The zkbd utility, found in Functions/Misc, can[0m
[38;2;248;248;242m       help you quickly create key bindings for your configuration.[0m

[38;2;248;248;242m       Run zkbd either as an autoloaded function, or as a shell script:[0m

[38;2;248;248;242m              zsh -f ~/zsh-5.9/Functions/Misc/zkbd[0m

[38;2;248;248;242m       When you run zkbd, it first asks you to enter your terminal type; if the  default[0m
[38;2;248;248;242m       it  offers  is correct, just press return.  It then asks you to press a number of[0m
[38;2;248;248;242m       different keys to determine characteristics of your keyboard and  terminal;  zkbd[0m
[38;2;248;248;242m       warns  you  if  it  finds anything out of the ordinary, such as a Delete key that[0m
[38;2;248;248;242m       sends neither ^H nor ^?.[0m

[38;2;248;248;242m       The keystrokes read by zkbd are recorded as a definition for an associative array[0m
[38;2;248;248;242m       named key, written to a file in the subdirectory .zkbd within either your HOME or[0m
[38;2;248;248;242m       ZDOTDIR directory.  The name of the file is composed from the  TERM,  VENDOR  and[0m
[38;2;248;248;242m       OSTYPE parameters, joined by hyphens.[0m

[38;2;248;248;242m       You may read this file into your .zshrc or another startup file with the `source'[0m
[38;2;248;248;242m       or `.' commands, then reference the key parameter in bindkey commands, like this:[0m

[38;2;248;248;242m              source ${ZDOTDIR:-$HOME}/.zkbd/$TERM-$VENDOR-$OSTYPE[0m
[38;2;248;248;242m              [[ -n ${key[Left]} ]] && bindkey "${key[Left]}" backward-char[0m
[38;2;248;248;242m              [[ -n ${key[Right]} ]] && bindkey "${key[Right]}" forward-char[0m
[38;2;248;248;242m              # etc.[0m

[38;2;248;248;242m       Note  that  in order for `autoload zkbd' to work, the zkdb file must be in one of[0m
[38;2;248;248;242m       the directories named in your fpath array (see zshparam(1)).  This should already[0m
[38;2;248;248;242m       be the case if you have a standard zsh installation; if it  is  not,  copy  Func‐[0m
[38;2;248;248;242m       tions/Misc/zkbd to an appropriate directory.[0m

[38;2;248;248;242m   Dumping Shell State[0m
[38;2;248;248;242m       Occasionally  you  may  encounter what appears to be a bug in the shell, particu‐[0m
[38;2;248;248;242m       larly if you are using a beta version of zsh or a development  release.   Usually[0m
[38;2;248;248;242m       it  is  sufficient to send a description of the problem to one of the zsh mailing[0m
[38;2;248;248;242m       lists (see zsh(1)), but sometimes one of the zsh developers will need to recreate[0m
[38;2;248;248;242m       your environment in order to track the problem down.[0m

[38;2;248;248;242m       The script named reporter, found in the Util directory of  the  distribution,  is[0m
[38;2;248;248;242m       provided  for  this  purpose.  (It is also possible to autoload reporter, but re‐[0m
[38;2;248;248;242m       porter is not installed in fpath by default.)  This  script  outputs  a  detailed[0m
[38;2;248;248;242m       dump of the shell state, in the form of another script that can be read with `zsh[0m
[38;2;248;248;242m       -f' to recreate that state.[0m

[38;2;248;248;242m       To  use  reporter, read the script into your shell with the `.' command and redi‐[0m
[38;2;248;248;242m       rect the output into a file:[0m

[38;2;248;248;242m              . ~/zsh-5.9/Util/reporter > zsh.report[0m

[38;2;248;248;242m       You should check the zsh.report file for any sensitive information such as  pass‐[0m
[38;2;248;248;242m       words and delete them by hand before sending the script to the developers.  Also,[0m
[38;2;248;248;242m       as  the output can be voluminous, it's best to wait for the developers to ask for[0m
[38;2;248;248;242m       this information before sending it.[0m

[38;2;248;248;242m       You can also use reporter to dump only a subset of  the  shell  state.   This  is[0m
[38;2;248;248;242m       sometimes useful for creating startup files for the first time.  Most of the out‐[0m
[38;2;248;248;242m       put  from  reporter  is far more detailed than usually is necessary for a startup[0m
[38;2;248;248;242m       file, but the aliases, options, and zstyles states may be useful because they in‐[0m
[38;2;248;248;242m       clude only changes from the defaults.  The bindings state may be  useful  if  you[0m
[38;2;248;248;242m       have  created  any  of  your  own  keymaps, because reporter arranges to dump the[0m
[38;2;248;248;242m       keymap creation commands as well as the bindings for every keymap.[0m

[38;2;248;248;242m       As is usual with automated tools, if you create a startup file with reporter, you[0m
[38;2;248;248;242m       should edit the results to remove unnecessary commands.  Note that if you're  us‐[0m
[38;2;248;248;242m       ing  the  new  completion system, you should not dump the functions state to your[0m
[38;2;248;248;242m       startup files with reporter; use the  compdump  function  instead  (see  zshcomp‐[0m
[38;2;248;248;242m       sys(1)).[0m

[38;2;248;248;242m       reporter [ state ... ][0m
[38;2;248;248;242m              Print  to standard output the indicated subset of the current shell state.[0m
[38;2;248;248;242m              The state arguments may be one or more of:[0m

[38;2;248;248;242m              all    Output everything listed below.[0m
[38;2;248;248;242m              aliases[0m
[38;2;248;248;242m                     Output alias definitions.[0m
[38;2;248;248;242m              bindings[0m
[38;2;248;248;242m                     Output ZLE key maps and bindings.[0m
[38;2;248;248;242m              completion[0m
[38;2;248;248;242m                     Output old-style compctl commands.  New completion  is  covered  by[0m
[38;2;248;248;242m                     functions and zstyles.[0m
[38;2;248;248;242m              functions[0m
[38;2;248;248;242m                     Output autoloads and function definitions.[0m
[38;2;248;248;242m              limits Output limit commands.[0m
[38;2;248;248;242m              options[0m
[38;2;248;248;242m                     Output setopt commands.[0m
[38;2;248;248;242m              styles Same as zstyles.[0m
[38;2;248;248;242m              variables[0m
[38;2;248;248;242m                     Output  shell  parameter  assignments, plus export commands for any[0m
[38;2;248;248;242m                     environment variables.[0m
[38;2;248;248;242m              zstyles[0m
[38;2;248;248;242m                     Output zstyle commands.[0m

[38;2;248;248;242m              If the state is omitted, all is assumed.[0m

[38;2;248;248;242m       With the exception of `all', every state can be abbreviated by any prefix, even a[0m
[38;2;248;248;242m       single letter; thus a is the same as aliases, z is the same as zstyles, etc.[0m

[38;2;248;248;242m   Manipulating Hook Functions[0m
[38;2;248;248;242m       add-zsh-hook [ -L | -dD ] [ -Uzk ] hook function[0m
[38;2;248;248;242m              Several functions are special to the shell, as described  in  the  section[0m
[38;2;248;248;242m              SPECIAL  FUNCTIONS,  see zshmisc(1), in that they are automatically called[0m
[38;2;248;248;242m              at specific points during shell execution.  Each has an  associated  array[0m
[38;2;248;248;242m              consisting of names of functions to be called at the same point; these are[0m
[38;2;248;248;242m              so-called  `hook  functions'.   The shell function add-zsh-hook provides a[0m
[38;2;248;248;242m              simple way of adding or removing functions from the array.[0m

[38;2;248;248;242m              hook is one of chpwd, periodic, precmd, preexec,  zshaddhistory,  zshexit,[0m
[38;2;248;248;242m              or  zsh_directory_name,  the  special  functions  in  question.  Note that[0m
[38;2;248;248;242m              zsh_directory_name is called in a different way from the other  functions,[0m
[38;2;248;248;242m              but may still be manipulated as a hook.[0m

[38;2;248;248;242m              function  is  name of an ordinary shell function.  If no options are given[0m
[38;2;248;248;242m              this will be added to the array of functions to be executed in  the  given[0m
[38;2;248;248;242m              context.  Functions are invoked in the order they were added.[0m

[38;2;248;248;242m              If  the  option  -L  is  given, the current values for the hook arrays are[0m
[38;2;248;248;242m              listed with typeset.[0m

[38;2;248;248;242m              If the option -d is given, the function is removed from the array of func‐[0m
[38;2;248;248;242m              tions to be executed.[0m

[38;2;248;248;242m              If the option -D is given, the function is treated as a  pattern  and  any[0m
[38;2;248;248;242m              matching  names of functions are removed from the array of functions to be[0m
[38;2;248;248;242m              executed.[0m

[38;2;248;248;242m              The options -U, -z and -k are passed as arguments to  autoload  for  func‐[0m
[38;2;248;248;242m              tion.   For  functions contributed with zsh, the options -Uz are appropri‐[0m
[38;2;248;248;242m              ate.[0m

[38;2;248;248;242m       add-zle-hook-widget [ -L | -dD ] [ -Uzk ] hook widgetname[0m
[38;2;248;248;242m              Several widget names are special to the line editor, as described  in  the[0m
[38;2;248;248;242m              section  Special  Widgets,  see  zshzle(1), in that they are automatically[0m
[38;2;248;248;242m              called at specific points during editing.  Unlike function hooks, these do[0m
[38;2;248;248;242m              not use a predefined array of other names to call at the same  point;  the[0m
[38;2;248;248;242m              shell  function add-zle-hook-widget maintains a similar array and arranges[0m
[38;2;248;248;242m              for the special widget to invoke those additional widgets.[0m

[38;2;248;248;242m              hook is one of isearch-exit, isearch-update,  line-pre-redraw,  line-init,[0m
[38;2;248;248;242m              line-finish,  history-line-set, or keymap-select, corresponding to each of[0m
[38;2;248;248;242m              the special widgets zle-isearch-exit, etc.  The special widget  names  are[0m
[38;2;248;248;242m              also accepted as the hook argument.[0m

[38;2;248;248;242m              widgetname  is  the name of a ZLE widget.  If no options are given this is[0m
[38;2;248;248;242m              added to the array of widgets to be invoked in  the  given  hook  context.[0m
[38;2;248;248;242m              Widgets are invoked in the order they were added, with[0m
[38;2;248;248;242m                     zle widgetname -Nw -f "nolast" -- "$@"[0m

[38;2;248;248;242m              Note  that  this means that the `WIDGET' special parameter tracks the wid‐[0m
[38;2;248;248;242m              getname when the widget function is called, rather than tracking the  name[0m
[38;2;248;248;242m              of the corresponding special hook widget.[0m

[38;2;248;248;242m              If  the  option  -d  is given, the widgetname is removed from the array of[0m
[38;2;248;248;242m              widgets to be executed.[0m

[38;2;248;248;242m              If the option -D is given, the widgetname is treated as a pattern and  any[0m
[38;2;248;248;242m              matching names of widgets are removed from the array.[0m

[38;2;248;248;242m              If widgetname does not name an existing widget when added to the array, it[0m
[38;2;248;248;242m              is assumed that a shell function also named widgetname is meant to provide[0m
[38;2;248;248;242m              the  implementation  of the widget.  This name is therefore marked for au‐[0m
[38;2;248;248;242m              toloading, and the options -U, -z and -k are passed as  arguments  to  au‐[0m
[38;2;248;248;242m              toload as with add-zsh-hook.  The widget is also created with `zle -N wid‐[0m
[38;2;248;248;242m              getname'  to  cause the corresponding function to be loaded the first time[0m
[38;2;248;248;242m              the hook is called.[0m

[38;2;248;248;242m              The arrays of widgetname are currently maintained in zstyle contexts,  one[0m
[38;2;248;248;242m              for  each  hook  context,  with a style of `widgets'.  If the -L option is[0m
[38;2;248;248;242m              given, this set of styles is listed with `zstyle -L'.  This implementation[0m
[38;2;248;248;242m              may change, and the special widgets that refer to the styles  are  created[0m
[38;2;248;248;242m              only  if  add-zle-hook-widget  is called to add at least one widget, so if[0m
[38;2;248;248;242m              this function is used for any hooks, then all hooks should be managed only[0m
[38;2;248;248;242m              via this function.[0m

[38;2;248;248;242mREMEMBERING RECENT DIRECTORIES[0m
[38;2;248;248;242m       The function cdr allows you to change the working directory to a previous working[0m
[38;2;248;248;242m       directory from a list maintained automatically.  It is similar in concept to  the[0m
[38;2;248;248;242m       directory stack controlled by the pushd, popd and dirs builtins, but is more con‐[0m
[38;2;248;248;242m       figurable, and as it stores all entries in files it is maintained across sessions[0m
[38;2;248;248;242m       and  (by  default) between terminal emulators in the current session.  Duplicates[0m
[38;2;248;248;242m       are automatically removed, so that the list reflects the single most  recent  use[0m
[38;2;248;248;242m       of each directory.[0m

[38;2;248;248;242m       Note  that  the pushd directory stack is not actually modified or used by cdr un‐[0m
[38;2;248;248;242m       less you configure it to do so as described in the configuration section below.[0m

[38;2;248;248;242m   Installation[0m
[38;2;248;248;242m       The system works by means of a hook function that is called every time the direc‐[0m
[38;2;248;248;242m       tory changes.  To install the system, autoload the required functions and use the[0m
[38;2;248;248;242m       add-zsh-hook function described above:[0m

[38;2;248;248;242m              autoload -Uz chpwd_recent_dirs cdr add-zsh-hook[0m
[38;2;248;248;242m              add-zsh-hook chpwd chpwd_recent_dirs[0m

[38;2;248;248;242m       Now every time you change directly interactively, no  matter  which  command  you[0m
[38;2;248;248;242m       use,  the  directory  to which you change will be remembered in most-recent-first[0m
[38;2;248;248;242m       order.[0m

[38;2;248;248;242m   Use[0m
[38;2;248;248;242m       All direct user interaction is via the cdr function.[0m

[38;2;248;248;242m       The argument to cdr is  a  number  N  corresponding  to  the  Nth  most  recently[0m
[38;2;248;248;242m       changed-to  directory.  1 is the immediately preceding directory; the current di‐[0m
[38;2;248;248;242m       rectory is remembered but is not offered as a destination.  Note that if you have[0m
[38;2;248;248;242m       multiple windows open 1 may refer to a directory changed to  in  another  window;[0m
[38;2;248;248;242m       you  can  avoid  this  by  having per-terminal files for storing directory as de‐[0m
[38;2;248;248;242m       scribed for the recent-dirs-file style below.[0m

[38;2;248;248;242m       If you set the recent-dirs-default style described below cdr will behave the same[0m
[38;2;248;248;242m       as cd if given a non-numeric argument, or more than one argument.  The recent di‐[0m
[38;2;248;248;242m       rectory list is updated just the same however you change directory.[0m

[38;2;248;248;242m       If the argument is omitted, 1 is assumed.  This is similar to  pushd's  behaviour[0m
[38;2;248;248;242m       of swapping the two most recent directories on the stack.[0m

[38;2;248;248;242m       Completion  for  the  argument to cdr is available if compinit has been run; menu[0m
[38;2;248;248;242m       selection is recommended, using:[0m

[38;2;248;248;242m              zstyle ':completion:*:*:cdr:*:*' menu selection[0m

[38;2;248;248;242m       to allow you to cycle through recent directories; the order is preserved, so  the[0m
[38;2;248;248;242m       first  choice  is  the most recent directory before the current one.  The verbose[0m
[38;2;248;248;242m       style is also recommended to ensure the directory is shown; this style is  on  by[0m
[38;2;248;248;242m       default so no action is required unless you have changed it.[0m

[38;2;248;248;242m   Options[0m
[38;2;248;248;242m       The behaviour of cdr may be modified by the following options.[0m

[38;2;248;248;242m       -l     lists  the  numbers  and the corresponding directories in abbreviated form[0m
[38;2;248;248;242m              (i.e. with ~ substitution reapplied), one per line.  The directories  here[0m
[38;2;248;248;242m              are  not quoted (this would only be an issue if a directory name contained[0m
[38;2;248;248;242m              a newline).  This is used by the completion system.[0m

[38;2;248;248;242m       -r     sets the variable reply to the current set  of  directories.   Nothing  is[0m
[38;2;248;248;242m              printed and the directory is not changed.[0m

[38;2;248;248;242m       -e     allows you to edit the list of directories, one per line.  The list can be[0m
[38;2;248;248;242m              edited  to  any extent you like; no sanity checking is performed.  Comple‐[0m
[38;2;248;248;242m              tion is available.  No quoting is necessary (except for newlines, where  I[0m
[38;2;248;248;242m              have  in  any case no sympathy); directories are in unabbreviated form and[0m
[38;2;248;248;242m              contain an absolute path, i.e. they start with /.  Usually the first entry[0m
[38;2;248;248;242m              should be left as the current directory.[0m

[38;2;248;248;242m       -p 'pattern'[0m
[38;2;248;248;242m              Prunes any items in the directory list that match the given extended  glob[0m
[38;2;248;248;242m              pattern;  the  pattern  needs to be quoted from immediate expansion on the[0m
[38;2;248;248;242m              command line.  The pattern is matched  against  each  completely  expanded[0m
[38;2;248;248;242m              file name in the list; the full string must match, so wildcards at the end[0m
[38;2;248;248;242m              (e.g. '*removeme*') are needed to remove entries with a given substring.[0m

[38;2;248;248;242m              If  output is to a terminal, then the function will print the new list af‐[0m
[38;2;248;248;242m              ter pruning and prompt for confirmation by the user.  This output and con‐[0m
[38;2;248;248;242m              firmation step can be skipped by using -P instead of -p.[0m

[38;2;248;248;242m   Configuration[0m
[38;2;248;248;242m       Configuration is by means of the styles mechanism that should  be  familiar  from[0m
[38;2;248;248;242m       completion;  if  not,  see  the  description of the zstyle command in see zshmod‐[0m
[38;2;248;248;242m       ules(1).  The context for setting styles should be ':chpwd:*' in case the meaning[0m
[38;2;248;248;242m       of the context is extended in future, for example:[0m

[38;2;248;248;242m              zstyle ':chpwd:*' recent-dirs-max 0[0m

[38;2;248;248;242m       sets the value of the recent-dirs-max style to 0.  In practice the style name  is[0m
[38;2;248;248;242m       specific enough that a context of '*' should be fine.[0m

[38;2;248;248;242m       An  exception  is recent-dirs-insert, which is used exclusively by the completion[0m
[38;2;248;248;242m       system and so has the usual completion system context (':completion:*' if nothing[0m
[38;2;248;248;242m       more specific is needed), though again '*' should be fine in practice.[0m

[38;2;248;248;242m       recent-dirs-default[0m
[38;2;248;248;242m              If true, and the command is expecting a recent directory index, and either[0m
[38;2;248;248;242m              there is more than one argument or the argument is not  an  integer,  then[0m
[38;2;248;248;242m              fall  through  to  "cd".  This allows the lazy to use only one command for[0m
[38;2;248;248;242m              directory changing.  Completion recognises this, too; see  recent-dirs-in‐[0m
[38;2;248;248;242m              sert for how to control completion when this option is in use.[0m

[38;2;248;248;242m       recent-dirs-file[0m
[38;2;248;248;242m              The  file  where the list of directories is saved.  The default is ${ZDOT‐[0m
[38;2;248;248;242m              DIR:-$HOME}/.chpwd-recent-dirs, i.e. this is in your home directory unless[0m
[38;2;248;248;242m              you have set the variable ZDOTDIR  to  point  somewhere  else.   Directory[0m
[38;2;248;248;242m              names  are  saved  in  $'...' quoted form, so each line in the file can be[0m
[38;2;248;248;242m              supplied directly to the shell as an argument.[0m

[38;2;248;248;242m              The value of this style may be an array.  In this case, the first file  in[0m
[38;2;248;248;242m              the  list will always be used for saving directories while any other files[0m
[38;2;248;248;242m              are left untouched.  When reading the recent directory list, if there  are[0m
[38;2;248;248;242m              fewer  than  the maximum number of entries in the first file, the contents[0m
[38;2;248;248;242m              of later files in the array will be appended with duplicates removed  from[0m
[38;2;248;248;242m              the  list  shown.   The contents of the two files are not sorted together,[0m
[38;2;248;248;242m              i.e. all the entries in the first file are shown first.  The special value[0m
[38;2;248;248;242m              + can appear in the list to indicate the default file should  be  read  at[0m
[38;2;248;248;242m              that point.  This allows effects like the following:[0m

[38;2;248;248;242m                     zstyle ':chpwd:*' recent-dirs-file \[0m
[38;2;248;248;242m                     ~/.chpwd-recent-dirs-${TTY##*/} +[0m

[38;2;248;248;242m              Recent  directories  are read from a file numbered according to the termi‐[0m
[38;2;248;248;242m              nal.  If there are insufficient entries the list is supplemented from  the[0m
[38;2;248;248;242m              default file.[0m

[38;2;248;248;242m              It  is possible to use zstyle -e to make the directory configurable at run[0m
[38;2;248;248;242m              time:[0m

[38;2;248;248;242m                     zstyle -e ':chpwd:*' recent-dirs-file pick-recent-dirs-file[0m
[38;2;248;248;242m                     pick-recent-dirs-file() {[0m
[38;2;248;248;242m                       if [[ $PWD = ~/text/writing(|/*) ]]; then[0m
[38;2;248;248;242m                         reply=(~/.chpwd-recent-dirs-writing)[0m
[38;2;248;248;242m                       else[0m
[38;2;248;248;242m                         reply=(+)[0m
[38;2;248;248;242m                       fi[0m
[38;2;248;248;242m                     }[0m

[38;2;248;248;242m              In this example, if the current directory is ~/text/writing or a directory[0m
[38;2;248;248;242m              under it, then use a special file for saving recent directories, else  use[0m
[38;2;248;248;242m              the default.[0m

[38;2;248;248;242m       recent-dirs-insert[0m
[38;2;248;248;242m              Used  by completion.  If recent-dirs-default is true, then setting this to[0m
[38;2;248;248;242m              true causes the actual directory, rather than its index, to be inserted on[0m
[38;2;248;248;242m              the command line; this has the same effect as using the corresponding  in‐[0m
[38;2;248;248;242m              dex, but makes the history clearer and the line easier to edit.  With this[0m
[38;2;248;248;242m              setting,  if  part of an argument was already typed, normal directory com‐[0m
[38;2;248;248;242m              pletion rather than recent directory completion is done; this  is  because[0m
[38;2;248;248;242m              recent  directory completion is expected to be done by cycling through en‐[0m
[38;2;248;248;242m              tries menu fashion.[0m

[38;2;248;248;242m              If the value of the style is always, then only recent directories will  be[0m
[38;2;248;248;242m              completed;  in  that  case,  use  the cd command when you want to complete[0m
[38;2;248;248;242m              other directories.[0m

[38;2;248;248;242m              If the value is fallback, recent directories will  be  tried  first,  then[0m
[38;2;248;248;242m              normal  directory  completion  is performed if recent directory completion[0m
[38;2;248;248;242m              failed to find a match.[0m

[38;2;248;248;242m              Finally, if the value is both then both sets of completions are presented;[0m
[38;2;248;248;242m              the usual tag mechanism can be used to distinguish  results,  with  recent[0m
[38;2;248;248;242m              directories  tagged  as recent-dirs.  Note that the recent directories in‐[0m
[38;2;248;248;242m              serted are abbreviated with directory names where appropriate.[0m

[38;2;248;248;242m       recent-dirs-max[0m
[38;2;248;248;242m              The maximum number of directories to save to the file.  If this is zero or[0m
[38;2;248;248;242m              negative there is no maximum.  The default is 20.  Note this includes  the[0m
[38;2;248;248;242m              current  directory, which isn't offered, so the highest number of directo‐[0m
[38;2;248;248;242m              ries you will be offered is one less than the maximum.[0m

[38;2;248;248;242m       recent-dirs-prune[0m
[38;2;248;248;242m              This style is an array determining what directories should (or should not)[0m
[38;2;248;248;242m              be added to the recent list.  Elements of the array can include:[0m

[38;2;248;248;242m              parent Prune parents (more accurately, ancestors) from  the  recent  list.[0m
[38;2;248;248;242m                     If  present,  changing  directly  down by any number of directories[0m
[38;2;248;248;242m                     causes the current  directory  to  be  overwritten.   For  example,[0m
[38;2;248;248;242m                     changing  from  ~pws  to  ~pws/some/other/dir causes ~pws not to be[0m
[38;2;248;248;242m                     left on the recent directory stack.  This only  applies  to  direct[0m
[38;2;248;248;242m                     changes  to descendant directories; earlier directories on the list[0m
[38;2;248;248;242m                     are not pruned.  For example,  changing  from  ~pws/yet/another  to[0m
[38;2;248;248;242m                     ~pws/some/other/dir does not cause ~pws to be pruned.[0m

[38;2;248;248;242m              pattern:pattern[0m
[38;2;248;248;242m                     Gives a zsh pattern for directories that should not be added to the[0m
[38;2;248;248;242m                     recent  list  (if not already there).  This element can be repeated[0m
[38;2;248;248;242m                     to add different patterns.  For example, 'pattern:/tmp(|/*)'  stops[0m
[38;2;248;248;242m                     /tmp or its descendants from being added.  The EXTENDED_GLOB option[0m
[38;2;248;248;242m                     is always turned on for these patterns.[0m

[38;2;248;248;242m       recent-dirs-pushd[0m
[38;2;248;248;242m              If  set to true, cdr will use pushd instead of cd to change the directory,[0m
[38;2;248;248;242m              so the directory is saved on the directory stack.  As the directory  stack[0m
[38;2;248;248;242m              is  completely separate from the list of files saved by the mechanism used[0m
[38;2;248;248;242m              in this file there is no obvious reason to do this.[0m

[38;2;248;248;242m   Use with dynamic directory naming[0m
[38;2;248;248;242m       It is possible to refer to recent directories using the  dynamic  directory  name[0m
[38;2;248;248;242m       syntax by using the supplied function zsh_directory_name_cdr a hook:[0m

[38;2;248;248;242m              autoload -Uz add-zsh-hook[0m
[38;2;248;248;242m              add-zsh-hook -Uz zsh_directory_name zsh_directory_name_cdr[0m

[38;2;248;248;242m       When  this is done, ~[1] will refer to the most recent directory other than $PWD,[0m
[38;2;248;248;242m       and so on.  Completion after ~[...  also works.[0m

[38;2;248;248;242m   Details of directory handling[0m
[38;2;248;248;242m       This section is for the curious or confused; most users will  not  need  to  know[0m
[38;2;248;248;242m       this information.[0m

[38;2;248;248;242m       Recent directories are saved to a file immediately and hence are preserved across[0m
[38;2;248;248;242m       sessions.  Note currently no file locking is applied: the list is updated immedi‐[0m
[38;2;248;248;242m       ately  on  interactive  commands and nowhere else (unlike history), and it is as‐[0m
[38;2;248;248;242m       sumed you are only going to change directory in one window at once.  This is  not[0m
[38;2;248;248;242m       safe  on  shared  accounts,  but  in any case the system has limited utility when[0m
[38;2;248;248;242m       someone else is changing to a different set of directories behind your back.[0m

[38;2;248;248;242m       To make this a little safer, only directory changes instituted from  the  command[0m
[38;2;248;248;242m       line, either directly or indirectly through shell function calls (but not through[0m
[38;2;248;248;242m       subshells,  evals,  traps,  completion  functions and the like) are saved.  Shell[0m
[38;2;248;248;242m       functions should use cd -q or pushd -q to avoid side effects if the change to the[0m
[38;2;248;248;242m       directory is to be invisible at the command line.  See the contents of the  func‐[0m
[38;2;248;248;242m       tion chpwd_recent_dirs for more details.[0m

[38;2;248;248;242mABBREVIATED DYNAMIC REFERENCES TO DIRECTORIES[0m
[38;2;248;248;242m       The  dynamic directory naming system is described in the subsection Dynamic named[0m
[38;2;248;248;242m       directories of the section Filename Expansion in zshexpn(1).  In this,  a  refer‐[0m
[38;2;248;248;242m       ence to ~[...] is expanded by a function found by the hooks mechanism.[0m

[38;2;248;248;242m       The  contributed  function  zsh_directory_name_generic provides a system allowing[0m
[38;2;248;248;242m       the user to refer to directories with only a limited amount of new code.  It sup‐[0m
[38;2;248;248;242m       ports all three of the standard interfaces for directory naming: converting  from[0m
[38;2;248;248;242m       a  name to a directory, converting in the reverse direction to find a short name,[0m
[38;2;248;248;242m       and completion of names.[0m

[38;2;248;248;242m       The main feature of this function is a path-like syntax, combining  abbreviations[0m
[38;2;248;248;242m       at multiple levels separated by ":".  As an example, ~[g:p:s] might specify:[0m
[38;2;248;248;242m       g      The  top  level  directory for your git area.  This first component has to[0m
[38;2;248;248;242m              match, or the function will return indicating another directory name  hook[0m
[38;2;248;248;242m              function should be tried.[0m

[38;2;248;248;242m       p      The name of a project within your git area.[0m

[38;2;248;248;242m       s      The  source  area within that project.  This allows you to collapse refer‐[0m
[38;2;248;248;242m              ences to long hierarchies to a very compact form, particularly if the  hi‐[0m
[38;2;248;248;242m              erarchies are similar across different areas of the disk.[0m

[38;2;248;248;242m       Name  components  may  be  completed: if a description is shown at the top of the[0m
[38;2;248;248;242m       list of completions, it includes the path to which  previous  components  expand,[0m
[38;2;248;248;242m       while  the  description  for  an  individual completion shows the path segment it[0m
[38;2;248;248;242m       would add.  No additional configuration is needed for this as the completion sys‐[0m
[38;2;248;248;242m       tem is aware of the dynamic directory name mechanism.[0m

[38;2;248;248;242m   Usage[0m
[38;2;248;248;242m       To use the function, first define a wrapper  function  for  your  specific  case.[0m
[38;2;248;248;242m       We'll assume it's to be autoloaded.  This can have any name but we'll refer to it[0m
[38;2;248;248;242m       as  zdn_mywrapper.   This wrapper function will define various variables and then[0m
[38;2;248;248;242m       call this function with the same arguments that the wrapper function gets.   This[0m
[38;2;248;248;242m       configuration is described below.[0m

[38;2;248;248;242m       Then arrange for the wrapper to be run as a zsh_directory_name hook:[0m

[38;2;248;248;242m              autoload -Uz add-zsh-hook zsh_directory_name_generic zdn_mywrapper[0m
[38;2;248;248;242m              add-zsh-hook -U zsh_directory_name zdn_mywrapper[0m

[38;2;248;248;242m   Configuration[0m
[38;2;248;248;242m       The  wrapper  function should define a local associative array zdn_top.  Alterna‐[0m
[38;2;248;248;242m       tively, this can be set with a style called mapping.  The context for  the  style[0m
[38;2;248;248;242m       is  :zdn:wrapper-name  where  wrapper-name  is  the  function  calling zsh_direc‐[0m
[38;2;248;248;242m       tory_name_generic; for example:[0m

[38;2;248;248;242m              zstyle :zdn:zdn_mywrapper: mapping zdn_mywrapper_top[0m

[38;2;248;248;242m       The keys in this associative array correspond to the first component of the name.[0m
[38;2;248;248;242m       The values are matching directories.  They may have an  optional  suffix  with  a[0m
[38;2;248;248;242m       slash  followed  by a colon and the name of a variable in the same format to give[0m
[38;2;248;248;242m       the next component.  (The slash before the colon  is  to  disambiguate  the  case[0m
[38;2;248;248;242m       where  a  colon  is needed in the path for a drive.  There is otherwise no syntax[0m
[38;2;248;248;242m       for escaping this, so path components whose names start with a colon are not sup‐[0m
[38;2;248;248;242m       ported.)  A special component :default: specifies a variable in  the  form  /:var[0m
[38;2;248;248;242m       (the  path  section is ignored and so is usually empty) that will be used for the[0m
[38;2;248;248;242m       next component if no variable is given  for  the  path.   Variables  referred  to[0m
[38;2;248;248;242m       within  zdn_top  have  the  same  format  as zdn_top itself, but contain relative[0m
[38;2;248;248;242m       paths.[0m

[38;2;248;248;242m       For example,[0m

[38;2;248;248;242m              local -A zdn_top=([0m
[38;2;248;248;242m                g   ~/git[0m
[38;2;248;248;242m                ga  ~/alternate/git[0m
[38;2;248;248;242m                gs  /scratch/$USER/git/:second2[0m
[38;2;248;248;242m                :default: /:second1[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m       This specifies the behaviour of a directory referred to as ~[g:...]  or ~[ga:...][0m
[38;2;248;248;242m       or ~[gs:...].  Later path components are optional; in that case ~[g]  expands  to[0m
[38;2;248;248;242m       ~/git,  and so on.  gs expands to /scratch/$USER/git and uses the associative ar‐[0m
[38;2;248;248;242m       ray second2 to match the second component; g and ga  use  the  associative  array[0m
[38;2;248;248;242m       second1 to match the second component.[0m

[38;2;248;248;242m       When  expanding  a  name to a directory, if the first component is not g or ga or[0m
[38;2;248;248;242m       gs, it is not an error; the function simply returns 1 so that a later hook  func‐[0m
[38;2;248;248;242m       tion  can  be tried.  However, matching the first component commits the function,[0m
[38;2;248;248;242m       so if a later component does not match, an error is printed  (though  this  still[0m
[38;2;248;248;242m       does not stop later hooks from being executed).[0m

[38;2;248;248;242m       For components after the first, a relative path is expected, but note that multi‐[0m
[38;2;248;248;242m       ple levels may still appear.  Here is an example of second1:[0m

[38;2;248;248;242m              local -A second1=([0m
[38;2;248;248;242m                p   myproject[0m
[38;2;248;248;242m                s   somproject[0m
[38;2;248;248;242m                os  otherproject/subproject/:third[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m       The path as found from zdn_top is extended with the matching directory, so ~[g:p][0m
[38;2;248;248;242m       becomes ~/git/myproject.  The slash between is added automatically (it's not pos‐[0m
[38;2;248;248;242m       sible  to have a later component modify the name of a directory already matched).[0m
[38;2;248;248;242m       Only os specifies a variable for a third component, and there's no :default:,  so[0m
[38;2;248;248;242m       it's an error to use a name like ~[g:p:x] or ~[ga:s:y] because there's nowhere to[0m
[38;2;248;248;242m       look up the x or y.[0m

[38;2;248;248;242m       The associative arrays need to be visible within this function; the generic func‐[0m
[38;2;248;248;242m       tion  therefore  uses  internal  variable names beginning _zdn_ in order to avoid[0m
[38;2;248;248;242m       clashes.  Note that the variable reply needs to be passed back to the  shell,  so[0m
[38;2;248;248;242m       should not be local in the calling function.[0m

[38;2;248;248;242m       The  function  does  not test whether directories assembled by component actually[0m
[38;2;248;248;242m       exist; this allows the system to work across automounted file systems.  The error[0m
[38;2;248;248;242m       from the command trying to use a non-existent directory should be  sufficient  to[0m
[38;2;248;248;242m       indicate the problem.[0m

[38;2;248;248;242m   Complete example[0m
[38;2;248;248;242m       Here is a full fictitious but usable autoloadable definition of the example func‐[0m
[38;2;248;248;242m       tion    defined    by    the    code    above.     So    ~[gs:p:s]   expands   to[0m
[38;2;248;248;242m       /scratch/$USER/git/myscratchproject/top/srcdir (with $USER also expanded).[0m

[38;2;248;248;242m              local -A zdn_top=([0m
[38;2;248;248;242m                g   ~/git[0m
[38;2;248;248;242m                ga  ~/alternate/git[0m
[38;2;248;248;242m                gs  /scratch/$USER/git/:second2[0m
[38;2;248;248;242m                :default: /:second1[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m              local -A second1=([0m
[38;2;248;248;242m                p   myproject[0m
[38;2;248;248;242m                s   somproject[0m
[38;2;248;248;242m                os  otherproject/subproject/:third[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m              local -A second2=([0m
[38;2;248;248;242m                p   myscratchproject[0m
[38;2;248;248;242m                s   somescratchproject[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m              local -A third=([0m
[38;2;248;248;242m                s   top/srcdir[0m
[38;2;248;248;242m                d   top/documentation[0m
[38;2;248;248;242m              )[0m

[38;2;248;248;242m              # autoload not needed if you did this at initialisation...[0m
[38;2;248;248;242m              autoload -Uz zsh_directory_name_generic[0m
[38;2;248;248;242m              zsh_directory_name_generic "$@[0m

[38;2;248;248;242m       It is also possible to use global associative arrays, suitably named, and set the[0m
[38;2;248;248;242m       style for the context of your wrapper function to refer to this.  Then  your  set[0m
[38;2;248;248;242m       up code would contain the following:[0m

[38;2;248;248;242m              typeset -A zdn_mywrapper_top=(...)[0m
[38;2;248;248;242m              # ... and so on for other associative arrays ...[0m
[38;2;248;248;242m              zstyle ':zdn:zdn_mywrapper:' mapping zdn_mywrapper_top[0m
[38;2;248;248;242m              autoload -Uz add-zsh-hook zsh_directory_name_generic zdn_mywrapper[0m
[38;2;248;248;242m              add-zsh-hook -U zsh_directory_name zdn_mywrapper[0m

[38;2;248;248;242m       and the function zdn_mywrapper would contain only the following:[0m

[38;2;248;248;242m              zsh_directory_name_generic "$@"[0m

[38;2;248;248;242mGATHERING INFORMATION FROM VERSION CONTROL SYSTEMS[0m
[38;2;248;248;242m       In  a lot of cases, it is nice to automatically retrieve information from version[0m
[38;2;248;248;242m       control systems (VCSs), such as subversion, CVS or git, to be able to provide  it[0m
[38;2;248;248;242m       to  the user; possibly in the user's prompt. So that you can instantly tell which[0m
[38;2;248;248;242m       branch you are currently on, for example.[0m

[38;2;248;248;242m       In order to do that, you may use the vcs_info function.[0m

[38;2;248;248;242m       The following VCSs are supported, showing the abbreviated name by which they  are[0m
[38;2;248;248;242m       referred to within the system:[0m
[38;2;248;248;242m       Bazaar (bzr)[0m
[38;2;248;248;242m              https://bazaar.canonical.com/[0m
[38;2;248;248;242m       Codeville (cdv)[0m
[38;2;248;248;242m              http://freecode.com/projects/codeville/[0m
[38;2;248;248;242m       Concurrent Versioning System (cvs)[0m
[38;2;248;248;242m              https://www.nongnu.org/cvs/[0m
[38;2;248;248;242m       Darcs (darcs)[0m
[38;2;248;248;242m              http://darcs.net/[0m
[38;2;248;248;242m       Fossil (fossil)[0m
[38;2;248;248;242m              https://fossil-scm.org/[0m
[38;2;248;248;242m       Git (git)[0m
[38;2;248;248;242m              https://git-scm.com/[0m
[38;2;248;248;242m       GNU arch (tla)[0m
[38;2;248;248;242m              https://www.gnu.org/software/gnu-arch/[0m
[38;2;248;248;242m       Mercurial (hg)[0m
[38;2;248;248;242m              https://www.mercurial-scm.org/[0m
[38;2;248;248;242m       Monotone (mtn)[0m
[38;2;248;248;242m              https://monotone.ca/[0m
[38;2;248;248;242m       Perforce (p4)[0m
[38;2;248;248;242m              https://www.perforce.com/[0m
[38;2;248;248;242m       Subversion (svn)[0m
[38;2;248;248;242m              https://subversion.apache.org/[0m
[38;2;248;248;242m       SVK (svk)[0m
[38;2;248;248;242m              https://svk.bestpractical.com/[0m

[38;2;248;248;242m       There  is  also  support  for  the  patch management system quilt (https://savan‐[0m
[38;2;248;248;242m       nah.nongnu.org/projects/quilt). See Quilt Support below for details.[0m

[38;2;248;248;242m       To load vcs_info:[0m

[38;2;248;248;242m              autoload -Uz vcs_info[0m

[38;2;248;248;242m       It can be used in any existing prompt, because it does not require  any  specific[0m
[38;2;248;248;242m       $psvar entries to be available.[0m

[38;2;248;248;242m   Quickstart[0m
[38;2;248;248;242m       To  get this feature working quickly (including colors), you can do the following[0m
[38;2;248;248;242m       (assuming, you loaded vcs_info properly - see above):[0m

[38;2;248;248;242m              zstyle ':vcs_info:*' actionformats \[0m
[38;2;248;248;242m                  '%F{5}(%f%s%F{5})%F{3}-%F{5}[%F{2}%b%F{3}|%F{1}%a%F{5}]%f '[0m
[38;2;248;248;242m              zstyle ':vcs_info:*' formats       \[0m
[38;2;248;248;242m                  '%F{5}(%f%s%F{5})%F{3}-%F{5}[%F{2}%b%F{5}]%f '[0m
[38;2;248;248;242m              zstyle ':vcs_info:(sv[nk]|bzr):*' branchformat '%b%F{1}:%F{3}%r'[0m
[38;2;248;248;242m              precmd () { vcs_info }[0m
[38;2;248;248;242m              PS1='%F{5}[%F{2}%n%F{5}] %F{3}%3~ ${vcs_info_msg_0_}%f%# '[0m

[38;2;248;248;242m       Obviously, the last two lines are there  for  demonstration.  You  need  to  call[0m
[38;2;248;248;242m       vcs_info  from  your  precmd function. Once that is done you need a single quoted[0m
[38;2;248;248;242m       '${vcs_info_msg_0_}' in your prompt.[0m

[38;2;248;248;242m       To be able to use '${vcs_info_msg_0_}' directly in your  prompt  like  this,  you[0m
[38;2;248;248;242m       will need to have the PROMPT_SUBST option enabled.[0m

[38;2;248;248;242m       Now call the vcs_info_printsys utility from the command line:[0m

[38;2;248;248;242m              % vcs_info_printsys[0m
[38;2;248;248;242m              ## list of supported version control backends:[0m
[38;2;248;248;242m              ## disabled systems are prefixed by a hash sign (#)[0m
[38;2;248;248;242m              bzr[0m
[38;2;248;248;242m              cdv[0m
[38;2;248;248;242m              cvs[0m
[38;2;248;248;242m              darcs[0m
[38;2;248;248;242m              fossil[0m
[38;2;248;248;242m              git[0m
[38;2;248;248;242m              hg[0m
[38;2;248;248;242m              mtn[0m
[38;2;248;248;242m              p4[0m
[38;2;248;248;242m              svk[0m
[38;2;248;248;242m              svn[0m
[38;2;248;248;242m              tla[0m
[38;2;248;248;242m              ## flavours (cannot be used in the enable or disable styles; they[0m
[38;2;248;248;242m              ## are enabled and disabled with their master [git-svn -> git])[0m
[38;2;248;248;242m              ## they *can* be used in contexts: ':vcs_info:git-svn:*'.[0m
[38;2;248;248;242m              git-p4[0m
[38;2;248;248;242m              git-svn[0m
[38;2;248;248;242m              hg-git[0m
[38;2;248;248;242m              hg-hgsubversion[0m
[38;2;248;248;242m              hg-hgsvn[0m

[38;2;248;248;242m       You  may  not  want all of these because there is no point in running the code to[0m
[38;2;248;248;242m       detect systems you do not use.  So there is a way to disable some backends  alto‐[0m
[38;2;248;248;242m       gether:[0m

[38;2;248;248;242m              zstyle ':vcs_info:*' disable bzr cdv darcs mtn svk tla[0m

[38;2;248;248;242m       You may also pick a few from that list and enable only those:[0m

[38;2;248;248;242m              zstyle ':vcs_info:*' enable git cvs svn[0m

[38;2;248;248;242m       If  you  rerun  vcs_info_printsys  after  one of these commands, you will see the[0m
[38;2;248;248;242m       backends listed in the disable style (or backends not in the enable  style  -  if[0m
[38;2;248;248;242m       you  used  that)  marked as disabled by a hash sign.  That means the detection of[0m
[38;2;248;248;242m       these systems is skipped completely. No wasted time there.[0m

[38;2;248;248;242m   Configuration[0m
[38;2;248;248;242m       The vcs_info feature can be configured via zstyle.[0m

[38;2;248;248;242m       First, the context in which we are working:[0m
[38;2;248;248;242m              :vcs_info:vcs-string:user-context:repo-root-name[0m

[38;2;248;248;242m       vcs-string[0m
[38;2;248;248;242m              is one of: git, git-svn, git-p4, hg,  hg-git,  hg-hgsubversion,  hg-hgsvn,[0m
[38;2;248;248;242m              darcs,  bzr, cdv, mtn, svn, cvs, svk, tla, p4 or fossil.  This is followed[0m
[38;2;248;248;242m              by `.quilt-quilt-mode' in Quilt mode (see Quilt Support for  details)  and[0m
[38;2;248;248;242m              by  `+hook-name'  while  hooks  are  active (see Hooks in vcs_info for de‐[0m
[38;2;248;248;242m              tails).[0m

[38;2;248;248;242m              Currently, hooks in quilt mode don't add the `.quilt-quilt-mode'  informa‐[0m
[38;2;248;248;242m              tion.  This may change in the future.[0m

[38;2;248;248;242m       user-context[0m
[38;2;248;248;242m              is a freely configurable string, assignable by the user as the first argu‐[0m
[38;2;248;248;242m              ment to vcs_info (see its description below).[0m

[38;2;248;248;242m       repo-root-name[0m
[38;2;248;248;242m              is the name of a repository in which you want a style to match. So, if you[0m
[38;2;248;248;242m              want  a  setting specific to /usr/src/zsh, with that being a CVS checkout,[0m
[38;2;248;248;242m              you can set repo-root-name to zsh to make it so.[0m

[38;2;248;248;242m       There are three special values for vcs-string: The first is named -init-, that is[0m
[38;2;248;248;242m       in effect as long as there was no decision what VCS backend to use. The second is[0m
[38;2;248;248;242m       -preinit-; it is used before vcs_info is run, when initializing the data  export‐[0m
[38;2;248;248;242m       ing   variables.  The  third  special  value  is  formats  and  is  used  by  the[0m
[38;2;248;248;242m       vcs_info_lastmsg for looking up its styles.[0m

[38;2;248;248;242m       The initial value of repo-root-name is -all- and it is replaced with  the  actual[0m
[38;2;248;248;242m       name,  as soon as it is known. Only use this part of the context for defining the[0m
[38;2;248;248;242m       formats,  actionformats  or  branchformat  styles,  as  it  is  guaranteed   that[0m
[38;2;248;248;242m       repo-root-name is set up correctly for these only. For all other styles, just use[0m
[38;2;248;248;242m       '*' instead.[0m

[38;2;248;248;242m       There are two pre-defined values for user-context:[0m
[38;2;248;248;242m       default[0m
[38;2;248;248;242m              the one used if none is specified[0m
[38;2;248;248;242m       command[0m
[38;2;248;248;242m              used by vcs_info_lastmsg to lookup its styles[0m

[38;2;248;248;242m       You  can  of  course  use ':vcs_info:*' to match all VCSs in all user-contexts at[0m
[38;2;248;248;242m       once.[0m

[38;2;248;248;242m       This is a description of all styles that are looked up.[0m

[38;2;248;248;242m       formats[0m
[38;2;248;248;242m              A list of formats, used when actionformats is not used (which is  most  of[0m
[38;2;248;248;242m              the time).[0m

[38;2;248;248;242m       actionformats[0m
[38;2;248;248;242m              A list of formats, used if there is a special action going on in your cur‐[0m
[38;2;248;248;242m              rent repository; like an interactive rebase or a merge conflict.[0m

[38;2;248;248;242m       branchformat[0m
[38;2;248;248;242m              Some  backends  replace  %b in the formats and actionformats styles above,[0m
[38;2;248;248;242m              not only by a branch name but also by a revision number. This  style  lets[0m
[38;2;248;248;242m              you modify how that string should look.[0m

[38;2;248;248;242m       nvcsformats[0m
[38;2;248;248;242m              These "formats" are set when we didn't detect a version control system for[0m
[38;2;248;248;242m              the current directory or vcs_info was disabled. This is useful if you want[0m
[38;2;248;248;242m              vcs_info  to completely take over the generation of your prompt. You would[0m
[38;2;248;248;242m              do something like PS1='${vcs_info_msg_0_}' to accomplish that.[0m

[38;2;248;248;242m       hgrevformat[0m
[38;2;248;248;242m              hg uses both a hash and a revision number to reference a specific  change‐[0m
[38;2;248;248;242m              set  in  a  repository. With this style you can format the revision string[0m
[38;2;248;248;242m              (see branchformat) to include  either  or  both.  It's  only  useful  when[0m
[38;2;248;248;242m              get-revision  is  true.  Note,  the  full  40-character revision id is not[0m
[38;2;248;248;242m              available (except when using the use-simple option) because  executing  hg[0m
[38;2;248;248;242m              more than once per prompt is too slow; you may customize this behavior us‐[0m
[38;2;248;248;242m              ing hooks.[0m

[38;2;248;248;242m       max-exports[0m
[38;2;248;248;242m              Defines the maximum number of vcs_info_msg_*_ variables vcs_info will set.[0m

[38;2;248;248;242m       enable A list of backends you want to use. Checked in the -init- context. If this[0m
[38;2;248;248;242m              list  contains  an item called NONE no backend is used at all and vcs_info[0m
[38;2;248;248;242m              will do nothing. If this list contains ALL, vcs_info will  use  all  known[0m
[38;2;248;248;242m              backends.  Only with ALL in enable will the disable style have any effect.[0m
[38;2;248;248;242m              ALL and NONE are case insensitive.[0m

[38;2;248;248;242m       disable[0m
[38;2;248;248;242m              A list of VCSs you don't want vcs_info to test for  repositories  (checked[0m
[38;2;248;248;242m              in the -init- context, too). Only used if enable contains ALL.[0m

[38;2;248;248;242m       disable-patterns[0m
[38;2;248;248;242m              A  list  of  patterns that are checked against $PWD. If a pattern matches,[0m
[38;2;248;248;242m              vcs_info   will   be   disabled.   This   style   is   checked   in    the[0m
[38;2;248;248;242m              :vcs_info:-init-:*:-all- context.[0m

[38;2;248;248;242m              Say, ~/.zsh is a directory under version control, in which you do not want[0m
[38;2;248;248;242m              vcs_info to be active, do:[0m
[38;2;248;248;242m                     zstyle ':vcs_info:*' disable-patterns "${(b)HOME}/.zsh(|/*)"[0m

[38;2;248;248;242m       use-quilt[0m
[38;2;248;248;242m              If  enabled,  the quilt support code is active in `addon' mode.  See Quilt[0m
[38;2;248;248;242m              Support for details.[0m

[38;2;248;248;242m       quilt-standalone[0m
[38;2;248;248;242m              If enabled, `standalone' mode detection is attempted if no VCS  is  active[0m
[38;2;248;248;242m              in a given directory. See Quilt Support for details.[0m

[38;2;248;248;242m       quilt-patch-dir[0m
[38;2;248;248;242m              Overwrite  the value of the $QUILT_PATCHES environment variable. See Quilt[0m
[38;2;248;248;242m              Support for details.[0m

[38;2;248;248;242m       quiltcommand[0m
[38;2;248;248;242m              When quilt itself is called in quilt support, the value of this  style  is[0m
[38;2;248;248;242m              used as the command name.[0m

[38;2;248;248;242m       check-for-changes[0m
[38;2;248;248;242m              If  enabled,  this  style causes the %c and %u format escapes to show when[0m
[38;2;248;248;242m              the working directory has uncommitted changes. The  strings  displayed  by[0m
[38;2;248;248;242m              these  escapes can be controlled via the stagedstr and unstagedstr styles.[0m
[38;2;248;248;242m              The only backends that currently support this option are git, hg, and  bzr[0m
[38;2;248;248;242m              (the latter two only support unstaged).[0m

[38;2;248;248;242m              For this style to be evaluated with the hg backend, the get-revision style[0m
[38;2;248;248;242m              needs  to be set and the use-simple style needs to be unset. The latter is[0m
[38;2;248;248;242m              the default; the former is not.[0m

[38;2;248;248;242m              With the bzr backend, lightweight checkouts only honor this style  if  the[0m
[38;2;248;248;242m              use-server style is set.[0m

[38;2;248;248;242m              Note, the actions taken if this style is enabled are potentially expensive[0m
[38;2;248;248;242m              (read:  they may be slow, depending on how big the current repository is).[0m
[38;2;248;248;242m              Therefore, it is disabled by default.[0m

[38;2;248;248;242m       check-for-staged-changes[0m
[38;2;248;248;242m              This style is like check-for-changes, but it  never  checks  the  worktree[0m
[38;2;248;248;242m              files,  only  the metadata in the .${vcs} dir.  Therefore, this style ini‐[0m
[38;2;248;248;242m              tializes only the %c escape (with stagedstr) but not the %u escape.   This[0m
[38;2;248;248;242m              style is faster than check-for-changes.[0m

[38;2;248;248;242m              In  the  git  backend,  this style checks for changes in the index.  Other[0m
[38;2;248;248;242m              backends do not currently implement this style.[0m

[38;2;248;248;242m              This style is disabled by default.[0m

[38;2;248;248;242m       stagedstr[0m
[38;2;248;248;242m              This string will be used in the %c escape if there are staged  changes  in[0m
[38;2;248;248;242m              the repository.[0m

[38;2;248;248;242m       unstagedstr[0m
[38;2;248;248;242m              This string will be used in the %u escape if there are unstaged changes in[0m
[38;2;248;248;242m              the repository.[0m

[38;2;248;248;242m       command[0m
[38;2;248;248;242m              This  style  causes  vcs_info to use the supplied string as the command to[0m
[38;2;248;248;242m              use as the VCS's binary. Note, that setting this in ':vcs_info:*' is not a[0m
[38;2;248;248;242m              good idea.[0m

[38;2;248;248;242m              If the value of this style is empty (which is the default), the  used  bi‐[0m
[38;2;248;248;242m              nary  name  is  the name of the backend in use (e.g. svn is used in an svn[0m
[38;2;248;248;242m              repository).[0m

[38;2;248;248;242m              The repo-root-name part in the context is always the  default  -all-  when[0m
[38;2;248;248;242m              this style is looked up.[0m

[38;2;248;248;242m              For  example,  this style can be used to use binaries from non-default in‐[0m
[38;2;248;248;242m              stallation directories. Assume, git is  installed  in  /usr/bin  but  your[0m
[38;2;248;248;242m              sysadmin  installed a newer version in /usr/local/bin. Instead of changing[0m
[38;2;248;248;242m              the order of your $PATH parameter, you can do this:[0m
[38;2;248;248;242m                     zstyle ':vcs_info:git:*:-all-' command /usr/local/bin/git[0m

[38;2;248;248;242m       use-server[0m
[38;2;248;248;242m              This is used by the Perforce backend (p4) to decide if it  should  contact[0m
[38;2;248;248;242m              the  Perforce  server  to  find out if a directory is managed by Perforce.[0m
[38;2;248;248;242m              This is the only reliable way of doing this, but runs the risk of a  delay[0m
[38;2;248;248;242m              if the server name cannot be found.  If the server (more specifically, the[0m
[38;2;248;248;242m              host:port pair describing the server) cannot be contacted, its name is put[0m
[38;2;248;248;242m              into  the  associative array vcs_info_p4_dead_servers and is not contacted[0m
[38;2;248;248;242m              again during the session until it is removed by hand.  If you do  not  set[0m
[38;2;248;248;242m              this  style, the p4 backend is only usable if you have set the environment[0m
[38;2;248;248;242m              variable P4CONFIG to a file name and have corresponding files in the  root[0m
[38;2;248;248;242m              directories  of  each  Perforce  client.   See  comments  in  the function[0m
[38;2;248;248;242m              VCS_INFO_detect_p4 for more detail.[0m

[38;2;248;248;242m              The Bazaar backend (bzr) uses this to permit contacting the  server  about[0m
[38;2;248;248;242m              lightweight checkouts, see the check-for-changes style.[0m

[38;2;248;248;242m       use-simple[0m
[38;2;248;248;242m              If  there  are two different ways of gathering information, you can select[0m
[38;2;248;248;242m              the simpler one by setting this style to true; the default is to  use  the[0m
[38;2;248;248;242m              not-that-simple  code, which is potentially a lot slower but might be more[0m
[38;2;248;248;242m              accurate in all possible cases. This style is used by the bzr, hg, and git[0m
[38;2;248;248;242m              backends. In the case of hg it will invoke the external hexdump program to[0m
[38;2;248;248;242m              parse the binary dirstate cache file; this method will not return the  lo‐[0m
[38;2;248;248;242m              cal revision number.[0m

[38;2;248;248;242m       get-revision[0m
[38;2;248;248;242m              If set to true, vcs_info goes the extra mile to figure out the revision of[0m
[38;2;248;248;242m              a  repository's  work  tree  (currently for the git and hg backends, where[0m
[38;2;248;248;242m              this kind of information is not always vital). For git, the hash value  of[0m
[38;2;248;248;242m              the  currently  checked out commit is available via the %i expansion. With[0m
[38;2;248;248;242m              hg, the local revision number and the corresponding global hash are avail‐[0m
[38;2;248;248;242m              able via %i.[0m

[38;2;248;248;242m       get-mq If set to true, the hg backend will look for a Mercurial Queue (mq)  patch[0m
[38;2;248;248;242m              directory. Information will be available via the `%m' replacement.[0m

[38;2;248;248;242m       get-bookmarks[0m
[38;2;248;248;242m              If  set  to  true,  the hg backend will try to get a list of current book‐[0m
[38;2;248;248;242m              marks. They will be available via the `%m' replacement.[0m

[38;2;248;248;242m              The default is to generate a comma-separated list of  all  bookmark  names[0m
[38;2;248;248;242m              that  refer  to  the currently checked out revision.  If a bookmark is ac‐[0m
[38;2;248;248;242m              tive, its name is suffixed an asterisk and placed first in the list.[0m

[38;2;248;248;242m       use-prompt-escapes[0m
[38;2;248;248;242m              Determines if we assume that the assembled string from  vcs_info  includes[0m
[38;2;248;248;242m              prompt escapes. (Used by vcs_info_lastmsg.)[0m

[38;2;248;248;242m       debug  Enable  debugging  output to track possible problems. Currently this style[0m
[38;2;248;248;242m              is only used by vcs_info's hooks system.[0m

[38;2;248;248;242m       hooks  A list style that defines hook-function names. See Hooks in vcs_info below[0m
[38;2;248;248;242m              for details.[0m

[38;2;248;248;242m       patch-format[0m
[38;2;248;248;242m       nopatch-format[0m
[38;2;248;248;242m              This pair of styles format the patch information used by the %m expando in[0m
[38;2;248;248;242m              formats and actionformats for the git and hg backends.  The value is  sub‐[0m
[38;2;248;248;242m              ject  to certain %-expansions described below.  The expanded value is made[0m
[38;2;248;248;242m              available in the global  backend_misc  array  as  ${backend_misc[patches]}[0m
[38;2;248;248;242m              (also if a set-patch-format hook is used).[0m

[38;2;248;248;242m       get-unapplied[0m
[38;2;248;248;242m              This  boolean  style controls whether a backend should attempt to gather a[0m
[38;2;248;248;242m              list of unapplied patches (for example with Mercurial Queue patches).[0m

[38;2;248;248;242m              Used by the quilt, hg, and git backends.[0m

[38;2;248;248;242m       The default values for these styles in all contexts are:[0m

[38;2;248;248;242m       formats[0m
[38;2;248;248;242m              " (%s)-[%b]%u%c-"[0m
[38;2;248;248;242m       actionformats[0m
[38;2;248;248;242m              " (%s)-[%b|%a]%u%c-"[0m
[38;2;248;248;242m       branchformat[0m
[38;2;248;248;242m              "%b:%r" (for bzr, svn, svk and hg)[0m
[38;2;248;248;242m       nvcsformats[0m
[38;2;248;248;242m              ""[0m
[38;2;248;248;242m       hgrevformat[0m
[38;2;248;248;242m              "%r:%h"[0m
[38;2;248;248;242m       max-exports[0m
[38;2;248;248;242m              2[0m
[38;2;248;248;242m       enable ALL[0m
[38;2;248;248;242m       disable[0m
[38;2;248;248;242m              (empty list)[0m
[38;2;248;248;242m       disable-patterns[0m
[38;2;248;248;242m              (empty list)[0m
[38;2;248;248;242m       check-for-changes[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       check-for-staged-changes[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       stagedstr[0m
[38;2;248;248;242m              (string: "S")[0m
[38;2;248;248;242m       unstagedstr[0m
[38;2;248;248;242m              (string: "U")[0m
[38;2;248;248;242m       command[0m
[38;2;248;248;242m              (empty string)[0m
[38;2;248;248;242m       use-server[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       use-simple[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       get-revision[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       get-mq true[0m
[38;2;248;248;242m       get-bookmarks[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       use-prompt-escapes[0m
[38;2;248;248;242m              true[0m
[38;2;248;248;242m       debug  false[0m
[38;2;248;248;242m       hooks  (empty list)[0m
[38;2;248;248;242m       use-quilt[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       quilt-standalone[0m
[38;2;248;248;242m              false[0m
[38;2;248;248;242m       quilt-patch-dir[0m
[38;2;248;248;242m              empty - use $QUILT_PATCHES[0m
[38;2;248;248;242m       quiltcommand[0m
[38;2;248;248;242m              quilt[0m
[38;2;248;248;242m       patch-format[0m
[38;2;248;248;242m              backend dependent[0m
[38;2;248;248;242m       nopatch-format[0m
[38;2;248;248;242m              backend dependent[0m
[38;2;248;248;242m       get-unapplied[0m
[38;2;248;248;242m              false[0m

[38;2;248;248;242m       In normal formats and actionformats the following replacements are done:[0m

[38;2;248;248;242m       %s     The VCS in use (git, hg, svn, etc.).[0m
[38;2;248;248;242m       %b     Information about the current branch.[0m
[38;2;248;248;242m       %a     An identifier that describes the action. Only makes  sense  in  actionfor‐[0m
[38;2;248;248;242m              mats.[0m
[38;2;248;248;242m       %i     The  current  revision  number or identifier. For hg the hgrevformat style[0m
[38;2;248;248;242m              may be used to customize the output.[0m
[38;2;248;248;242m       %c     The string from the stagedstr style if there are  staged  changes  in  the[0m
[38;2;248;248;242m              repository.[0m
[38;2;248;248;242m       %u     The string from the unstagedstr style if there are unstaged changes in the[0m
[38;2;248;248;242m              repository.[0m
[38;2;248;248;242m       %R     The base directory of the repository.[0m
[38;2;248;248;242m       %r     The repository name. If %R is /foo/bar/repoXY, %r is repoXY.[0m
[38;2;248;248;242m       %S     A subdirectory within a repository. If $PWD is /foo/bar/repoXY/beer/tasty,[0m
[38;2;248;248;242m              %S is beer/tasty.[0m
[38;2;248;248;242m       %m     A  "misc"  replacement.  It  is at the discretion of the backend to decide[0m
[38;2;248;248;242m              what this replacement expands to.[0m

[38;2;248;248;242m              The hg and git backends use this expando to display patch information.  hg[0m
[38;2;248;248;242m              sources patch information from the mq extensions; git from in-progress re‐[0m
[38;2;248;248;242m              base and  cherry-pick  operations  and  from  the  stgit  extension.   The[0m
[38;2;248;248;242m              patch-format  and nopatch-format styles control the generated string.  The[0m
[38;2;248;248;242m              former is used when at least one patch from the patch queue has  been  ap‐[0m
[38;2;248;248;242m              plied, and the latter otherwise.[0m

[38;2;248;248;242m              The  hg backend displays bookmark information in this expando (in addition[0m
[38;2;248;248;242m              to mq information).  See the get-mq and  get-bookmarks  styles.   Both  of[0m
[38;2;248;248;242m              these  styles  may be enabled at the same time.  If both are enabled, both[0m
[38;2;248;248;242m              resulting strings will be shown separated by a semicolon (that cannot cur‐[0m
[38;2;248;248;242m              rently be customized).[0m

[38;2;248;248;242m              The quilt `standalone' backend sets this expando to the same value as  the[0m
[38;2;248;248;242m              %Q expando.[0m

[38;2;248;248;242m       %Q     Quilt  series  information.  When quilt is used (either in `addon' mode or[0m
[38;2;248;248;242m              as a `standalone' backend), this expando  is  set  to  the  quilt  series'[0m
[38;2;248;248;242m              patch-format  string.   The set-patch-format hook and nopatch-format style[0m
[38;2;248;248;242m              are honoured.[0m

[38;2;248;248;242m              See Quilt Support below for details.[0m

[38;2;248;248;242m       In branchformat these replacements are done:[0m

[38;2;248;248;242m       %b     The branch name. For hg, the branch name can include a topic name.[0m
[38;2;248;248;242m       %r     The current revision number or the hgrevformat style for hg.[0m

[38;2;248;248;242m       In hgrevformat these replacements are done:[0m

[38;2;248;248;242m       %r     The current local revision number.[0m
[38;2;248;248;242m       %h     The current global revision identifier.[0m

[38;2;248;248;242m       In patch-format and nopatch-format these replacements are done:[0m

[38;2;248;248;242m       %p     The name of the top-most applied patch;  may  be  overridden  by  the  ap‐[0m
[38;2;248;248;242m              plied-string hook.[0m
[38;2;248;248;242m       %u     The number of unapplied patches; may be overridden by the unapplied-string[0m
[38;2;248;248;242m              hook.[0m
[38;2;248;248;242m       %n     The number of applied patches.[0m
[38;2;248;248;242m       %c     The number of unapplied patches.[0m
[38;2;248;248;242m       %a     The number of all patches (%a = %n + %c).[0m
[38;2;248;248;242m       %g     The names of active mq guards (hg backend).[0m
[38;2;248;248;242m       %G     The number of active mq guards (hg backend).[0m

[38;2;248;248;242m       Not  all  VCS  backends  have to support all replacements. For nvcsformats no re‐[0m
[38;2;248;248;242m       placements are performed at all, it is just a string.[0m

[38;2;248;248;242m   Oddities[0m
[38;2;248;248;242m       If you want to use the %b (bold off) prompt expansion in formats,  which  expands[0m
[38;2;248;248;242m       %b  itself,  use  %%b. That will cause the vcs_info expansion to replace %%b with[0m
[38;2;248;248;242m       %b, so that zsh's prompt expansion mechanism can handle it.  Similarly,  to  hand[0m
[38;2;248;248;242m       down %b from branchformat, use %%%%b. Sorry for this inconvenience, but it cannot[0m
[38;2;248;248;242m       be  easily  avoided.  Luckily we do not clash with a lot of prompt expansions and[0m
[38;2;248;248;242m       this only needs to be done for those.[0m

[38;2;248;248;242m       When one of the gen-applied-string,  gen-unapplied-string,  and  set-patch-format[0m
[38;2;248;248;242m       hooks  is defined, applying %-escaping (`foo=${foo//'%'/%%}') to the interpolated[0m
[38;2;248;248;242m       values for use in the prompt is the responsibility of those hooks (jointly); when[0m
[38;2;248;248;242m       neither of those hooks is defined, vcs_info handles escaping by itself.   We  re‐[0m
[38;2;248;248;242m       gret this coupling, but it was required for backwards compatibility.[0m

[38;2;248;248;242m   Quilt Support[0m
[38;2;248;248;242m       Quilt  is  not  a  version control system, therefore this is not implemented as a[0m
[38;2;248;248;242m       backend. It can help keeping track of a series of patches. People use it to  keep[0m
[38;2;248;248;242m       a  set  of changes they want to use on top of software packages (which is tightly[0m
[38;2;248;248;242m       integrated into the package build process - the Debian project does  this  for  a[0m
[38;2;248;248;242m       large  number  of packages). Quilt can also help individual developers keep track[0m
[38;2;248;248;242m       of their own patches on top of real version control systems.[0m

[38;2;248;248;242m       The vcs_info integration tries to support both ways of using quilt by having  two[0m
[38;2;248;248;242m       slightly different modes of operation: `addon' mode and `standalone' mode).[0m

[38;2;248;248;242m       Quilt  integration  is off by default; to enable it, set the use-quilt style, and[0m
[38;2;248;248;242m       add %Q to your formats or actionformats style:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*' use-quilt true[0m

[38;2;248;248;242m       Styles looked up from the Quilt support code include `.quilt-quilt-mode'  in  the[0m
[38;2;248;248;242m       vcs-string  part  of the context, where quilt-mode is either addon or standalone.[0m
[38;2;248;248;242m       Example: :vcs_info:git.quilt-addon:default:repo-root-name.[0m

[38;2;248;248;242m       For `addon' mode to become active vcs_info must have already detected a real ver‐[0m
[38;2;248;248;242m       sion control system controlling the directory. If that is the case,  a  directory[0m
[38;2;248;248;242m       that  holds quilt's patches needs to be found. That directory is configurable via[0m
[38;2;248;248;242m       the `QUILT_PATCHES' environment variable. If that variable exists  its  value  is[0m
[38;2;248;248;242m       used, otherwise the value `patches' is assumed. The value from $QUILT_PATCHES can[0m
[38;2;248;248;242m       be  overwritten using the `quilt-patch-dir' style. (Note: you can use vcs_info to[0m
[38;2;248;248;242m       keep the value of $QUILT_PATCHES correct all the time via the post-quilt hook).[0m

[38;2;248;248;242m       When the directory in question is found, quilt is assumed to be active. To gather[0m
[38;2;248;248;242m       more information, vcs_info looks for a directory called `.pc';  Quilt  uses  that[0m
[38;2;248;248;242m       directory  to  track  its current state. If this directory does not exist we know[0m
[38;2;248;248;242m       that quilt has not done anything to the working directory (read: no patches  have[0m
[38;2;248;248;242m       been applied yet).[0m

[38;2;248;248;242m       If  patches are applied, vcs_info will try to find out which. If you want to know[0m
[38;2;248;248;242m       which patches of a series are not yet applied, you need to activate the get-unap‐[0m
[38;2;248;248;242m       plied style in the appropriate context.[0m

[38;2;248;248;242m       vcs_info allows for very detailed control over how the  gathered  information  is[0m
[38;2;248;248;242m       presented  (see  the  Configuration and Hooks in vcs_info sections), all of which[0m
[38;2;248;248;242m       are documented below. Note there are a number of  other  patch  tracking  systems[0m
[38;2;248;248;242m       that  work  on top of a certain version control system (like stgit for git, or mq[0m
[38;2;248;248;242m       for hg); the configuration for systems like that  are  generally  configured  the[0m
[38;2;248;248;242m       same way as the quilt support.[0m

[38;2;248;248;242m       If the quilt support is working in `addon' mode, the produced string is available[0m
[38;2;248;248;242m       as  a  simple format replacement (%Q to be precise), which can be used in formats[0m
[38;2;248;248;242m       and actionformats; see below for details).[0m

[38;2;248;248;242m       If, on the other hand, the support code is working in `standalone' mode, vcs_info[0m
[38;2;248;248;242m       will pretend as if quilt were an actual version control system. That  means  that[0m
[38;2;248;248;242m       the  version  control  system identifier (which otherwise would be something like[0m
[38;2;248;248;242m       `svn' or `cvs') will be set to `-quilt-'. This has implications on the used style[0m
[38;2;248;248;242m       context where this identifier is the second element. vcs_info will have filled in[0m
[38;2;248;248;242m       a proper value for the "repository's" root directory and  the  string  containing[0m
[38;2;248;248;242m       the  information  about quilt's state will be available as the `misc' replacement[0m
[38;2;248;248;242m       (and %Q for compatibility with `addon' mode).[0m

[38;2;248;248;242m       What is left to discuss is how `standalone' mode is detected. The  detection  it‐[0m
[38;2;248;248;242m       self is a series of searches for directories. You can have this detection enabled[0m
[38;2;248;248;242m       all  the  time in every directory that is not otherwise under version control. If[0m
[38;2;248;248;242m       you know there is only a limited set of trees where you would  like  vcs_info  to[0m
[38;2;248;248;242m       try  and  look for Quilt in `standalone' mode to minimise the amount of searching[0m
[38;2;248;248;242m       on every call to vcs_info, there are a number of ways to do that:[0m

[38;2;248;248;242m       Essentially,  `standalone'  mode  detection  is  controlled  by  a  style  called[0m
[38;2;248;248;242m       `quilt-standalone'.  It  is  a  string style and its value can have different ef‐[0m
[38;2;248;248;242m       fects. The simplest values are: `always' to run detection every time vcs_info  is[0m
[38;2;248;248;242m       run, and `never' to turn the detection off entirely.[0m

[38;2;248;248;242m       If  the  value  of  quilt-standalone is something else, it is interpreted differ‐[0m
[38;2;248;248;242m       ently. If the value is the name of a scalar variable the value of  that  variable[0m
[38;2;248;248;242m       is  checked  and that value is used in the same `always'/`never' way as described[0m
[38;2;248;248;242m       above.[0m

[38;2;248;248;242m       If the value of quilt-standalone is an array, the elements of that array are used[0m
[38;2;248;248;242m       as directory names under which you want the detection to be active.[0m

[38;2;248;248;242m       If quilt-standalone is an associative array, the  keys  are  taken  as  directory[0m
[38;2;248;248;242m       names  under  which  you  want the detection to be active, but only if the corre‐[0m
[38;2;248;248;242m       sponding value is the string `true'.[0m

[38;2;248;248;242m       Last, but not least, if the value of quilt-standalone is the name of a  function,[0m
[38;2;248;248;242m       the function is called without arguments and the return value decides whether de‐[0m
[38;2;248;248;242m       tection  should be active. A `0' return value is true; a non-zero return value is[0m
[38;2;248;248;242m       interpreted as false.[0m

[38;2;248;248;242m       Note, if there is both a function and a variable by the name of quilt-standalone,[0m
[38;2;248;248;242m       the function will take precedence.[0m

[38;2;248;248;242m   Function Descriptions (Public API)[0m
[38;2;248;248;242m       vcs_info [user-context][0m
[38;2;248;248;242m              The main function, that runs all backends  and  assembles  all  data  into[0m
[38;2;248;248;242m              ${vcs_info_msg_*_}.  This  is the function you want to call from precmd if[0m
[38;2;248;248;242m              you want to include up-to-date information in your  prompt  (see  Variable[0m
[38;2;248;248;242m              Description below).  If an argument is given, that string will be used in‐[0m
[38;2;248;248;242m              stead of default in the user-context field of the style context.[0m

[38;2;248;248;242m       vcs_info_hookadd[0m
[38;2;248;248;242m              Statically registers a number of functions to a given hook. The hook needs[0m
[38;2;248;248;242m              to be given as the first argument; what follows is a list of hook-function[0m
[38;2;248;248;242m              names  to  register  to  the  hook. The `+vi-' prefix needs to be left out[0m
[38;2;248;248;242m              here. See Hooks in vcs_info below for details.[0m

[38;2;248;248;242m       vcs_info_hookdel[0m
[38;2;248;248;242m              Remove hook-functions from a given hook. The hook needs to be given as the[0m
[38;2;248;248;242m              first non-option argument; what follows is a list of  hook-function  names[0m
[38;2;248;248;242m              to  un-register  from the hook. If `-a' is used as the first argument, all[0m
[38;2;248;248;242m              occurrences of the functions are unregistered. Otherwise only the last oc‐[0m
[38;2;248;248;242m              currence is removed (if a function was registered  to  a  hook  more  than[0m
[38;2;248;248;242m              once). The `+vi-' prefix needs to be left out here.  See Hooks in vcs_info[0m
[38;2;248;248;242m              below for details.[0m

[38;2;248;248;242m       vcs_info_lastmsg[0m
[38;2;248;248;242m              Outputs  the current values of ${vcs_info_msg_*_}.  Takes into account the[0m
[38;2;248;248;242m              value  of  the   use-prompt-escapes   style   in   ':vcs_info:formats:com‐[0m
[38;2;248;248;242m              mand:-all-'. It also only prints max-exports values.[0m

[38;2;248;248;242m       vcs_info_printsys [user-context][0m
[38;2;248;248;242m              Prints a list of all supported version control systems. Useful to find out[0m
[38;2;248;248;242m              possible  contexts  (and which of them are enabled) or values for the dis‐[0m
[38;2;248;248;242m              able style.[0m

[38;2;248;248;242m       vcs_info_setsys[0m
[38;2;248;248;242m              Initializes vcs_info's internal list  of  available  backends.  With  this[0m
[38;2;248;248;242m              function, you can add support for new VCSs without restarting the shell.[0m

[38;2;248;248;242m       All functions named VCS_INFO_* are for internal use only.[0m

[38;2;248;248;242m   Variable Description[0m
[38;2;248;248;242m       ${vcs_info_msg_N_} (Note the trailing underscore)[0m
[38;2;248;248;242m              Where  N  is  an  integer,  e.g., vcs_info_msg_0_. These variables are the[0m
[38;2;248;248;242m              storage for the informational message the last vcs_info  call  has  assem‐[0m
[38;2;248;248;242m              bled. These are strongly connected to the formats, actionformats and nvcs‐[0m
[38;2;248;248;242m              formats  styles  described above. Those styles are lists. The first member[0m
[38;2;248;248;242m              of that list  gets  expanded  into  ${vcs_info_msg_0_},  the  second  into[0m
[38;2;248;248;242m              ${vcs_info_msg_1_} and the Nth into ${vcs_info_msg_N-1_}. (See the max-ex‐[0m
[38;2;248;248;242m              ports style above.)[0m

[38;2;248;248;242m       All variables named VCS_INFO_* are for internal use only.[0m

[38;2;248;248;242m   Hooks in vcs_info[0m
[38;2;248;248;242m       Hooks  are places in vcs_info where you can run your own code. That code can com‐[0m
[38;2;248;248;242m       municate with the code that called it and through that, change the  system's  be‐[0m
[38;2;248;248;242m       haviour.[0m

[38;2;248;248;242m       For configuration, hooks change the style context:[0m
[38;2;248;248;242m              :vcs_info:vcs-string+hook-name:user-context:repo-root-name[0m

[38;2;248;248;242m       To  register functions to a hook, you need to list them in the hooks style in the[0m
[38;2;248;248;242m       appropriate context.[0m

[38;2;248;248;242m       Example:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*+foo:*' hooks bar baz[0m

[38;2;248;248;242m       This registers functions to the hook `foo' for all backends. In  order  to  avoid[0m
[38;2;248;248;242m       namespace  problems,  all registered function names are prepended by a `+vi-', so[0m
[38;2;248;248;242m       the actual functions called for the `foo' hook are `+vi-bar' and `+vi-baz'.[0m

[38;2;248;248;242m       If you would like to register a function to a hook regardless of the current con‐[0m
[38;2;248;248;242m       text, you may use the vcs_info_hookadd function. To remove a  function  that  was[0m
[38;2;248;248;242m       added like that, the vcs_info_hookdel function can be used.[0m

[38;2;248;248;242m       If  something seems weird, you can enable the `debug' boolean style in the proper[0m
[38;2;248;248;242m       context and the hook-calling code will print what it tried to execute and whether[0m
[38;2;248;248;242m       the function in question existed.[0m

[38;2;248;248;242m       When you register more than one function to a hook, all  functions  are  executed[0m
[38;2;248;248;242m       one after another until one function returns non-zero or until all functions have[0m
[38;2;248;248;242m       been called. Context-sensitive hook functions are executed before statically reg‐[0m
[38;2;248;248;242m       istered ones (the ones added by vcs_info_hookadd).[0m

[38;2;248;248;242m       You may pass data between functions via an associative array, user_data.  For ex‐[0m
[38;2;248;248;242m       ample:[0m
[38;2;248;248;242m              +vi-git-myfirsthook(){[0m
[38;2;248;248;242m                  user_data[myval]=$myval[0m
[38;2;248;248;242m              }[0m
[38;2;248;248;242m              +vi-git-mysecondhook(){[0m
[38;2;248;248;242m                  # do something with ${user_data[myval]}[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       There are a number of variables that are special in hook contexts:[0m

[38;2;248;248;242m       ret    The  return value that the hooks system will return to the caller. The de‐[0m
[38;2;248;248;242m              fault is an integer `zero'. If and how a changed ret value changes the ex‐[0m
[38;2;248;248;242m              ecution of the caller depends on the specific hook. See the hook  documen‐[0m
[38;2;248;248;242m              tation below for details.[0m

[38;2;248;248;242m       hook_com[0m
[38;2;248;248;242m              An associated array which is used for bidirectional communication from the[0m
[38;2;248;248;242m              caller to hook functions. The used keys depend on the specific hook.[0m

[38;2;248;248;242m       context[0m
[38;2;248;248;242m              The  active  context of the hook. Functions that wish to change this vari‐[0m
[38;2;248;248;242m              able should make it local scope first.[0m

[38;2;248;248;242m       vcs    The current VCS after it was detected. The  same  values  as  in  the  en‐[0m
[38;2;248;248;242m              able/disable style are used. Available in all hooks except start-up.[0m

[38;2;248;248;242m       Finally, the full list of currently available hooks:[0m

[38;2;248;248;242m       start-up[0m
[38;2;248;248;242m              Called after starting vcs_info but before the VCS in this directory is de‐[0m
[38;2;248;248;242m              termined.  It can be used to deactivate vcs_info temporarily if necessary.[0m
[38;2;248;248;242m              When ret is set to 1, vcs_info aborts and does nothing;  when  set  to  2,[0m
[38;2;248;248;242m              vcs_info  sets  up everything as if no version control were active and ex‐[0m
[38;2;248;248;242m              its.[0m

[38;2;248;248;242m       pre-get-data[0m
[38;2;248;248;242m              Same as start-up but after the VCS was detected.[0m

[38;2;248;248;242m       gen-hg-bookmark-string[0m
[38;2;248;248;242m              Called in the Mercurial backend when a bookmark string is  generated;  the[0m
[38;2;248;248;242m              get-revision and get-bookmarks styles must be true.[0m

[38;2;248;248;242m              This  hook  gets  the  names of the Mercurial bookmarks that vcs_info col‐[0m
[38;2;248;248;242m              lected from `hg'.[0m

[38;2;248;248;242m              If a bookmark is active, the key ${hook_com[hg-active-bookmark]} is set to[0m
[38;2;248;248;242m              its name.  The key is otherwise unset.[0m

[38;2;248;248;242m              When  setting  ret  to  non-zero,  the   string   in   ${hook_com[hg-book‐[0m
[38;2;248;248;242m              mark-string]}  will  be used in the %m escape in formats and actionformats[0m
[38;2;248;248;242m              and will  be  available  in  the  global  backend_misc  array  as  ${back‐[0m
[38;2;248;248;242m              end_misc[bookmarks]}.[0m

[38;2;248;248;242m       gen-applied-string[0m
[38;2;248;248;242m              Called in the git (with stgit or during rebase or merge), and hg (with mq)[0m
[38;2;248;248;242m              backends  and  in  quilt support when the applied-string is generated; the[0m
[38;2;248;248;242m              use-quilt zstyle must be true for quilt (the mq and stgit backends are ac‐[0m
[38;2;248;248;242m              tive by default).[0m

[38;2;248;248;242m              The arguments to this hook describe applied patches in the opposite order,[0m
[38;2;248;248;242m              which means that the first argument is the top-most patch and so forth.[0m

[38;2;248;248;242m              When the patches' log messages can be extracted, those are embedded within[0m
[38;2;248;248;242m              each argument after a space, so each argument is of the  form  `patch-name[0m
[38;2;248;248;242m              first  line  of the log message', where patch-name contains no whitespace.[0m
[38;2;248;248;242m              The mq backend passes arguments of the form `patch  name',  with  possible[0m
[38;2;248;248;242m              embedded spaces, but without extracting the patch's log message.[0m

[38;2;248;248;242m              When  setting  ret  to non-zero, the string in ${hook_com[applied-string]}[0m
[38;2;248;248;242m              will be available as %p in the  patch-format  and  nopatch-format  styles.[0m
[38;2;248;248;242m              This hook is, in concert with set-patch-format, responsible for %-escaping[0m
[38;2;248;248;242m              that value for use in the prompt.  (See the Oddities section.)[0m

[38;2;248;248;242m              The     quilt     backend    passes    to    this    hook    the    inputs[0m
[38;2;248;248;242m              ${hook_com[quilt-patches-dir]}   and,   if   it   has   been   determined,[0m
[38;2;248;248;242m              ${hook_com[quilt-pc-dir]}.[0m

[38;2;248;248;242m       gen-unapplied-string[0m
[38;2;248;248;242m              Called  in the git (with stgit or during rebase), and hg (with mq) backend[0m
[38;2;248;248;242m              and in quilt support when the unapplied-string is generated; the get-unap‐[0m
[38;2;248;248;242m              plied style must be true.[0m

[38;2;248;248;242m              This hook gets the names of all unapplied patches which vcs_info in order,[0m
[38;2;248;248;242m              which means that the first argument is the patch next-in-line  to  be  ap‐[0m
[38;2;248;248;242m              plied and so forth.[0m

[38;2;248;248;242m              The format of each argument is as for gen-applied-string, above.[0m

[38;2;248;248;242m              When  setting ret to non-zero, the string in ${hook_com[unapplied-string]}[0m
[38;2;248;248;242m              will be available as %u in the  patch-format  and  nopatch-format  styles.[0m
[38;2;248;248;242m              This hook is, in concert with set-patch-format, responsible for %-escaping[0m
[38;2;248;248;242m              that value for use in the prompt.  (See the Oddities section.)[0m

[38;2;248;248;242m              The     quilt     backend    passes    to    this    hook    the    inputs[0m
[38;2;248;248;242m              ${hook_com[quilt-patches-dir]}   and,   if   it   has   been   determined,[0m
[38;2;248;248;242m              ${hook_com[quilt-pc-dir]}.[0m

[38;2;248;248;242m       gen-mqguards-string[0m
[38;2;248;248;242m              Called in the hg backend when guards-string is generated; the get-mq style[0m
[38;2;248;248;242m              must be true (default).[0m

[38;2;248;248;242m              This hook gets the names of any active mq guards.[0m

[38;2;248;248;242m              When  setting  ret  to  non-zero, the string in ${hook_com[guards-string]}[0m
[38;2;248;248;242m              will be used in the %g  escape  in  the  patch-format  and  nopatch-format[0m
[38;2;248;248;242m              styles.[0m

[38;2;248;248;242m       no-vcs This hooks is called when no version control system was detected.[0m

[38;2;248;248;242m              The `hook_com' parameter is not used.[0m

[38;2;248;248;242m       post-backend[0m
[38;2;248;248;242m              Called as soon as the backend has finished collecting information.[0m

[38;2;248;248;242m              The `hook_com' keys available are as for the set-message hook.[0m

[38;2;248;248;242m       post-quilt[0m
[38;2;248;248;242m              Called  after  the  quilt  support  is  done. The following information is[0m
[38;2;248;248;242m              passed as arguments to the hook: 1. the  quilt-support  mode  (`addon'  or[0m
[38;2;248;248;242m              `standalone'); 2. the directory that contains the patch series; 3. the di‐[0m
[38;2;248;248;242m              rectory that holds quilt's status information (the `.pc' directory) or the[0m
[38;2;248;248;242m              string "-nopc-" if that directory wasn't found.[0m

[38;2;248;248;242m              The `hook_com' parameter is not used.[0m

[38;2;248;248;242m       set-branch-format[0m
[38;2;248;248;242m              Called  before `branchformat' is set. The only argument to the hook is the[0m
[38;2;248;248;242m              format that is configured at this point.[0m

[38;2;248;248;242m              The `hook_com' keys considered are `branch' and `revision'.  They are  set[0m
[38;2;248;248;242m              to  the  values figured out so far by vcs_info and any change will be used[0m
[38;2;248;248;242m              directly when the actual replacement is done.[0m

[38;2;248;248;242m              If ret is set to non-zero, the string in ${hook_com[branch-replace]}  will[0m
[38;2;248;248;242m              be  used  unchanged  as  the  `%b'  replacement  in  the  variables set by[0m
[38;2;248;248;242m              vcs_info.[0m

[38;2;248;248;242m       set-hgrev-format[0m
[38;2;248;248;242m              Called before a `hgrevformat' is set. The only argument to the hook is the[0m
[38;2;248;248;242m              format that is configured at this point.[0m

[38;2;248;248;242m              The `hook_com' keys considered are `hash' and `localrev'.  They are set to[0m
[38;2;248;248;242m              the values figured out so far by vcs_info and any change will be used  di‐[0m
[38;2;248;248;242m              rectly when the actual replacement is done.[0m

[38;2;248;248;242m              If  ret is set to non-zero, the string in ${hook_com[rev-replace]} will be[0m
[38;2;248;248;242m              used unchanged as the `%i' replacement in the variables set by vcs_info.[0m

[38;2;248;248;242m       pre-addon-quilt[0m
[38;2;248;248;242m              This hook is used when vcs_info's quilt functionality is active in "addon"[0m
[38;2;248;248;242m              mode (quilt used on top of a real version control system). It is activated[0m
[38;2;248;248;242m              right before any quilt specific action is taken.[0m

[38;2;248;248;242m              Setting the `ret' variable in this hook to a  non-zero  value  avoids  any[0m
[38;2;248;248;242m              quilt specific actions from being run at all.[0m

[38;2;248;248;242m       set-patch-format[0m
[38;2;248;248;242m              This hook is used to control some of the possible expansions in patch-for‐[0m
[38;2;248;248;242m              mat  and  nopatch-format  styles  with  patch queue systems such as quilt,[0m
[38;2;248;248;242m              mqueue and the like.[0m

[38;2;248;248;242m              This hook is used in the git, hg and quilt backends.[0m

[38;2;248;248;242m              The hook allows the  control  of  the  %p  (${hook_com[applied]})  and  %u[0m
[38;2;248;248;242m              (${hook_com[unapplied]}) expansion in all backends that use the hook. With[0m
[38;2;248;248;242m              the  mercurial backend, the %g (${hook_com[guards]}) expansion is control‐[0m
[38;2;248;248;242m              lable in addition to that.[0m

[38;2;248;248;242m              If ret is set to non-zero, the string in  ${hook_com[patch-replace]}  will[0m
[38;2;248;248;242m              be  used  unchanged  instead  of  an  expanded format from patch-format or[0m
[38;2;248;248;242m              nopatch-format.[0m

[38;2;248;248;242m              This  hook  is,  in  concert  with  the  gen-applied-string  or  gen-unap‐[0m
[38;2;248;248;242m              plied-string hooks if they are defined, responsible for %-escaping the fi‐[0m
[38;2;248;248;242m              nal patch-format value for use in the prompt.  (See the Oddities section.)[0m

[38;2;248;248;242m              The     quilt     backend    passes    to    this    hook    the    inputs[0m
[38;2;248;248;242m              ${hook_com[quilt-patches-dir]}   and,   if   it   has   been   determined,[0m
[38;2;248;248;242m              ${hook_com[quilt-pc-dir]}.[0m

[38;2;248;248;242m       set-message[0m
[38;2;248;248;242m              Called  each time before a `vcs_info_msg_N_' message is set.  It takes two[0m
[38;2;248;248;242m              arguments; the first being the `N' in the message variable name, the  sec‐[0m
[38;2;248;248;242m              ond is the currently configured formats or actionformats.[0m

[38;2;248;248;242m              There  are  a  number  of  `hook_com'  keys, that are used here: `action',[0m
[38;2;248;248;242m              `branch', `base', `base-name', `subdir', `staged', `unstaged', `revision',[0m
[38;2;248;248;242m              `misc', `vcs' and one `miscN' entry for each backend-specific  data  field[0m
[38;2;248;248;242m              (N  starting  at  zero).  They are set to the values figured out so far by[0m
[38;2;248;248;242m              vcs_info and any change will be used directly when the actual  replacement[0m
[38;2;248;248;242m              is done.[0m

[38;2;248;248;242m              Since this hook is triggered multiple times (once for each configured for‐[0m
[38;2;248;248;242m              mats  or  actionformats), each of the `hook_com' keys mentioned above (ex‐[0m
[38;2;248;248;242m              cept for the miscN entries) has an `_orig' counterpart,  so  even  if  you[0m
[38;2;248;248;242m              changed a value to your liking you can still get the original value in the[0m
[38;2;248;248;242m              next run. Changing the `_orig' values is probably not a good idea.[0m

[38;2;248;248;242m              If ret is set to non-zero, the string in ${hook_com[message]} will be used[0m
[38;2;248;248;242m              unchanged as the message by vcs_info.[0m

[38;2;248;248;242m       If all of this sounds rather confusing, take a look at the Examples section below[0m
[38;2;248;248;242m       and also in the Misc/vcs_info-examples file in the Zsh source.  They contain some[0m
[38;2;248;248;242m       explanatory code.[0m

[38;2;248;248;242m   Examples[0m
[38;2;248;248;242m       Don't use vcs_info at all (even though it's in your prompt):[0m
[38;2;248;248;242m              zstyle ':vcs_info:*' enable NONE[0m

[38;2;248;248;242m       Disable the backends for bzr and svk:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*' disable bzr svk[0m

[38;2;248;248;242m       Disable everything but bzr and svk:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*' enable bzr svk[0m

[38;2;248;248;242m       Provide a special formats for git:[0m
[38;2;248;248;242m              zstyle ':vcs_info:git:*' formats       ' GIT, BABY! [%b]'[0m
[38;2;248;248;242m              zstyle ':vcs_info:git:*' actionformats ' GIT ACTION! [%b|%a]'[0m

[38;2;248;248;242m       All  %x  expansion in all sorts of formats (formats, actionformats, branchformat,[0m
[38;2;248;248;242m       you name it) are done using the `zformat' builtin from  the  `zsh/zutil'  module.[0m
[38;2;248;248;242m       That  means  you  can do everything with these %x items what zformat supports. In[0m
[38;2;248;248;242m       particular, if you want something that is really long to have a fixed width, like[0m
[38;2;248;248;242m       a hash in a mercurial branchformat, you can do this: %12.12i. That'll shrink  the[0m
[38;2;248;248;242m       40 character hash to its 12 leading characters. The form is actually `%min.maxx'.[0m
[38;2;248;248;242m       More  is  possible.   See the section `The zsh/zutil Module' in zshmodules(1) for[0m
[38;2;248;248;242m       details.[0m

[38;2;248;248;242m       Use the quicker bzr backend[0m
[38;2;248;248;242m              zstyle ':vcs_info:bzr:*' use-simple true[0m

[38;2;248;248;242m       If you do use use-simple, please report if it does `the-right-thing[tm]'.[0m

[38;2;248;248;242m       Display the revision number in yellow for bzr and svn:[0m
[38;2;248;248;242m              zstyle ':vcs_info:(svn|bzr):*' \[0m
[38;2;248;248;242m                     branchformat '%b%%F{yellow}:%r'[0m

[38;2;248;248;242m       The doubled percent sign is explained in the Oddities section.[0m

[38;2;248;248;242m       Alternatively, one can use the raw colour codes directly:[0m

[38;2;248;248;242m              zstyle ':vcs_info:(svn|bzr):*' \[0m
[38;2;248;248;242m                     branchformat '%b%{'${fg[yellow]}'%}:%r'[0m

[38;2;248;248;242m       Normally when a variable is interpolated into a format string, the variable needs[0m
[38;2;248;248;242m       to be %-escaped. In this example we skipped that because we assume the  value  of[0m
[38;2;248;248;242m       ${fg[yellow]} doesn't contain any % signs.[0m

[38;2;248;248;242m       Make  sure  you  enclose the color codes in %{...%} if you want to use the string[0m
[38;2;248;248;242m       provided by vcs_info in prompts.[0m

[38;2;248;248;242m       Here is how to print the VCS information as a command (not in a prompt):[0m
[38;2;248;248;242m              vcsi() { vcs_info interactive; vcs_info_lastmsg }[0m

[38;2;248;248;242m       This way, you can even define different formats for output  via  vcs_info_lastmsg[0m
[38;2;248;248;242m       in the ':vcs_info:*:interactive:*' namespace.[0m

[38;2;248;248;242m       Now as promised, some code that uses hooks: say, you'd like to replace the string[0m
[38;2;248;248;242m       `svn' by `subversion' in vcs_info's %s formats replacement.[0m

[38;2;248;248;242m       First, we will tell vcs_info to call a function when populating the message vari‐[0m
[38;2;248;248;242m       ables with the gathered information:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*+set-message:*' hooks svn2subversion[0m

[38;2;248;248;242m       Nothing  happens. Which is reasonable, since we didn't define the actual function[0m
[38;2;248;248;242m       yet. To see what the hooks subsystem is trying to do, enable the `debug' style:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*+*:*' debug true[0m

[38;2;248;248;242m       That should give you an idea what is going on. Specifically, the function that we[0m
[38;2;248;248;242m       are looking for is `+vi-svn2subversion'. Note, the `+vi-' prefix. So,  everything[0m
[38;2;248;248;242m       is  in  order,  just  as documented. When you are done checking out the debugging[0m
[38;2;248;248;242m       output, disable it again:[0m
[38;2;248;248;242m              zstyle ':vcs_info:*+*:*' debug false[0m

[38;2;248;248;242m       Now, let's define the function:[0m
[38;2;248;248;242m              function +vi-svn2subversion() {[0m
[38;2;248;248;242m                  [[ ${hook_com[vcs_orig]} == svn ]] && hook_com[vcs]=subversion[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       Simple enough. And it could have even been simpler, if only we had registered our[0m
[38;2;248;248;242m       function in a less generic context. If we do it only in the `svn' backend's  con‐[0m
[38;2;248;248;242m       text, we don't need to test which the active backend is:[0m
[38;2;248;248;242m              zstyle ':vcs_info:svn+set-message:*' hooks svn2subversion[0m
[38;2;248;248;242m              function +vi-svn2subversion() {[0m
[38;2;248;248;242m                  hook_com[vcs]=subversion[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       And  finally  a  little more elaborate example, that uses a hook to create a cus‐[0m
[38;2;248;248;242m       tomised bookmark string for the hg backend.[0m

[38;2;248;248;242m       Again, we start off by registering a function:[0m
[38;2;248;248;242m              zstyle ':vcs_info:hg+gen-hg-bookmark-string:*' hooks hgbookmarks[0m

[38;2;248;248;242m       And then we define the `+vi-hgbookmarks' function:[0m
[38;2;248;248;242m              function +vi-hgbookmarks() {[0m
[38;2;248;248;242m                  # The default is to connect all bookmark names by[0m
[38;2;248;248;242m                  # commas. This mixes things up a little.[0m
[38;2;248;248;242m                  # Imagine, there's one type of bookmarks that is[0m
[38;2;248;248;242m                  # special to you. Say, because it's *your* work.[0m
[38;2;248;248;242m                  # Those bookmarks look always like this: "sh/*"[0m
[38;2;248;248;242m                  # (because your initials are sh, for example).[0m
[38;2;248;248;242m                  # This makes the bookmarks string use only those[0m
[38;2;248;248;242m                  # bookmarks. If there's more than one, it[0m
[38;2;248;248;242m                  # concatenates them using commas.[0m
[38;2;248;248;242m                  # The bookmarks returned by `hg' are available in[0m
[38;2;248;248;242m                  # the function's positional parameters.[0m
[38;2;248;248;242m                  local s="${(Mj:,:)@:#sh/*}"[0m
[38;2;248;248;242m                  # Now, the communication with the code that calls[0m
[38;2;248;248;242m                  # the hook functions is done via the hook_com[][0m
[38;2;248;248;242m                  # hash. The key at which the `gen-hg-bookmark-string'[0m
[38;2;248;248;242m                  # hook looks is `hg-bookmark-string'. So:[0m
[38;2;248;248;242m                  hook_com[hg-bookmark-string]=$s[0m
[38;2;248;248;242m                  # And to signal that we want to use the string we[0m
[38;2;248;248;242m                  # just generated, set the special variable `ret' to[0m
[38;2;248;248;242m                  # something other than the default zero:[0m
[38;2;248;248;242m                  ret=1[0m
[38;2;248;248;242m                  return 0[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       Some longer examples and code snippets which might be useful are available in the[0m
[38;2;248;248;242m       examples file located at Misc/vcs_info-examples in the Zsh source directory.[0m

[38;2;248;248;242m       This concludes our guided tour through zsh's vcs_info.[0m

[38;2;248;248;242mPROMPT THEMES[0m
[38;2;248;248;242m   Installation[0m
[38;2;248;248;242m       You should make sure all the functions from the  Functions/Prompts  directory  of[0m
[38;2;248;248;242m       the  source  distribution are available; they all begin with the string `prompt_'[0m
[38;2;248;248;242m       except for the special function `promptinit'.  You also  need  the  `colors'  and[0m
[38;2;248;248;242m       `add-zsh-hook' functions from Functions/Misc.  All these functions may already be[0m
[38;2;248;248;242m       installed  on your system; if not, you will need to find them and copy them.  The[0m
[38;2;248;248;242m       directory should appear as one of the elements of the fpath  array  (this  should[0m
[38;2;248;248;242m       already be the case if they were installed), and at least the function promptinit[0m
[38;2;248;248;242m       should  be autoloaded; it will autoload the rest.  Finally, to initialize the use[0m
[38;2;248;248;242m       of the system you need to call the promptinit function.  The  following  code  in[0m
[38;2;248;248;242m       your  .zshrc will arrange for this; assume the functions are stored in the direc‐[0m
[38;2;248;248;242m       tory ~/myfns:[0m

[38;2;248;248;242m              fpath=(~/myfns $fpath)[0m
[38;2;248;248;242m              autoload -U promptinit[0m
[38;2;248;248;242m              promptinit[0m

[38;2;248;248;242m   Theme Selection[0m
[38;2;248;248;242m       Use the prompt command to select your preferred theme.  This command may be added[0m
[38;2;248;248;242m       to your .zshrc following the call to promptinit in order  to  start  zsh  with  a[0m
[38;2;248;248;242m       theme already selected.[0m

[38;2;248;248;242m       prompt [ -c | -l ][0m
[38;2;248;248;242m       prompt [ -p | -h ] [ theme ... ][0m
[38;2;248;248;242m       prompt [ -s ] theme [ arg ... ][0m
[38;2;248;248;242m              Set  or  examine  the prompt theme.  With no options and a theme argument,[0m
[38;2;248;248;242m              the theme with that name is set  as  the  current  theme.   The  available[0m
[38;2;248;248;242m              themes  are  determined at run time; use the -l option to see a list.  The[0m
[38;2;248;248;242m              special theme `random' selects at random one of the available  themes  and[0m
[38;2;248;248;242m              sets your prompt to that.[0m

[38;2;248;248;242m              In  some  cases  the theme may be modified by one or more arguments, which[0m
[38;2;248;248;242m              should be given after the theme name.  See the help for each theme for de‐[0m
[38;2;248;248;242m              scriptions of these arguments.[0m

[38;2;248;248;242m              Options are:[0m

[38;2;248;248;242m              -c     Show the currently selected theme and its parameters, if any.[0m
[38;2;248;248;242m              -l     List all available prompt themes.[0m
[38;2;248;248;242m              -p     Preview the theme named by theme, or all  themes  if  no  theme  is[0m
[38;2;248;248;242m                     given.[0m
[38;2;248;248;242m              -h     Show  help for the theme named by theme, or for the prompt function[0m
[38;2;248;248;242m                     if no theme is given.[0m
[38;2;248;248;242m              -s     Set theme as the current theme and save state.[0m

[38;2;248;248;242m       prompt_theme_setup[0m
[38;2;248;248;242m              Each available theme has a setup function which is called  by  the  prompt[0m
[38;2;248;248;242m              function  to install that theme.  This function may define other functions[0m
[38;2;248;248;242m              as necessary to maintain the prompt, including functions used  to  preview[0m
[38;2;248;248;242m              the  prompt  or  provide help for its use.  You should not normally call a[0m
[38;2;248;248;242m              theme's setup function directly.[0m

[38;2;248;248;242m   Utility Themes[0m
[38;2;248;248;242m       prompt off[0m
[38;2;248;248;242m              The theme `off' sets all the prompt variables to minimal  values  with  no[0m
[38;2;248;248;242m              special effects.[0m

[38;2;248;248;242m       prompt default[0m
[38;2;248;248;242m              The  theme  `default' sets all prompt variables to the same state as if an[0m
[38;2;248;248;242m              interactive zsh was started with no initialization files.[0m

[38;2;248;248;242m       prompt restore[0m
[38;2;248;248;242m              The special theme `restore' erases all  theme  settings  and  sets  prompt[0m
[38;2;248;248;242m              variables  to  their state before the first time the `prompt' function was[0m
[38;2;248;248;242m              run, provided each theme has properly defined its cleanup (see below).[0m

[38;2;248;248;242m              Note that you can undo `prompt off' and `prompt default' with `prompt  re‐[0m
[38;2;248;248;242m              store', but a second restore does not undo the first.[0m

[38;2;248;248;242m   Writing Themes[0m
[38;2;248;248;242m       The first step for adding your own theme is to choose a name for it, and create a[0m
[38;2;248;248;242m       file `prompt_name_setup' in a directory in your fpath, such as ~/myfns in the ex‐[0m
[38;2;248;248;242m       ample above.  The file should at minimum contain assignments for the prompt vari‐[0m
[38;2;248;248;242m       ables  that  your  theme  wishes  to modify.  By convention, themes use PS1, PS2,[0m
[38;2;248;248;242m       RPS1, etc., rather than the longer PROMPT and RPROMPT.[0m

[38;2;248;248;242m       The file is autoloaded as a function in the current shell context, so it may con‐[0m
[38;2;248;248;242m       tain any necessary commands to customize your  theme,  including  defining  addi‐[0m
[38;2;248;248;242m       tional  functions.   To  make  some complex tasks easier, your setup function may[0m
[38;2;248;248;242m       also do any of the following:[0m

[38;2;248;248;242m       Assign prompt_opts[0m
[38;2;248;248;242m              The array prompt_opts may be assigned  any  of  "bang",  "cr",  "percent",[0m
[38;2;248;248;242m              "sp",  and/or  "subst"  as values.  The corresponding setopts (promptbang,[0m
[38;2;248;248;242m              etc.) are turned on, all other prompt-related options are turned off.  The[0m
[38;2;248;248;242m              prompt_opts array preserves setopts even beyond the scope of localoptions,[0m
[38;2;248;248;242m              should your function need that.[0m

[38;2;248;248;242m       Modify hooks[0m
[38;2;248;248;242m              Use of add-zsh-hook and add-zle-hook-widget is recommended (see the Manip‐[0m
[38;2;248;248;242m              ulating Hook Functions section above).  All hooks that follow  the  naming[0m
[38;2;248;248;242m              pattern  prompt_theme_hook are automatically removed when the prompt theme[0m
[38;2;248;248;242m              changes or is disabled.[0m

[38;2;248;248;242m       Declare cleanup[0m
[38;2;248;248;242m              If your function makes any other changes that should be  undone  when  the[0m
[38;2;248;248;242m              theme is disabled, your setup function may call[0m

[38;2;248;248;242m                     prompt_cleanup command[0m

[38;2;248;248;242m              where  command  should be suitably quoted.  If your theme is ever disabled[0m
[38;2;248;248;242m              or replaced by another, command is executed with eval.   You  may  declare[0m
[38;2;248;248;242m              more than one such cleanup hook.[0m

[38;2;248;248;242m       Define preview[0m
[38;2;248;248;242m              Define  or  autoload a function prompt_name_preview to display a simulated[0m
[38;2;248;248;242m              version of your prompt.  A simple default previewer is defined  by  promp‐[0m
[38;2;248;248;242m              tinit  for  themes that do not define their own.  This preview function is[0m
[38;2;248;248;242m              called by `prompt -p'.[0m

[38;2;248;248;242m       Provide help[0m
[38;2;248;248;242m              Define or autoload a function prompt_name_help to display documentation or[0m
[38;2;248;248;242m              help text for your theme.  This help function is called by `prompt -h'.[0m

[38;2;248;248;242mZLE FUNCTIONS[0m
[38;2;248;248;242m   Widgets[0m
[38;2;248;248;242m       These functions all implement user-defined ZLE widgets (see zshzle(1)) which  can[0m
[38;2;248;248;242m       be  bound  to  keystrokes in interactive shells.  To use them, your .zshrc should[0m
[38;2;248;248;242m       contain lines of the form[0m

[38;2;248;248;242m              autoload function[0m
[38;2;248;248;242m              zle -N function[0m

[38;2;248;248;242m       followed by an appropriate bindkey command to associate the function with  a  key[0m
[38;2;248;248;242m       sequence.  Suggested bindings are described below.[0m

[38;2;248;248;242m       bash-style word functions[0m
[38;2;248;248;242m              If  you  are  looking  for  functions to implement moving over and editing[0m
[38;2;248;248;242m              words in the manner of bash, where only alphanumeric characters  are  con‐[0m
[38;2;248;248;242m              sidered  word  characters, you can use the functions described in the next[0m
[38;2;248;248;242m              section.  The following is sufficient:[0m

[38;2;248;248;242m                     autoload -U select-word-style[0m
[38;2;248;248;242m                     select-word-style bash[0m

[38;2;248;248;242m       forward-word-match, backward-word-match[0m
[38;2;248;248;242m       kill-word-match, backward-kill-word-match[0m
[38;2;248;248;242m       transpose-words-match, capitalize-word-match[0m
[38;2;248;248;242m       up-case-word-match, down-case-word-match[0m
[38;2;248;248;242m       delete-whole-word-match, select-word-match[0m
[38;2;248;248;242m       select-word-style, match-word-context, match-words-by-style[0m
[38;2;248;248;242m              The first eight  `-match'  functions  are  drop-in  replacements  for  the[0m
[38;2;248;248;242m              builtin  widgets  without the suffix.  By default they behave in a similar[0m
[38;2;248;248;242m              way.  However, by the use of styles and  the  function  select-word-style,[0m
[38;2;248;248;242m              the way words are matched can be altered. select-word-match is intended to[0m
[38;2;248;248;242m              be  used as a text object in vi mode but with custom word styles. For com‐[0m
[38;2;248;248;242m              parison, the widgets described in zshzle(1) under Text Objects  use  fixed[0m
[38;2;248;248;242m              definitions of words, compatible with the vim editor.[0m

[38;2;248;248;242m              The simplest way of configuring the functions is to use select-word-style,[0m
[38;2;248;248;242m              which can either be called as a normal function with the appropriate argu‐[0m
[38;2;248;248;242m              ment,  or  invoked as a user-defined widget that will prompt for the first[0m
[38;2;248;248;242m              character of the word style to be used.  The first time it is invoked, the[0m
[38;2;248;248;242m              first eight -match functions will automatically replace the  builtin  ver‐[0m
[38;2;248;248;242m              sions, so they do not need to be loaded explicitly.[0m

[38;2;248;248;242m              The word styles available are as follows.  Only the first character is ex‐[0m
[38;2;248;248;242m              amined.[0m

[38;2;248;248;242m              bash   Word characters are alphanumeric characters only.[0m

[38;2;248;248;242m              normal As  in  normal  shell  operation:  word characters are alphanumeric[0m
[38;2;248;248;242m                     characters plus any characters present in the string given  by  the[0m
[38;2;248;248;242m                     parameter $WORDCHARS.[0m

[38;2;248;248;242m              shell  Words are complete shell command arguments, possibly including com‐[0m
[38;2;248;248;242m                     plete quoted strings, or any tokens special to the shell.[0m

[38;2;248;248;242m              whitespace[0m
[38;2;248;248;242m                     Words are any set of characters delimited by whitespace.[0m

[38;2;248;248;242m              default[0m
[38;2;248;248;242m                     Restore the default settings; this is usually the same as `normal'.[0m

[38;2;248;248;242m              All  but  `default' can be input as an upper case character, which has the[0m
[38;2;248;248;242m              same effect but with subword matching turned on.  In this case, words with[0m
[38;2;248;248;242m              upper case characters are treated specially: each separate  run  of  upper[0m
[38;2;248;248;242m              case  characters,  or  an  upper  case character followed by any number of[0m
[38;2;248;248;242m              other characters, is considered a word.  The style subword-range can  sup‐[0m
[38;2;248;248;242m              ply  an  alternative character range to the default `[:upper:]'; the value[0m
[38;2;248;248;242m              of the style is treated as the contents of a `[...]'  pattern  (note  that[0m
[38;2;248;248;242m              the  outer  brackets  should not be supplied, only those surrounding named[0m
[38;2;248;248;242m              ranges).[0m

[38;2;248;248;242m              More control can be obtained using the zstyle  command,  as  described  in[0m
[38;2;248;248;242m              zshmodules(1).   Each  style is looked up in the context :zle:widget where[0m
[38;2;248;248;242m              widget is the name of the user-defined widget, not the name of  the  func‐[0m
[38;2;248;248;242m              tion  implementing  it,  so in the case of the definitions supplied by se‐[0m
[38;2;248;248;242m              lect-word-style the appropriate contexts are :zle:forward-word, and so on.[0m
[38;2;248;248;242m              The function select-word-style itself always defines styles for  the  con‐[0m
[38;2;248;248;242m              text  `:zle:*'  which can be overridden by more specific (longer) patterns[0m
[38;2;248;248;242m              as well as explicit contexts.[0m

[38;2;248;248;242m              The style word-style specifies the rules to use.  This may have  the  fol‐[0m
[38;2;248;248;242m              lowing values.[0m

[38;2;248;248;242m              normal Use  the  standard  shell rules, i.e. alphanumerics and $WORDCHARS,[0m
[38;2;248;248;242m                     unless overridden by the styles word-chars or word-class.[0m

[38;2;248;248;242m              specified[0m
[38;2;248;248;242m                     Similar to normal, but only the specified characters, and not  also[0m
[38;2;248;248;242m                     alphanumerics, are considered word characters.[0m

[38;2;248;248;242m              unspecified[0m
[38;2;248;248;242m                     The  negation  of  specified.  The given characters are those which[0m
[38;2;248;248;242m                     will not be considered part of a word.[0m

[38;2;248;248;242m              shell  Words are obtained by using  the  syntactic  rules  for  generating[0m
[38;2;248;248;242m                     shell  command  arguments.   In  addition, special tokens which are[0m
[38;2;248;248;242m                     never command arguments such as `()' are also treated as words.[0m

[38;2;248;248;242m              whitespace[0m
[38;2;248;248;242m                     Words are whitespace-delimited strings of characters.[0m

[38;2;248;248;242m              The first three of those rules usually use $WORDCHARS, but  the  value  in[0m
[38;2;248;248;242m              the  parameter  can  be overridden by the style word-chars, which works in[0m
[38;2;248;248;242m              exactly the same way as $WORDCHARS.  In  addition,  the  style  word-class[0m
[38;2;248;248;242m              uses  character class syntax to group characters and takes precedence over[0m
[38;2;248;248;242m              word-chars if both are set.  The word-class style  does  not  include  the[0m
[38;2;248;248;242m              surrounding brackets of the character class; for example, `-:[:alnum:]' is[0m
[38;2;248;248;242m              a  valid  word-class  to include all alphanumerics plus the characters `-'[0m
[38;2;248;248;242m              and `:'.  Be careful including `]', `^' and `-' as these are  special  in‐[0m
[38;2;248;248;242m              side character classes.[0m

[38;2;248;248;242m              word-style  may also have `-subword' appended to its value to turn on sub‐[0m
[38;2;248;248;242m              word matching, as described above.[0m

[38;2;248;248;242m              The style skip-chars is mostly  useful  for  transpose-words  and  similar[0m
[38;2;248;248;242m              functions.   If set, it gives a count of characters starting at the cursor[0m
[38;2;248;248;242m              position which will not be considered part of the word and are treated  as[0m
[38;2;248;248;242m              space, regardless of what they actually are.  For example, if[0m

[38;2;248;248;242m                     zstyle ':zle:transpose-words' skip-chars 1[0m

[38;2;248;248;242m              has been set, and transpose-words-match is called with the cursor on the X[0m
[38;2;248;248;242m              of fooXbar, where X can be any character, then the resulting expression is[0m
[38;2;248;248;242m              barXfoo.[0m

[38;2;248;248;242m              Finer grained control can be obtained by setting the style word-context to[0m
[38;2;248;248;242m              an  array of pairs of entries.  Each pair of entries consists of a pattern[0m
[38;2;248;248;242m              and a subcontext.  The shell argument the cursor is on is matched  against[0m
[38;2;248;248;242m              each  pattern  in  turn  until one matches; if it does, the context is ex‐[0m
[38;2;248;248;242m              tended by a colon and the corresponding subcontext.  Note that the test is[0m
[38;2;248;248;242m              made against the original word on the line, with no stripping  of  quotes.[0m
[38;2;248;248;242m              Special  handling  is  done between words: the current context is examined[0m
[38;2;248;248;242m              and if it contains the string between the word is set to a  single  space;[0m
[38;2;248;248;242m              else if it is contains the string back, the word before the cursor is con‐[0m
[38;2;248;248;242m              sidered, else the word after cursor is considered. Some examples are given[0m
[38;2;248;248;242m              below.[0m

[38;2;248;248;242m              The style skip-whitespace-first is only used with the forward-word widget.[0m
[38;2;248;248;242m              If  it  is  set  to true, then forward-word skips any non-word-characters,[0m
[38;2;248;248;242m              followed by any non-word-characters: this is similar to the  behaviour  of[0m
[38;2;248;248;242m              other  word-orientated  widgets, and also that used by other editors, how‐[0m
[38;2;248;248;242m              ever  it  differs  from  the  standard  zsh  behaviour.   When  using  se‐[0m
[38;2;248;248;242m              lect-word-style  the  widget  is  set in the context :zle:* to true if the[0m
[38;2;248;248;242m              word style is bash and false otherwise.  It may be overridden  by  setting[0m
[38;2;248;248;242m              it in the more specific context :zle:forward-word*.[0m

[38;2;248;248;242m              It is possible to create widgets with specific behaviour by defining a new[0m
[38;2;248;248;242m              widget  implemented  by  the  appropriate generic function, then setting a[0m
[38;2;248;248;242m              style for the context of the specific widget.  For example, the  following[0m
[38;2;248;248;242m              defines  a widget backward-kill-space-word using backward-kill-word-match,[0m
[38;2;248;248;242m              the generic widget implementing backward-kill-word behaviour, and  ensures[0m
[38;2;248;248;242m              that the new widget always implements space-delimited behaviour.[0m

[38;2;248;248;242m                     zle -N backward-kill-space-word backward-kill-word-match[0m
[38;2;248;248;242m                     zstyle :zle:backward-kill-space-word word-style space[0m

[38;2;248;248;242m              The widget backward-kill-space-word can now be bound to a key.[0m

[38;2;248;248;242m              Here  are  some further examples of use of the styles, actually taken from[0m
[38;2;248;248;242m              the simplified interface in select-word-style:[0m

[38;2;248;248;242m                     zstyle ':zle:*' word-style standard[0m
[38;2;248;248;242m                     zstyle ':zle:*' word-chars ''[0m

[38;2;248;248;242m              Implements bash-style word handling for all widgets, i.e. only alphanumer‐[0m
[38;2;248;248;242m              ics are word characters; equivalent to  setting  the  parameter  WORDCHARS[0m
[38;2;248;248;242m              empty for the given context.[0m

[38;2;248;248;242m                     style ':zle:*kill*' word-style space[0m

[38;2;248;248;242m              Uses  space-delimited  words for widgets with the word `kill' in the name.[0m
[38;2;248;248;242m              Neither of the styles word-chars nor word-class is used in this case.[0m

[38;2;248;248;242m              Here are some examples of use of the word-context style to extend the con‐[0m
[38;2;248;248;242m              text.[0m

[38;2;248;248;242m                     zstyle ':zle:*' word-context \[0m
[38;2;248;248;242m                            "*/*" filename "[[:space:]]" whitespace[0m
[38;2;248;248;242m                     zstyle ':zle:transpose-words:whitespace' word-style shell[0m
[38;2;248;248;242m                     zstyle ':zle:transpose-words:filename' word-style normal[0m
[38;2;248;248;242m                     zstyle ':zle:transpose-words:filename' word-chars ''[0m

[38;2;248;248;242m              This provides two different ways of  using  transpose-words  depending  on[0m
[38;2;248;248;242m              whether  the  cursor is on whitespace between words or on a filename, here[0m
[38;2;248;248;242m              any word containing a /.  On whitespace, complete arguments as defined  by[0m
[38;2;248;248;242m              standard  shell rules will be transposed.  In a filename, only alphanumer‐[0m
[38;2;248;248;242m              ics will be transposed.  Elsewhere, words will be transposed using the de‐[0m
[38;2;248;248;242m              fault style for :zle:transpose-words.[0m

[38;2;248;248;242m              The word matching and all the handling of zstyle settings is actually  im‐[0m
[38;2;248;248;242m              plemented  by the function match-words-by-style.  This can be used to cre‐[0m
[38;2;248;248;242m              ate new user-defined widgets.  The calling function should set  the  local[0m
[38;2;248;248;242m              parameter   curcontext   to   :zle:widget,   create  the  local  parameter[0m
[38;2;248;248;242m              matched_words and call match-words-by-style with no arguments.  On return,[0m
[38;2;248;248;242m              matched_words will be set to an array with the elements: (1) the start  of[0m
[38;2;248;248;242m              the  line  (2)  the word before the cursor (3) any non-word characters be‐[0m
[38;2;248;248;242m              tween that word and the cursor (4) any non-word character  at  the  cursor[0m
[38;2;248;248;242m              position  plus any remaining non-word characters before the next word, in‐[0m
[38;2;248;248;242m              cluding all characters specified by the skip-chars style, (5) the word  at[0m
[38;2;248;248;242m              or  following  the  cursor (6) any non-word characters following that word[0m
[38;2;248;248;242m              (7) the remainder of the line.  Any  of  the  elements  may  be  an  empty[0m
[38;2;248;248;242m              string; the calling function should test for this to decide whether it can[0m
[38;2;248;248;242m              perform its function.[0m

[38;2;248;248;242m              If   the   variable   matched_words   is   defined   by   the   caller  to[0m
[38;2;248;248;242m              match-words-by-style as an associative  array  (local  -A  matched_words),[0m
[38;2;248;248;242m              then  the seven values given above should be retrieved from it as elements[0m
[38;2;248;248;242m              named  start,   word-before-cursor,   ws-before-cursor,   ws-after-cursor,[0m
[38;2;248;248;242m              word-after-cursor,  ws-after-word,  and  end.   In  addition  the  element[0m
[38;2;248;248;242m              is-word-start is 1 if the cursor is on the start of a word or subword,  or[0m
[38;2;248;248;242m              on  white  space  before it (the cases can be distinguished by testing the[0m
[38;2;248;248;242m              ws-after-cursor element) and 0 otherwise.  This form  is  recommended  for[0m
[38;2;248;248;242m              future compatibility.[0m

[38;2;248;248;242m              It  is  possible to pass options with arguments to match-words-by-style to[0m
[38;2;248;248;242m              override the use of styles.  The options are:[0m
[38;2;248;248;242m              -w     word-style[0m
[38;2;248;248;242m              -s     skip-chars[0m
[38;2;248;248;242m              -c     word-class[0m
[38;2;248;248;242m              -C     word-chars[0m
[38;2;248;248;242m              -r     subword-range[0m

[38;2;248;248;242m              For example, match-words-by-style -w shell -c 0 may be used to extract the[0m
[38;2;248;248;242m              command argument around the cursor.[0m

[38;2;248;248;242m              The word-context style is implemented by the function  match-word-context.[0m
[38;2;248;248;242m              This should not usually need to be called directly.[0m

[38;2;248;248;242m       bracketed-paste-magic[0m
[38;2;248;248;242m              The  bracketed-paste  widget  (see  the subsection `Miscellaneous' in zsh‐[0m
[38;2;248;248;242m              zle(1)) inserts pasted text literally into the editor buffer  rather  than[0m
[38;2;248;248;242m              interpret  it  as  keystrokes.  This disables some common usages where the[0m
[38;2;248;248;242m              self-insert widget is replaced in order to accomplish some extra  process‐[0m
[38;2;248;248;242m              ing.   An  example is the contributed url-quote-magic widget described be‐[0m
[38;2;248;248;242m              low.[0m

[38;2;248;248;242m              The bracketed-paste-magic widget is meant to replace bracketed-paste  with[0m
[38;2;248;248;242m              a  wrapper that re-enables these self-insert actions, and other actions as[0m
[38;2;248;248;242m              selected by zstyles.  Therefore this widget is installed with[0m

[38;2;248;248;242m                     autoload -Uz bracketed-paste-magic[0m
[38;2;248;248;242m                     zle -N bracketed-paste bracketed-paste-magic[0m

[38;2;248;248;242m              Other than enabling some widget processing, bracketed-paste-magic attempts[0m
[38;2;248;248;242m              to replicate bracketed-paste as faithfully as possible.[0m

[38;2;248;248;242m              The following zstyles may be set to control  processing  of  pasted  text.[0m
[38;2;248;248;242m              All are looked up in the context `:bracketed-paste-magic'.[0m

[38;2;248;248;242m              active-widgets[0m
[38;2;248;248;242m                     A  list  of patterns matching widget names that should be activated[0m
[38;2;248;248;242m                     during the  paste.   All  other  key  sequences  are  processed  as[0m
[38;2;248;248;242m                     self-insert-unmeta.   The  default  is `self-*' so any user-defined[0m
[38;2;248;248;242m                     widgets named with that prefix are active along  with  the  builtin[0m
[38;2;248;248;242m                     self-insert.[0m

[38;2;248;248;242m                     If  this  style  is not set (explicitly deleted) or set to an empty[0m
[38;2;248;248;242m                     value, no widgets are active and the pasted text is inserted liter‐[0m
[38;2;248;248;242m                     ally.  If the value includes `undefined-key', any unknown sequences[0m
[38;2;248;248;242m                     are discarded from the pasted text.[0m

[38;2;248;248;242m              inactive-keys[0m
[38;2;248;248;242m                     The inverse of active-widgets, a list of key sequences that  always[0m
[38;2;248;248;242m                     use  self-insert-unmeta  even when bound to an active widget.  Note[0m
[38;2;248;248;242m                     that this is a list of literal key sequences, not patterns.[0m

[38;2;248;248;242m              paste-init[0m
[38;2;248;248;242m                     A list of function names, called in widget context (but not as wid‐[0m
[38;2;248;248;242m                     gets).  The functions are called in order until one of them returns[0m
[38;2;248;248;242m                     a non-zero status.  The parameter  `PASTED'  contains  the  initial[0m
[38;2;248;248;242m                     state  of  the  pasted  text.   All  other  ZLE  parameters such as[0m
[38;2;248;248;242m                     `BUFFER' have their normal values and side-effects, and  full  his‐[0m
[38;2;248;248;242m                     tory  is  available,  so  for example paste-init functions may move[0m
[38;2;248;248;242m                     words from BUFFER into PASTED to make those words  visible  to  the[0m
[38;2;248;248;242m                     active-widgets.[0m

[38;2;248;248;242m                     A  non-zero  return from a paste-init function does not prevent the[0m
[38;2;248;248;242m                     paste itself from proceeding.[0m

[38;2;248;248;242m                     Loading  bracketed-paste-magic  defines  backward-extend-paste,   a[0m
[38;2;248;248;242m                     helper function for use in paste-init.[0m

[38;2;248;248;242m                            zstyle :bracketed-paste-magic paste-init \[0m
[38;2;248;248;242m                                   backward-extend-paste[0m

[38;2;248;248;242m                     When  a paste would insert into the middle of a word or append text[0m
[38;2;248;248;242m                     to a word already on the line, backward-extend-paste moves the pre‐[0m
[38;2;248;248;242m                     fix from LBUFFER into PASTED so that  the  active-widgets  see  the[0m
[38;2;248;248;242m                     full word so far.  This may be useful with url-quote-magic.[0m

[38;2;248;248;242m              paste-finish[0m
[38;2;248;248;242m                     Another  list  of  function names called in order until one returns[0m
[38;2;248;248;242m                     non-zero.  These functions are called after  the  pasted  text  has[0m
[38;2;248;248;242m                     been  processed  by  the  active-widgets, but before it is inserted[0m
[38;2;248;248;242m                     into  `BUFFER'.   ZLE  parameters  have  their  normal  values  and[0m
[38;2;248;248;242m                     side-effects.[0m

[38;2;248;248;242m                     A non-zero return from a paste-finish function does not prevent the[0m
[38;2;248;248;242m                     paste itself from proceeding.[0m

[38;2;248;248;242m                     Loading  bracketed-paste-magic  also  defines quote-paste, a helper[0m
[38;2;248;248;242m                     function for use in paste-finish.[0m

[38;2;248;248;242m                            zstyle :bracketed-paste-magic paste-finish \[0m
[38;2;248;248;242m                                   quote-paste[0m
[38;2;248;248;242m                            zstyle :bracketed-paste-magic:finish quote-style \[0m
[38;2;248;248;242m                                   qqq[0m

[38;2;248;248;242m                     When the pasted text is inserted into BUFFER, it is quoted per  the[0m
[38;2;248;248;242m                     quote-style  value.  To forcibly turn off the built-in numeric pre‐[0m
[38;2;248;248;242m                     fix quoting of bracketed-paste, use:[0m

[38;2;248;248;242m                            zstyle :bracketed-paste-magic:finish quote-style \[0m
[38;2;248;248;242m                                   none[0m

[38;2;248;248;242m              Important: During active-widgets processing of the paste (after paste-init[0m
[38;2;248;248;242m              and before paste-finish), BUFFER starts empty and history  is  restricted,[0m
[38;2;248;248;242m              so cursor motions, etc., may not pass outside of the pasted content.  Text[0m
[38;2;248;248;242m              assigned to BUFFER by the active widgets is copied back into PASTED before[0m
[38;2;248;248;242m              paste-finish.[0m

[38;2;248;248;242m       copy-earlier-word[0m

[38;2;248;248;242m              words on the relevant history line.  With a numeric argument N, insert the[0m

[38;2;248;248;242m              If insert-last-word has been used to retrieve the last word on a  previous[0m

[38;2;248;248;242m              Otherwise, the widget applies to words on the line currently being edited.[0m

[38;2;248;248;242m       cycle-completion-positions[0m
[38;2;248;248;242m              After inserting an unambiguous string into the command line, the new func‐[0m
[38;2;248;248;242m              tion based completion system may know about multiple places in this string[0m
[38;2;248;248;242m              where characters are missing or differ from at least one of  the  possible[0m
[38;2;248;248;242m              matches.  It will then place the cursor on the position it considers to be[0m
[38;2;248;248;242m              the  most interesting one, i.e. the one where one can disambiguate between[0m
[38;2;248;248;242m              as many matches as possible with as little typing as possible.[0m

[38;2;248;248;242m              This widget allows the cursor to be easily moved to the other  interesting[0m
[38;2;248;248;242m              spots.   It  can  be invoked repeatedly to cycle between all positions re‐[0m
[38;2;248;248;242m              ported by the completion system.[0m

[38;2;248;248;242m       delete-whole-word-match[0m
[38;2;248;248;242m              This is another function which works like the -match  functions  described[0m
[38;2;248;248;242m              immediately  above, i.e. using styles to decide the word boundaries.  How‐[0m
[38;2;248;248;242m              ever, it is not a replacement for any existing function.[0m

[38;2;248;248;242m              The basic behaviour is to delete the word around the cursor.  There is  no[0m
[38;2;248;248;242m              numeric  argument handling; only the single word around the cursor is con‐[0m
[38;2;248;248;242m              sidered.  If the widget contains the string kill, the removed text will be[0m

[38;2;248;248;242m                     zle -N kill-whole-word-match delete-whole-word-match[0m

[38;2;248;248;242m              and then binding the widget kill-whole-word-match.[0m

[38;2;248;248;242m       up-line-or-beginning-search, down-line-or-beginning-search[0m
[38;2;248;248;242m              These  widgets  are similar to the builtin functions up-line-or-search and[0m
[38;2;248;248;242m              down-line-or-search:  if in a multiline buffer they move up or down within[0m

[38;2;248;248;242m       edit-command-line[0m

[38;2;248;248;242m              The  editor to be used can also be specified using the editor style in the[0m

[38;2;248;248;242m              ments:[0m

[38;2;248;248;242m       expand-absolute-path[0m

[38;2;248;248;242m       history-search-end[0m

[38;2;248;248;242m       history-beginning-search-menu[0m
[38;2;248;248;242m              This function implements yet another form of history searching.  The  text[0m

[38;2;248;248;242m              Note that leading zeroes must be typed (they are only shown when necessary[0m

[38;2;248;248;242m       insert-composed-char[0m
[38;2;248;248;242m              This function allows you to compose characters that don't  appear  on  the[0m
[38;2;248;248;242m              keyboard to be inserted into the command line.  The command is followed by[0m
[38;2;248;248;242m              two  keys corresponding to ASCII characters (there is no prompt).  For ac‐[0m
[38;2;248;248;242m              cented characters, the two keys are a base character followed  by  a  code[0m
[38;2;248;248;242m              for  the accent, while for other special characters the two characters to‐[0m
[38;2;248;248;242m              gether form a mnemonic for the character to be inserted.  The  two-charac‐[0m
[38;2;248;248;242m              ter  codes  are  a  subset  of  those  given  by RFC 1345 (see for example[0m
[38;2;248;248;242m              http://www.faqs.org/rfcs/rfc1345.html).[0m

[38;2;248;248;242m              The function may optionally be followed by up to two characters which  re‐[0m
[38;2;248;248;242m              place  one or both of the characters read from the keyboard; if both char‐[0m
[38;2;248;248;242m              acters are supplied, no input is read.  For example,  insert-composed-char[0m
[38;2;248;248;242m              a: can be used within a widget to insert an a with umlaut into the command[0m
[38;2;248;248;242m              line.   This has the advantages over use of a literal character that it is[0m
[38;2;248;248;242m              more portable.[0m

[38;2;248;248;242m              For best results zsh should have been built  with  support  for  multibyte[0m
[38;2;248;248;242m              characters  (configured  with  --enable-multibyte);  however, the function[0m
[38;2;248;248;242m              works for the limited range of characters available in single-byte charac‐[0m
[38;2;248;248;242m              ter sets such as ISO-8859-1.[0m

[38;2;248;248;242m              The character is converted into the local representation and inserted into[0m
[38;2;248;248;242m              the command line at the cursor position.  (The conversion is  done  within[0m
[38;2;248;248;242m              the  shell, using whatever facilities the C library provides.)  With a nu‐[0m
[38;2;248;248;242m              meric argument, the character and its code are  previewed  in  the  status[0m
[38;2;248;248;242m              line[0m

[38;2;248;248;242m              The  function may be run outside zle in which case it prints the character[0m
[38;2;248;248;242m              (together with a newline) to standard output.  Input is  still  read  from[0m
[38;2;248;248;242m              keystrokes.[0m

[38;2;248;248;242m              See  insert-unicode-char for an alternative way of inserting Unicode char‐[0m

[38;2;248;248;242m              The set of accented characters is reasonably complete up to Unicode  char‐[0m

[38;2;248;248;242m       insert-files[0m
[38;2;248;248;242m              This  function  allows you type a file pattern, and see the results of the[0m
[38;2;248;248;242m              expansion at each step.  When you hit return, all expansions are  inserted[0m
[38;2;248;248;242m              into the command line.[0m

[38;2;248;248;242m                     bindkey '^Xf' insert-files[0m

[38;2;248;248;242m       insert-unicode-char[0m
[38;2;248;248;242m              When first executed, the user inputs a set of hexadecimal digits.  This is[0m
[38;2;248;248;242m              terminated  with another call to insert-unicode-char.  The digits are then[0m
[38;2;248;248;242m              turned into the corresponding Unicode character.  For example, if the wid‐[0m
[38;2;248;248;242m              get is bound to ^XU, the character sequence `^XU 4 c ^XU' inserts L  (Uni‐[0m
[38;2;248;248;242m              code U+004c).[0m

[38;2;248;248;242m              See  insert-composed-char  for  a  way  of  inserting  characters  using a[0m
[38;2;248;248;242m              two-character mnemonic.[0m

[38;2;248;248;242m       narrow-to-region [ -p pre ] [ -P post ][0m
[38;2;248;248;242m                        [ -S statepm | -R statepm | [ -l lbufvar ] [ -r rbufvar ] ][0m
[38;2;248;248;242m                        [ -n ] [ start end ][0m
[38;2;248;248;242m       narrow-to-region-invisible[0m
[38;2;248;248;242m              Narrow the editable portion of the buffer to the region between the cursor[0m
[38;2;248;248;242m              and the mark, which may be in either order.  The region may not be empty.[0m

[38;2;248;248;242m              narrow-to-region may be used as a widget or called as a  function  from  a[0m
[38;2;248;248;242m              user-defined  widget;  by  default, the text outside the editable area re‐[0m
[38;2;248;248;242m              mains visible.  A recursive-edit is performed and  the  original  widening[0m
[38;2;248;248;242m              status is then restored.  Various options and arguments are available when[0m
[38;2;248;248;242m              it is called as a function.[0m

[38;2;248;248;242m              The options -p pretext and -P posttext may be used to replace the text be‐[0m
[38;2;248;248;242m              fore  and  after  the  display for the duration of the function; either or[0m
[38;2;248;248;242m              both may be an empty string.[0m

[38;2;248;248;242m              If the option -n is also given, pretext or posttext will only be  inserted[0m
[38;2;248;248;242m              if  there  is  text  before or after the region respectively which will be[0m
[38;2;248;248;242m              made invisible.[0m

[38;2;248;248;242m              Two numeric arguments may be given which will be used instead of the  cur‐[0m
[38;2;248;248;242m              sor and mark positions.[0m

[38;2;248;248;242m              The  option  -S  statepm  is used to narrow according to the other options[0m
[38;2;248;248;242m              while saving the original state in the parameter with name statepm,  while[0m
[38;2;248;248;242m              the  option  -R  statepm  is used to restore the state from the parameter;[0m
[38;2;248;248;242m              note in both cases the name of the parameter is required.  In  the  second[0m
[38;2;248;248;242m              case,  other  options  and  arguments are irrelevant.  When this method is[0m
[38;2;248;248;242m              used, no recursive-edit is performed; the calling widget should call  this[0m
[38;2;248;248;242m              function  with  the option -S, perform its own editing on the command line[0m
[38;2;248;248;242m              or pass control to the user via `zle recursive-edit', then call this func‐[0m
[38;2;248;248;242m              tion with the option -R.  The argument statepm must be a suitable name for[0m
[38;2;248;248;242m              an ordinary parameter, except that parameters beginning  with  the  prefix[0m
[38;2;248;248;242m              _ntr_ are reserved for use within narrow-to-region.  Typically the parame‐[0m
[38;2;248;248;242m              ter will be local to the calling function.[0m

[38;2;248;248;242m              The  options  -l  lbufvar and -r rbufvar may be used to specify parameters[0m
[38;2;248;248;242m              where the widget will store the resulting text from  the  operation.   The[0m
[38;2;248;248;242m              parameter  lbufvar  will contain LBUFFER and rbufvar will contain RBUFFER.[0m
[38;2;248;248;242m              Neither of these two options may be used with -S or -R.[0m

[38;2;248;248;242m              narrow-to-region-invisible is a simple widget which calls narrow-to-region[0m
[38;2;248;248;242m              with arguments which replace any text outside the region with  `...'.   It[0m
[38;2;248;248;242m              does not take any arguments.[0m

[38;2;248;248;242m              The  display  is  restored  (and  the widget returns) upon any zle command[0m
[38;2;248;248;242m              which would usually cause the line to be accepted or  aborted.   Hence  an[0m
[38;2;248;248;242m              additional such command is required to accept or abort the current line.[0m

[38;2;248;248;242m              The  return  status of both widgets is zero if the line was accepted, else[0m
[38;2;248;248;242m              non-zero.[0m

[38;2;248;248;242m              Here is a trivial example of a widget using this feature.[0m
[38;2;248;248;242m                     local state[0m
[38;2;248;248;242m                     narrow-to-region -p $'Editing restricted region\n' \[0m
[38;2;248;248;242m                       -P '' -S state[0m
[38;2;248;248;242m                     zle recursive-edit[0m
[38;2;248;248;242m                     narrow-to-region -R state[0m

[38;2;248;248;242m       predict-on[0m
[38;2;248;248;242m              This set of functions implements predictive typing using  history  search.[0m
[38;2;248;248;242m              After  predict-on, typing characters causes the editor to look backward in[0m
[38;2;248;248;242m              the history for the first line beginning with what you have typed so  far.[0m
[38;2;248;248;242m              After predict-off, editing returns to normal for the line found.  In fact,[0m
[38;2;248;248;242m              you  often don't even need to use predict-off, because if the line doesn't[0m
[38;2;248;248;242m              match something in the history, adding a key performs standard completion,[0m
[38;2;248;248;242m              and then inserts itself if no completions were found.  However, editing in[0m
[38;2;248;248;242m              the middle of a line is liable to confuse prediction; see the toggle style[0m
[38;2;248;248;242m              below.[0m

[38;2;248;248;242m              With the function based completion system (which is needed for this),  you[0m
[38;2;248;248;242m              should  be  able  to type TAB at almost any point to advance the cursor to[0m
[38;2;248;248;242m              the next ``interesting'' character position (usually the end of  the  cur‐[0m
[38;2;248;248;242m              rent  word,  but  sometimes  somewhere in the middle of the word).  And of[0m
[38;2;248;248;242m              course as soon as the entire line is what you want, you  can  accept  with[0m
[38;2;248;248;242m              return, without needing to move the cursor to the end first.[0m

[38;2;248;248;242m              The  first  time  predict-on is used, it creates several additional widget[0m
[38;2;248;248;242m              functions:[0m

[38;2;248;248;242m              delete-backward-and-predict[0m
[38;2;248;248;242m                     Replaces the backward-delete-char widget.  You do not need to  bind[0m
[38;2;248;248;242m                     this yourself.[0m
[38;2;248;248;242m              insert-and-predict[0m
[38;2;248;248;242m                     Implements  predictive  typing by replacing the self-insert widget.[0m
[38;2;248;248;242m                     You do not need to bind this yourself.[0m
[38;2;248;248;242m              predict-off[0m
[38;2;248;248;242m                     Turns off predictive typing.[0m

[38;2;248;248;242m              Although you autoload only the predict-on function,  it  is  necessary  to[0m
[38;2;248;248;242m              create a keybinding for predict-off as well.[0m

[38;2;248;248;242m                     zle -N predict-on[0m
[38;2;248;248;242m                     zle -N predict-off[0m
[38;2;248;248;242m                     bindkey '^X^Z' predict-on[0m
[38;2;248;248;242m                     bindkey '^Z' predict-off[0m

[38;2;248;248;242m       read-from-minibuffer[0m
[38;2;248;248;242m              This  is  most  useful when called as a function from inside a widget, but[0m
[38;2;248;248;242m              will work correctly as a widget in its own right.  It prompts for a  value[0m
[38;2;248;248;242m              below  the  current  command  line;  a value may be input using all of the[0m
[38;2;248;248;242m              standard zle operations (and not merely the restricted set available  when[0m
[38;2;248;248;242m              executing, for example, execute-named-cmd).  The value is then returned to[0m
[38;2;248;248;242m              the  calling  function  in the parameter $REPLY and the editing buffer re‐[0m
[38;2;248;248;242m              stored to its previous state.  If the read was aborted by a keyboard break[0m
[38;2;248;248;242m              (typically ^G), the function returns status 1 and $REPLY is not set.[0m

[38;2;248;248;242m              If one argument is supplied to the function it is taken as a prompt,  oth‐[0m
[38;2;248;248;242m              erwise  `?  ' is used.  If two arguments are supplied, they are the prompt[0m
[38;2;248;248;242m              and the initial value of $LBUFFER, and if a third argument is given it  is[0m
[38;2;248;248;242m              the initial value of $RBUFFER.  This provides a default value and starting[0m
[38;2;248;248;242m              cursor placement.  Upon return the entire buffer is the value of $REPLY.[0m

[38;2;248;248;242m              One  option is available: `-k num' specifies that num characters are to be[0m
[38;2;248;248;242m              read instead of a whole line.  The line editor is not invoked  recursively[0m
[38;2;248;248;242m              in  this  case, so depending on the terminal settings the input may not be[0m
[38;2;248;248;242m              visible, and only the input keys are placed  in  $REPLY,  not  the  entire[0m
[38;2;248;248;242m              buffer.   Note that unlike the read builtin num must be given; there is no[0m
[38;2;248;248;242m              default.[0m

[38;2;248;248;242m              The name is a slight misnomer, as in fact the shell's  own  minibuffer  is[0m
[38;2;248;248;242m              not used.  Hence it is still possible to call executed-named-cmd and simi‐[0m
[38;2;248;248;242m              lar functions while reading a value.[0m

[38;2;248;248;242m       replace-argument, replace-argument-edit[0m
[38;2;248;248;242m              The  function replace-argument can be used to replace a command line argu‐[0m
[38;2;248;248;242m              ment in the current command line or, if the current command line is empty,[0m
[38;2;248;248;242m              in the last command line executed (the new command line is not  executed).[0m
[38;2;248;248;242m              Arguments are as delimited by standard shell syntax,[0m

[38;2;248;248;242m              If  a  numeric  argument  is  given, that specifies the argument to be re‐[0m
[38;2;248;248;242m              placed.  0 means the command name, as in history  expansion.   A  negative[0m
[38;2;248;248;242m              numeric argument counts backward from the last word.[0m

[38;2;248;248;242m              If no numeric argument is given, the current argument is replaced; this is[0m
[38;2;248;248;242m              the last argument if the previous history line is being used.[0m

[38;2;248;248;242m              The function prompts for a replacement argument.[0m

[38;2;248;248;242m              If the widget contains the string edit, for example is defined as[0m

[38;2;248;248;242m                     zle -N replace-argument-edit replace-argument[0m

[38;2;248;248;242m              then  the function presents the current value of the argument for editing,[0m
[38;2;248;248;242m              otherwise the editing buffer for the replacement is initially empty.[0m

[38;2;248;248;242m       replace-string, replace-pattern[0m
[38;2;248;248;242m       replace-string-again, replace-pattern-again[0m
[38;2;248;248;242m              The function replace-string implements three widgets.   If  defined  under[0m
[38;2;248;248;242m              the  same  name  as  the  function,  it prompts for two strings; the first[0m
[38;2;248;248;242m              (source) string will be replaced by the second everywhere it occurs in the[0m
[38;2;248;248;242m              line editing buffer.[0m

[38;2;248;248;242m              If the widget name contains the word `pattern', for  example  by  defining[0m
[38;2;248;248;242m              the widget using the command `zle -N replace-pattern replace-string', then[0m
[38;2;248;248;242m              the  matching  is performed using zsh patterns.  All zsh extended globbing[0m
[38;2;248;248;242m              patterns can be used in the source string; note that unlike filename  gen‐[0m
[38;2;248;248;242m              eration  the  pattern  does  not need to match an entire word, nor do glob[0m
[38;2;248;248;242m              qualifiers have any effect.  In addition, the replacement string can  con‐[0m
[38;2;248;248;242m              tain  parameter  or  command substitutions.  Furthermore, a `&' in the re‐[0m
[38;2;248;248;242m              placement string will be replaced with the matched source  string,  and  a[0m
[38;2;248;248;242m              backquoted digit `\N' will be replaced by the Nth parenthesised expression[0m
[38;2;248;248;242m              matched.   The form `\{N}' may be used to protect the digit from following[0m
[38;2;248;248;242m              digits.[0m

[38;2;248;248;242m              If the widget instead contains the word `regex' (or  `regexp'),  then  the[0m
[38;2;248;248;242m              matching is performed using regular expressions, respecting the setting of[0m
[38;2;248;248;242m              the  option  RE_MATCH_PCRE (see the description of the function regexp-re‐[0m
[38;2;248;248;242m              place below).  The special replacement facilities described above for pat‐[0m
[38;2;248;248;242m              tern matching are available.[0m

[38;2;248;248;242m              By default the previous source or replacement string will not  be  offered[0m
[38;2;248;248;242m              for  editing.  However, this feature can be activated by setting the style[0m
[38;2;248;248;242m              edit-previous  in  the  context   :zle:widget   (for   example,   :zle:re‐[0m
[38;2;248;248;242m              place-string)  to  true.   In addition, a positive numeric argument forces[0m
[38;2;248;248;242m              the previous values to be offered, a negative or zero argument forces them[0m
[38;2;248;248;242m              not to be.[0m

[38;2;248;248;242m              The function replace-string-again can be used to repeat the  previous  re‐[0m
[38;2;248;248;242m              placement;  no  prompting is done.  As with replace-string, if the name of[0m
[38;2;248;248;242m              the widget contains the word `pattern' or `regex', pattern or regular  ex‐[0m
[38;2;248;248;242m              pression  matching  is performed, else a literal string replacement.  Note[0m
[38;2;248;248;242m              that the previous source and replacement text are the  same  whether  pat‐[0m
[38;2;248;248;242m              tern, regular expression or string matching is used.[0m

[38;2;248;248;242m              In  addition,  replace-string  shows  the  previous  replacement above the[0m
[38;2;248;248;242m              prompt, so long as there was one during the current session; if the source[0m
[38;2;248;248;242m              string is empty, that replacement will  be  repeated  without  the  widget[0m
[38;2;248;248;242m              prompting for a replacement string.[0m

[38;2;248;248;242m              For example, starting from the line:[0m

[38;2;248;248;242m                     print This line contains fan and fond[0m

[38;2;248;248;242m              and  invoking  replace-pattern  with the source string `f(?)n' and the re‐[0m
[38;2;248;248;242m              placement string `c\1r' produces the not very useful line:[0m

[38;2;248;248;242m                     print This line contains car and cord[0m

[38;2;248;248;242m              The range of the replacement string can  be  limited  by  using  the  nar‐[0m
[38;2;248;248;242m              row-to-region-invisible  widget.  One limitation of the current version is[0m
[38;2;248;248;242m              that undo will cycle through changes to the replacement and source strings[0m
[38;2;248;248;242m              before undoing the replacement itself.[0m

[38;2;248;248;242m       send-invisible[0m
[38;2;248;248;242m              This is similar to read-from-minibuffer in that it  may  be  called  as  a[0m
[38;2;248;248;242m              function  from a widget or as a widget of its own, and interactively reads[0m
[38;2;248;248;242m              input from the keyboard.  However, the input being typed is concealed  and[0m
[38;2;248;248;242m              a  string  of asterisks (`*') is shown instead.  The value is saved in the[0m
[38;2;248;248;242m              parameter $INVISIBLE to which a reference is  inserted  into  the  editing[0m
[38;2;248;248;242m              buffer at the restored cursor position.  If the read was aborted by a key‐[0m
[38;2;248;248;242m              board  break  (typically  ^G)  or  another  escape  from  editing  such as[0m
[38;2;248;248;242m              push-line, $INVISIBLE is set to empty and the original buffer is  restored[0m
[38;2;248;248;242m              unchanged.[0m

[38;2;248;248;242m              If  one argument is supplied to the function it is taken as a prompt, oth‐[0m
[38;2;248;248;242m              erwise `Non-echoed text: ' is used (as in emacs).  If a second  and  third[0m
[38;2;248;248;242m              argument are supplied they are used to begin and end the reference to $IN‐[0m
[38;2;248;248;242m              VISIBLE that is inserted into the buffer.  The default is to open with ${,[0m
[38;2;248;248;242m              then INVISIBLE, and close with }, but many other effects are possible.[0m

[38;2;248;248;242m       smart-insert-last-word[0m
[38;2;248;248;242m              This function may replace the insert-last-word widget, like so:[0m

[38;2;248;248;242m                     zle -N insert-last-word smart-insert-last-word[0m

[38;2;248;248;242m              With  a  numeric argument, or when passed command line arguments in a call[0m
[38;2;248;248;242m              from another widget, it behaves like insert-last-word, except  that  words[0m
[38;2;248;248;242m              in comments are ignored when INTERACTIVE_COMMENTS is set.[0m

[38;2;248;248;242m              Otherwise, the rightmost ``interesting'' word from the previous command is[0m
[38;2;248;248;242m              found and inserted.  The default definition of ``interesting'' is that the[0m
[38;2;248;248;242m              word  contains  at  least  one  alphabetic character, slash, or backslash.[0m
[38;2;248;248;242m              This definition may be overridden by use of the match style.  The  context[0m
[38;2;248;248;242m              used  to  look  up the style is the widget name, so usually the context is[0m
[38;2;248;248;242m              :insert-last-word.  However, you can bind this function to different  wid‐[0m
[38;2;248;248;242m              gets to use different patterns:[0m

[38;2;248;248;242m                     zle -N insert-last-assignment smart-insert-last-word[0m
[38;2;248;248;242m                     zstyle :insert-last-assignment match '[[:alpha:]][][[:alnum:]]#=*'[0m
[38;2;248;248;242m                     bindkey '\e=' insert-last-assignment[0m

[38;2;248;248;242m              If  no  interesting  word is found and the auto-previous style is set to a[0m
[38;2;248;248;242m              true value,  the  search  continues  upward  through  the  history.   When[0m
[38;2;248;248;242m              auto-previous  is unset or false (the default), the widget must be invoked[0m
[38;2;248;248;242m              repeatedly in order to search earlier history lines.[0m

[38;2;248;248;242m       transpose-lines[0m
[38;2;248;248;242m              Only useful with a multi-line editing buffer; the  lines  here  are  lines[0m
[38;2;248;248;242m              within  the  current  on-screen  buffer, not history lines.  The effect is[0m
[38;2;248;248;242m              similar to the function of the same name in Emacs.[0m

[38;2;248;248;242m              Transpose the current line with the previous line and move the  cursor  to[0m
[38;2;248;248;242m              the  start of the next line.  Repeating this (which can be done by provid‐[0m
[38;2;248;248;242m              ing a positive numeric argument) has the effect of moving the  line  above[0m
[38;2;248;248;242m              the cursor down by a number of lines.[0m

[38;2;248;248;242m              With  a  negative  numeric  argument, requires two lines above the cursor.[0m
[38;2;248;248;242m              These two lines are transposed and the cursor moved to the  start  of  the[0m
[38;2;248;248;242m              previous  line.   Using  a numeric argument less than -1 has the effect of[0m
[38;2;248;248;242m              moving the line above the cursor up by minus that number of lines.[0m

[38;2;248;248;242m       url-quote-magic[0m
[38;2;248;248;242m              This widget replaces the built-in self-insert to make it  easier  to  type[0m
[38;2;248;248;242m              URLs  as command line arguments.  As you type, the input character is ana‐[0m
[38;2;248;248;242m              lyzed and, if it may need quoting, the current word is checked for  a  URI[0m
[38;2;248;248;242m              scheme.   If one is found and the current word is not already in quotes, a[0m
[38;2;248;248;242m              backslash is inserted before the input character.[0m

[38;2;248;248;242m              Styles to control quoting behavior:[0m

[38;2;248;248;242m              url-metas[0m
[38;2;248;248;242m                     This style is looked up in  the  context  `:url-quote-magic:scheme'[0m
[38;2;248;248;242m                     (where  scheme  is that of the current URL, e.g. "ftp").  The value[0m
[38;2;248;248;242m                     is a string listing  the  characters  to  be  treated  as  globbing[0m
[38;2;248;248;242m                     metacharacters  when appearing in a URL using that scheme.  The de‐[0m
[38;2;248;248;242m                     fault is to quote all zsh extended globbing  characters,  excluding[0m
[38;2;248;248;242m                     '<' and '>' but including braces (as in brace expansion).  See also[0m
[38;2;248;248;242m                     url-seps.[0m

[38;2;248;248;242m              url-seps[0m
[38;2;248;248;242m                     Like url-metas, but lists characters that should be considered com‐[0m
[38;2;248;248;242m                     mand  separators,  redirections,  history references, etc.  The de‐[0m
[38;2;248;248;242m                     fault is to quote the standard set of shell  separators,  excluding[0m
[38;2;248;248;242m                     those  that  overlap with the extended globbing characters, but in‐[0m
[38;2;248;248;242m                     cluding '<' and '>' and the first character of $histchars.[0m

[38;2;248;248;242m              url-globbers[0m
[38;2;248;248;242m                     This style is looked up in  the  context  `:url-quote-magic'.   The[0m
[38;2;248;248;242m                     values  form  a list of command names that are expected to do their[0m
[38;2;248;248;242m                     own globbing on the URL string.  This implies that they are aliased[0m
[38;2;248;248;242m                     to use the `noglob' modifier.  When the  first  word  on  the  line[0m
[38;2;248;248;242m                     matches  one  of the values and the URL refers to a local file (see[0m
[38;2;248;248;242m                     url-local-schema), only the url-seps  characters  are  quoted;  the[0m
[38;2;248;248;242m                     url-metas  are  left  alone,  allowing  them to affect command-line[0m
[38;2;248;248;242m                     parsing,  completion,  etc.   The  default  values  are  a  literal[0m
[38;2;248;248;242m                     `noglob' plus (when the zsh/parameter module is available) any com‐[0m
[38;2;248;248;242m                     mands  aliased  to  the  helper  function `urlglobber' or its alias[0m
[38;2;248;248;242m                     `globurl'.[0m

[38;2;248;248;242m              url-local-schema[0m
[38;2;248;248;242m                     This style is always looked up in the context  `:urlglobber',  even[0m
[38;2;248;248;242m                     though it is used by both url-quote-magic and urlglobber.  The val‐[0m
[38;2;248;248;242m                     ues  form  a list of URI schema that should be treated as referring[0m
[38;2;248;248;242m                     to local files by their real local path names, as opposed to  files[0m
[38;2;248;248;242m                     which are specified relative to a web-server-defined document root.[0m
[38;2;248;248;242m                     The defaults are "ftp" and "file".[0m

[38;2;248;248;242m              url-other-schema[0m
[38;2;248;248;242m                     Like  url-local-schema,  but  lists all other URI schema upon which[0m
[38;2;248;248;242m                     urlglobber and url-quote-magic should act.  If the URI on the  com‐[0m
[38;2;248;248;242m                     mand  line  does not have a scheme appearing either in this list or[0m
[38;2;248;248;242m                     in url-local-schema, it is not magically quoted.  The default  val‐[0m
[38;2;248;248;242m                     ues  are  "http",  "https",  and "ftp".  When a scheme appears both[0m
[38;2;248;248;242m                     here and in url-local-schema, it is quoted differently depending on[0m
[38;2;248;248;242m                     whether the command name appears in url-globbers.[0m

[38;2;248;248;242m              Loading url-quote-magic also defines a helper  function  `urlglobber'  and[0m
[38;2;248;248;242m              aliases `globurl' to `noglob urlglobber'.  This function takes a local URL[0m
[38;2;248;248;242m              apart,  attempts  to pattern-match the local file portion of the URL path,[0m
[38;2;248;248;242m              and then puts the results back into URL format again.[0m

[38;2;248;248;242m       vi-pipe[0m
[38;2;248;248;242m              This function reads a movement command from the keyboard and then  prompts[0m
[38;2;248;248;242m              for an external command. The part of the buffer covered by the movement is[0m
[38;2;248;248;242m              piped  to  the external command and then replaced by the command's output.[0m
[38;2;248;248;242m              If the movement command is bound to vi-pipe, the current line is used.[0m

[38;2;248;248;242m              The function serves as an example for reading a vi movement  command  from[0m
[38;2;248;248;242m              within a user-defined widget.[0m

[38;2;248;248;242m       which-command[0m
[38;2;248;248;242m              This  function  is a drop-in replacement for the builtin widget which-com‐[0m
[38;2;248;248;242m              mand.  It has enhanced behaviour, in that it correctly detects whether  or[0m
[38;2;248;248;242m              not the command word needs to be expanded as an alias; if so, it continues[0m
[38;2;248;248;242m              tracing the command word from the expanded alias until it reaches the com‐[0m
[38;2;248;248;242m              mand that will be executed.[0m

[38;2;248;248;242m              The style whence is available in the context :zle:$WIDGET; this may be set[0m
[38;2;248;248;242m              to  an array to give the command and options that will be used to investi‐[0m
[38;2;248;248;242m              gate the command word found.  The default is whence -c.[0m

[38;2;248;248;242m       zcalc-auto-insert[0m
[38;2;248;248;242m              This function is useful together with the zcalc function described in  the[0m
[38;2;248;248;242m              section  `Mathematical Functions'.  It should be bound to a key represent‐[0m
[38;2;248;248;242m              ing a binary operator such as `+', `-',  `*'  or  `/'.   When  running  in[0m
[38;2;248;248;242m              zcalc, if the key occurs at the start of the line or immediately following[0m
[38;2;248;248;242m              an open parenthesis, the text "ans " is inserted before the representation[0m
[38;2;248;248;242m              of  the  key itself.  This allows easy use of the answer from the previous[0m
[38;2;248;248;242m              calculation in the current line.  The text to be inserted before the  sym‐[0m
[38;2;248;248;242m              bol  typed  can be modified by setting the variable ZCALC_AUTO_INSERT_PRE‐[0m
[38;2;248;248;242m              FIX.[0m

[38;2;248;248;242m              Hence, for example, typing `+12' followed by return adds 12 to the  previ‐[0m
[38;2;248;248;242m              ous result.[0m

[38;2;248;248;242m              If zcalc is in RPN mode (-r option) the effect of this binding is automat‐[0m
[38;2;248;248;242m              ically suppressed as operators alone on a line are meaningful.[0m

[38;2;248;248;242m              When not in zcalc, the key simply inserts the symbol itself.[0m

[38;2;248;248;242m   Utility Functions[0m
[38;2;248;248;242m       These  functions  are useful in constructing widgets.  They should be loaded with[0m
[38;2;248;248;242m       `autoload -U function' and called as indicated from user-defined widgets.[0m

[38;2;248;248;242m       split-shell-arguments[0m
[38;2;248;248;242m              This function splits the line currently being edited into shell  arguments[0m
[38;2;248;248;242m              and  whitespace.  The result is stored in the array reply.  The array con‐[0m
[38;2;248;248;242m              tains all the parts of the line in order, starting with any whitespace be‐[0m
[38;2;248;248;242m              fore the first argument, and finishing with any whitespace after the  last[0m
[38;2;248;248;242m              argument.   Hence (so long as the option KSH_ARRAYS is not set) whitespace[0m
[38;2;248;248;242m              is given by odd indices in the array and arguments by even indices.   Note[0m
[38;2;248;248;242m              that  no stripping of quotes is done; joining together all the elements of[0m
[38;2;248;248;242m              reply in order is guaranteed to produce the original line.[0m

[38;2;248;248;242m              The parameter REPLY is set to the index of the word in  reply  which  con‐[0m
[38;2;248;248;242m              tains the character after the cursor, where the first element has index 1.[0m
[38;2;248;248;242m              The parameter REPLY2 is set to the index of the character under the cursor[0m
[38;2;248;248;242m              in that word, where the first character has index 1.[0m

[38;2;248;248;242m              Hence  reply,  REPLY  and REPLY2 should all be made local to the enclosing[0m
[38;2;248;248;242m              function.[0m

[38;2;248;248;242m              See the function modify-current-argument, described below, for an  example[0m
[38;2;248;248;242m              of how to call this function.[0m

[38;2;248;248;242m       modify-current-argument [ expr-using-$ARG | func ][0m
[38;2;248;248;242m              This function provides a simple method of allowing user-defined widgets to[0m
[38;2;248;248;242m              modify  the  command line argument under the cursor (or immediately to the[0m
[38;2;248;248;242m              left of the cursor if the cursor is between arguments).[0m

[38;2;248;248;242m              The argument can be an expression which when  evaluated  operates  on  the[0m
[38;2;248;248;242m              shell parameter ARG, which will have been set to the command line argument[0m
[38;2;248;248;242m              under  the cursor.  The expression should be suitably quoted to prevent it[0m
[38;2;248;248;242m              being evaluated too early.[0m

[38;2;248;248;242m              Alternatively, if the argument does not contain the string ARG, it is  as‐[0m
[38;2;248;248;242m              sumed  to  be a shell function, to which the current command line argument[0m
[38;2;248;248;242m              is passed as the only argument.  The function should set the variable  RE‐[0m
[38;2;248;248;242m              PLY  to  the new value for the command line argument.  If the function re‐[0m
[38;2;248;248;242m              turns non-zero status, so does the calling function.[0m

[38;2;248;248;242m              For example, a user-defined widget containing the following code  converts[0m
[38;2;248;248;242m              the characters in the argument under the cursor into all upper case:[0m

[38;2;248;248;242m                     modify-current-argument '${(U)ARG}'[0m

[38;2;248;248;242m              The  following  strips  any  quoting  from the current word (whether back‐[0m
[38;2;248;248;242m              slashes or one of the styles of quotes), and replaces it with single quot‐[0m
[38;2;248;248;242m              ing throughout:[0m

[38;2;248;248;242m                     modify-current-argument '${(qq)${(Q)ARG}}'[0m

[38;2;248;248;242m              The following performs directory expansion on the  command  line  argument[0m
[38;2;248;248;242m              and replaces it by the absolute path:[0m

[38;2;248;248;242m                     expand-dir() {[0m
[38;2;248;248;242m                       REPLY=${~1}[0m
[38;2;248;248;242m                       REPLY=${REPLY:a}[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     modify-current-argument expand-dir[0m

[38;2;248;248;242m              In  practice  the function expand-dir would probably not be defined within[0m
[38;2;248;248;242m              the widget where modify-current-argument is called.[0m

[38;2;248;248;242m   Styles[0m
[38;2;248;248;242m       The behavior of several of the above widgets can be controlled by the use of  the[0m
[38;2;248;248;242m       zstyle  mechanism.  In particular, widgets that interact with the completion sys‐[0m
[38;2;248;248;242m       tem pass along their context to any completions that they invoke.[0m

[38;2;248;248;242m       break-keys[0m
[38;2;248;248;242m              This style is used by  the  incremental-complete-word  widget.  Its  value[0m
[38;2;248;248;242m              should  be  a  pattern,  and all keys matching this pattern will cause the[0m
[38;2;248;248;242m              widget to stop incremental completion without the key having  any  further[0m
[38;2;248;248;242m              effect.  Like  all styles used directly by incremental-complete-word, this[0m
[38;2;248;248;242m              style is looked up using the context `:incremental'.[0m

[38;2;248;248;242m       completer[0m
[38;2;248;248;242m              The incremental-complete-word and insert-and-predict widgets set up  their[0m
[38;2;248;248;242m              top-level  context name before calling completion.  This allows one to de‐[0m
[38;2;248;248;242m              fine different sets of completer functions for normal completion  and  for[0m
[38;2;248;248;242m              these  widgets.  For example, to use completion, approximation and correc‐[0m
[38;2;248;248;242m              tion for normal completion, completion and correction for incremental com‐[0m
[38;2;248;248;242m              pletion and only completion for prediction one could use:[0m

[38;2;248;248;242m                     zstyle ':completion:*' completer \[0m
[38;2;248;248;242m                             _complete _correct _approximate[0m
[38;2;248;248;242m                     zstyle ':completion:incremental:*' completer \[0m
[38;2;248;248;242m                             _complete _correct[0m
[38;2;248;248;242m                     zstyle ':completion:predict:*' completer \[0m
[38;2;248;248;242m                             _complete[0m

[38;2;248;248;242m              It is a good idea to restrict the completers used in  prediction,  because[0m
[38;2;248;248;242m              they  may  be automatically invoked as you type.  The _list and _menu com‐[0m
[38;2;248;248;242m              pleters should never be used with prediction.  The _approximate, _correct,[0m
[38;2;248;248;242m              _expand, and _match completers may be used, but be  aware  that  they  may[0m
[38;2;248;248;242m              change  characters  anywhere in the word behind the cursor, so you need to[0m
[38;2;248;248;242m              watch carefully that the result is what you intended.[0m

[38;2;248;248;242m       cursor The insert-and-predict widget uses this style, in the context  `:predict',[0m
[38;2;248;248;242m              to decide where to place the cursor after completion has been tried.  Val‐[0m
[38;2;248;248;242m              ues are:[0m

[38;2;248;248;242m              complete[0m
[38;2;248;248;242m                     The  cursor is left where it was when completion finished, but only[0m
[38;2;248;248;242m                     if it is after a character equal to the one just  inserted  by  the[0m
[38;2;248;248;242m                     user.   If it is after another character, this value is the same as[0m
[38;2;248;248;242m                     `key'.[0m

[38;2;248;248;242m              key    The cursor is left after the nth occurrence of the  character  just[0m
[38;2;248;248;242m                     inserted, where n is the number of times that character appeared in[0m
[38;2;248;248;242m                     the  word  before completion was attempted.  In short, this has the[0m
[38;2;248;248;242m                     effect of leaving the cursor after the character just typed even if[0m
[38;2;248;248;242m                     the completion code found out that no other characters need  to  be[0m
[38;2;248;248;242m                     inserted at that position.[0m

[38;2;248;248;242m              Any  other  value  for this style unconditionally leaves the cursor at the[0m
[38;2;248;248;242m              position where the completion code left it.[0m

[38;2;248;248;242m       list   When using the incremental-complete-word widget, this style  says  if  the[0m
[38;2;248;248;242m              matches  should  be listed on every key press (if they fit on the screen).[0m
[38;2;248;248;242m              Use the context prefix `:completion:incremental'.[0m

[38;2;248;248;242m              The insert-and-predict widget uses this style to decide if the  completion[0m
[38;2;248;248;242m              should  be  shown  even if there is only one possible completion.  This is[0m
[38;2;248;248;242m              done if the value of this style is the string always.  In  this  case  the[0m
[38;2;248;248;242m              context is `:predict' (not `:completion:predict').[0m

[38;2;248;248;242m       match  This  style  is used by smart-insert-last-word to provide a pattern (using[0m
[38;2;248;248;242m              full EXTENDED_GLOB syntax) that matches an interesting word.  The  context[0m
[38;2;248;248;242m              is  the  name  of the widget to which smart-insert-last-word is bound (see[0m
[38;2;248;248;242m              above).  The default behavior of smart-insert-last-word is equivalent to:[0m

[38;2;248;248;242m                     zstyle :insert-last-word match '*[[:alpha:]/\\]*'[0m

[38;2;248;248;242m              However, you might want to include words that contain spaces:[0m

[38;2;248;248;242m                     zstyle :insert-last-word match '*[[:alpha:][:space:]/\\]*'[0m

[38;2;248;248;242m              Or include numbers as long as the word is at least two characters long:[0m

[38;2;248;248;242m                     zstyle :insert-last-word match '*([[:digit:]]?|[[:alpha:]/\\])*'[0m

[38;2;248;248;242m              The above example causes redirections like "2>" to be included.[0m

[38;2;248;248;242m       prompt The incremental-complete-word widget shows the value of this style in  the[0m
[38;2;248;248;242m              status  line  during incremental completion.  The string value may contain[0m
[38;2;248;248;242m              any of the following substrings in the manner of the PS1 and other  prompt[0m
[38;2;248;248;242m              parameters:[0m

[38;2;248;248;242m              %c     Replaced  by  the name of the completer function that generated the[0m
[38;2;248;248;242m                     matches (without the leading underscore).[0m

[38;2;248;248;242m              %l     When the list style is set,  replaced  by  `...'  if  the  list  of[0m
[38;2;248;248;242m                     matches  is  too long to fit on the screen and with an empty string[0m
[38;2;248;248;242m                     otherwise.  If the list style is `false' or not set, `%l' is always[0m
[38;2;248;248;242m                     removed.[0m

[38;2;248;248;242m              %n     Replaced by the number of matches generated.[0m

[38;2;248;248;242m              %s     Replaced by `-no match-', `-no prefix-',  or  an  empty  string  if[0m
[38;2;248;248;242m                     there  is  no  completion  matching  the  word  on the line, if the[0m
[38;2;248;248;242m                     matches have no common prefix different from the word on the  line,[0m
[38;2;248;248;242m                     or if there is such a common prefix, respectively.[0m

[38;2;248;248;242m              %u     Replaced  by  the unambiguous part of all matches, if there is any,[0m
[38;2;248;248;242m                     and if it is different from the word on the line.[0m

[38;2;248;248;242m              Like `break-keys', this uses the `:incremental' context.[0m

[38;2;248;248;242m       stop-keys[0m
[38;2;248;248;242m              This style is used by the incremental-complete-word widget.  Its value  is[0m
[38;2;248;248;242m              treated  similarly  to the one for the break-keys style (and uses the same[0m
[38;2;248;248;242m              context: `:incremental').  However, in this case  all  keys  matching  the[0m
[38;2;248;248;242m              pattern  given as its value will stop incremental completion and will then[0m
[38;2;248;248;242m              execute their usual function.[0m

[38;2;248;248;242m       toggle This boolean style is used by predict-on and its related  widgets  in  the[0m
[38;2;248;248;242m              context  `:predict'.  If set to one of the standard `true' values, predic‐[0m
[38;2;248;248;242m              tive typing is automatically toggled off in situations  where  it  is  un‐[0m
[38;2;248;248;242m              likely  to  be  useful,  such as when editing a multi-line buffer or after[0m
[38;2;248;248;242m              moving into the middle of a line and then deleting a character.   The  de‐[0m
[38;2;248;248;242m              fault  is  to  leave  prediction  turned on until an explicit call to pre‐[0m
[38;2;248;248;242m              dict-off.[0m

[38;2;248;248;242m       verbose[0m
[38;2;248;248;242m              This boolean style is used by predict-on and its related  widgets  in  the[0m
[38;2;248;248;242m              context  `:predict'.   If  set to one of the standard `true' values, these[0m
[38;2;248;248;242m              widgets display a message below the prompt when the  predictive  state  is[0m
[38;2;248;248;242m              toggled.   This  is most useful in combination with the toggle style.  The[0m
[38;2;248;248;242m              default does not display these messages.[0m

[38;2;248;248;242m       widget This style is similar to the command style: For widget functions that  use[0m
[38;2;248;248;242m              zle  to  call  other widgets, this style can sometimes be used to override[0m
[38;2;248;248;242m              the widget which is called.  The context for this style is the name of the[0m
[38;2;248;248;242m              calling widget (not the name of the calling function, because one function[0m
[38;2;248;248;242m              may be bound to multiple widget names).[0m

[38;2;248;248;242m                     zstyle :copy-earlier-word widget smart-insert-last-word[0m

[38;2;248;248;242m              Check the documentation for the calling widget or  function  to  determine[0m
[38;2;248;248;242m              whether the widget style is used.[0m

[38;2;248;248;242mEXCEPTION HANDLING[0m
[38;2;248;248;242m       Two  functions are provided to enable zsh to provide exception handling in a form[0m
[38;2;248;248;242m       that should be familiar from other languages.[0m

[38;2;248;248;242m       throw exception[0m
[38;2;248;248;242m              The function throw throws the named exception.  The name is  an  arbitrary[0m
[38;2;248;248;242m              string and is only used by the throw and catch functions.  An exception is[0m
[38;2;248;248;242m              for the most part treated the same as a shell error, i.e. an unhandled ex‐[0m
[38;2;248;248;242m              ception  will  cause  the  shell  to abort all processing in a function or[0m
[38;2;248;248;242m              script and to return to the top level in an interactive shell.[0m

[38;2;248;248;242m       catch exception-pattern[0m
[38;2;248;248;242m              The function catch returns status zero if an exception was thrown and  the[0m
[38;2;248;248;242m              pattern  exception-pattern  matches its name.  Otherwise it returns status[0m
[38;2;248;248;242m              1.  exception-pattern is a standard shell pattern, respecting the  current[0m
[38;2;248;248;242m              setting  of  the  EXTENDED_GLOB option.  An alias catch is also defined to[0m
[38;2;248;248;242m              prevent the argument to the function from matching filenames, so  patterns[0m
[38;2;248;248;242m              may  be used unquoted.  Note that as exceptions are not fundamentally dif‐[0m
[38;2;248;248;242m              ferent from other shell errors it is possible to catch shell errors by us‐[0m
[38;2;248;248;242m              ing an empty string as the exception name.  The shell variable  CAUGHT  is[0m
[38;2;248;248;242m              set  by  catch  to  the  name  of the exception caught.  It is possible to[0m
[38;2;248;248;242m              rethrow an exception by calling the throw function again once an exception[0m
[38;2;248;248;242m              has been caught.[0m

[38;2;248;248;242m       The functions are designed to be used together  with  the  always  construct  de‐[0m
[38;2;248;248;242m       scribed in zshmisc(1).  This is important as only this construct provides the re‐[0m
[38;2;248;248;242m       quired support for exceptions.  A typical example is as follows.[0m

[38;2;248;248;242m              {[0m
[38;2;248;248;242m                # "try" block[0m
[38;2;248;248;242m                # ... nested code here calls "throw MyExcept"[0m
[38;2;248;248;242m              } always {[0m
[38;2;248;248;242m                # "always" block[0m
[38;2;248;248;242m                if catch MyExcept; then[0m
[38;2;248;248;242m                  print "Caught exception MyExcept"[0m
[38;2;248;248;242m                elif catch ''; then[0m
[38;2;248;248;242m                  print "Caught a shell error.  Propagating..."[0m
[38;2;248;248;242m                  throw ''[0m
[38;2;248;248;242m                fi[0m
[38;2;248;248;242m                # Other exceptions are not handled but may be caught further[0m
[38;2;248;248;242m                # up the call stack.[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       If all exceptions should be caught, the following idiom might be preferable.[0m

[38;2;248;248;242m              {[0m
[38;2;248;248;242m                # ... nested code here throws an exception[0m
[38;2;248;248;242m              } always {[0m
[38;2;248;248;242m                if catch *; then[0m
[38;2;248;248;242m                  case $CAUGHT in[0m
[38;2;248;248;242m                    (MyExcept)[0m
[38;2;248;248;242m                    print "Caught my own exception"[0m
[38;2;248;248;242m                    ;;[0m
[38;2;248;248;242m                    (*)[0m
[38;2;248;248;242m                    print "Caught some other exception"[0m
[38;2;248;248;242m                    ;;[0m
[38;2;248;248;242m                  esac[0m
[38;2;248;248;242m                fi[0m
[38;2;248;248;242m              }[0m

[38;2;248;248;242m       In common with exception handling in other languages, the exception may be thrown[0m
[38;2;248;248;242m       by  code  deeply  nested  inside  the `try' block.  However, note that it must be[0m
[38;2;248;248;242m       thrown inside the current shell, not in a subshell forked for a pipeline,  paren‐[0m
[38;2;248;248;242m       thesised  current-shell  construct,  or some form of command or process substitu‐[0m
[38;2;248;248;242m       tion.[0m

[38;2;248;248;242m       The system internally uses the shell variable EXCEPTION to record the name of the[0m
[38;2;248;248;242m       exception between throwing and catching.  One drawback of this scheme is that  if[0m
[38;2;248;248;242m       the exception is not handled the variable EXCEPTION remains set and may be incor‐[0m
[38;2;248;248;242m       rectly  recognised  as the name of an exception if a shell error subsequently oc‐[0m
[38;2;248;248;242m       curs.  Adding unset EXCEPTION at the start of the outermost  layer  of  any  code[0m
[38;2;248;248;242m       that uses exception handling will eliminate this problem.[0m

[38;2;248;248;242mMIME FUNCTIONS[0m
[38;2;248;248;242m       Three  functions  are available to provide handling of files recognised by exten‐[0m
[38;2;248;248;242m       sion, for example to dispatch a file text.ps when executed as a command to an ap‐[0m
[38;2;248;248;242m       propriate viewer.[0m

[38;2;248;248;242m       zsh-mime-setup [ -fv ] [ -l [ suffix ... ] ][0m
[38;2;248;248;242m       zsh-mime-handler [ -l ] command argument ...[0m
[38;2;248;248;242m              These two functions use the files ~/.mime.types and /etc/mime.types, which[0m
[38;2;248;248;242m              associate types and extensions, as well  as  ~/.mailcap  and  /etc/mailcap[0m
[38;2;248;248;242m              files, which associate types and the programs that handle them.  These are[0m
[38;2;248;248;242m              provided on many systems with the Multimedia Internet Mail Extensions.[0m

[38;2;248;248;242m              To enable the system, the function zsh-mime-setup should be autoloaded and[0m
[38;2;248;248;242m              run.   This allows files with extensions to be treated as executable; such[0m
[38;2;248;248;242m              files be completed  by  the  function  completion  system.   The  function[0m
[38;2;248;248;242m              zsh-mime-handler should not need to be called by the user.[0m

[38;2;248;248;242m              The  system  works  by  setting up suffix aliases with `alias -s'.  Suffix[0m
[38;2;248;248;242m              aliases already installed by the user will not be overwritten.[0m

[38;2;248;248;242m              For suffixes defined in lower case, upper case variants will also automat‐[0m
[38;2;248;248;242m              ically be handled (e.g. PDF is automatically handled if handling  for  the[0m
[38;2;248;248;242m              suffix pdf is defined), but not vice versa.[0m

[38;2;248;248;242m              Repeated  calls to zsh-mime-setup do not override the existing mapping be‐[0m
[38;2;248;248;242m              tween suffixes and executable files unless the option -f is given.   Note,[0m
[38;2;248;248;242m              however,  that  this does not override existing suffix aliases assigned to[0m
[38;2;248;248;242m              handlers other than zsh-mime-handler.[0m

[38;2;248;248;242m              Calling zsh-mime-setup with the option  -l  lists  the  existing  mappings[0m
[38;2;248;248;242m              without  altering them.  Suffixes to list (which may contain pattern char‐[0m
[38;2;248;248;242m              acters that should be quoted from immediate interpretation on the  command[0m
[38;2;248;248;242m              line)  may  be  given  as additional arguments, otherwise all suffixes are[0m
[38;2;248;248;242m              listed.[0m

[38;2;248;248;242m              Calling zsh-mime-setup with the option -v  causes  verbose  output  to  be[0m
[38;2;248;248;242m              shown during the setup operation.[0m

[38;2;248;248;242m              The system respects the mailcap flags needsterminal and copiousoutput; see[0m
[38;2;248;248;242m              mailcap(4) or mailcap(5) (the man page's name varies across platforms).[0m

[38;2;248;248;242m              The  functions use the following styles, which are defined with the zstyle[0m
[38;2;248;248;242m              builtin command  (see  zshmodules(1)).   They  should  be  defined  before[0m
[38;2;248;248;242m              zsh-mime-setup  is run.  The contexts used all start with :mime:, with ad‐[0m
[38;2;248;248;242m              ditional components in some cases.  It is recommended that  a  trailing  *[0m
[38;2;248;248;242m              (suitably  quoted) be appended to style patterns in case the system is ex‐[0m
[38;2;248;248;242m              tended in future.  Some examples are given below.[0m

[38;2;248;248;242m              For files that have multiple suffixes, e.g. .pdf.gz, where the context in‐[0m
[38;2;248;248;242m              cludes the suffix it will be looked up starting with the longest  possible[0m
[38;2;248;248;242m              suffix until a match for the style is found.  For example, if .pdf.gz pro‐[0m
[38;2;248;248;242m              duces  a  match  for the handler, that will be used; otherwise the handler[0m
[38;2;248;248;242m              for .gz will be used.  Note that, owing to the way suffix aliases work, it[0m
[38;2;248;248;242m              is always required that there be a handler for the shortest possible  suf‐[0m
[38;2;248;248;242m              fix, so in this example .pdf.gz can only be handled if .gz is also handled[0m
[38;2;248;248;242m              (though  not  necessarily in the same way).  Alternatively, if no handling[0m
[38;2;248;248;242m              for .gz on its own is needed, simply adding the command[0m

[38;2;248;248;242m                     alias -s gz=zsh-mime-handler[0m

[38;2;248;248;242m              to the initialisation code is sufficient; .gz will not be handled  on  its[0m
[38;2;248;248;242m              own, but may be in combination with other suffixes.[0m

[38;2;248;248;242m              current-shell[0m
[38;2;248;248;242m                     If  this boolean style is true, the mailcap handler for the context[0m
[38;2;248;248;242m                     in question is run using the eval builtin instead of by starting  a[0m
[38;2;248;248;242m                     new  sh  process.   This is more efficient, but may not work in the[0m
[38;2;248;248;242m                     occasional cases where the mailcap handler uses strict  POSIX  syn‐[0m
[38;2;248;248;242m                     tax.[0m

[38;2;248;248;242m              disown If  this  boolean  style  is  true, mailcap handlers started in the[0m
[38;2;248;248;242m                     background will be disowned, i.e. not subject to job control within[0m
[38;2;248;248;242m                     the parent shell.  Such handlers nearly always  produce  their  own[0m
[38;2;248;248;242m                     windows,  so  the  only  likely  harmful side effect of setting the[0m
[38;2;248;248;242m                     style is that it becomes harder to kill jobs from within the shell.[0m

[38;2;248;248;242m              execute-as-is[0m
[38;2;248;248;242m                     This style gives a list of patterns to  be  matched  against  files[0m
[38;2;248;248;242m                     passed  for  execution with a handler program.  If the file matches[0m
[38;2;248;248;242m                     the pattern, the entire command line is  executed  in  its  current[0m
[38;2;248;248;242m                     form,  with  no handler.  This is useful for files which might have[0m
[38;2;248;248;242m                     suffixes but nonetheless be executable in their own right.  If  the[0m
[38;2;248;248;242m                     style  is  not set, the pattern *(*) *(/) is used; hence executable[0m
[38;2;248;248;242m                     files are executed directly and not passed to a  handler,  and  the[0m
[38;2;248;248;242m                     option  AUTO_CD may be used to change to directories that happen to[0m
[38;2;248;248;242m                     have MIME suffixes.[0m

[38;2;248;248;242m              execute-never[0m
[38;2;248;248;242m                     This style is useful in combination with execute-as-is.  It is  set[0m
[38;2;248;248;242m                     to  an  array of patterns corresponding to full paths to files that[0m
[38;2;248;248;242m                     should never be treated as executable, even if the file  passed  to[0m
[38;2;248;248;242m                     the  MIME  handler  matches execute-as-is.  This is useful for file[0m
[38;2;248;248;242m                     systems that don't handle execute permission or that  contain  exe‐[0m
[38;2;248;248;242m                     cutables  from another operating system.  For example, if /mnt/win‐[0m
[38;2;248;248;242m                     dows is a Windows mount, then[0m

[38;2;248;248;242m                            zstyle ':mime:*' execute-never '/mnt/windows/*'[0m

[38;2;248;248;242m                     will ensure that any files found in that area will be  executed  as[0m
[38;2;248;248;242m                     MIME types even if they are executable.  As this example shows, the[0m
[38;2;248;248;242m                     complete  file  name  is matched against the pattern, regardless of[0m
[38;2;248;248;242m                     how the file was passed to the handler.  The file is resolved to  a[0m
[38;2;248;248;242m                     full  path using the :P modifier described in the subsection `Modi‐[0m
[38;2;248;248;242m                     fiers' in zshexpn(1); this means that symbolic links  are  resolved[0m
[38;2;248;248;242m                     where possible, so that links into other file systems behave in the[0m
[38;2;248;248;242m                     correct fashion.[0m

[38;2;248;248;242m              file-path[0m
[38;2;248;248;242m                     Used  if  the style find-file-in-path is true for the same context.[0m
[38;2;248;248;242m                     Set to an array of directories that are used for searching for  the[0m
[38;2;248;248;242m                     file  to  be  handled; the default is the command path given by the[0m
[38;2;248;248;242m                     special parameter path.  The shell option PATH_DIRS  is  respected;[0m
[38;2;248;248;242m                     if  that  is set, the appropriate path will be searched even if the[0m
[38;2;248;248;242m                     name of the file to be handled as it appears on  the  command  line[0m
[38;2;248;248;242m                     contains  a  `/'.  The full context is :mime:.suffix:, as described[0m
[38;2;248;248;242m                     for the style handler.[0m

[38;2;248;248;242m              find-file-in-path[0m
[38;2;248;248;242m                     If set, allows files whose names do not contain absolute  paths  to[0m
[38;2;248;248;242m                     be  searched  for  in the command path or the path specified by the[0m
[38;2;248;248;242m                     file-path style.  If the file is not  found  in  the  path,  it  is[0m
[38;2;248;248;242m                     looked  for locally (whether or not the current directory is in the[0m
[38;2;248;248;242m                     path); if it is not found locally, the handler  will  abort  unless[0m
[38;2;248;248;242m                     the  handle-nonexistent  style is set.  Files found in the path are[0m
[38;2;248;248;242m                     tested as described for the style execute-as-is.  The full  context[0m
[38;2;248;248;242m                     is :mime:.suffix:, as described for the style handler.[0m

[38;2;248;248;242m              flags  Defines  flags to go with a handler; the context is as for the han‐[0m
[38;2;248;248;242m                     dler style, and the format is as for the flags in mailcap.[0m

[38;2;248;248;242m              handle-nonexistent[0m
[38;2;248;248;242m                     By default, arguments that don't correspond to files are not passed[0m
[38;2;248;248;242m                     to the MIME handler in order to prevent it from  intercepting  com‐[0m
[38;2;248;248;242m                     mands  found  in the path that happen to have suffixes.  This style[0m
[38;2;248;248;242m                     may be set to an array of extended glob patterns for arguments that[0m
[38;2;248;248;242m                     will be passed to the handler even if they don't exist.  If  it  is[0m
[38;2;248;248;242m                     not explicitly set it defaults to [[:alpha:]]#:/* which allows URLs[0m
[38;2;248;248;242m                     to  be  passed  to the MIME handler even though they don't exist in[0m
[38;2;248;248;242m                     that format in the file system.  The full  context  is  :mime:.suf‐[0m
[38;2;248;248;242m                     fix:, as described for the style handler.[0m

[38;2;248;248;242m              handler[0m
[38;2;248;248;242m                     Specifies  a  handler for a suffix; the suffix is given by the con‐[0m
[38;2;248;248;242m                     text as :mime:.suffix:, and the format of the  handler  is  exactly[0m
[38;2;248;248;242m                     that  in mailcap.  Note in particular the `.' and trailing colon to[0m
[38;2;248;248;242m                     distinguish this use of the context.  This  overrides  any  handler[0m
[38;2;248;248;242m                     specified  by  the mailcap files.  If the handler requires a termi‐[0m
[38;2;248;248;242m                     nal, the flags style should be set to include the word  needstermi‐[0m
[38;2;248;248;242m                     nal,  or  if the output is to be displayed through a pager (but not[0m
[38;2;248;248;242m                     if the handler is itself a pager), it should include copiousoutput.[0m

[38;2;248;248;242m              mailcap[0m
[38;2;248;248;242m                     A list of files in the format of ~/.mailcap and /etc/mailcap to  be[0m
[38;2;248;248;242m                     read  during  setup,  replacing  the default list which consists of[0m
[38;2;248;248;242m                     those two files.  The context is :mime:.  A + in the list  will  be[0m
[38;2;248;248;242m                     replaced by the default files.[0m

[38;2;248;248;242m              mailcap-priorities[0m
[38;2;248;248;242m                     This style is used to resolve multiple mailcap entries for the same[0m
[38;2;248;248;242m                     MIME  type.   It consists of an array of the following elements, in[0m
[38;2;248;248;242m                     descending order of priority; later entries will be used if earlier[0m
[38;2;248;248;242m                     entries are unable to resolve the entries being compared.  If  none[0m
[38;2;248;248;242m                     of  the  tests  resolve the entries, the first entry encountered is[0m
[38;2;248;248;242m                     retained.[0m

[38;2;248;248;242m                     files  The order of files (entries  in  the  mailcap  style)  read.[0m
[38;2;248;248;242m                            Earlier  files  are  preferred.  (Note this does not resolve[0m
[38;2;248;248;242m                            entries in the same file.)[0m

[38;2;248;248;242m                     priority[0m
[38;2;248;248;242m                            The priority flag from the mailcap entry.  The  priority  is[0m
[38;2;248;248;242m                            an integer from 0 to 9 with the default value being 5.[0m

[38;2;248;248;242m                     flags  The  test  given by the mailcap-prio-flags option is used to[0m
[38;2;248;248;242m                            resolve entries.[0m

[38;2;248;248;242m                     place  Later entries are preferred; as the entries are strictly or‐[0m
[38;2;248;248;242m                            dered, this test always succeeds.[0m

[38;2;248;248;242m                     Note that as this style is handled during initialisation, the  con‐[0m
[38;2;248;248;242m                     text is always :mime:, with no discrimination by suffix.[0m

[38;2;248;248;242m              mailcap-prio-flags[0m
[38;2;248;248;242m                     This  style  is  used  when the keyword flags is encountered in the[0m
[38;2;248;248;242m                     list of tests specified by the mailcap-priorities style.  It should[0m
[38;2;248;248;242m                     be set to a list of patterns, each of which is tested  against  the[0m
[38;2;248;248;242m                     flags  specified  in the mailcap entry (in other words, the sets of[0m
[38;2;248;248;242m                     assignments found with some entries in the mailcap file).   Earlier[0m
[38;2;248;248;242m                     patterns  in the list are preferred to later ones, and matched pat‐[0m
[38;2;248;248;242m                     terns are preferred to unmatched ones.[0m

[38;2;248;248;242m              mime-types[0m
[38;2;248;248;242m                     A list of files in the format of ~/.mime.types and  /etc/mime.types[0m
[38;2;248;248;242m                     to  be read during setup, replacing the default list which consists[0m
[38;2;248;248;242m                     of those two files.  The context is :mime:.  A + in the  list  will[0m
[38;2;248;248;242m                     be replaced by the default files.[0m

[38;2;248;248;242m              never-background[0m
[38;2;248;248;242m                     If  this boolean style is set, the handler for the given context is[0m
[38;2;248;248;242m                     always run in the foreground, even if the  flags  provided  in  the[0m
[38;2;248;248;242m                     mailcap  entry  suggest it need not be (for example, it doesn't re‐[0m
[38;2;248;248;242m                     quire a terminal).[0m

[38;2;248;248;242m              pager  If set, will be used instead of $PAGER or more to  handle  suffixes[0m
[38;2;248;248;242m                     where  the  copiousoutput  flag is set.  The context is as for han‐[0m
[38;2;248;248;242m                     dler, i.e. :mime:.suffix: for handling a file with the  given  suf‐[0m
[38;2;248;248;242m                     fix.[0m

[38;2;248;248;242m              Examples:[0m

[38;2;248;248;242m                     zstyle ':mime:*' mailcap ~/.mailcap /usr/local/etc/mailcap[0m
[38;2;248;248;242m                     zstyle ':mime:.txt:' handler less %s[0m
[38;2;248;248;242m                     zstyle ':mime:.txt:' flags needsterminal[0m

[38;2;248;248;242m              When  zsh-mime-setup is subsequently run, it will look for mailcap entries[0m
[38;2;248;248;242m              in the two files given.  Files of suffix .txt will be handled  by  running[0m
[38;2;248;248;242m              `less  file.txt'.  The flag needsterminal is set to show that this program[0m
[38;2;248;248;242m              must run attached to a terminal.[0m

[38;2;248;248;242m              As there are several steps to dispatching a command, the following  should[0m
[38;2;248;248;242m              be checked if attempting to execute a file by extension .ext does not have[0m
[38;2;248;248;242m              the expected effect.[0m

[38;2;248;248;242m              The command `alias -s ext' should show `ps=zsh-mime-handler'.  If it shows[0m
[38;2;248;248;242m              something  else,  another  suffix  alias was already installed and was not[0m
[38;2;248;248;242m              overwritten.  If it shows nothing, no handler was installed:  this is most[0m
[38;2;248;248;242m              likely because no handler was found in the .mime.types and mailcap  combi‐[0m
[38;2;248;248;242m              nation for .ext files.  In that case, appropriate handling should be added[0m
[38;2;248;248;242m              to ~/.mime.types and mailcap.[0m

[38;2;248;248;242m              If the extension is handled by zsh-mime-handler but the file is not opened[0m
[38;2;248;248;242m              correctly,  either  the  handler defined for the type is incorrect, or the[0m
[38;2;248;248;242m              flags associated with it are in appropriate.   Running  zsh-mime-setup  -l[0m
[38;2;248;248;242m              will  show the handler and, if there are any, the flags.  A %s in the han‐[0m
[38;2;248;248;242m              dler is replaced by the file (suitably quoted if necessary).   Check  that[0m
[38;2;248;248;242m              the  handler  program  listed lists and can be run in the way shown.  Also[0m
[38;2;248;248;242m              check that the flags needsterminal or copiousoutput are set if the handler[0m
[38;2;248;248;242m              needs to be run under a terminal; the second flag is used  if  the  output[0m
[38;2;248;248;242m              should  be  sent  to  a pager.  An example of a suitable mailcap entry for[0m
[38;2;248;248;242m              such a program is:[0m

[38;2;248;248;242m                     text/html; /usr/bin/lynx '%s'; needsterminal[0m

[38;2;248;248;242m              Running `zsh-mime-handler -l command line' prints the  command  line  that[0m
[38;2;248;248;242m              would  be  executed,  simplified  to  remove  the effect of any flags, and[0m
[38;2;248;248;242m              quoted so that the output can be run as a complete zsh command line.  This[0m
[38;2;248;248;242m              is used by the completion system to decide how to complete  after  a  file[0m
[38;2;248;248;242m              handled by zsh-mime-setup.[0m

[38;2;248;248;242m       pick-web-browser[0m
[38;2;248;248;242m              This  function is separate from the two MIME functions described above and[0m
[38;2;248;248;242m              can be assigned directly to a suffix:[0m

[38;2;248;248;242m                     autoload -U pick-web-browser[0m
[38;2;248;248;242m                     alias -s html=pick-web-browser[0m

[38;2;248;248;242m              It is provided as an intelligent front end to dispatch a web browser.   It[0m
[38;2;248;248;242m              may  be run as either a function or a shell script.  The status 255 is re‐[0m
[38;2;248;248;242m              turned if no browser could be started.[0m

[38;2;248;248;242m              Various styles are available to customize the choice of browsers:[0m

[38;2;248;248;242m              browser-style[0m
[38;2;248;248;242m                     The value of the style is an array giving preferences in decreasing[0m
[38;2;248;248;242m                     order for the type of browser to use.  The values of  elements  may[0m
[38;2;248;248;242m                     be[0m

[38;2;248;248;242m                     running[0m
[38;2;248;248;242m                            Use  a  GUI browser that is already running when an X Window[0m
[38;2;248;248;242m                            display is available.  The browsers listed in the x-browsers[0m
[38;2;248;248;242m                            style are tried in order until one is found; if it  is,  the[0m
[38;2;248;248;242m                            file will be displayed in that browser, so the user may need[0m
[38;2;248;248;242m                            to  check whether it has appeared.  If no running browser is[0m
[38;2;248;248;242m                            found, one is not started.   Browsers  other  than  Firefox,[0m
[38;2;248;248;242m                            Opera  and  Konqueror  are assumed to understand the Mozilla[0m
[38;2;248;248;242m                            syntax for opening a URL remotely.[0m

[38;2;248;248;242m                     x      Start a new GUI browser when an X Window display  is  avail‐[0m
[38;2;248;248;242m                            able.   Search  for  the availability of one of the browsers[0m
[38;2;248;248;242m                            listed in the x-browsers style and start the first one  that[0m
[38;2;248;248;242m                            is found.  No check is made for an already running browser.[0m

[38;2;248;248;242m                     tty    Start a terminal-based browser.  Search for the availability[0m
[38;2;248;248;242m                            of  one of the browsers listed in the tty-browsers style and[0m
[38;2;248;248;242m                            start the first one that is found.[0m

[38;2;248;248;242m                     If the style is not set the default running x tty is used.[0m

[38;2;248;248;242m              x-browsers[0m
[38;2;248;248;242m                     An array in decreasing order of preference of browsers to use  when[0m
[38;2;248;248;242m                     running  under the X Window System.  The array consists of the com‐[0m
[38;2;248;248;242m                     mand name under which to start the browser.  They are looked up  in[0m
[38;2;248;248;242m                     the  context  :mime: (which may be extended in future, so appending[0m
[38;2;248;248;242m                     `*' is recommended).  For example,[0m

[38;2;248;248;242m                            zstyle ':mime:*' x-browsers opera konqueror firefox[0m

[38;2;248;248;242m                     specifies that pick-web-browser should first look for a running in‐[0m
[38;2;248;248;242m                     stance of Opera, Konqueror or Firefox, in that  order,  and  if  it[0m
[38;2;248;248;242m                     fails  to  find  any should attempt to start Opera.  The default is[0m
[38;2;248;248;242m                     firefox mozilla netscape opera konqueror.[0m

[38;2;248;248;242m              tty-browsers[0m
[38;2;248;248;242m                     An array similar to x-browsers, except that it  gives  browsers  to[0m
[38;2;248;248;242m                     use  when  no X Window display is available.  The default is elinks[0m
[38;2;248;248;242m                     links lynx.[0m

[38;2;248;248;242m              command[0m
[38;2;248;248;242m                     If it is set this style is used to pick the command used to open  a[0m
[38;2;248;248;242m                     page  for a browser.  The context is :mime:browser:new:$browser: to[0m
[38;2;248;248;242m                     start a new browser or :mime:browser:running:$browser:  to  open  a[0m
[38;2;248;248;242m                     URL  in  a  browser already running on the current X display, where[0m
[38;2;248;248;242m                     $browser is the value matched in  the  x-browsers  or  tty-browsers[0m
[38;2;248;248;242m                     style.   The  escape  sequence  %b in the style's value will be re‐[0m
[38;2;248;248;242m                     placed by the browser, while %u will be replaced by  the  URL.   If[0m
[38;2;248;248;242m                     the  style is not set, the default for all new instances is equiva‐[0m
[38;2;248;248;242m                     lent to %b %u and the  defaults  for  using  running  browsers  are[0m
[38;2;248;248;242m                     equivalent  to the values kfmclient openURL %u for Konqueror, fire‐[0m
[38;2;248;248;242m                     fox -new-tab %u for Firefox, opera -newpage %u for  Opera,  and  %b[0m
[38;2;248;248;242m                     -remote "openUrl(%u)" for all others.[0m

[38;2;248;248;242mMATHEMATICAL FUNCTIONS[0m
[38;2;248;248;242m       zcalc [ -erf ] [ expression ... ][0m
[38;2;248;248;242m              A  reasonably powerful calculator based on zsh's arithmetic evaluation fa‐[0m
[38;2;248;248;242m              cility.  The syntax is similar to that of  formulae  in  most  programming[0m
[38;2;248;248;242m              languages;  see  the section `Arithmetic Evaluation' in zshmisc(1) for de‐[0m
[38;2;248;248;242m              tails.[0m

[38;2;248;248;242m              Non-programmers should note that, as in many other programming  languages,[0m
[38;2;248;248;242m              expressions  involving  only  integers  (whether  constants without a `.',[0m
[38;2;248;248;242m              variables containing such constants as strings, or variables  declared  to[0m
[38;2;248;248;242m              be  integers)  are by default evaluated using integer arithmetic, which is[0m
[38;2;248;248;242m              not how an ordinary desk calculator operates.  To force floating point op‐[0m
[38;2;248;248;242m              eration, pass the option -f; see further notes below.[0m

[38;2;248;248;242m              If the file ~/.zcalcrc exists it will be sourced inside the function  once[0m
[38;2;248;248;242m              it is set up and about to process the command line.  This can be used, for[0m
[38;2;248;248;242m              example,  to set shell options; emulate -L zsh and setopt extendedglob are[0m
[38;2;248;248;242m              in effect at this point.  Any failure to source the file if it  exists  is[0m
[38;2;248;248;242m              treated  as  fatal.   As  with  other  initialisation files, the directory[0m
[38;2;248;248;242m              $ZDOTDIR is used instead of $HOME if it is set.[0m

[38;2;248;248;242m              The mathematical library zsh/mathfunc will be loaded if it  is  available;[0m
[38;2;248;248;242m              see the section `The zsh/mathfunc Module' in zshmodules(1).  The mathemat‐[0m
[38;2;248;248;242m              ical  functions  correspond  to the raw system libraries, so trigonometric[0m
[38;2;248;248;242m              functions are evaluated using radians, and so on.[0m

[38;2;248;248;242m              Each line typed is evaluated as an expression.  The prompt shows a number,[0m
[38;2;248;248;242m              which corresponds to a positional parameter where the result of that  cal‐[0m
[38;2;248;248;242m              culation  is  stored.   For  example, the result of the calculation on the[0m
[38;2;248;248;242m              line preceded by `4> ' is available as $4.  The last value  calculated  is[0m
[38;2;248;248;242m              available  as  ans.   Full  command line editing, including the history of[0m
[38;2;248;248;242m              previous calculations, is available; the history  is  saved  in  the  file[0m
[38;2;248;248;242m              ~/.zcalc_history.   To  exit,  enter  a blank line or type `:q' on its own[0m
[38;2;248;248;242m              (`q' is allowed for historical compatibility).[0m

[38;2;248;248;242m              A line ending with a single backslash is treated in the same fashion as it[0m
[38;2;248;248;242m              is in command line  editing:   the  backslash  is  removed,  the  function[0m
[38;2;248;248;242m              prompts for more input (the prompt is preceded by `...' to indicate this),[0m
[38;2;248;248;242m              and the lines are combined into one to get the final result.  In addition,[0m
[38;2;248;248;242m              if  the  input so far contains more open than close parentheses zcalc will[0m
[38;2;248;248;242m              prompt for more input.[0m

[38;2;248;248;242m              If arguments are given to zcalc on start up, they are used  to  prime  the[0m
[38;2;248;248;242m              first  few  positional  parameters.   A visual indication of this is given[0m
[38;2;248;248;242m              when the calculator starts.[0m

[38;2;248;248;242m              The constants PI (3.14159...) and E (2.71828...) are provided.   Parameter[0m
[38;2;248;248;242m              assignment  is possible, but note that all parameters will be put into the[0m
[38;2;248;248;242m              global namespace unless the :local special command is used.  The  function[0m
[38;2;248;248;242m              creates  local  variables  whose names start with _, so users should avoid[0m
[38;2;248;248;242m              doing so.  The variables ans (the last answer) and stack (the stack in RPN[0m
[38;2;248;248;242m              mode) may be referred to directly; stack is an array but  elements  of  it[0m
[38;2;248;248;242m              are  numeric.  Various other special variables are used locally with their[0m
[38;2;248;248;242m              standard meaning, for example compcontext, match, mbegin, mend, psvar.[0m

[38;2;248;248;242m              The output base can be initialised by passing the option `-#base', for ex‐[0m
[38;2;248;248;242m              ample `zcalc -#16' (the `#' may have to be quoted, depending on the  glob‐[0m
[38;2;248;248;242m              bing options set).[0m

[38;2;248;248;242m              If  the option `-e' is set, the function runs non-interactively: the argu‐[0m
[38;2;248;248;242m              ments are treated as expressions to be evaluated as  if  entered  interac‐[0m
[38;2;248;248;242m              tively line by line.[0m

[38;2;248;248;242m              If  the  option  `-f'  is  set, all numbers are treated as floating point,[0m
[38;2;248;248;242m              hence for example the expression `3/4' evaluates to 0.75  rather  than  0.[0m
[38;2;248;248;242m              Options must appear in separate words.[0m

[38;2;248;248;242m              If  the option `-r' is set, RPN (Reverse Polish Notation) mode is entered.[0m
[38;2;248;248;242m              This has various additional properties:[0m
[38;2;248;248;242m              Stack  Evaluated values are maintained in a stack; this is contained in an[0m
[38;2;248;248;242m                     array named stack with the most recent value in ${stack[1]}.[0m

[38;2;248;248;242m              Operators and functions[0m
[38;2;248;248;242m                     If the line entered matches an operator (+, -, *, /, **, ^, | or &)[0m
[38;2;248;248;242m                     or a function supplied by the zsh/mathfunc library, the bottom ele‐[0m
[38;2;248;248;242m                     ment or elements of the stack are popped to use as the argument  or[0m
[38;2;248;248;242m                     arguments.  The higher elements of stack (least recent) are used as[0m
[38;2;248;248;242m                     earlier arguments.  The result is then pushed into ${stack[1]}.[0m

[38;2;248;248;242m              Expressions[0m
[38;2;248;248;242m                     Other expressions are evaluated normally, printed, and added to the[0m
[38;2;248;248;242m                     stack as numeric values.  The syntax within expressions on a single[0m
[38;2;248;248;242m                     line is normal shell arithmetic (not RPN).[0m

[38;2;248;248;242m              Stack listing[0m
[38;2;248;248;242m                     If  an  integer  follows the option -r with no space, then on every[0m
[38;2;248;248;242m                     evaluation that many elements of the stack,  where  available,  are[0m
[38;2;248;248;242m                     printed  instead  of just the most recent result.  Hence, for exam‐[0m
[38;2;248;248;242m                     ple, zcalc -r4 shows $stack[4] to $stack[1] each time  results  are[0m
[38;2;248;248;242m                     printed.[0m

[38;2;248;248;242m              Duplication: =[0m
[38;2;248;248;242m                     The  pseudo-operator  = causes the most recent element of the stack[0m
[38;2;248;248;242m                     to be duplicated onto the stack.[0m

[38;2;248;248;242m              pop    The pseudo-function pop causes the most recent element of the stack[0m
[38;2;248;248;242m                     to be popped.  A `>' on its own has the same effect.[0m

[38;2;248;248;242m              >ident The expression > followed (with no space)  by  a  shell  identifier[0m
[38;2;248;248;242m                     causes  the  most  recent element of the stack to be popped and as‐[0m
[38;2;248;248;242m                     signed to the variable with that name.  The variable  is  local  to[0m
[38;2;248;248;242m                     the zcalc function.[0m

[38;2;248;248;242m              <ident The  expression  <  followed  (with no space) by a shell identifier[0m
[38;2;248;248;242m                     causes the value of the variable with that name to be  pushed  onto[0m
[38;2;248;248;242m                     the stack.  ident may be an integer, in which case the previous re‐[0m
[38;2;248;248;242m                     sult  with that number (as shown before the > in the standard zcalc[0m
[38;2;248;248;242m                     prompt) is put on the stack.[0m

[38;2;248;248;242m              Exchange: xy[0m
[38;2;248;248;242m                     The pseudo-function xy causes the most recent two elements  of  the[0m
[38;2;248;248;242m                     stack to be exchanged.  `<>' has the same effect.[0m

[38;2;248;248;242m              The  prompt is configurable via the parameter ZCALCPROMPT, which undergoes[0m
[38;2;248;248;242m              standard prompt expansion.  The index of the current entry is  stored  lo‐[0m
[38;2;248;248;242m              cally in the first element of the array psvar, which can be referred to in[0m
[38;2;248;248;242m              ZCALCPROMPT as `%1v'.  The default prompt is `%1v> '.[0m

[38;2;248;248;242m              The  variable ZCALC_ACTIVE is set within the function and can be tested by[0m
[38;2;248;248;242m              nested functions; it has the value rpn if RPN mode is active, else 1.[0m

[38;2;248;248;242m              A few special commands are available; these are  introduced  by  a  colon.[0m
[38;2;248;248;242m              For backward compatibility, the colon may be omitted for certain commands.[0m
[38;2;248;248;242m              Completion is available if compinit has been run.[0m

[38;2;248;248;242m              The output precision may be specified within zcalc by special commands fa‐[0m
[38;2;248;248;242m              miliar from many calculators.[0m
[38;2;248;248;242m              :norm  The  default output format.  It corresponds to the printf %g speci‐[0m
[38;2;248;248;242m                     fication.  Typically this shows six decimal digits.[0m

[38;2;248;248;242m              :sci digits[0m
[38;2;248;248;242m                     Scientific notation, corresponding to the printf %g  output  format[0m
[38;2;248;248;242m                     with  the  precision  given  by digits.  This produces either fixed[0m
[38;2;248;248;242m                     point or exponential notation depending on the value output.[0m

[38;2;248;248;242m              :fix digits[0m
[38;2;248;248;242m                     Fixed point notation, corresponding to the printf %f output  format[0m
[38;2;248;248;242m                     with the precision given by digits.[0m

[38;2;248;248;242m              :eng digits[0m
[38;2;248;248;242m                     Exponential  notation, corresponding to the printf %E output format[0m
[38;2;248;248;242m                     with the precision given by digits.[0m

[38;2;248;248;242m              :raw   Raw output:  this is the default form of the  output  from  a  math[0m
[38;2;248;248;242m                     evaluation.   This may show more precision than the number actually[0m
[38;2;248;248;242m                     possesses.[0m

[38;2;248;248;242m              Other special commands:[0m
[38;2;248;248;242m              :!line...[0m
[38;2;248;248;242m                     Execute line... as a normal shell command line.  Note  that  it  is[0m
[38;2;248;248;242m                     executed in the context of the function, i.e. with local variables.[0m
[38;2;248;248;242m                     Space is optional after :!.[0m

[38;2;248;248;242m              :local arg ...[0m
[38;2;248;248;242m                     Declare  variables  local  to the function.  Other variables may be[0m
[38;2;248;248;242m                     used, too, but they will be taken  from  or  put  into  the  global[0m
[38;2;248;248;242m                     scope.[0m

[38;2;248;248;242m              :function name [ body ][0m
[38;2;248;248;242m                     Define a mathematical function or (with no body) delete it.  :func‐[0m
[38;2;248;248;242m                     tion  may  be abbreviated to :func or simply :f.  The name may con‐[0m
[38;2;248;248;242m                     tain the same characters as a shell function name.  The function is[0m
[38;2;248;248;242m                     defined using zmathfuncdef, see below.[0m

[38;2;248;248;242m                     Note that zcalc takes care of all quoting.  Hence for example:[0m

[38;2;248;248;242m                            :f cube $1 * $1 * $1[0m

[38;2;248;248;242m                     defines a function to cube the sole  argument.   Functions  so  de‐[0m
[38;2;248;248;242m                     fined, or indeed any functions defined directly or indirectly using[0m
[38;2;248;248;242m                     functions  -M,  are available to execute by typing only the name on[0m
[38;2;248;248;242m                     the line in RPN mode; this pops the appropriate number of arguments[0m
[38;2;248;248;242m                     off the stack to pass to the function, i.e. 1 in the  case  of  the[0m
[38;2;248;248;242m                     example  cube  function.   If there are optional arguments only the[0m
[38;2;248;248;242m                     mandatory arguments are supplied by this means.[0m

[38;2;248;248;242m              [#base][0m
[38;2;248;248;242m                     This is not a special command, rather  part  of  normal  arithmetic[0m
[38;2;248;248;242m                     syntax; however, when this form appears on a line by itself the de‐[0m
[38;2;248;248;242m                     fault  output  radix  is set to base.  Use, for example, `[#16]' to[0m
[38;2;248;248;242m                     display hexadecimal output preceded by an indication of  the  base,[0m
[38;2;248;248;242m                     or  `[##16]'  just  to  display  the  raw number in the given base.[0m
[38;2;248;248;242m                     Bases themselves are always specified in  decimal.  `[#]'  restores[0m
[38;2;248;248;242m                     the  normal  output  format.  Note that setting an output base sup‐[0m
[38;2;248;248;242m                     presses floating point output; use `[#]' to return to normal opera‐[0m
[38;2;248;248;242m                     tion.[0m

[38;2;248;248;242m              $var   Print out the value of var literally; does not affect the  calcula‐[0m
[38;2;248;248;242m                     tion.  To use the value of var, omit the leading `$'.[0m

[38;2;248;248;242m              See the comments in the function for a few extra tips.[0m

[38;2;248;248;242m       min(arg, ...)[0m
[38;2;248;248;242m       max(arg, ...)[0m
[38;2;248;248;242m       sum(arg, ...)[0m
[38;2;248;248;242m       zmathfunc[0m
[38;2;248;248;242m              The  function zmathfunc defines the three mathematical functions min, max,[0m
[38;2;248;248;242m              and sum.  The functions min and max take one or more arguments.  The func‐[0m
[38;2;248;248;242m              tion sum takes zero or more arguments.   Arguments  can  be  of  different[0m
[38;2;248;248;242m              types (ints and floats).[0m

[38;2;248;248;242m              Not  to be confused with the zsh/mathfunc module, described in the section[0m
[38;2;248;248;242m              `The zsh/mathfunc Module' in zshmodules(1).[0m

[38;2;248;248;242m       zmathfuncdef [ mathfunc [ body ] ][0m
[38;2;248;248;242m              A convenient front end to functions -M.[0m

[38;2;248;248;242m              With two arguments, define a mathematical function  named  mathfunc  which[0m
[38;2;248;248;242m              can  be used in any form of arithmetic evaluation.  body is a mathematical[0m
[38;2;248;248;242m              expression to implement the function.  It may contain references to  posi‐[0m
[38;2;248;248;242m              tion parameters $1, $2, ...  to refer to mandatory parameters and ${1:-de‐[0m
[38;2;248;248;242m              fvalue} ...  to refer to optional parameters.  Note that the forms must be[0m
[38;2;248;248;242m              strictly  adhered  to  for the function to calculate the correct number of[0m
[38;2;248;248;242m              arguments.   The  implementation  is  held  in  a  shell  function   named[0m
[38;2;248;248;242m              zsh_math_func_mathfunc;  usually  the  user  will not need to refer to the[0m
[38;2;248;248;242m              shell function directly.  Any  existing  function  of  the  same  name  is[0m
[38;2;248;248;242m              silently replaced.[0m

[38;2;248;248;242m              With  one  argument,  remove the mathematical function mathfunc as well as[0m
[38;2;248;248;242m              the shell function implementation.[0m

[38;2;248;248;242m              With no arguments, list all mathfunc functions  in  a  form  suitable  for[0m
[38;2;248;248;242m              restoring the definition.  The functions have not necessarily been defined[0m
[38;2;248;248;242m              by zmathfuncdef.[0m

[38;2;248;248;242mUSER CONFIGURATION FUNCTIONS[0m
[38;2;248;248;242m       The  zsh/newuser module comes with a function to aid in configuring shell options[0m
[38;2;248;248;242m       for new users.  If the module is installed, this function  can  also  be  run  by[0m
[38;2;248;248;242m       hand.  It is available even if the module's default behaviour, namely running the[0m
[38;2;248;248;242m       function for a new user logging in without startup files, is inhibited.[0m

[38;2;248;248;242m       zsh-newuser-install [ -f ][0m
[38;2;248;248;242m              The  function presents the user with various options for customizing their[0m
[38;2;248;248;242m              initialization scripts.   Currently  only  ~/.zshrc  is  handled.   $ZDOT‐[0m
[38;2;248;248;242m              DIR/.zshrc  is used instead if the parameter ZDOTDIR is set; this provides[0m
[38;2;248;248;242m              a way for the user to  configure  a  file  without  altering  an  existing[0m
[38;2;248;248;242m              .zshrc.[0m

[38;2;248;248;242m              By  default  the  function  exits immediately if it finds any of the files[0m
[38;2;248;248;242m              .zshenv, .zprofile, .zshrc, or .zlogin in the appropriate directory.   The[0m
[38;2;248;248;242m              option  -f  is  required in order to force the function to continue.  Note[0m
[38;2;248;248;242m              this may happen even if .zshrc itself does not exist.[0m

[38;2;248;248;242m              As currently configured, the function will exit immediately  if  the  user[0m
[38;2;248;248;242m              has root privileges; this behaviour cannot be overridden.[0m

[38;2;248;248;242m              Once  activated,  the function's behaviour is supposed to be self-explana‐[0m
[38;2;248;248;242m              tory.  Menus are present allowing the user to alter the value  of  options[0m
[38;2;248;248;242m              and parameters.  Suggestions for improvements are always welcome.[0m

[38;2;248;248;242m              When  the  script exits, the user is given the opportunity to save the new[0m
[38;2;248;248;242m              file or not; changes are not irreversible until this point.  However,  the[0m
[38;2;248;248;242m              script  is  careful to restrict changes to the file only to a group marked[0m
[38;2;248;248;242m              by the lines `# Lines configured by zsh-newuser-install'  and  `#  End  of[0m
[38;2;248;248;242m              lines configured by zsh-newuser-install'.  In addition, the old version of[0m
[38;2;248;248;242m              .zshrc is saved to a file with the suffix .zni appended.[0m

[38;2;248;248;242m              If  the  function edits an existing .zshrc, it is up to the user to ensure[0m
[38;2;248;248;242m              that the changes made will take effect.  For example, if  control  usually[0m
[38;2;248;248;242m              returns  early from the existing .zshrc the lines will not be executed; or[0m
[38;2;248;248;242m              a later initialization file may override options or parameters, and so on.[0m
[38;2;248;248;242m              The function itself does not attempt to detect any such conflicts.[0m

[38;2;248;248;242mOTHER FUNCTIONS[0m
[38;2;248;248;242m       There are a large number of helpful functions in the Functions/Misc directory  of[0m
[38;2;248;248;242m       the  zsh  distribution.   Most  are  very simple and do not require documentation[0m
[38;2;248;248;242m       here, but a few are worthy of special mention.[0m

[38;2;248;248;242m   Descriptions[0m
[38;2;248;248;242m       colors This function initializes several associative arrays to map color names to[0m
[38;2;248;248;242m              (and from) the ANSI standard eight-color terminal codes.  These  are  used[0m
[38;2;248;248;242m              by  the  prompt  theme  system (see above).  You seldom should need to run[0m
[38;2;248;248;242m              colors more than once.[0m

[38;2;248;248;242m              The eight base colors are: black, red, green, yellow, blue, magenta, cyan,[0m
[38;2;248;248;242m              and white.  Each of these has codes for foreground and background.  In ad‐[0m
[38;2;248;248;242m              dition there are seven intensity attributes: bold, faint, standout, under‐[0m
[38;2;248;248;242m              line, blink, reverse, and conceal.  Finally, there are seven codes used to[0m
[38;2;248;248;242m              negate attributes: none (reset all attributes  to  the  defaults),  normal[0m
[38;2;248;248;242m              (neither bold nor faint), no-standout, no-underline, no-blink, no-reverse,[0m
[38;2;248;248;242m              and no-conceal.[0m

[38;2;248;248;242m              Some terminals do not support all combinations of colors and intensities.[0m

[38;2;248;248;242m              The associative arrays are:[0m

[38;2;248;248;242m              color[0m
[38;2;248;248;242m              colour Map  all  the color names to their integer codes, and integer codes[0m
[38;2;248;248;242m                     to the color names.  The eight base names  map  to  the  foreground[0m
[38;2;248;248;242m                     color  codes,  as  do  names prefixed with `fg-', such as `fg-red'.[0m
[38;2;248;248;242m                     Names prefixed with `bg-', such as `bg-blue', refer  to  the  back‐[0m
[38;2;248;248;242m                     ground  codes.   The reverse mapping from code to color yields base[0m
[38;2;248;248;242m                     name for foreground codes and the bg- form for backgrounds.[0m

[38;2;248;248;242m                     Although it is a misnomer to call them `colors', these arrays  also[0m
[38;2;248;248;242m                     map  the other fourteen attributes from names to codes and codes to[0m
[38;2;248;248;242m                     names.[0m

[38;2;248;248;242m              fg[0m
[38;2;248;248;242m              fg_bold[0m
[38;2;248;248;242m              fg_no_bold[0m
[38;2;248;248;242m                     Map the eight basic color names to ANSI terminal  escape  sequences[0m
[38;2;248;248;242m                     that  set the corresponding foreground text properties.  The fg se‐[0m
[38;2;248;248;242m                     quences change the color without changing the eight  intensity  at‐[0m
[38;2;248;248;242m                     tributes.[0m

[38;2;248;248;242m              bg[0m
[38;2;248;248;242m              bg_bold[0m
[38;2;248;248;242m              bg_no_bold[0m
[38;2;248;248;242m                     Map  the  eight basic color names to ANSI terminal escape sequences[0m
[38;2;248;248;242m                     that set the corresponding background properties.  The bg sequences[0m
[38;2;248;248;242m                     change the color without changing the eight intensity attributes.[0m

[38;2;248;248;242m              In addition, the scalar parameters reset_color and bold_color are  set  to[0m
[38;2;248;248;242m              the  ANSI  terminal  escapes that turn off all attributes and turn on bold[0m
[38;2;248;248;242m              intensity, respectively.[0m

[38;2;248;248;242m       fned [ -x num ] name[0m
[38;2;248;248;242m              Same as zed -f.  This function does not appear in  the  zsh  distribution,[0m
[38;2;248;248;242m              but  can  be  created by linking zed to the name fned in some directory in[0m
[38;2;248;248;242m              your fpath.[0m

[38;2;248;248;242m       histed [ [ name ] size ][0m
[38;2;248;248;242m              Same as zed -h.  This function does not appear in  the  zsh  distribution,[0m
[38;2;248;248;242m              but  can be created by linking zed to the name histed in some directory in[0m
[38;2;248;248;242m              your fpath.[0m

[38;2;248;248;242m       is-at-least needed [ present ][0m
[38;2;248;248;242m              Perform a greater-than-or-equal-to comparison of two  strings  having  the[0m
[38;2;248;248;242m              format of a zsh version number; that is, a string of numbers and text with[0m
[38;2;248;248;242m              segments  separated  by dots or dashes.  If the present string is not pro‐[0m
[38;2;248;248;242m              vided, $ZSH_VERSION is used.  Segments are paired left-to-right in the two[0m
[38;2;248;248;242m              strings with leading non-number parts ignored.  If one  string  has  fewer[0m
[38;2;248;248;242m              segments than the other, the missing segments are considered zero.[0m

[38;2;248;248;242m              This  is  useful  in startup files to set options and other state that are[0m
[38;2;248;248;242m              not available in all versions of zsh.[0m

[38;2;248;248;242m                     is-at-least 3.1.6-15 && setopt NO_GLOBAL_RCS[0m
[38;2;248;248;242m                     is-at-least 3.1.0 && setopt HIST_REDUCE_BLANKS[0m
[38;2;248;248;242m                     is-at-least 2.6-17 || print "You can't use is-at-least here."[0m

[38;2;248;248;242m       nslookup [ arg ... ][0m
[38;2;248;248;242m              This wrapper function for the nslookup command requires the zsh/zpty  mod‐[0m
[38;2;248;248;242m              ule  (see  zshmodules(1)).   It behaves exactly like the standard nslookup[0m
[38;2;248;248;242m              except that it  provides  customizable  prompts  (including  a  right-side[0m
[38;2;248;248;242m              prompt)  and completion of nslookup commands, host names, etc. (if you use[0m
[38;2;248;248;242m              the function-based completion system).  Completion styles may be set  with[0m
[38;2;248;248;242m              the context prefix `:completion:nslookup'.[0m

[38;2;248;248;242m              See also the pager, prompt and rprompt styles below.[0m

[38;2;248;248;242m       regexp-replace var regexp replace[0m
[38;2;248;248;242m              Use  regular  expressions to perform a global search and replace operation[0m
[38;2;248;248;242m              on a variable.  POSIX extended regular expressions (ERE) are used,  unless[0m
[38;2;248;248;242m              the option RE_MATCH_PCRE has been set, in which case Perl-compatible regu‐[0m
[38;2;248;248;242m              lar expressions are used (this requires the shell to be linked against the[0m
[38;2;248;248;242m              pcre library).[0m

[38;2;248;248;242m              var  is the name of the variable containing the string to be matched.  The[0m
[38;2;248;248;242m              variable will be modified directly by the function.  The variables  MATCH,[0m
[38;2;248;248;242m              MBEGIN,  MEND,  match, mbegin, mend should be avoided as these are used by[0m
[38;2;248;248;242m              the regular expression code.[0m

[38;2;248;248;242m              regexp is the regular expression to match against the string.[0m

[38;2;248;248;242m              replace is the replacement text.  This can contain parameter, command  and[0m
[38;2;248;248;242m              arithmetic expressions which will be replaced:  in particular, a reference[0m
[38;2;248;248;242m              to $MATCH will be replaced by the text matched by the pattern.[0m

[38;2;248;248;242m              The return status is 0 if at least one match was performed, else 1.[0m

[38;2;248;248;242m              Note  that  if  using  POSIX EREs, the ^ or word boundary operators (where[0m
[38;2;248;248;242m              available) may not work properly.[0m

[38;2;248;248;242m       run-help cmd[0m
[38;2;248;248;242m              This function is designed to be invoked by the  run-help  ZLE  widget,  in[0m
[38;2;248;248;242m              place  of the default alias.  See `Accessing On-Line Help' above for setup[0m
[38;2;248;248;242m              instructions.[0m

[38;2;248;248;242m              In the discussion which follows, if cmd is a file system path, it is first[0m
[38;2;248;248;242m              reduced to its rightmost component (the file name).[0m

[38;2;248;248;242m              Help is first sought by looking for a file  named  cmd  in  the  directory[0m
[38;2;248;248;242m              named  by  the HELPDIR parameter.  If no file is found, an assistant func‐[0m
[38;2;248;248;242m              tion, alias, or command named run-help-cmd is sought.  If found,  the  as‐[0m
[38;2;248;248;242m              sistant  is executed with the rest of the current command line (everything[0m
[38;2;248;248;242m              after the command name cmd) as its arguments.  When neither file  nor  as‐[0m
[38;2;248;248;242m              sistant is found, the external command `man cmd' is run.[0m

[38;2;248;248;242m              An example assistant for the "ssh" command:[0m

[38;2;248;248;242m                     run-help-ssh() {[0m
[38;2;248;248;242m                         emulate -LR zsh[0m
[38;2;248;248;242m                         local -a args[0m
[38;2;248;248;242m                         # Delete the "-l username" option[0m
[38;2;248;248;242m                         zparseopts -D -E -a args l:[0m
[38;2;248;248;242m                         # Delete other options, leaving: host command[0m
[38;2;248;248;242m                         args=(${@:#-*})[0m
[38;2;248;248;242m                         if [[ ${#args} -lt 2 ]]; then[0m
[38;2;248;248;242m                             man ssh[0m
[38;2;248;248;242m                         else[0m
[38;2;248;248;242m                             run-help $args[2][0m
[38;2;248;248;242m                         fi[0m
[38;2;248;248;242m                     }[0m

[38;2;248;248;242m              Several  of these assistants are provided in the Functions/Misc directory.[0m
[38;2;248;248;242m              These must be autoloaded, or placed as executable scripts in  your  search[0m
[38;2;248;248;242m              path, in order to be found and used by run-help.[0m

[38;2;248;248;242m              run-help-btrfs[0m
[38;2;248;248;242m              run-help-git[0m
[38;2;248;248;242m              run-help-ip[0m
[38;2;248;248;242m              run-help-openssl[0m
[38;2;248;248;242m              run-help-p4[0m
[38;2;248;248;242m              run-help-sudo[0m
[38;2;248;248;242m              run-help-svk[0m
[38;2;248;248;242m              run-help-svn[0m
[38;2;248;248;242m                     Assistant functions for the btrfs, git, ip, openssl, p4, sudo, svk,[0m
[38;2;248;248;242m                     and svn, commands.[0m

[38;2;248;248;242m       tetris Zsh  was once accused of not being as complete as Emacs, because it lacked[0m
[38;2;248;248;242m              a Tetris game.  This function was written to refute this vicious slander.[0m

[38;2;248;248;242m              This function must be used as a ZLE widget:[0m

[38;2;248;248;242m                     autoload -U tetris[0m
[38;2;248;248;242m                     zle -N tetris[0m
[38;2;248;248;242m                     bindkey keys tetris[0m

[38;2;248;248;242m              To start a game, execute the widget by typing the keys.  Whatever  command[0m
[38;2;248;248;242m              line you were editing disappears temporarily, and your keymap is also tem‐[0m
[38;2;248;248;242m              porarily  replaced  by the Tetris control keys.  The previous editor state[0m
[38;2;248;248;242m              is restored when you quit the game (by pressing `q') or when you lose.[0m

[38;2;248;248;242m              If you quit in the middle of a game, the next  invocation  of  the  tetris[0m
[38;2;248;248;242m              widget will continue where you left off.  If you lost, it will start a new[0m
[38;2;248;248;242m              game.[0m

[38;2;248;248;242m       tetriscurses[0m
[38;2;248;248;242m              This  is a port of the above to zcurses.  The input handling is improved a[0m
[38;2;248;248;242m              bit so that moving  a  block  sideways  doesn't  automatically  advance  a[0m
[38;2;248;248;242m              timestep, and the graphics use unicode block graphics.[0m

[38;2;248;248;242m              This  version does not save the game state between invocations, and is not[0m
[38;2;248;248;242m              invoked as a widget, but rather as:[0m

[38;2;248;248;242m                     autoload -U tetriscurses[0m
[38;2;248;248;242m                     tetriscurses[0m

[38;2;248;248;242m       zargs [ option ... -- ] [ input ... ] [ -- command [ arg ... ] ][0m
[38;2;248;248;242m              This function has a similar purpose to  GNU  xargs.   Instead  of  reading[0m
[38;2;248;248;242m              lines of arguments from the standard input, it takes them from the command[0m
[38;2;248;248;242m              line.   This  is useful because zsh, especially with recursive glob opera‐[0m
[38;2;248;248;242m              tors, often can construct a command line for  a  shell  function  that  is[0m
[38;2;248;248;242m              longer than can be accepted by an external command.[0m

[38;2;248;248;242m              The  option list represents options of the zargs command itself, which are[0m
[38;2;248;248;242m              the same as those of xargs.  The input list is the collection  of  strings[0m
[38;2;248;248;242m              (often  file names) that become the arguments of the command, analogous to[0m
[38;2;248;248;242m              the standard input of xargs.  Finally, the arg list consists of those  ar‐[0m
[38;2;248;248;242m              guments  (usually  options)  that  are  passed to the command each time it[0m
[38;2;248;248;242m              runs.  The arg list precedes the elements from the input list in each run.[0m
[38;2;248;248;242m              If no command is provided, then no arg list may be provided, and  in  that[0m
[38;2;248;248;242m              event the default command is `print' with arguments `-r --'.[0m

[38;2;248;248;242m              For example, to get a long ls listing of all non-hidden plain files in the[0m
[38;2;248;248;242m              current directory or its subdirectories:[0m

[38;2;248;248;242m                     autoload -U zargs[0m
[38;2;248;248;242m                     zargs -- **/*(.) -- ls -ld --[0m

[38;2;248;248;242m              The  first  and  third occurrences of `--' are used to mark the end of op‐[0m
[38;2;248;248;242m              tions for zargs and ls respectively to guard  against  filenames  starting[0m
[38;2;248;248;242m              with  `-', while the second is used to separate the list of files from the[0m
[38;2;248;248;242m              command to run (`ls -ld --').[0m

[38;2;248;248;242m              The first `--' would also be needed if there was a chance the  list  might[0m
[38;2;248;248;242m              be empty as in:[0m

[38;2;248;248;242m                     zargs -r -- ./*.back(#qN) -- rm -f[0m

[38;2;248;248;242m              In the event that the string `--' is or may be an input, the -e option may[0m
[38;2;248;248;242m              be  used  to  change  the  end-of-inputs  marker.  Note that this does not[0m
[38;2;248;248;242m              change the end-of-options marker.  For example, to use `..' as the marker:[0m

[38;2;248;248;242m                     zargs -e.. -- **/*(.) .. ls -ld --[0m

[38;2;248;248;242m              This is a good choice in that example because no plain file can  be  named[0m
[38;2;248;248;242m              `..', but the best end-marker depends on the circumstances.[0m

[38;2;248;248;242m              The  options  -i,  -I,  -l, -L, and -n differ slightly from their usage in[0m
[38;2;248;248;242m              xargs.  There are no input lines for zargs to count, so -l  and  -L  count[0m
[38;2;248;248;242m              through  the  input  list, and -n counts the number of arguments passed to[0m
[38;2;248;248;242m              each execution of command, including any arg list.  Also, any time  -i  or[0m
[38;2;248;248;242m              -I is used, each input is processed separately as if by `-L 1'.[0m

[38;2;248;248;242m              For  details  of  the  other zargs options, see the xargs(1) man page (but[0m
[38;2;248;248;242m              note the difference in function between zargs and xargs) or run zargs with[0m
[38;2;248;248;242m              the --help option.[0m

[38;2;248;248;242m       zed [ -f [ -x num ] ] name[0m
[38;2;248;248;242m       zed [ -h [ name ] size ][0m
[38;2;248;248;242m       zed -b This function uses the ZLE editor to edit a file or function.[0m

[38;2;248;248;242m              Only one name argument is allowed.  If the -f option is given, the name is[0m
[38;2;248;248;242m              taken to be that of a function; if the function is marked for autoloading,[0m
[38;2;248;248;242m              zed searches for it in the fpath and loads it.  Note that functions edited[0m
[38;2;248;248;242m              this way are installed into the current shell, but not written back to the[0m
[38;2;248;248;242m              autoload file.  In this case the -x option specifies that leading tabs in‐[0m
[38;2;248;248;242m              denting the function according to syntax  should  be  converted  into  the[0m
[38;2;248;248;242m              given  number of spaces; `-x 2' is consistent with the layout of functions[0m
[38;2;248;248;242m              distributed with the shell.[0m

[38;2;248;248;242m              Without -f, name is the path name of the file to edit, which need not  ex‐[0m
[38;2;248;248;242m              ist;  it is created on write, if necessary.  With -h, the file is presumed[0m
[38;2;248;248;242m              to contain history events.[0m

[38;2;248;248;242m              When no file name is provided for -h the current shell history  is  edited[0m
[38;2;248;248;242m              in place.  The history is renumbered when zed exits successfully.[0m

[38;2;248;248;242m              When  editing history, multi-line events must have a trailing backslash on[0m
[38;2;248;248;242m              every line before the last.[0m

[38;2;248;248;242m              While editing, the function sets the main keymap to zed and the vi command[0m
[38;2;248;248;242m              keymap to zed-vicmd.  These will be copied  from  the  existing  main  and[0m
[38;2;248;248;242m              vicmd keymaps if they do not exist the first time zed is run.  They can be[0m
[38;2;248;248;242m              used to provide special key bindings used only in zed.[0m

[38;2;248;248;242m              If  it  creates  the  keymap,  zed rebinds the return key to insert a line[0m
[38;2;248;248;242m              break and `^X^W' to accept the edit in the zed keymap, and binds  `ZZ'  to[0m
[38;2;248;248;242m              accept the edit in the zed-vicmd keymap.[0m

[38;2;248;248;242m              The bindings alone can be installed by running `zed -b'.  This is suitable[0m
[38;2;248;248;242m              for putting into a startup file.  Note that, if rerun, this will overwrite[0m
[38;2;248;248;242m              the existing zed and zed-vicmd keymaps.[0m

[38;2;248;248;242m              Completion  is  available,  and  styles may be set with the context prefix[0m
[38;2;248;248;242m              `:completion:zed:'.[0m

[38;2;248;248;242m              A zle widget zed-set-file-name is available.  This can be called  by  name[0m
[38;2;248;248;242m              from  within zed using `\ex zed-set-file-name' or can be bound to a key in[0m
[38;2;248;248;242m              either of the zed or zed-vicmd keymaps after `zed -b' has been run.   When[0m
[38;2;248;248;242m              the widget is called, it prompts for a new name for the file being edited.[0m
[38;2;248;248;242m              When  zed  exits the file will be written under that name and the original[0m
[38;2;248;248;242m              file will be left alone.  The widget has no effect when invoked from  `zed[0m
[38;2;248;248;242m              -f'.      The    completion    context    is    changed    to    `:comple‐[0m
[38;2;248;248;242m              tion:zed-set-file-name:'.  When editing the current history with `zed -h',[0m
[38;2;248;248;242m              the history is first updated and then the file is written, but the  global[0m
[38;2;248;248;242m              setting of HISTFILE is not altered.[0m

[38;2;248;248;242m              While zed-set-file-name is running, zed uses the keymap zed-normal-keymap,[0m
[38;2;248;248;242m              which is linked from the main keymap in effect at the time zed initialised[0m
[38;2;248;248;242m              its bindings.  (This is to make the return key operate normally.)  The re‐[0m
[38;2;248;248;242m              sult is that if the main keymap has been changed, the widget won't notice.[0m
[38;2;248;248;242m              This is not a concern for most users.[0m

[38;2;248;248;242m       zcp [ -finqQvwW ] srcpat dest[0m
[38;2;248;248;242m       zln [ -finqQsvwW ] srcpat dest[0m
[38;2;248;248;242m              Same as zmv -C and zmv -L, respectively.  These functions do not appear in[0m
[38;2;248;248;242m              the  zsh  distribution, but can be created by linking zmv to the names zcp[0m
[38;2;248;248;242m              and zln in some directory in your fpath.[0m

[38;2;248;248;242m       zkbd   See `Keyboard Definition' above.[0m

[38;2;248;248;242m       zmv [ -finqQsvwW ] [ -C | -L | -M | -{p|P} program ] [ -o optstring ][0m
[38;2;248;248;242m           srcpat dest[0m
[38;2;248;248;242m              Move (usually, rename) files matching the pattern srcpat to  corresponding[0m
[38;2;248;248;242m              files having names of the form given by dest, where srcpat contains paren‐[0m
[38;2;248;248;242m              theses  surrounding patterns which will be replaced in turn by $1, $2, ...[0m
[38;2;248;248;242m              in dest.  For example,[0m

[38;2;248;248;242m                     zmv '(*).lis' '$1.txt'[0m

[38;2;248;248;242m              renames `foo.lis' to `foo.txt', `my.old.stuff.lis' to  `my.old.stuff.txt',[0m
[38;2;248;248;242m              and so on.[0m

[38;2;248;248;242m              The pattern is always treated as an EXTENDED_GLOB pattern.  Any file whose[0m
[38;2;248;248;242m              name  is  not changed by the substitution is simply ignored.  Any error (a[0m
[38;2;248;248;242m              substitution resulted in an empty string, two substitutions gave the  same[0m
[38;2;248;248;242m              result, the destination was an existing regular file and -f was not given)[0m
[38;2;248;248;242m              causes the entire function to abort without doing anything.[0m

[38;2;248;248;242m              In  addition to pattern replacement, the variable $f can be referred to in[0m
[38;2;248;248;242m              the second (replacement) argument.  This makes it possible to use variable[0m
[38;2;248;248;242m              substitution to alter the argument; see examples below.[0m

[38;2;248;248;242m              Options:[0m

[38;2;248;248;242m              -f     Force overwriting of destination files.  Not currently passed  down[0m
[38;2;248;248;242m                     to the mv/cp/ln command due to vagaries of implementations (but you[0m
[38;2;248;248;242m                     can use -o-f to do that).[0m
[38;2;248;248;242m              -i     Interactive: show each line to be executed and ask the user whether[0m
[38;2;248;248;242m                     to execute it.  `Y' or `y' will execute it, anything else will skip[0m
[38;2;248;248;242m                     it.  Note that you just need to type one character.[0m
[38;2;248;248;242m              -n     No execution: print what would happen, but don't do it.[0m
[38;2;248;248;242m              -q     Turn  bare glob qualifiers off: now assumed by default, so this has[0m
[38;2;248;248;242m                     no effect.[0m
[38;2;248;248;242m              -Q     Force bare glob qualifiers on.  Don't turn this on unless  you  are[0m
[38;2;248;248;242m                     actually using glob qualifiers in a pattern.[0m
[38;2;248;248;242m              -s     Symbolic, passed down to ln; only works with -L.[0m
[38;2;248;248;242m              -v     Verbose: print each command as it's being executed.[0m
[38;2;248;248;242m              -w     Pick out wildcard parts of the pattern, as described above, and im‐[0m
[38;2;248;248;242m                     plicitly add parentheses for referring to them.[0m
[38;2;248;248;242m              -W     Just  like  -w,  with  the addition of turning wildcards in the re‐[0m
[38;2;248;248;242m                     placement pattern into sequential ${1} .. ${N} references.[0m
[38;2;248;248;242m              -C[0m
[38;2;248;248;242m              -L[0m
[38;2;248;248;242m              -M     Force cp, ln or mv, respectively, regardless of  the  name  of  the[0m
[38;2;248;248;242m                     function.[0m
[38;2;248;248;242m              -p program[0m
[38;2;248;248;242m                     Call  program instead of cp, ln or mv.  Whatever it does, it should[0m
[38;2;248;248;242m                     at least understand the form `program  --  oldname  newname'  where[0m
[38;2;248;248;242m                     oldname  and  newname are filenames generated by zmv.  program will[0m
[38;2;248;248;242m                     be split into words, so might be e.g. the name of an  archive  tool[0m
[38;2;248;248;242m                     plus a copy or rename subcommand.[0m
[38;2;248;248;242m              -P program[0m
[38;2;248;248;242m                     As  -p  program, except that program does not accept a following --[0m
[38;2;248;248;242m                     to indicate the end of options.  In this case  filenames  must  al‐[0m
[38;2;248;248;242m                     ready be in a sane form for the program in question.[0m
[38;2;248;248;242m              -o optstring[0m
[38;2;248;248;242m                     The  optstring  is split into words and passed down verbatim to the[0m
[38;2;248;248;242m                     cp, ln or mv command called to perform the work.  It should  proba‐[0m
[38;2;248;248;242m                     bly begin with a `-'.[0m

[38;2;248;248;242m              Further examples:[0m

[38;2;248;248;242m                     zmv -v '(* *)' '${1// /_}'[0m

[38;2;248;248;242m              For any file in the current directory with at least one space in the name,[0m
[38;2;248;248;242m              replace every space by an underscore and display the commands executed.[0m

[38;2;248;248;242m                     zmv -v '* *' '${f// /_}'[0m

[38;2;248;248;242m              This does exactly the same by referring to the file name stored in $f.[0m

[38;2;248;248;242m              For  more  complete examples and other implementation details, see the zmv[0m
[38;2;248;248;242m              source file, usually located in one  of  the  directories  named  in  your[0m
[38;2;248;248;242m              fpath, or in Functions/Misc/zmv in the zsh distribution.[0m

[38;2;248;248;242m       zrecompile[0m
[38;2;248;248;242m              See `Recompiling Functions' above.[0m

[38;2;248;248;242m       zstyle+ context style value [ + subcontext style value ... ][0m
[38;2;248;248;242m              This  makes  defining styles a bit simpler by using a single `+' as a spe‐[0m
[38;2;248;248;242m              cial token that allows you to append a context name to the previously used[0m
[38;2;248;248;242m              context name.  Like this:[0m

[38;2;248;248;242m                     zstyle+ ':foo:bar' style1 value1 \[0m
[38;2;248;248;242m                            +':baz'     style2 value2 \[0m
[38;2;248;248;242m                            +':frob'    style3 value3[0m

[38;2;248;248;242m              This defines style1 with value1 for the context :foo:bar as usual, but  it[0m
[38;2;248;248;242m              also  defines  style2  with value2 for the context :foo:bar:baz and style3[0m
[38;2;248;248;242m              with value3 for :foo:bar:frob.  Any subcontext may be the empty string  to[0m
[38;2;248;248;242m              re-use the first context unchanged.[0m

[38;2;248;248;242m   Styles[0m
[38;2;248;248;242m       insert-tab[0m
[38;2;248;248;242m              The  zed  function  sets this style in context `:completion:zed:*' to turn[0m
[38;2;248;248;242m              off completion when TAB is typed at the beginning  of  a  line.   You  may[0m
[38;2;248;248;242m              override this by setting your own value for this context and style.[0m

[38;2;248;248;242m       pager  The  nslookup  function  looks up this style in the context `:nslookup' to[0m
[38;2;248;248;242m              determine the program used to display output that does not fit on a single[0m
[38;2;248;248;242m              screen.[0m

[38;2;248;248;242m       prompt[0m
[38;2;248;248;242m       rprompt[0m
[38;2;248;248;242m              The nslookup function looks up this style in the  context  `:nslookup'  to[0m
[38;2;248;248;242m              set  the prompt and the right-side prompt, respectively.  The usual expan‐[0m
[38;2;248;248;242m              sions for the PS1 and RPS1 parameters may be used (see EXPANSION OF PROMPT[0m
[38;2;248;248;242m              SEQUENCES in zshmisc(1)).[0m
