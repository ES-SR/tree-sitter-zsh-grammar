[38;2;248;248;242mSHELL BUILTIN COMMANDS[0m
[38;2;248;248;242m       Some  shell  builtin  commands  take  options as described in individual entries;[0m
[38;2;248;248;242m       these are often referred to in the list below as `flags' to avoid confusion  with[0m
[38;2;248;248;242m       shell  options,  which  may  also have an effect on the behaviour of builtin com‐[0m
[38;2;248;248;242m       mands.  In this introductory section, `option' always has the meaning of  an  op‐[0m
[38;2;248;248;242m       tion to a command that should be familiar to most command line users.[0m

[38;2;248;248;242m       Typically,  options  are  single  letters preceded by a hyphen (-).  Options that[0m
[38;2;248;248;242m       take an argument accept it either immediately following the option letter or  af‐[0m
[38;2;248;248;242m       ter white space, for example `print -C3 {1..9}' or `print -C 3 {1..9}' are equiv‐[0m
[38;2;248;248;242m       alent.   Arguments  to  options are not the same as arguments to the command; the[0m
[38;2;248;248;242m       documentation indicates which is which.  Options that do not take an argument may[0m
[38;2;248;248;242m       be combined in a single word, for example `print -rca -- *' and `print -r  -c  -a[0m
[38;2;248;248;242m       -- *' are equivalent.[0m

[38;2;248;248;242m       Some shell builtin commands also take options that begin with `+' instead of `-'.[0m
[38;2;248;248;242m       The list below makes clear which commands these are.[0m

[38;2;248;248;242m       Options (together with their individual arguments, if any) must appear in a group[0m
[38;2;248;248;242m       before  any  non-option  arguments;  once  the first non-option argument has been[0m
[38;2;248;248;242m       found, option processing is terminated.[0m

[38;2;248;248;242m       All builtin commands other than `echo' and precommand modifiers, even those  that[0m
[38;2;248;248;242m       have  no  options, can be given the argument `--' to terminate option processing.[0m
[38;2;248;248;242m       This indicates that the following words are non-option arguments, but  is  other‐[0m
[38;2;248;248;242m       wise  ignored.   This is useful in cases where arguments to the command may begin[0m
[38;2;248;248;242m       with `-'.  For historical reasons, most builtin commands (including `echo')  also[0m
[38;2;248;248;242m       recognize  a  single  `-'  in a separate word for this purpose; note that this is[0m
[38;2;248;248;242m       less standard and use of `--' is recommended.[0m

[38;2;248;248;242m       - simple command[0m
[38;2;248;248;242m              See the section `Precommand Modifiers' in zshmisc(1).[0m

[38;2;248;248;242m       . file [ arg ... ][0m
[38;2;248;248;242m              Read commands from file and execute them in the current shell environment.[0m

[38;2;248;248;242m              If file does not contain a slash, or if PATH_DIRS is set, the shell  looks[0m
[38;2;248;248;242m              in  the  components of $path to find the directory containing file.  Files[0m
[38;2;248;248;242m              in the current directory are not read  unless  `.'  appears  somewhere  in[0m
[38;2;248;248;242m              $path.   If  a  file named `file.zwc' is found, is newer than file, and is[0m
[38;2;248;248;242m              the compiled form (created with the zcompile builtin) of file,  then  com‐[0m
[38;2;248;248;242m              mands are read from that file instead of file.[0m

[38;2;248;248;242m              If any arguments arg are given, they become the positional parameters; the[0m
[38;2;248;248;242m              old  positional  parameters  are restored when the file is done executing.[0m
[38;2;248;248;242m              However, if no arguments are given, the positional parameters remain those[0m
[38;2;248;248;242m              of the calling context, and no restoring is done.[0m

[38;2;248;248;242m              If file was not found the return status is 127; if file was found but con‐[0m
[38;2;248;248;242m              tained a syntax error the return status is 126; else the return status  is[0m
[38;2;248;248;242m              the exit status of the last command executed.[0m

[38;2;248;248;242m       : [ arg ... ][0m
[38;2;248;248;242m              This  command  does  nothing,  although normal argument expansions is per‐[0m
[38;2;248;248;242m              formed which may have effects on shell parameters.  A zero exit status  is[0m
[38;2;248;248;242m              returned.[0m

[38;2;248;248;242m       alias [ {+|-}gmrsL ] [ name[=value] ... ][0m
[38;2;248;248;242m              For each name with a corresponding value, define an alias with that value.[0m
[38;2;248;248;242m              A trailing space in value causes the next word to be checked for alias ex‐[0m
[38;2;248;248;242m              pansion.  If the -g flag is present, define a global alias; global aliases[0m
[38;2;248;248;242m              are expanded even if they do not occur in command position:[0m

[38;2;248;248;242m                     % perldoc --help 2>&1 | grep 'built-in functions'[0m
[38;2;248;248;242m                         -f   Search Perl built-in functions[0m
[38;2;248;248;242m                     % alias -g HG='--help 2>&1 | grep'[0m
[38;2;248;248;242m                     % perldoc HG 'built-in functions'[0m
[38;2;248;248;242m                         -f   Search Perl built-in functions[0m

[38;2;248;248;242m              If the -s flag is present, define a suffix alias: if the command word on a[0m
[38;2;248;248;242m              command  line  is  in  the  form  `text.name', where text is any non-empty[0m
[38;2;248;248;242m              string, it is replaced by the text `value text.name'.  Note that  name  is[0m
[38;2;248;248;242m              treated  as a literal string, not a pattern.  A trailing space in value is[0m
[38;2;248;248;242m              not special in this case.  For example,[0m

[38;2;248;248;242m                     alias -s ps='gv --'[0m

[38;2;248;248;242m              will cause the command `*.ps' to be expanded to `gv --  *.ps'.   As  alias[0m
[38;2;248;248;242m              expansion  is  carried  out earlier than globbing, the `*.ps' will then be[0m
[38;2;248;248;242m              expanded.  Suffix aliases constitute a different  name  space  from  other[0m
[38;2;248;248;242m              aliases  (so  in the above example it is still possible to create an alias[0m
[38;2;248;248;242m              for the command ps) and the two sets are never listed together.[0m

[38;2;248;248;242m              For each name with no value, print the value of name, if any.  With no ar‐[0m
[38;2;248;248;242m              guments, print all currently defined aliases other  than  suffix  aliases.[0m
[38;2;248;248;242m              If  the  -m flag is given the arguments are taken as patterns (they should[0m
[38;2;248;248;242m              be quoted to preserve them from being interpreted as glob  patterns),  and[0m
[38;2;248;248;242m              the  aliases  matching  these patterns are printed.  When printing aliases[0m
[38;2;248;248;242m              and one of the -g, -r or -s flags is present,  restrict  the  printing  to[0m
[38;2;248;248;242m              global,  regular  or  suffix aliases, respectively; a regular alias is one[0m
[38;2;248;248;242m              which is neither a global nor a suffix alias.   Using `+' instead of  `-',[0m
[38;2;248;248;242m              or  ending  the  option list with a single `+', prevents the values of the[0m
[38;2;248;248;242m              aliases from being printed.[0m

[38;2;248;248;242m              If the -L flag is present, then print each alias in a manner suitable  for[0m
[38;2;248;248;242m              putting  in  a startup script.  The exit status is nonzero if a name (with[0m
[38;2;248;248;242m              no value) is given for which no alias has been defined.[0m

[38;2;248;248;242m              For more on aliases, include common problems, see the section ALIASING  in[0m
[38;2;248;248;242m              zshmisc(1).[0m

[38;2;248;248;242m       autoload [ {+|-}RTUXdkmrtWz ] [ -w ] [ name ... ][0m
[38;2;248;248;242m              See  the  section  `Autoloading Functions' in zshmisc(1) for full details.[0m
[38;2;248;248;242m              The fpath parameter will be searched to find the function definition  when[0m
[38;2;248;248;242m              the function is first referenced.[0m

[38;2;248;248;242m              If name consists of an absolute path, the function is defined to load from[0m
[38;2;248;248;242m              the  file given (searching as usual for dump files in the given location).[0m
[38;2;248;248;242m              The name of the function is the basename (non-directory part) of the file.[0m
[38;2;248;248;242m              It is normally an error if the function is not found in  the  given  loca‐[0m
[38;2;248;248;242m              tion;  however,  if the option -d is given, searching for the function de‐[0m
[38;2;248;248;242m              faults to $fpath.  If a function is loaded by absolute path, any functions[0m
[38;2;248;248;242m              loaded from it that are marked for autoload without an absolute path  have[0m
[38;2;248;248;242m              the load path of the parent function temporarily prepended to $fpath.[0m

[38;2;248;248;242m              If  the option -r or -R is given, the function is searched for immediately[0m
[38;2;248;248;242m              and the location is recorded internally for use when the function is  exe‐[0m
[38;2;248;248;242m              cuted; a relative path is expanded using the value of $PWD.  This protects[0m
[38;2;248;248;242m              against  a  change  to $fpath after the call to autoload.  With -r, if the[0m
[38;2;248;248;242m              function is not found, it is silently  left  unresolved  until  execution;[0m
[38;2;248;248;242m              with  -R, an error message is printed and command processing aborted imme‐[0m
[38;2;248;248;242m              diately the search fails, i.e. at the  autoload  command  rather  than  at[0m
[38;2;248;248;242m              function execution..[0m

[38;2;248;248;242m              The flag -X may be used only inside a shell function.  It causes the call‐[0m
[38;2;248;248;242m              ing  function to be marked for autoloading and then immediately loaded and[0m
[38;2;248;248;242m              executed, with the current array of positional  parameters  as  arguments.[0m
[38;2;248;248;242m              This replaces the previous definition of the function.  If no function de‐[0m
[38;2;248;248;242m              finition  is found, an error is printed and the function remains undefined[0m
[38;2;248;248;242m              and marked for autoloading.  If an argument is given, it is used as a  di‐[0m
[38;2;248;248;242m              rectory  (i.e.  it does not include the name of the function) in which the[0m
[38;2;248;248;242m              function is to be found; this may be combined with the -d option to  allow[0m
[38;2;248;248;242m              the  function  search to default to $fpath if it is not in the given loca‐[0m
[38;2;248;248;242m              tion.[0m

[38;2;248;248;242m              The flag +X attempts to load each name as an autoloaded function, but does[0m
[38;2;248;248;242m              not execute it.  The exit status is zero (success) if the function was not[0m
[38;2;248;248;242m              previously defined and a definition for it was found.  This does  not  re‐[0m
[38;2;248;248;242m              place any existing definition of the function.  The exit status is nonzero[0m
[38;2;248;248;242m              (failure)  if  the  function was already defined or when no definition was[0m
[38;2;248;248;242m              found.  In the latter case the function remains undefined and  marked  for[0m
[38;2;248;248;242m              autoloading.   If  ksh-style  autoloading is enabled, the function created[0m
[38;2;248;248;242m              will contain the contents of the file plus a call to the  function  itself[0m
[38;2;248;248;242m              appended  to it, thus giving normal ksh autoloading behaviour on the first[0m
[38;2;248;248;242m              call to the function.  If the -m flag is also given each name  is  treated[0m
[38;2;248;248;242m              as  a pattern and all functions already marked for autoload that match the[0m
[38;2;248;248;242m              pattern are loaded.[0m

[38;2;248;248;242m              With the -t flag, turn on execution tracing; with -T,  turn  on  execution[0m
[38;2;248;248;242m              tracing  only  for  the  current  function, turning it off on entry to any[0m
[38;2;248;248;242m              called functions that do not also have tracing enabled.[0m

[38;2;248;248;242m              With the -U flag, alias expansion  is  suppressed  when  the  function  is[0m
[38;2;248;248;242m              loaded.[0m

[38;2;248;248;242m              With  the -w flag, the names are taken as names of files compiled with the[0m
[38;2;248;248;242m              zcompile builtin, and all functions defined in them  are  marked  for  au‐[0m
[38;2;248;248;242m              toloading.[0m

[38;2;248;248;242m              The  flags  -z  and -k mark the function to be autoloaded using the zsh or[0m
[38;2;248;248;242m              ksh style, as if the option KSH_AUTOLOAD were unset or were  set,  respec‐[0m
[38;2;248;248;242m              tively.   The  flags  override  the  setting of the option at the time the[0m
[38;2;248;248;242m              function is loaded.[0m

[38;2;248;248;242m              Note that the autoload command makes no attempt to ensure  the  shell  op‐[0m
[38;2;248;248;242m              tions  set during the loading or execution of the file have any particular[0m
[38;2;248;248;242m              value.  For this, the emulate command can be used:[0m

[38;2;248;248;242m                     emulate zsh -c 'autoload -Uz func'[0m

[38;2;248;248;242m              arranges that when func is loaded the shell is in  native  zsh  emulation,[0m
[38;2;248;248;242m              and this emulation is also applied when func is run.[0m

[38;2;248;248;242m              Some  of  the  functions  of autoload are also provided by functions -u or[0m
[38;2;248;248;242m              functions -U, but autoload is a more comprehensive interface.[0m

[38;2;248;248;242m       cd [ -qsLP ] [ arg ][0m
[38;2;248;248;242m       cd [ -qsLP ] old new[0m
[38;2;248;248;242m       cd [ -qsLP ] {+|-}n[0m
[38;2;248;248;242m              Change  the  current directory.  In the first form, change the current di‐[0m
[38;2;248;248;242m              rectory to arg, or to the value of $HOME if arg is not specified.  If  arg[0m
[38;2;248;248;242m              is `-', change to the previous directory.[0m

[38;2;248;248;242m              Otherwise,  if arg begins with a slash, attempt to change to the directory[0m
[38;2;248;248;242m              given by arg.[0m

[38;2;248;248;242m              If arg does not begin with a slash, the behaviour depends on  whether  the[0m
[38;2;248;248;242m              current  directory  `.' occurs in the list of directories contained in the[0m
[38;2;248;248;242m              shell parameter cdpath.  If it does not, first attempt to  change  to  the[0m
[38;2;248;248;242m              directory arg under the current directory, and if that fails but cdpath is[0m
[38;2;248;248;242m              set  and  contains at least one element attempt to change to the directory[0m
[38;2;248;248;242m              arg under each component of cdpath in turn until successful.  If  `.'  oc‐[0m
[38;2;248;248;242m              curs  in  cdpath, then cdpath is searched strictly in order so that `.' is[0m
[38;2;248;248;242m              only tried at the appropriate point.[0m

[38;2;248;248;242m              The order of testing cdpath is modified if the option POSIX_CD is set,  as[0m
[38;2;248;248;242m              described in the documentation for the option.[0m

[38;2;248;248;242m              If  no  directory is found, the option CDABLE_VARS is set, and a parameter[0m
[38;2;248;248;242m              named arg exists whose value begins with a slash, treat its value  as  the[0m
[38;2;248;248;242m              directory.   In  that  case, the parameter is added to the named directory[0m
[38;2;248;248;242m              hash table.[0m

[38;2;248;248;242m              The second form of cd substitutes the string new for the string old in the[0m
[38;2;248;248;242m              name of the current directory, and tries to change to this new directory.[0m

[38;2;248;248;242m              The third form of cd extracts an  entry  from  the  directory  stack,  and[0m
[38;2;248;248;242m              changes  to  that  directory.   An  argument of the form `+n' identifies a[0m
[38;2;248;248;242m              stack entry by counting from the left of the list shown by the  dirs  com‐[0m
[38;2;248;248;242m              mand,  starting  with  zero.  An argument of the form `-n' counts from the[0m
[38;2;248;248;242m              right.  If the PUSHD_MINUS option is set, the meanings of `+' and  `-'  in[0m
[38;2;248;248;242m              this  context are swapped.  If the POSIX_CD option is set, this form of cd[0m
[38;2;248;248;242m              is not recognised and will be interpreted as the first form.[0m

[38;2;248;248;242m              If the -q (quiet) option is specified, the hook  function  chpwd  and  the[0m
[38;2;248;248;242m              functions in the array chpwd_functions are not called.  This is useful for[0m
[38;2;248;248;242m              calls  to  cd  that  do  not change the environment seen by an interactive[0m
[38;2;248;248;242m              user.[0m

[38;2;248;248;242m              If the -s option is specified, cd refuses to change the current  directory[0m
[38;2;248;248;242m              if the given pathname contains symlinks.  If the -P option is given or the[0m
[38;2;248;248;242m              CHASE_LINKS  option is set, symbolic links are resolved to their true val‐[0m
[38;2;248;248;242m              ues.  If the -L option is given symbolic links are retained in the  direc‐[0m
[38;2;248;248;242m              tory (and not resolved) regardless of the state of the CHASE_LINKS option.[0m

[38;2;248;248;242m       chdir  Same as cd.[0m

[38;2;248;248;242m       disable [ -afmprs ] name ...[0m
[38;2;248;248;242m              Temporarily disable the named hash table elements or  patterns.   The  de‐[0m
[38;2;248;248;242m              fault  is to disable builtin commands.  This allows you to use an external[0m
[38;2;248;248;242m              command with the same name as a builtin command.   The  -a  option  causes[0m
[38;2;248;248;242m              disable to act on regular or global aliases.  The -s option causes disable[0m
[38;2;248;248;242m              to  act  on  suffix aliases.  The -f option causes disable to act on shell[0m
[38;2;248;248;242m              functions.  The -r options causes disable to act on reserved words.  With‐[0m
[38;2;248;248;242m              out arguments all disabled hash table elements from the corresponding hash[0m
[38;2;248;248;242m              table are printed.  With the -m flag the arguments are taken  as  patterns[0m
[38;2;248;248;242m              (which  should  be  quoted to prevent them from undergoing filename expan‐[0m
[38;2;248;248;242m              sion), and all hash table  elements  from  the  corresponding  hash  table[0m
[38;2;248;248;242m              matching  these  patterns  are  disabled.  Disabled objects can be enabled[0m
[38;2;248;248;242m              with the enable command.[0m

[38;2;248;248;242m              With the option -p, name ... refer to elements of the shell's pattern syn‐[0m
[38;2;248;248;242m              tax as described in the section `Filename Generation'.   Certain  elements[0m
[38;2;248;248;242m              can be disabled separately, as given below.[0m

[38;2;248;248;242m              Note that patterns not allowed by the current settings for the options EX‐[0m
[38;2;248;248;242m              TENDED_GLOB,  KSH_GLOB  and  SH_GLOB  are never enabled, regardless of the[0m
[38;2;248;248;242m              setting here.  For example, if EXTENDED_GLOB is not active, the pattern  ^[0m
[38;2;248;248;242m              is ineffective even if `disable -p "^"' has not been issued.  The list be‐[0m
[38;2;248;248;242m              low  indicates  any  option settings that restrict the use of the pattern.[0m
[38;2;248;248;242m              It should be noted that setting SH_GLOB has a  wider  effect  than  merely[0m
[38;2;248;248;242m              disabling  patterns  as certain expressions, in particular those involving[0m
[38;2;248;248;242m              parentheses, are parsed differently.[0m

[38;2;248;248;242m              The following patterns may be disabled; all the strings  need  quoting  on[0m
[38;2;248;248;242m              the  command  line  to  prevent them from being interpreted immediately as[0m
[38;2;248;248;242m              patterns and the patterns are shown below in single quotes as a reminder.[0m

[38;2;248;248;242m              '?'    The pattern character ? wherever it occurs, including when  preced‐[0m
[38;2;248;248;242m                     ing a parenthesis with KSH_GLOB.[0m

[38;2;248;248;242m              '*'    The  pattern  character  *  wherever it occurs, including recursive[0m
[38;2;248;248;242m                     globbing and when preceding a parenthesis with KSH_GLOB.[0m

[38;2;248;248;242m              '['    Character classes.[0m

[38;2;248;248;242m              '<' (NO_SH_GLOB)[0m
[38;2;248;248;242m                     Numeric ranges.[0m

[38;2;248;248;242m              '|' (NO_SH_GLOB)[0m
[38;2;248;248;242m                     Alternation in  grouped  patterns,  case  statements,  or  KSH_GLOB[0m
[38;2;248;248;242m                     parenthesised expressions.[0m

[38;2;248;248;242m              '(' (NO_SH_GLOB)[0m
[38;2;248;248;242m                     Grouping using single parentheses.  Disabling this does not disable[0m
[38;2;248;248;242m                     the  use of parentheses for KSH_GLOB where they are introduced by a[0m
[38;2;248;248;242m                     special  character,  nor   for   glob   qualifiers   (use   `setopt[0m
[38;2;248;248;242m                     NO_BARE_GLOB_QUAL'  to disable glob qualifiers that use parentheses[0m
[38;2;248;248;242m                     only).[0m

[38;2;248;248;242m              '~' (EXTENDED_GLOB)[0m
[38;2;248;248;242m                     Exclusion in the form A~B.[0m

[38;2;248;248;242m              '^' (EXTENDED_GLOB)[0m
[38;2;248;248;242m                     Exclusion in the form A^B.[0m

[38;2;248;248;242m              '#' (EXTENDED_GLOB)[0m
[38;2;248;248;242m                     The pattern character # wherever it occurs, both for repetition  of[0m
[38;2;248;248;242m                     a previous pattern and for indicating globbing flags.[0m

[38;2;248;248;242m              '?(' (KSH_GLOB)[0m
[38;2;248;248;242m                     The  grouping  form  ?(...).   Note this is also disabled if '?' is[0m
[38;2;248;248;242m                     disabled.[0m

[38;2;248;248;242m              '*(' (KSH_GLOB)[0m
[38;2;248;248;242m                     The grouping form *(...).  Note this is also  disabled  if  '*'  is[0m
[38;2;248;248;242m                     disabled.[0m

[38;2;248;248;242m              '+(' (KSH_GLOB)[0m
[38;2;248;248;242m                     The grouping form +(...).[0m

[38;2;248;248;242m              '!(' (KSH_GLOB)[0m
[38;2;248;248;242m                     The grouping form !(...).[0m

[38;2;248;248;242m              '@(' (KSH_GLOB)[0m
[38;2;248;248;242m                     The grouping form @(...).[0m

[38;2;248;248;242m       disown [ job ... ][0m
[38;2;248;248;242m       job ... &|[0m
[38;2;248;248;242m       job ... &![0m
[38;2;248;248;242m              Remove the specified jobs from the job table; the shell will no longer re‐[0m
[38;2;248;248;242m              port their status, and will not complain if you try to exit an interactive[0m
[38;2;248;248;242m              shell  with  them  running or stopped.  If no job is specified, disown the[0m
[38;2;248;248;242m              current job.[0m

[38;2;248;248;242m              If the jobs are currently stopped and the AUTO_CONTINUE option is not set,[0m
[38;2;248;248;242m              a warning is printed containing information about how to make them running[0m
[38;2;248;248;242m              after they have been disowned.  If one of the latter two  forms  is  used,[0m
[38;2;248;248;242m              the jobs will automatically be made running, independent of the setting of[0m
[38;2;248;248;242m              the AUTO_CONTINUE option.[0m

[38;2;248;248;242m       echo [ -neE ] [ arg ... ][0m
[38;2;248;248;242m              Write  each  arg on the standard output, with a space separating each one.[0m
[38;2;248;248;242m              If the -n flag is not present, print a newline at the  end.   echo  recog‐[0m
[38;2;248;248;242m              nizes the following escape sequences:[0m

[38;2;248;248;242m              \a     bell character[0m
[38;2;248;248;242m              \b     backspace[0m
[38;2;248;248;242m              \c     suppress subsequent characters and final newline[0m
[38;2;248;248;242m              \e     escape[0m
[38;2;248;248;242m              \f     form feed[0m
[38;2;248;248;242m              \n     linefeed (newline)[0m
[38;2;248;248;242m              \r     carriage return[0m
[38;2;248;248;242m              \t     horizontal tab[0m
[38;2;248;248;242m              \v     vertical tab[0m
[38;2;248;248;242m              \\     backslash[0m
[38;2;248;248;242m              \0NNN  character code in octal[0m
[38;2;248;248;242m              \xNN   character code in hexadecimal[0m
[38;2;248;248;242m              \uNNNN unicode character code in hexadecimal[0m
[38;2;248;248;242m              \UNNNNNNNN[0m
[38;2;248;248;242m                     unicode character code in hexadecimal[0m

[38;2;248;248;242m              The  -E  flag, or the BSD_ECHO option, can be used to disable these escape[0m
[38;2;248;248;242m              sequences.  In the latter case, -e flag can be used to enable them.[0m

[38;2;248;248;242m              Note that for standards compliance a double dash does not terminate option[0m
[38;2;248;248;242m              processing; instead, it is printed directly.  However, a single dash  does[0m
[38;2;248;248;242m              terminate  option  processing,  so  the first dash, possibly following op‐[0m
[38;2;248;248;242m              tions, is not printed, but everything following it is printed as an  argu‐[0m
[38;2;248;248;242m              ment.   The  single  dash behaviour is different from other shells.  For a[0m
[38;2;248;248;242m              more portable way of printing text, see printf, and for  a  more  control‐[0m
[38;2;248;248;242m              lable way of printing text within zsh, see print.[0m

[38;2;248;248;242m       echotc See the section `The zsh/termcap Module' in zshmodules(1).[0m

[38;2;248;248;242m       echoti See the section `The zsh/terminfo Module' in zshmodules(1).[0m

[38;2;248;248;242m       emulate [ -lLR ] [ {zsh|sh|ksh|csh} [ flags ... ] ][0m
[38;2;248;248;242m              Without any argument print current emulation mode.[0m

[38;2;248;248;242m              With  single argument set up zsh options to emulate the specified shell as[0m
[38;2;248;248;242m              much as possible.  csh will never be fully emulated.  If the  argument  is[0m
[38;2;248;248;242m              not  one  of  the shells listed above, zsh will be used as a default; more[0m
[38;2;248;248;242m              precisely, the tests performed on the argument are the same as those  used[0m
[38;2;248;248;242m              to  determine  the  emulation  at startup based on the shell name, see the[0m
[38;2;248;248;242m              section COMPATIBILITY in zsh(1) .  In addition to setting  shell  options,[0m
[38;2;248;248;242m              the command also restores the pristine state of pattern enables, as if all[0m
[38;2;248;248;242m              patterns had been enabled using enable -p.[0m

[38;2;248;248;242m              If  the  emulate command occurs inside a function that has been marked for[0m
[38;2;248;248;242m              execution tracing with functions -t then the xtrace option will be  turned[0m
[38;2;248;248;242m              on regardless of emulation mode or other options.  Note that code executed[0m
[38;2;248;248;242m              inside  the  function by the ., source, or eval commands is not considered[0m
[38;2;248;248;242m              to be running directly from the function, hence does not provoke this  be‐[0m
[38;2;248;248;242m              haviour.[0m

[38;2;248;248;242m              If the -R switch is given, all settable options are reset to their default[0m
[38;2;248;248;242m              value  corresponding  to  the specified emulation mode, except for certain[0m
[38;2;248;248;242m              options describing the interactive environment; otherwise, only those  op‐[0m
[38;2;248;248;242m              tions  likely  to  cause portability problems in scripts and functions are[0m
[38;2;248;248;242m              altered.  If the -L switch is given, the options LOCAL_OPTIONS, LOCAL_PAT‐[0m
[38;2;248;248;242m              TERNS and LOCAL_TRAPS will be set as well, causing the effects of the emu‐[0m
[38;2;248;248;242m              late command and any setopt, disable -p or enable -p, and trap commands to[0m
[38;2;248;248;242m              be local to the immediately surrounding shell function, if  any;  normally[0m
[38;2;248;248;242m              these  options  are  turned  off in all emulation modes except ksh. The -L[0m
[38;2;248;248;242m              switch is mutually exclusive with the use of -c in flags.[0m

[38;2;248;248;242m              If there is a single argument and the -l switch is given, the options that[0m
[38;2;248;248;242m              would be set or unset (the latter indicated  with  the  prefix  `no')  are[0m
[38;2;248;248;242m              listed.  -l can be combined with -L or -R and the list will be modified in[0m
[38;2;248;248;242m              the appropriate way.  Note the list does not depend on the current setting[0m
[38;2;248;248;242m              of options, i.e. it includes all options that may in principle change, not[0m
[38;2;248;248;242m              just those that would actually change.[0m

[38;2;248;248;242m              The flags may be any of the invocation-time flags described in the section[0m
[38;2;248;248;242m              INVOCATION  in zsh(1), except that `-o EMACS' and `-o VI' may not be used.[0m
[38;2;248;248;242m              Flags such as `+r'/`+o RESTRICTED'  may  be  prohibited  in  some  circum‐[0m
[38;2;248;248;242m              stances.[0m

[38;2;248;248;242m              If -c arg appears in flags, arg is evaluated while the requested emulation[0m
[38;2;248;248;242m              is temporarily in effect.  In this case the emulation mode and all options[0m
[38;2;248;248;242m              are  restored  to  their  previous  values before emulate returns.  The -R[0m
[38;2;248;248;242m              switch may precede the name of the shell to emulate; note this has a mean‐[0m
[38;2;248;248;242m              ing distinct from including -R in flags.[0m

[38;2;248;248;242m              Use of -c enables `sticky' emulation mode for functions defined within the[0m
[38;2;248;248;242m              evaluated expression:  the emulation mode is  associated  thereafter  with[0m
[38;2;248;248;242m              the  function so that whenever the function is executed the emulation (re‐[0m
[38;2;248;248;242m              specting the -R switch, if present) and all options are set  (and  pattern[0m
[38;2;248;248;242m              disables  cleared) before entry to the function, and the state is restored[0m
[38;2;248;248;242m              after exit.  If the function is called when the sticky  emulation  is  al‐[0m
[38;2;248;248;242m              ready  in effect, either within an `emulate shell -c' expression or within[0m
[38;2;248;248;242m              another function with the same sticky emulation, entry and exit  from  the[0m
[38;2;248;248;242m              function  do  not cause options to be altered (except due to standard pro‐[0m
[38;2;248;248;242m              cessing such as the LOCAL_OPTIONS option).  This also applies to functions[0m
[38;2;248;248;242m              marked for autoload within the sticky emulation; the  appropriate  set  of[0m
[38;2;248;248;242m              options  will  be  applied  at the point the function is loaded as well as[0m
[38;2;248;248;242m              when it is run.[0m

[38;2;248;248;242m              For example:[0m

[38;2;248;248;242m                     emulate sh -c 'fni() { setopt cshnullglob; }[0m
[38;2;248;248;242m                     fno() { fni; }'[0m
[38;2;248;248;242m                     fno[0m

[38;2;248;248;242m              The two functions fni and fno are defined with sticky sh  emulation.   fno[0m
[38;2;248;248;242m              is  then executed, causing options associated with emulations to be set to[0m
[38;2;248;248;242m              their values in sh.  fno then calls fni; because fni is  also  marked  for[0m
[38;2;248;248;242m              sticky sh emulation, no option changes take place on entry to or exit from[0m
[38;2;248;248;242m              it.   Hence  the  option  cshnullglob, turned off by sh emulation, will be[0m
[38;2;248;248;242m              turned on within fni and remain on return to fno.  On exit from  fno,  the[0m
[38;2;248;248;242m              emulation  mode and all options will be restored to the state they were in[0m
[38;2;248;248;242m              before entry to the temporary emulation.[0m

[38;2;248;248;242m              The documentation above is typically sufficient for the  intended  purpose[0m
[38;2;248;248;242m              of  executing  code  designed  for other shells in a suitable environment.[0m
[38;2;248;248;242m              More detailed rules follow.[0m
[38;2;248;248;242m              1.     The sticky emulation environment provided by `emulate shell -c'  is[0m
[38;2;248;248;242m                     identical to that provided by entry to a function marked for sticky[0m
[38;2;248;248;242m                     emulation as a consequence of being defined in such an environment.[0m
[38;2;248;248;242m                     Hence,  for  example, the sticky emulation is inherited by subfunc‐[0m
[38;2;248;248;242m                     tions defined within functions with sticky emulation.[0m
[38;2;248;248;242m              2.     No change of options takes place on entry to or exit from functions[0m
[38;2;248;248;242m                     that are not marked for sticky emulation,  other  than  those  that[0m
[38;2;248;248;242m                     would  normally  take  place,  even  if  those functions are called[0m
[38;2;248;248;242m                     within sticky emulation.[0m
[38;2;248;248;242m              3.     No special handling is provided for functions marked  for  autoload[0m
[38;2;248;248;242m                     nor  for functions present in wordcode created by the zcompile com‐[0m
[38;2;248;248;242m                     mand.[0m
[38;2;248;248;242m              4.     The presence or absence of the -R switch to emulate corresponds  to[0m
[38;2;248;248;242m                     different  sticky  emulation modes, so for example `emulate sh -c',[0m
[38;2;248;248;242m                     `emulate -R sh -c' and `emulate csh -c' are treated as  three  dis‐[0m
[38;2;248;248;242m                     tinct sticky emulations.[0m
[38;2;248;248;242m              5.     Difference  in shell options supplied in addition to the basic emu‐[0m
[38;2;248;248;242m                     lation also mean the sticky emulations are different, so for  exam‐[0m
[38;2;248;248;242m                     ple  `emulate zsh -c' and `emulate zsh -o cbases -c' are treated as[0m
[38;2;248;248;242m                     distinct sticky emulations.[0m

[38;2;248;248;242m       enable [ -afmprs ] name ...[0m
[38;2;248;248;242m              Enable the named hash table elements,  presumably  disabled  earlier  with[0m
[38;2;248;248;242m              disable.  The default is to enable builtin commands.  The -a option causes[0m
[38;2;248;248;242m              enable  to  act on regular or global aliases.  The -s option causes enable[0m
[38;2;248;248;242m              to act on suffix aliases.  The -f option causes enable  to  act  on  shell[0m
[38;2;248;248;242m              functions.  The -r option causes enable to act on reserved words.  Without[0m
[38;2;248;248;242m              arguments  all enabled hash table elements from the corresponding hash ta‐[0m

[38;2;248;248;242m              table  matching  these  patterns are enabled.  Enabled objects can be dis‐[0m
[38;2;248;248;242m              abled with the disable builtin command.[0m

[38;2;248;248;242m       eval [ arg ... ][0m

[38;2;248;248;242m              With  -c clear the environment; with -l prepend - to the argv[0] string of[0m

[38;2;248;248;242m              If  the  option  POSIX_BUILTINS  is set, command is never interpreted as a[0m

[38;2;248;248;242m       exit [ n ][0m

[38;2;248;248;242m              See  notes  at the end of the section JOBS in zshmisc(1) for some possibly[0m

[38;2;248;248;242m       export [ name[=value] ... ][0m
[38;2;248;248;242m              The specified names are marked for automatic export to the environment  of[0m

[38;2;248;248;242m              ter specified does not already exist, it is created in the global scope.[0m

[38;2;248;248;242m       false [ arg ... ][0m
[38;2;248;248;242m              Do nothing and return an exit status of 1.[0m

[38;2;248;248;242m       fc [ -e ename ] [ -s ] [ -LI ] [ -m match ] [ old=new ... ] [ first [ last ] ][0m
[38;2;248;248;242m       fc -l [ -LI ] [ -nrdfEiD ] [ -t timefmt ] [ -m match ][0m
[38;2;248;248;242m             [ old=new ... ] [ first [ last ] ][0m
[38;2;248;248;242m       fc -p [ -a ] [ filename [ histsize [ savehistsize ] ] ][0m
[38;2;248;248;242m       fc -P[0m
[38;2;248;248;242m       fc -ARWI [ filename ][0m
[38;2;248;248;242m              The fc command controls the  interactive  history  mechanism.   Note  that[0m
[38;2;248;248;242m              reading  and  writing of history options is only performed if the shell is[0m
[38;2;248;248;242m              interactive.  Usually this is detected automatically, but it can be forced[0m
[38;2;248;248;242m              by setting the interactive option when starting the shell.[0m

[38;2;248;248;242m              The first two forms of this command select a range of events from first to[0m
[38;2;248;248;242m              last from the history list.  The arguments first and last may be specified[0m
[38;2;248;248;242m              as a number or as a string.  A negative number is used as an offset to the[0m
[38;2;248;248;242m              current history event number.  A string specifies the  most  recent  event[0m
[38;2;248;248;242m              beginning  with  the given string.  All substitutions old=new, if any, are[0m
[38;2;248;248;242m              then performed on the text of the events.[0m

[38;2;248;248;242m              The range of events selected by numbers can be  narrowed  further  by  the[0m
[38;2;248;248;242m              following flags.[0m
[38;2;248;248;242m              -I     restricts to only internal events (not from $HISTFILE)[0m
[38;2;248;248;242m              -L     restricts  to  only  local  events  (not  from  other  shells,  see[0m
[38;2;248;248;242m                     SHARE_HISTORY in zshoptions(1) -- note that $HISTFILE is considered[0m
[38;2;248;248;242m                     local when read at startup)[0m
[38;2;248;248;242m              -m     takes the first argument as a pattern (which should be quoted)  and[0m
[38;2;248;248;242m                     only the history events matching this pattern are considered[0m

[38;2;248;248;242m              If  first  is not specified, it will be set to -1 (the most recent event),[0m
[38;2;248;248;242m              or to -16 if the -l flag is given.  If last is not specified, it  will  be[0m
[38;2;248;248;242m              set  to  first, or to -1 if the -l flag is given.  However, if the current[0m
[38;2;248;248;242m              event has added entries to the history with `print -s' or  `fc  -R',  then[0m
[38;2;248;248;242m              the default last for -l includes all new history entries since the current[0m
[38;2;248;248;242m              event began.[0m

[38;2;248;248;242m              When  the  -l  flag  is given, the resulting events are listed on standard[0m
[38;2;248;248;242m              output.  Otherwise the editor program specified by -e ename is invoked  on[0m
[38;2;248;248;242m              a  file containing these history events.  If -e is not given, the value of[0m
[38;2;248;248;242m              the parameter FCEDIT is used; if that is not set the value of the  parame‐[0m
[38;2;248;248;242m              ter  EDITOR is used; if that is not set a builtin default, usually `vi' is[0m
[38;2;248;248;242m              used.  If ename is `-', no editor is invoked.  When editing  is  complete,[0m
[38;2;248;248;242m              the edited command is executed.[0m

[38;2;248;248;242m              The  flag `-s' is equivalent to `-e -'.  The flag -r reverses the order of[0m
[38;2;248;248;242m              the events and the flag -n suppresses event numbers when listing.[0m

[38;2;248;248;242m              Also when listing,[0m
[38;2;248;248;242m              -d     prints timestamps for each event[0m
[38;2;248;248;242m              -f     prints full time-date stamps in the US `MM/DD/YY hh:mm' format[0m
[38;2;248;248;242m              -E     prints full time-date stamps in  the  European  `dd.mm.yyyy  hh:mm'[0m
[38;2;248;248;242m                     format[0m
[38;2;248;248;242m              -i     prints full time-date stamps in ISO8601 `yyyy-mm-dd hh:mm' format[0m
[38;2;248;248;242m              -t fmt prints  time  and date stamps in the given format; fmt is formatted[0m
[38;2;248;248;242m                     with the strftime function with the zsh  extensions  described  for[0m
[38;2;248;248;242m                     the %D{string} prompt format in the section EXPANSION OF PROMPT SE‐[0m
[38;2;248;248;242m                     QUENCES  in  zshmisc(1).  The resulting formatted string must be no[0m
[38;2;248;248;242m                     more than 256 characters or will not be printed[0m
[38;2;248;248;242m              -D     prints elapsed times; may be combined with one of the options above[0m

[38;2;248;248;242m              `fc -p' pushes the current history list onto a stack and switches to a new[0m
[38;2;248;248;242m              history list.  If the -a option is also specified, this history list  will[0m
[38;2;248;248;242m              be  automatically  popped when the current function scope is exited, which[0m
[38;2;248;248;242m              is a much better solution than creating a trap function to  call  `fc  -P'[0m
[38;2;248;248;242m              manually.   If no arguments are specified, the history list is left empty,[0m
[38;2;248;248;242m              $HISTFILE is unset, and $HISTSIZE & $SAVEHIST are  set  to  their  default[0m
[38;2;248;248;242m              values.   If  one  argument  is  given, $HISTFILE is set to that filename,[0m
[38;2;248;248;242m              $HISTSIZE & $SAVEHIST are left unchanged, and the history file is read  in[0m
[38;2;248;248;242m              (if it exists) to initialize the new list.  If a second argument is speci‐[0m
[38;2;248;248;242m              fied,  $HISTSIZE  &  $SAVEHIST are instead set to the single specified nu‐[0m
[38;2;248;248;242m              meric value.  Finally, if a third argument is specified, $SAVEHIST is  set[0m
[38;2;248;248;242m              to a separate value from $HISTSIZE.  You are free to change these environ‐[0m
[38;2;248;248;242m              ment values for the new history list however you desire in order to manip‐[0m
[38;2;248;248;242m              ulate the new history list.[0m

[38;2;248;248;242m              `fc -P' pops the history list back to an older list saved by `fc -p'.  The[0m
[38;2;248;248;242m              current  list  is  saved to its $HISTFILE before it is destroyed (assuming[0m
[38;2;248;248;242m              that $HISTFILE and $SAVEHIST are set appropriately, of course).  The  val‐[0m
[38;2;248;248;242m              ues of $HISTFILE, $HISTSIZE, and $SAVEHIST are restored to the values they[0m
[38;2;248;248;242m              had when `fc -p' was called.  Note that this restoration can conflict with[0m
[38;2;248;248;242m              making  these variables "local", so your best bet is to avoid local decla‐[0m
[38;2;248;248;242m              rations for these variables in functions that use `fc -p'.  The one  other[0m
[38;2;248;248;242m              guaranteed-safe  combination  is  declaring these variables to be local at[0m
[38;2;248;248;242m              the top of your function and using the automatic option (-a) with `fc -p'.[0m
[38;2;248;248;242m              Finally, note that it is legal to manually pop a push marked for automatic[0m
[38;2;248;248;242m              popping if you need to do so before the function exits.[0m

[38;2;248;248;242m              `fc -R' reads the history from the given file, `fc -W' writes the  history[0m
[38;2;248;248;242m              out  to  the  given file, and `fc -A' appends the history out to the given[0m
[38;2;248;248;242m              file.  If no filename is specified, the $HISTFILE is assumed.  If  the  -I[0m
[38;2;248;248;242m              option  is  added  to -R, only those events that are not already contained[0m
[38;2;248;248;242m              within the internal history list are added.  If the -I option is added  to[0m
[38;2;248;248;242m              -A  or  -W,  only  those  events  that  are new since last incremental ap‐[0m
[38;2;248;248;242m              pend/write to the history file are appended/written.   In  any  case,  the[0m
[38;2;248;248;242m              created file will have no more than $SAVEHIST entries.[0m

[38;2;248;248;242m       float [ {+|-}Hghlprtux ] [ {+|-}EFLRZ [ n ] ] [ name[=value] ... ][0m

[38;2;248;248;242m       functions [ {+|-}UkmtTuWz ] [ -x num ] [ name ... ][0m
[38;2;248;248;242m       functions -c oldfn newfn[0m
[38;2;248;248;242m       functions -M [-s] mathfn [ min [ max [ shellfn ] ] ][0m
[38;2;248;248;242m       functions -M [ -m pattern ... ][0m
[38;2;248;248;242m       functions +M [ -m ] mathfn ...[0m
[38;2;248;248;242m              Equivalent to typeset -f, with the exception of the -c, -x, -M and -W  op‐[0m
[38;2;248;248;242m              tions.   For  functions  -u and functions -U, see autoload, which provides[0m
[38;2;248;248;242m              additional options.  For functions -t and functions -T, see typeset -f.[0m

[38;2;248;248;242m              The -x option indicates that any functions output will have  each  leading[0m
[38;2;248;248;242m              tab  for  indentation, added by the shell to show syntactic structure, ex‐[0m
[38;2;248;248;242m              panded to the given number num of spaces.  num can also be 0  to  suppress[0m
[38;2;248;248;242m              all indentation.[0m

[38;2;248;248;242m              The  -W  option turns on the option WARN_NESTED_VAR for the named function[0m
[38;2;248;248;242m              or functions only.  The option is turned off at the start of nested  func‐[0m
[38;2;248;248;242m              tions  (apart  from  anonoymous functions) unless the called function also[0m
[38;2;248;248;242m              has the -W attribute.[0m

[38;2;248;248;242m              The -c option causes oldfn to be copied to newfn.  The copy is efficiently[0m
[38;2;248;248;242m              handled internally by reference counting.  If oldfn  was  marked  for  au‐[0m
[38;2;248;248;242m              toload  it is first loaded and if this fails the copy fails.  Either func‐[0m
[38;2;248;248;242m              tion may subsequently be redefined without affecting the other.  A typical[0m
[38;2;248;248;242m              idiom is that oldfn is the name of a library shell function which is  then[0m
[38;2;248;248;242m              redefined  to  call  newfn,  thereby  installing a modified version of the[0m
[38;2;248;248;242m              function.[0m

[38;2;248;248;242m              The -M and +M flags[0m

[38;2;248;248;242m              Use of the -M option may not be combined with any of the  options  handled[0m
[38;2;248;248;242m              by typeset -f.[0m

[38;2;248;248;242m              functions  -M mathfn defines mathfn as the name of a mathematical function[0m
[38;2;248;248;242m              recognised in all forms  of  arithmetical  expressions;  see  the  section[0m
[38;2;248;248;242m              `Arithmetic  Evaluation'  in  zshmisc(1).   By default mathfn may take any[0m
[38;2;248;248;242m              number of comma-separated arguments.  If min is given, it  must  have  ex‐[0m
[38;2;248;248;242m              actly  min  args; if min and max are both given, it must have at least min[0m
[38;2;248;248;242m              and at most max args.  max may be -1 to indicate that there  is  no  upper[0m
[38;2;248;248;242m              limit.[0m

[38;2;248;248;242m              By  default  the  function  is implemented by a shell function of the same[0m
[38;2;248;248;242m              name; if shellfn is specified it gives the name of the corresponding shell[0m
[38;2;248;248;242m              function while mathfn remains the name used in  arithmetical  expressions.[0m
[38;2;248;248;242m              The  name of the function in $0 is mathfn (not shellfn as would usually be[0m
[38;2;248;248;242m              the case), provided the option FUNCTION_ARGZERO is in effect.   The  posi‐[0m
[38;2;248;248;242m              tional parameters in the shell function correspond to the arguments of the[0m
[38;2;248;248;242m              mathematical function call.[0m

[38;2;248;248;242m              The  result of the last arithmetical expression evaluated inside the shell[0m
[38;2;248;248;242m              function gives the result of the mathematical function.  This is not  lim‐[0m
[38;2;248;248;242m              ited  to  arithmetic substitutions of the form $((...)), but also includes[0m
[38;2;248;248;242m              arithmetical expressions evaluated in any other way, including by the  let[0m
[38;2;248;248;242m              builtin,  by ((...)) statements, and even by the return builtin and by ar‐[0m
[38;2;248;248;242m              ray subscripts.  Therefore, care must be taken not to use syntactical con‐[0m
[38;2;248;248;242m              structs that perform arithmetic evaluation after evaluating what is to  be[0m
[38;2;248;248;242m              the result of the function.  For example:[0m

[38;2;248;248;242m                     # WRONG[0m
[38;2;248;248;242m                     zmath_cube() {[0m
[38;2;248;248;242m                       (( $1 * $1 * $1 ))[0m
[38;2;248;248;242m                       return 0[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     functions -M cube 1 1 zmath_cube[0m
[38;2;248;248;242m                     print $(( cube(3) ))[0m

[38;2;248;248;242m              This will print `0' because of the return.[0m

[38;2;248;248;242m              Commenting  the  return out would lead to a different problem: the ((...))[0m
[38;2;248;248;242m              statement would become the last statement in the function, so  the  return[0m
[38;2;248;248;242m              status  ($?)  of the function would be non-zero (indicating failure) when‐[0m
[38;2;248;248;242m              ever the arithmetic result of the function would happen to be zero (numer‐[0m
[38;2;248;248;242m              ically):[0m

[38;2;248;248;242m                     # WRONG[0m
[38;2;248;248;242m                     zmath_cube() {[0m
[38;2;248;248;242m                       (( $1 * $1 * $1 ))[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     functions -M cube 1 1 zmath_cube[0m
[38;2;248;248;242m                     print $(( cube(0) ))[0m

[38;2;248;248;242m              Instead, the true builtin can be used:[0m

[38;2;248;248;242m                     # RIGHT[0m
[38;2;248;248;242m                     zmath_cube() {[0m
[38;2;248;248;242m                       (( $1 * $1 * $1 ))[0m
[38;2;248;248;242m                       true[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     functions -M cube 1 1 zmath_cube[0m
[38;2;248;248;242m                     print $(( cube(3) ))[0m

[38;2;248;248;242m              If the additional option -s is given to functions -M, the argument to  the[0m
[38;2;248;248;242m              function  is  a  single  string: anything between the opening and matching[0m
[38;2;248;248;242m              closing parenthesis is passed to the function as a single  argument,  even[0m
[38;2;248;248;242m              if  it  includes  commas or white space.  The minimum and maximum argument[0m
[38;2;248;248;242m              specifiers must therefore be 1 if given.  An empty argument list is passed[0m
[38;2;248;248;242m              as a zero-length string.  Thus, the following string function takes a sin‐[0m
[38;2;248;248;242m              gle argument, including the commas, and prints 11:[0m

[38;2;248;248;242m                     stringfn() { (( $#1 )); true }[0m
[38;2;248;248;242m                     functions -Ms stringfn[0m
[38;2;248;248;242m                     print $(( stringfn(foo,bar,rod) ))[0m

[38;2;248;248;242m              functions -M with no arguments lists all such  user-defined  functions  in[0m
[38;2;248;248;242m              the  same  form as a definition.  With the additional option -m and a list[0m
[38;2;248;248;242m              of arguments, all functions whose mathfn matches one of the pattern  argu‐[0m
[38;2;248;248;242m              ments are listed.[0m

[38;2;248;248;242m              function  +M  removes  the  list of mathematical functions; with the addi‐[0m
[38;2;248;248;242m              tional option -m the arguments are treated as patterns and  all  functions[0m
[38;2;248;248;242m              whose  mathfn  matches the pattern are removed.  Note that the shell func‐[0m
[38;2;248;248;242m              tion implementing the behaviour is not removed (regardless of whether  its[0m
[38;2;248;248;242m              name coincides with mathfn).[0m

[38;2;248;248;242m       getopts optstring name [ arg ... ][0m
[38;2;248;248;242m              Checks the args for legal options.  If the args are omitted, use the posi‐[0m
[38;2;248;248;242m              tional  parameters.   A  valid option argument begins with a `+' or a `-'.[0m
[38;2;248;248;242m              An argument not beginning with a `+' or a `-', or the argument `--',  ends[0m
[38;2;248;248;242m              the  options.  Note that a single `-' is not considered a valid option ar‐[0m
[38;2;248;248;242m              gument.  optstring contains the letters that  getopts  recognizes.   If  a[0m
[38;2;248;248;242m              letter  is  followed  by a `:', that option requires an argument.  The op‐[0m
[38;2;248;248;242m              tions can be separated from the argument by blanks.[0m

[38;2;248;248;242m              Each time it is invoked, getopts places the option letter it finds in  the[0m
[38;2;248;248;242m              shell  parameter  name,  prepended  with a `+' when arg begins with a `+'.[0m
[38;2;248;248;242m              The index of the next arg is stored in OPTIND.  The  option  argument,  if[0m
[38;2;248;248;242m              any, is stored in OPTARG.[0m

[38;2;248;248;242m              The  first option to be examined may be changed by explicitly assigning to[0m

[38;2;248;248;242m              A  leading  `:' in optstring causes getopts to store the letter of any in‐[0m

[38;2;248;248;242m       hash [ -Ldfmrv ] [ name[=value] ] ...[0m

[38;2;248;248;242m              Given no arguments, and neither the -r or -f options,  the  selected  hash[0m

[38;2;248;248;242m              The  -r  option  causes the selected hash table to be emptied.  It will be[0m
[38;2;248;248;242m              subsequently rebuilt in the normal fashion.  The -f option causes the  se‐[0m

[38;2;248;248;242m              The -m option causes the arguments to be taken as patterns  (which  should[0m

[38;2;248;248;242m              For  each name with a corresponding value, put `name' in the selected hash[0m

[38;2;248;248;242m              ble,  this  means  that whenever `name' is used as a command argument, the[0m
[38;2;248;248;242m              shell will try to execute the file given by `value'.  In the named  direc‐[0m
[38;2;248;248;242m              tory hash table, this means that `value' may be referred to as `~name'.[0m

[38;2;248;248;242m              For each name with no corresponding value, attempt to add name to the hash[0m
[38;2;248;248;242m              table,  checking  what  the  appropriate value is in the normal manner for[0m

[38;2;248;248;242m              The  -v option causes hash table entries to be listed as they are added by[0m

[38;2;248;248;242m              If the -L flag is present, then each hash table entry is  printed  in  the[0m

[38;2;248;248;242m              Defining a trap under either name causes any  trap  under  an  alternative[0m
[38;2;248;248;242m              name to be removed.  However, it is recommended that for consistency users[0m
[38;2;248;248;242m              stick exclusively to one name or another.[0m

[38;2;248;248;242m       true [ arg ... ][0m
[38;2;248;248;242m              Do nothing and return an exit status of 0.[0m

[38;2;248;248;242m       ttyctl [ -fu ][0m
[38;2;248;248;242m              The -f option freezes the tty (i.e. terminal or terminal emulator), and -u[0m
[38;2;248;248;242m              unfreezes it.  When the tty is frozen, no changes made to the tty settings[0m
[38;2;248;248;242m              by  external  programs will be honored by the shell, except for changes in[0m
[38;2;248;248;242m              the size of the screen; the shell will simply reset the settings to  their[0m
[38;2;248;248;242m              previous values as soon as each command exits or is suspended.  Thus, stty[0m
[38;2;248;248;242m              and  similar programs have no effect when the tty is frozen.  Freezing the[0m
[38;2;248;248;242m              tty does not cause the current state to be remembered: instead, it  causes[0m
[38;2;248;248;242m              future changes to the state to be blocked.[0m

[38;2;248;248;242m              Without options it reports whether the terminal is frozen or not.[0m

[38;2;248;248;242m              Note that, regardless of whether the tty is frozen or not, the shell needs[0m
[38;2;248;248;242m              to  change the settings when the line editor starts, so unfreezing the tty[0m
[38;2;248;248;242m              does not guarantee settings  made  on  the  command  line  are  preserved.[0m
[38;2;248;248;242m              Strings  of  commands run between editing the command line will see a con‐[0m
[38;2;248;248;242m              sistent tty state.  See also the shell variable STTY for a means  of  ini‐[0m
[38;2;248;248;242m              tialising the tty before running external commands and/or freezing the tty[0m
[38;2;248;248;242m              around a single command.[0m

[38;2;248;248;242m       type [ -wfpamsS ] name ...[0m
[38;2;248;248;242m              Equivalent to whence -v.[0m

[38;2;248;248;242m       typeset [ {+|-}AHUaghlmrtux ] [ {+|-}EFLRZip [ n ] ][0m
[38;2;248;248;242m               [ + ] [ name[=value] ... ][0m
[38;2;248;248;242m       typeset -T [ {+|-}Uglrux ] [ {+|-}LRZp [ n ] ][0m
[38;2;248;248;242m               [ + | SCALAR[=value] array[=(value ...)] [ sep ] ][0m
[38;2;248;248;242m       typeset -f [ {+|-}TUkmtuz ] [ + ] [ name ... ][0m
[38;2;248;248;242m              Set or display attributes and values for shell parameters.[0m

[38;2;248;248;242m              Except  as noted below for control flags that change the behavior, a para‐[0m
[38;2;248;248;242m              meter is created for each name that does not already refer to  one.   When[0m
[38;2;248;248;242m              inside  a  function, a new parameter is created for every name (even those[0m
[38;2;248;248;242m              that already exist), and is unset again when the function completes.   See[0m
[38;2;248;248;242m              `Local  Parameters' in zshparam(1).  The same rules apply to special shell[0m
[38;2;248;248;242m              parameters, which retain their special attributes when made local.[0m

[38;2;248;248;242m              For each name=value assignment, the parameter name is set  to  value.   If[0m
[38;2;248;248;242m              the  assignment  is omitted and name does not refer to an existing parame‐[0m
[38;2;248;248;242m              ter, a new parameter is intialized to empty string, zero, or  empty  array[0m
[38;2;248;248;242m              (as  appropriate),  unless the shell option TYPESET_TO_UNSET is set.  When[0m
[38;2;248;248;242m              that option is set, the parameter attributes are recorded but the  parame‐[0m
[38;2;248;248;242m              ter remains unset.[0m

[38;2;248;248;242m              If  the  shell  option  TYPESET_SILENT is not set, for each remaining name[0m
[38;2;248;248;242m              that refers to a parameter that is already set, the name and value of  the[0m
[38;2;248;248;242m              parameter  are  printed  in the form of an assignment.  Nothing is printed[0m
[38;2;248;248;242m              for newly-created parameters, or when any attribute flags listed below are[0m
[38;2;248;248;242m              given along with the name.  Using `+' instead of minus to introduce an at‐[0m
[38;2;248;248;242m              tribute turns it off.[0m

[38;2;248;248;242m              If no name is present, the names and values of all parameters are printed.[0m
[38;2;248;248;242m              In this case the attribute flags restrict the display to only those  para‐[0m
[38;2;248;248;242m              meters  that  have the specified attributes, and using `+' rather than `-'[0m
[38;2;248;248;242m              to introduce the flag suppresses printing of the values of parameters when[0m
[38;2;248;248;242m              there is no parameter name.[0m

[38;2;248;248;242m              All forms of the command handle scalar assignment.   Array  assignment  is[0m
[38;2;248;248;242m              possible if any of the reserved words declare, export, float, integer, lo‐[0m
[38;2;248;248;242m              cal, readonly or typeset is matched when the line is parsed (N.B. not when[0m
[38;2;248;248;242m              it  is  executed).   In this case the arguments are parsed as assignments,[0m
[38;2;248;248;242m              except that the `+=' syntax and the GLOB_ASSIGN option are not  supported,[0m
[38;2;248;248;242m              and  scalar  values  after = are not split further into words, even if ex‐[0m
[38;2;248;248;242m              panded (regardless of the setting of the KSH_TYPESET option;  this  option[0m
[38;2;248;248;242m              is obsolete).[0m

[38;2;248;248;242m              Examples of the differences between command and reserved word parsing:[0m

[38;2;248;248;242m                     # Reserved word parsing[0m
[38;2;248;248;242m                     typeset svar=$(echo one word) avar=(several words)[0m

[38;2;248;248;242m              The  above  creates a scalar parameter svar and an array parameter avar as[0m
[38;2;248;248;242m              if the assignments had been[0m

[38;2;248;248;242m                     svar="one word"[0m
[38;2;248;248;242m                     avar=(several words)[0m

[38;2;248;248;242m              On the other hand:[0m

[38;2;248;248;242m                     # Normal builtin interface[0m
[38;2;248;248;242m                     builtin typeset svar=$(echo two words)[0m

[38;2;248;248;242m              The builtin keyword causes the above to use the standard builtin interface[0m
[38;2;248;248;242m              to typeset in which argument parsing is performed in the same way  as  for[0m
[38;2;248;248;242m              other  commands.   This example creates a scalar svar containing the value[0m
[38;2;248;248;242m              two and another scalar parameter words with no value.  An array  value  in[0m
[38;2;248;248;242m              this  case  would either cause an error or be treated as an obscure set of[0m
[38;2;248;248;242m              glob qualifiers.[0m

[38;2;248;248;242m              Arbitrary arguments are allowed if they take the form of assignments after[0m
[38;2;248;248;242m              command line expansion; however, these only perform scalar assignment:[0m

[38;2;248;248;242m                     var='svar=val'[0m
[38;2;248;248;242m                     typeset $var[0m

[38;2;248;248;242m              The above sets the scalar parameter svar to the  value  val.   Parentheses[0m
[38;2;248;248;242m              around  the value within var would not cause array assignment as they will[0m
[38;2;248;248;242m              be treated as ordinary characters when $var is substituted.  Any non-triv‐[0m
[38;2;248;248;242m              ial expansion in the name part of the assignment causes the argument to be[0m
[38;2;248;248;242m              treated in this fashion:[0m

[38;2;248;248;242m                     typeset {var1,var2,var3}=name[0m

[38;2;248;248;242m              The above syntax is valid, and has the  expected  effect  of  setting  the[0m
[38;2;248;248;242m              three  parameters  to  the same value, but the command line is parsed as a[0m
[38;2;248;248;242m              set of three normal command line arguments  to  typeset  after  expansion.[0m
[38;2;248;248;242m              Hence it is not possible to assign to multiple arrays by this means.[0m

[38;2;248;248;242m              Note  that  each  interface  to  any of the commands may be disabled sepa‐[0m
[38;2;248;248;242m              rately.  For example, `disable -r typeset' disables the reserved word  in‐[0m
[38;2;248;248;242m              terface  to  typeset, exposing the builtin interface, while `disable type‐[0m
[38;2;248;248;242m              set' disables the builtin.  Note that disabling the reserved  word  inter‐[0m
[38;2;248;248;242m              face for typeset may cause problems with the output of `typeset -p', which[0m
[38;2;248;248;242m              assumes the reserved word interface is available in order to restore array[0m
[38;2;248;248;242m              and associative array values.[0m

[38;2;248;248;242m              Unlike  parameter  assignment  statements, typeset's exit status on an as‐[0m
[38;2;248;248;242m              signment that involves a command substitution does not  reflect  the  exit[0m
[38;2;248;248;242m              status  of the command substitution.  Therefore, to test for an error in a[0m
[38;2;248;248;242m              command substitution, separate the declaration of the parameter  from  its[0m
[38;2;248;248;242m              initialization:[0m

[38;2;248;248;242m                     # WRONG[0m
[38;2;248;248;242m                     typeset var1=$(exit 1) || echo "Trouble with var1"[0m

[38;2;248;248;242m                     # RIGHT[0m
[38;2;248;248;242m                     typeset var1 && var1=$(exit 1) || echo "Trouble with var1"[0m

[38;2;248;248;242m              To  initialize a parameter param to a command output and mark it readonly,[0m
[38;2;248;248;242m              use typeset -r param or readonly  param  after  the  parameter  assignment[0m
[38;2;248;248;242m              statement.[0m

[38;2;248;248;242m              If  no attribute flags are given, and either no name arguments are present[0m
[38;2;248;248;242m              or the flag +m is used, then each parameter name printed is preceded by  a[0m
[38;2;248;248;242m              list  of  the  attributes of that parameter (array, association, exported,[0m
[38;2;248;248;242m              float, integer, readonly, or undefined for autoloaded parameters  not  yet[0m
[38;2;248;248;242m              loaded).   If +m is used with attribute flags, and all those flags are in‐[0m
[38;2;248;248;242m              troduced with +, the matching parameter names are printed but their values[0m
[38;2;248;248;242m              are not.[0m

[38;2;248;248;242m              The following control flags change the behavior of typeset:[0m

[38;2;248;248;242m              +      If `+' appears by itself in a separate word  as  the  last  option,[0m
[38;2;248;248;242m                     then  the  names of all parameters (functions with -f) are printed,[0m
[38;2;248;248;242m                     but the values (function bodies) are not.  No  name  arguments  may[0m
[38;2;248;248;242m                     appear,  and  it  is  an error for any other options to follow `+'.[0m
[38;2;248;248;242m                     The effect of `+' is as if all attribute  flags  which  precede  it[0m
[38;2;248;248;242m                     were  given  with  a  `+'  prefix.   For example, `typeset -U +' is[0m
[38;2;248;248;242m                     equivalent to `typeset +U' and displays the  names  of  all  arrays[0m
[38;2;248;248;242m                     having the uniqueness attribute, whereas `typeset -f -U +' displays[0m
[38;2;248;248;242m                     the  names of all autoloadable functions.  If + is the only option,[0m
[38;2;248;248;242m                     then type information (array, readonly, etc.) is also  printed  for[0m
[38;2;248;248;242m                     each parameter, in the same manner as `typeset +m "*"'.[0m

[38;2;248;248;242m              -g     The  -g (global) means that any resulting parameter will not be re‐[0m
[38;2;248;248;242m                     stricted to local scope.  Note that this does not necessarily  mean[0m
[38;2;248;248;242m                     that  the  parameter  will be global, as the flag will apply to any[0m
[38;2;248;248;242m                     existing parameter (even if  unset)  from  an  enclosing  function.[0m
[38;2;248;248;242m                     This  flag  does  not affect the parameter after creation, hence it[0m
[38;2;248;248;242m                     has no effect when listing existing parameters, nor does  the  flag[0m
[38;2;248;248;242m                     +g have any effect except in combination with -m (see below).[0m

[38;2;248;248;242m              -m     If  the  -m  flag is given the name arguments are taken as patterns[0m
[38;2;248;248;242m                     (use quoting to prevent these from being interpreted as  file  pat‐[0m
[38;2;248;248;242m                     terns).  With no attribute flags, all parameters (or functions with[0m
[38;2;248;248;242m                     the  -f  flag)  with  matching  names are printed (the shell option[0m
[38;2;248;248;242m                     TYPESET_SILENT is not used in this case).[0m

[38;2;248;248;242m                     If the +g flag is combined with -m, a new local parameter  is  cre‐[0m
[38;2;248;248;242m                     ated  for every matching parameter that is not already local.  Oth‐[0m
[38;2;248;248;242m                     erwise -m applies all other flags or assignments  to  the  existing[0m
[38;2;248;248;242m                     parameters.[0m

[38;2;248;248;242m                     Except  when  assignments are made with name=value, using +m forces[0m
[38;2;248;248;242m                     the matching parameters and their attributes to  be  printed,  even[0m
[38;2;248;248;242m                     inside  a  function.   Note  that  -m is ignored if no patterns are[0m
[38;2;248;248;242m                     given, so `typeset -m' displays attributes but `typeset -a +m' does[0m
[38;2;248;248;242m                     not.[0m

[38;2;248;248;242m              -p [ n ][0m
[38;2;248;248;242m                     If the -p option is given, parameters and values are printed in the[0m
[38;2;248;248;242m                     form of a typeset command with an assignment, regardless  of  other[0m
[38;2;248;248;242m                     flags  and  options.   Note  that  the -H flag on parameters is re‐[0m
[38;2;248;248;242m                     spected; no value will be shown for these parameters.[0m

[38;2;248;248;242m                     -p may be followed by an optional integer argument.  Currently only[0m
[38;2;248;248;242m                     the value 1 is supported.  In this case arrays and associative  ar‐[0m
[38;2;248;248;242m                     rays  are printed with newlines between indented elements for read‐[0m
[38;2;248;248;242m                     ability.[0m

[38;2;248;248;242m              -T [ scalar[=value] array[=(value ...)] [ sep ] ][0m
[38;2;248;248;242m                     This flag has a different meaning when used  with  -f;  see  below.[0m
[38;2;248;248;242m                     Otherwise  the  -T option requires zero, two, or three arguments to[0m
[38;2;248;248;242m                     be present.  With no arguments, the list of parameters  created  in[0m
[38;2;248;248;242m                     this  fashion is shown.  With two or three arguments, the first two[0m
[38;2;248;248;242m                     are the name of a scalar and of an array parameter (in that  order)[0m
[38;2;248;248;242m                     that  will  be tied together in the manner of $PATH and $path.  The[0m
[38;2;248;248;242m                     optional third argument is a single-character separator which  will[0m
[38;2;248;248;242m                     be  used  to  join the elements of the array to form the scalar; if[0m
[38;2;248;248;242m                     absent, a colon is used, as with $PATH.  Only the  first  character[0m
[38;2;248;248;242m                     of  the  separator is significant; any remaining characters are ig‐[0m
[38;2;248;248;242m                     nored.  Multibyte characters are not yet supported.[0m

[38;2;248;248;242m                     Only one of the scalar and array parameters may be assigned an ini‐[0m
[38;2;248;248;242m                     tial value (the restrictions on assignment  forms  described  above[0m
[38;2;248;248;242m                     also apply).[0m

[38;2;248;248;242m                     Both the scalar and the array may be manipulated as normal.  If one[0m
[38;2;248;248;242m                     is  unset,  the other will automatically be unset too.  There is no[0m
[38;2;248;248;242m                     way of untying the variables without unsetting them,  nor  of  con‐[0m
[38;2;248;248;242m                     verting  the  type  of one of them with another typeset command; +T[0m
[38;2;248;248;242m                     does not work, assigning an array to scalar is an  error,  and  as‐[0m
[38;2;248;248;242m                     signing a scalar to array sets it to be a single-element array.[0m

[38;2;248;248;242m                     Note  that  both  `typeset  -xT ...'  and `export -T ...' work, but[0m
[38;2;248;248;242m                     only the scalar will be marked for export.  Setting the value using[0m
[38;2;248;248;242m                     the scalar version causes a split on all separators  (which  cannot[0m
[38;2;248;248;242m                     be  quoted).   It  is  possible  to apply -T to two previously tied[0m
[38;2;248;248;242m                     variables but with a different separator character, in  which  case[0m
[38;2;248;248;242m                     the variables remain joined as before but the separator is changed.[0m

[38;2;248;248;242m                     When  an  existing  scalar is tied to a new array, the value of the[0m
[38;2;248;248;242m                     scalar is preserved but no attribute other than export will be pre‐[0m
[38;2;248;248;242m                     served.[0m

[38;2;248;248;242m              Attribute flags that transform the final value (-L, -R, -Z,  -l,  -u)  are[0m
[38;2;248;248;242m              only  applied  to the expanded value at the point of a parameter expansion[0m
[38;2;248;248;242m              expression using `$'.  They are not applied when a parameter is  retrieved[0m
[38;2;248;248;242m              internally by the shell for any purpose.[0m

[38;2;248;248;242m              The following attribute flags may be specified:[0m

[38;2;248;248;242m              -A     The names refer to associative array parameters; see `Array Parame‐[0m
[38;2;248;248;242m                     ters' in zshparam(1).[0m

[38;2;248;248;242m              -L [ n ][0m
[38;2;248;248;242m                     Left  justify and remove leading blanks from the value when the pa‐[0m
[38;2;248;248;242m                     rameter is expanded.  If n is nonzero, it defines the width of  the[0m
[38;2;248;248;242m                     field.   If  n is zero, the width is determined by the width of the[0m
[38;2;248;248;242m                     value of the first assignment.  In the case of numeric  parameters,[0m
[38;2;248;248;242m                     the  length of the complete value assigned to the parameter is used[0m
[38;2;248;248;242m                     to determine the width, not the value that would be output.[0m

[38;2;248;248;242m                     The width is the count of characters, which may be multibyte  char‐[0m
[38;2;248;248;242m                     acters  if the MULTIBYTE option is in effect.  Note that the screen[0m
[38;2;248;248;242m                     width of the character is not taken into account; if  this  is  re‐[0m
[38;2;248;248;242m                     quired, use padding with parameter expansion flags ${(ml...)...} as[0m
[38;2;248;248;242m                     described in `Parameter Expansion Flags' in zshexpn(1).[0m

[38;2;248;248;242m                     When  the  parameter  is  expanded,  it is filled on the right with[0m
[38;2;248;248;242m                     blanks or truncated if necessary to fit the field.  Note truncation[0m
[38;2;248;248;242m                     can lead to unexpected results with  numeric  parameters.   Leading[0m
[38;2;248;248;242m                     zeros are removed if the -Z flag is also set.[0m

[38;2;248;248;242m              -R [ n ][0m
[38;2;248;248;242m                     Similar  to  -L,  except that right justification is used; when the[0m
[38;2;248;248;242m                     parameter is expanded, the field is  left  filled  with  blanks  or[0m
[38;2;248;248;242m                     truncated from the end.  May not be combined with the -Z flag.[0m

[38;2;248;248;242m              -U     For  arrays  (but  not for associative arrays), keep only the first[0m
[38;2;248;248;242m                     occurrence of each duplicated value.  This may also be set for tied[0m
[38;2;248;248;242m                     parameters (see -T) or colon-separated special parameters like PATH[0m
[38;2;248;248;242m                     or FIGNORE, etc.  Note the flag takes effect on assignment, and the[0m
[38;2;248;248;242m                     type of the variable being assigned to is determinative; for  vari‐[0m
[38;2;248;248;242m                     ables  with  shared  values  it is therefore recommended to set the[0m
[38;2;248;248;242m                     flag for all interfaces, e.g. `typeset -U PATH path'.[0m

[38;2;248;248;242m                     This flag has a different meaning when used with -f; see below.[0m

[38;2;248;248;242m              -Z [ n ][0m
[38;2;248;248;242m                     Specially handled if set along with the -L flag.  Otherwise,  simi‐[0m
[38;2;248;248;242m                     lar  to  -R, except that leading zeros are used for padding instead[0m
[38;2;248;248;242m                     of blanks if the first non-blank character is a digit.  Numeric pa‐[0m
[38;2;248;248;242m                     rameters are  specially  handled:  they  are  always  eligible  for[0m
[38;2;248;248;242m                     padding  with zeroes, and the zeroes are inserted at an appropriate[0m
[38;2;248;248;242m                     place in the output.[0m

[38;2;248;248;242m              -a     The names refer to array parameters.  An  array  parameter  may  be[0m
[38;2;248;248;242m                     created  this  way, but it may be assigned to in the typeset state‐[0m
[38;2;248;248;242m                     ment only if the reserved word form of typeset is enabled (as it is[0m
[38;2;248;248;242m                     by default).  When displaying, both normal and  associative  arrays[0m
[38;2;248;248;242m                     are shown.[0m

[38;2;248;248;242m              -f     The  names  refer  to functions rather than parameters.  No assign‐[0m
[38;2;248;248;242m                     ments can be made, and the only other valid flags are -t,  -T,  -k,[0m
[38;2;248;248;242m                     -u,  -U  and  -z.   The flag -t turns on execution tracing for this[0m
[38;2;248;248;242m                     function; the flag -T does the same, but turns off tracing for  any[0m
[38;2;248;248;242m                     named  (not anonymous) function called from the present one, unless[0m
[38;2;248;248;242m                     that function also has the -t or -T flag.   The  -u  and  -U  flags[0m
[38;2;248;248;242m                     cause  the  function  to  be marked for autoloading; -U also causes[0m
[38;2;248;248;242m                     alias expansion to be suppressed when the function is loaded.   See[0m
[38;2;248;248;242m                     the description of the `autoload' builtin for details.[0m

[38;2;248;248;242m                     Note  that  the builtin functions provides the same basic capabili‐[0m
[38;2;248;248;242m                     ties as typeset -f but gives access to a  few  extra  options;  au‐[0m
[38;2;248;248;242m                     toload  gives  further  additional options for the case typeset -fu[0m
[38;2;248;248;242m                     and typeset -fU.[0m

[38;2;248;248;242m              -h     Hide: only useful for special parameters (those marked `<S>' in the[0m
[38;2;248;248;242m                     table in zshparam(1)), and for local parameters with the same  name[0m
[38;2;248;248;242m                     as  a special parameter, though harmless for others.  A special pa‐[0m
[38;2;248;248;242m                     rameter with this attribute will not retain its special effect when[0m
[38;2;248;248;242m                     made local.  Thus after `typeset -h PATH',  a  function  containing[0m
[38;2;248;248;242m                     `typeset  PATH' will create an ordinary local parameter without the[0m

[38;2;248;248;242m              and  functions  marked  for autoloading may be given.  In either case, the[0m
[38;2;248;248;242m              functions in files written with the -c or -a option will be autoloaded  as[0m
[38;2;248;248;242m              if the KSH_AUTOLOAD option were unset.[0m

[38;2;248;248;242m              The reason for handling loaded and not-yet-loaded functions with different[0m
[38;2;248;248;242m              options  is  that  some  definition  files for autoloading define multiple[0m
[38;2;248;248;242m              functions, including the function with the same name as the file, and,  at[0m
[38;2;248;248;242m              the  end,  call  that function.  In such cases the output of `zcompile -c'[0m
[38;2;248;248;242m              does not include the additional functions defined in  the  file,  and  any[0m
[38;2;248;248;242m              other  initialization  code in the file is lost.  Using `zcompile -a' cap‐[0m
[38;2;248;248;242m              tures all this extra information.[0m

[38;2;248;248;242m              If the -m option is combined with -c or -a, the names are used as patterns[0m
[38;2;248;248;242m              and all functions whose names match one of these patterns will be written.[0m
[38;2;248;248;242m              If no name is given, the definitions of all functions currently defined or[0m
[38;2;248;248;242m              marked as autoloaded will be written.[0m

[38;2;248;248;242m              Note the second form cannot be used for compiling functions  that  include[0m
[38;2;248;248;242m              redirections  as part of the definition rather than within the body of the[0m
[38;2;248;248;242m              function; for example[0m

[38;2;248;248;242m                     fn1() { { ... } >~/logfile }[0m

[38;2;248;248;242m              can be compiled but[0m

[38;2;248;248;242m                     fn1() { ... } >~/logfile[0m

[38;2;248;248;242m              cannot.  It is possible to use the first form of zcompile to  compile  au‐[0m
[38;2;248;248;242m              toloadable  functions that include the full function definition instead of[0m
[38;2;248;248;242m              just the body of the function.[0m

[38;2;248;248;242m              The third form, with the -t option, examines an  existing  compiled  file.[0m
[38;2;248;248;242m              Without  further  arguments, the names of the original files compiled into[0m
[38;2;248;248;242m              it are listed.  The first line of output shows the version  of  the  shell[0m
[38;2;248;248;242m              which  compiled the file and how the file will be used (i.e. by reading it[0m
[38;2;248;248;242m              directly or by mapping it into memory).  With arguments, nothing is output[0m
[38;2;248;248;242m              and the return status is set to zero if definitions  for  all  names  were[0m
[38;2;248;248;242m              found  in  the  compiled file, and non-zero if the definition for at least[0m
[38;2;248;248;242m              one name was not found.[0m

[38;2;248;248;242m              Other options:[0m

[38;2;248;248;242m              -U     Aliases are not expanded when compiling the named files.[0m

[38;2;248;248;242m              -R     When the compiled file is read, its contents are  copied  into  the[0m
[38;2;248;248;242m                     shell's  memory,  rather than memory-mapped (see -M).  This happens[0m
[38;2;248;248;242m                     automatically on systems that do not support memory mapping.[0m

[38;2;248;248;242m                     When compiling scripts instead of autoloadable functions, it is of‐[0m
[38;2;248;248;242m                     ten desirable to use this option; otherwise the whole file, includ‐[0m
[38;2;248;248;242m                     ing the code to define functions which have already  been  defined,[0m
[38;2;248;248;242m                     will remain mapped, consequently wasting memory.[0m

[38;2;248;248;242m              -M     The compiled file is mapped into the shell's memory when read. This[0m
[38;2;248;248;242m                     is  done in such a way that multiple instances of the shell running[0m
[38;2;248;248;242m                     on the same host will share this mapped file.  If neither -R nor -M[0m
[38;2;248;248;242m                     is given, the zcompile builtin decides what to do based on the size[0m
[38;2;248;248;242m                     of the compiled file.[0m

[38;2;248;248;242m              -k[0m
[38;2;248;248;242m              -z     These options are used when the compiled  file  contains  functions[0m
[38;2;248;248;242m                     which  are  to  be autoloaded. If -z is given, the function will be[0m
[38;2;248;248;242m                     autoloaded as if the KSH_AUTOLOAD option is not set, even if it  is[0m
[38;2;248;248;242m                     set  at  the  time  the  compiled  file is read, while if the -k is[0m
[38;2;248;248;242m                     given, the function will be  loaded  as  if  KSH_AUTOLOAD  is  set.[0m
[38;2;248;248;242m                     These options also take precedence over any -k or -z options speci‐[0m
[38;2;248;248;242m                     fied to the autoload builtin. If neither of these options is given,[0m
[38;2;248;248;242m                     the  function  will  be  loaded as determined by the setting of the[0m
[38;2;248;248;242m                     KSH_AUTOLOAD option at the time the compiled file is read.[0m

[38;2;248;248;242m                     These options may also appear as many times  as  necessary  between[0m
[38;2;248;248;242m                     the  listed  names  to  specify  the loading style of all following[0m
[38;2;248;248;242m                     functions, up to the next -k or -z.[0m

[38;2;248;248;242m                     The created file always contains two versions of the compiled  for‐[0m
[38;2;248;248;242m                     mat, one for big-endian machines and one for small-endian machines.[0m
[38;2;248;248;242m                     The upshot of this is that the compiled file is machine independent[0m
[38;2;248;248;242m                     and  if it is read or mapped, only one half of the file is actually[0m
[38;2;248;248;242m                     used (and mapped).[0m

[38;2;248;248;242m       zformat[0m
[38;2;248;248;242m              See the section `The zsh/zutil Module' in zshmodules(1).[0m

[38;2;248;248;242m       zftp   See the section `The zsh/zftp Module' in zshmodules(1).[0m

[38;2;248;248;242m       zle    See the section `Zle Builtins' in zshzle(1).[0m

[38;2;248;248;242m       zmodload [ -dL ] [ -s ] [ ... ][0m
[38;2;248;248;242m       zmodload -F [ -alLme -P param ] module [ [+-]feature ... ][0m
[38;2;248;248;242m       zmodload -e [ -A ] [ ... ][0m
[38;2;248;248;242m       zmodload [ -a [ -bcpf [ -I ] ] ] [ -iL ] ...[0m
[38;2;248;248;242m       zmodload -u [ -abcdpf [ -I ] ] [ -iL ] ...[0m
[38;2;248;248;242m       zmodload -A [ -L ] [ modalias[=module] ... ][0m
[38;2;248;248;242m       zmodload -R modalias ...[0m
[38;2;248;248;242m              Performs operations relating to zsh's loadable modules.  Loading  of  mod‐[0m
[38;2;248;248;242m              ules  while the shell is running (`dynamical loading') is not available on[0m
[38;2;248;248;242m              all operating systems, or on all installations on a  particular  operating[0m
[38;2;248;248;242m              system,  although  the zmodload command itself is always available and can[0m
[38;2;248;248;242m              be used to manipulate modules built into versions of the shell  executable[0m
[38;2;248;248;242m              without dynamical loading.[0m

[38;2;248;248;242m              Without  arguments  the  names  of all currently loaded binary modules are[0m
[38;2;248;248;242m              printed.  The -L option causes this list to be in the form of a series  of[0m
[38;2;248;248;242m              zmodload commands.  Forms with arguments are:[0m

[38;2;248;248;242m              zmodload [ -is ] name ...[0m
[38;2;248;248;242m              zmodload -u [ -i ] name ...[0m
[38;2;248;248;242m                     In  the  simplest case, zmodload loads a binary module.  The module[0m
[38;2;248;248;242m                     must be in a file with a name consisting of the specified name fol‐[0m
[38;2;248;248;242m                     lowed by a standard suffix, usually `.so' (`.sl' on HPUX).  If  the[0m
[38;2;248;248;242m                     module  to  be loaded is already loaded the duplicate module is ig‐[0m
[38;2;248;248;242m                     nored.  If zmodload detects an inconsistency, such  as  an  invalid[0m
[38;2;248;248;242m                     module  name or circular dependency list, the current code block is[0m
[38;2;248;248;242m                     aborted.  If it is available, the module is  loaded  if  necessary,[0m
[38;2;248;248;242m                     while if it is not available, non-zero status is silently returned.[0m
[38;2;248;248;242m                     The option -i is accepted for compatibility but has no effect.[0m

[38;2;248;248;242m                     The  named module is searched for in the same way a command is, us‐[0m
[38;2;248;248;242m                     ing $module_path instead of $path.  However,  the  path  search  is[0m
[38;2;248;248;242m                     performed  even  when the module name contains a `/', which it usu‐[0m
[38;2;248;248;242m                     ally does.  There is no way to prevent the path search.[0m

[38;2;248;248;242m                     If the module supports features (see below), zmodload tries to  en‐[0m
[38;2;248;248;242m                     able  all  features  when loading a module.  If the module was suc‐[0m
[38;2;248;248;242m                     cessfully loaded but not all features could  be  enabled,  zmodload[0m
[38;2;248;248;242m                     returns status 2.[0m

[38;2;248;248;242m                     If  the  option  -s is given, no error is printed if the module was[0m
[38;2;248;248;242m                     not available (though other errors indicating a  problem  with  the[0m
[38;2;248;248;242m                     module are printed).  The return status indicates if the module was[0m
[38;2;248;248;242m                     loaded.  This is appropriate if the caller considers the module op‐[0m
[38;2;248;248;242m                     tional.[0m

[38;2;248;248;242m                     With  -u,  zmodload  unloads  modules.  The same name must be given[0m
[38;2;248;248;242m                     that was given when the module was loaded, but it is not  necessary[0m
[38;2;248;248;242m                     for  the  module  to  exist in the file system.  The -i option sup‐[0m
[38;2;248;248;242m                     presses the error if the module is already unloaded (or  was  never[0m
[38;2;248;248;242m                     loaded).[0m

[38;2;248;248;242m                     Each module has a boot and a cleanup function.  The module will not[0m
[38;2;248;248;242m                     be  loaded if its boot function fails.  Similarly a module can only[0m
[38;2;248;248;242m                     be unloaded if its cleanup function runs successfully.[0m

[38;2;248;248;242m              zmodload -F [ -almLe -P param ] module [ [+-]feature ... ][0m
[38;2;248;248;242m                     zmodload -F allows more selective control over  the  features  pro‐[0m
[38;2;248;248;242m                     vided  by modules.  With no options apart from -F, the module named[0m
[38;2;248;248;242m                     module is loaded, if it was not already loaded,  and  the  list  of[0m
[38;2;248;248;242m                     features  is  set to the required state.  If no features are speci‐[0m
[38;2;248;248;242m                     fied, the module is loaded, if it was not already loaded,  but  the[0m
[38;2;248;248;242m                     state  of features is unchanged.  Each feature may be preceded by a[0m
[38;2;248;248;242m                     + to turn the feature on, or - to turn it off; the + is assumed  if[0m
[38;2;248;248;242m                     neither character is present.  Any feature not explicitly mentioned[0m
[38;2;248;248;242m                     is  left  in  its  current  state; if the module was not previously[0m
[38;2;248;248;242m                     loaded this means any such features will remain disabled.  The  re‐[0m
[38;2;248;248;242m                     turn  status  is  zero  if  all  features were set, 1 if the module[0m
[38;2;248;248;242m                     failed to load, and 2 if some features could not be set (for  exam‐[0m
[38;2;248;248;242m                     ple,  a  parameter  couldn't be added because there was a different[0m
[38;2;248;248;242m                     parameter of the same name) but the module was loaded.[0m

[38;2;248;248;242m                     The standard features are builtins, conditions, parameters and math[0m
[38;2;248;248;242m                     functions; these are indicated by the prefix `b:', `c:'  (`C:'  for[0m
[38;2;248;248;242m                     an  infix  condition), `p:' and `f:', respectively, followed by the[0m
[38;2;248;248;242m                     name that the corresponding feature would have in the  shell.   For[0m
[38;2;248;248;242m                     example,  `b:strftime'  indicates  a  builtin  named  strftime  and[0m
[38;2;248;248;242m                     p:EPOCHSECONDS indicates a parameter named EPOCHSECONDS.  The  mod‐[0m
[38;2;248;248;242m                     ule may provide other (`abstract') features of its own as indicated[0m
[38;2;248;248;242m                     by its documentation; these have no prefix.[0m

[38;2;248;248;242m                     With -l or -L, features provided by the module are listed.  With -l[0m
[38;2;248;248;242m                     alone,  a list of features together with their states is shown, one[0m
[38;2;248;248;242m                     feature per line.  With -L alone, a zmodload -F command that  would[0m
[38;2;248;248;242m                     cause  enabled  features  of  the  module to be turned on is shown.[0m
[38;2;248;248;242m                     With -lL, a zmodload -F command that would cause all  the  features[0m
[38;2;248;248;242m                     to  be set to their current state is shown.  If one of these combi‐[0m
[38;2;248;248;242m                     nations is given with the option -P param then the parameter  param[0m
[38;2;248;248;242m                     is set to an array of features, either features together with their[0m
[38;2;248;248;242m                     state or (if -L alone is given) enabled features.[0m

[38;2;248;248;242m                     With  the  option -L the module name may be omitted; then a list of[0m
[38;2;248;248;242m                     all enabled features for all modules providing features is  printed[0m
[38;2;248;248;242m                     in  the  form  of  zmodload  -F commands.  If -l is also given, the[0m
[38;2;248;248;242m                     state of both enabled and disabled features is output in that form.[0m

[38;2;248;248;242m                     A set of features may be provided together with -l or -L and a mod‐[0m
[38;2;248;248;242m                     ule name; in that case only the state of those features is  consid‐[0m
[38;2;248;248;242m                     ered.  Each feature may be preceded by + or - but the character has[0m
[38;2;248;248;242m                     no  effect.   If  no  set of features is provided, all features are[0m
[38;2;248;248;242m                     considered.[0m

[38;2;248;248;242m                     With -e, the command first tests that the module is loaded;  if  it[0m
[38;2;248;248;242m                     is not, status 1 is returned.  If the module is loaded, the list of[0m
[38;2;248;248;242m                     features  given as an argument is examined.  Any feature given with[0m
[38;2;248;248;242m                     no prefix is simply tested to see if the module  provides  it;  any[0m
[38;2;248;248;242m                     feature  given with a prefix + or - is tested to see if is provided[0m
[38;2;248;248;242m                     and in the given state.  If the tests on all features in  the  list[0m
[38;2;248;248;242m                     succeed, status 0 is returned, else status 1.[0m

[38;2;248;248;242m                     With  -m,  each  entry  in the given list of features is taken as a[0m
[38;2;248;248;242m                     pattern to be matched against the list of features provided by  the[0m
[38;2;248;248;242m                     module.   An initial + or - must be given explicitly.  This may not[0m
[38;2;248;248;242m                     be combined with the -a option as autoloads must be  specified  ex‐[0m
[38;2;248;248;242m                     plicitly.[0m

[38;2;248;248;242m                     With -a, the given list of features is marked for autoload from the[0m
[38;2;248;248;242m                     specified  module,  which may not yet be loaded.  An optional + may[0m
[38;2;248;248;242m                     appear before the feature name.  If the feature is prefixed with -,[0m
[38;2;248;248;242m                     any existing autoload is removed.  The options -l  and  -L  may  be[0m
[38;2;248;248;242m                     used to list autoloads.  Autoloading is specific to individual fea‐[0m
[38;2;248;248;242m                     tures;  when the module is loaded only the requested feature is en‐[0m
[38;2;248;248;242m                     abled.  Autoload requests are preserved if  the  module  is  subse‐[0m
[38;2;248;248;242m                     quently  unloaded  until an explicit `zmodload -Fa module -feature'[0m
[38;2;248;248;242m                     is issued.  It is not an error to request an autoload for a feature[0m
[38;2;248;248;242m                     of a module that is already loaded.[0m

[38;2;248;248;242m                     When the module is loaded each autoload is checked against the fea‐[0m
[38;2;248;248;242m                     tures actually provided by the module; if the feature is  not  pro‐[0m
[38;2;248;248;242m                     vided  the  autoload request is deleted.  A warning message is out‐[0m
[38;2;248;248;242m                     put; if the module is being loaded to provide a different  feature,[0m
[38;2;248;248;242m                     and  that  autoload is successful, there is no effect on the status[0m
[38;2;248;248;242m                     of the current command.  If the module is  already  loaded  at  the[0m
[38;2;248;248;242m                     time when zmodload -Fa is run, an error message is printed and sta‐[0m
[38;2;248;248;242m                     tus 1 returned.[0m

[38;2;248;248;242m                     zmodload  -Fa  can  be  used with the -l, -L, -e and -P options for[0m
[38;2;248;248;242m                     listing and testing the existence  of  autoloadable  features.   In[0m
[38;2;248;248;242m                     this  case -l is ignored if -L is specified.  zmodload -FaL with no[0m
[38;2;248;248;242m                     module name lists autoloads for all modules.[0m

[38;2;248;248;242m                     Note that only standard features as  described  above  can  be  au‐[0m
[38;2;248;248;242m                     toloaded; other features require the module to be loaded before en‐[0m
[38;2;248;248;242m                     abling.[0m

[38;2;248;248;242m              zmodload -d [ -L ] [ name ][0m
[38;2;248;248;242m              zmodload -d name dep ...[0m
[38;2;248;248;242m              zmodload -ud name [ dep ... ][0m
[38;2;248;248;242m                     The -d option can be used to specify module dependencies.  The mod‐[0m
[38;2;248;248;242m                     ules  named  in  the second and subsequent arguments will be loaded[0m
[38;2;248;248;242m                     before the module named in the first argument.[0m

[38;2;248;248;242m                     With -d and one argument, all  dependencies  for  that  module  are[0m
[38;2;248;248;242m                     listed.   With  -d  and  no  arguments, all module dependencies are[0m
[38;2;248;248;242m                     listed.  This listing is by default in a Makefile-like format.  The[0m
[38;2;248;248;242m                     -L option changes this format to a list of zmodload -d commands.[0m

[38;2;248;248;242m                     If -d and -u are both used, dependencies are removed.  If only  one[0m
[38;2;248;248;242m                     argument is given, all dependencies for that module are removed.[0m

[38;2;248;248;242m              zmodload -ab [ -L ][0m
[38;2;248;248;242m              zmodload -ab [ -i ] name [ builtin ... ][0m
[38;2;248;248;242m              zmodload -ub [ -i ] builtin ...[0m
[38;2;248;248;242m                     The  -ab option defines autoloaded builtins.  It defines the speci‐[0m
[38;2;248;248;242m                     fied builtins.  When any of those builtins is  called,  the  module[0m
[38;2;248;248;242m                     specified  in the first argument is loaded and all its features are[0m
[38;2;248;248;242m                     enabled (for selective control of features use `zmodload -F -a'  as[0m
[38;2;248;248;242m                     described  above).   If  only the name is given, one builtin is de‐[0m
[38;2;248;248;242m                     fined, with the same name as the module.  -i suppresses  the  error[0m
[38;2;248;248;242m                     if the builtin is already defined or autoloaded, but not if another[0m
[38;2;248;248;242m                     builtin of the same name is already defined.[0m

[38;2;248;248;242m                     With -ab and no arguments, all autoloaded builtins are listed, with[0m
[38;2;248;248;242m                     the  module  name  (if  different)  shown  in parentheses after the[0m
[38;2;248;248;242m                     builtin name.  The -L option changes this format to a list of zmod‐[0m
[38;2;248;248;242m                     load -a commands.[0m

[38;2;248;248;242m                     If -b is used together with the -u option, it removes builtins pre‐[0m
[38;2;248;248;242m                     viously defined with -ab.  This is only possible if the builtin  is[0m
[38;2;248;248;242m                     not  yet loaded.  -i suppresses the error if the builtin is already[0m
[38;2;248;248;242m                     removed (or never existed).[0m

[38;2;248;248;242m                     Autoload requests are retained if the module  is  subsequently  un‐[0m
[38;2;248;248;242m                     loaded until an explicit `zmodload -ub builtin' is issued.[0m

[38;2;248;248;242m              zmodload -ac [ -IL ][0m
[38;2;248;248;242m              zmodload -ac [ -iI ] name [ cond ... ][0m
[38;2;248;248;242m              zmodload -uc [ -iI ] cond ...[0m
[38;2;248;248;242m                     The  -ac  option  is used to define autoloaded condition codes. The[0m
[38;2;248;248;242m                     cond strings give the names of the conditions defined by  the  mod‐[0m
[38;2;248;248;242m                     ule.  The  optional  -I  option  is  used to define infix condition[0m
[38;2;248;248;242m                     names. Without this option prefix condition names are defined.[0m

[38;2;248;248;242m                     If given no condition names, all defined names are listed (as a se‐[0m
[38;2;248;248;242m                     ries of zmodload commands if the -L option is given).[0m

[38;2;248;248;242m                     The -uc option removes definitions for autoloaded conditions.[0m

[38;2;248;248;242m              zmodload -ap [ -L ][0m
[38;2;248;248;242m              zmodload -ap [ -i ] name [ parameter ... ][0m
[38;2;248;248;242m              zmodload -up [ -i ] parameter ...[0m
[38;2;248;248;242m                     The -p option is like the -b and -c  options,  but  makes  zmodload[0m
[38;2;248;248;242m                     work on autoloaded parameters instead.[0m

[38;2;248;248;242m              zmodload -af [ -L ][0m
[38;2;248;248;242m              zmodload -af [ -i ] name [ function ... ][0m
[38;2;248;248;242m              zmodload -uf [ -i ] function ...[0m
[38;2;248;248;242m                     The  -f  option is like the -b, -p, and -c options, but makes zmod‐[0m
[38;2;248;248;242m                     load work on autoloaded math functions instead.[0m

[38;2;248;248;242m              zmodload -a [ -L ][0m
[38;2;248;248;242m              zmodload -a [ -i ] name [ builtin ... ][0m
[38;2;248;248;242m              zmodload -ua [ -i ] builtin ...[0m
[38;2;248;248;242m                     Equivalent to -ab and -ub.[0m

[38;2;248;248;242m              zmodload -e [ -A ] [ string ... ][0m
[38;2;248;248;242m                     The -e option without arguments lists all loaded modules; if the -A[0m
[38;2;248;248;242m                     option is also given, module aliases corresponding to  loaded  mod‐[0m
[38;2;248;248;242m                     ules  are  also  shown.   If  arguments  are  provided,  nothing is[0m
[38;2;248;248;242m                     printed; the return status is set to zero if all strings  given  as[0m
[38;2;248;248;242m                     arguments  are  names  of  loaded modules and to one if at least on[0m
[38;2;248;248;242m                     string is not the name of a loaded module.  This  can  be  used  to[0m
[38;2;248;248;242m                     test  for  the  availability  of things implemented by modules.  In[0m
[38;2;248;248;242m                     this case, any aliases are automatically resolved and the  -A  flag[0m
[38;2;248;248;242m                     is not used.[0m

[38;2;248;248;242m              zmodload -A [ -L ] [ modalias[=module] ... ][0m
[38;2;248;248;242m                     For  each  argument,  if both modalias and module are given, define[0m
[38;2;248;248;242m                     modalias to be an alias for  the  module  module.   If  the  module[0m
[38;2;248;248;242m                     modalias is ever subsequently requested, either via a call to zmod‐[0m
[38;2;248;248;242m                     load  or implicitly, the shell will attempt to load module instead.[0m
[38;2;248;248;242m                     If module is not given, show the definition of modalias.  If no ar‐[0m
[38;2;248;248;242m                     guments are given, list all defined module aliases.  When  listing,[0m
[38;2;248;248;242m                     if  the  -L  flag was also given, list the definition as a zmodload[0m
[38;2;248;248;242m                     command to recreate the alias.[0m

[38;2;248;248;242m                     The existence of aliases for modules is completely  independent  of[0m
[38;2;248;248;242m                     whether the name resolved is actually loaded as a module: while the[0m
[38;2;248;248;242m                     alias  exists, loading and unloading the module under any alias has[0m
[38;2;248;248;242m                     exactly the same effect as using the resolved name,  and  does  not[0m
[38;2;248;248;242m                     affect the connection between the alias and the resolved name which[0m
[38;2;248;248;242m                     can  be  removed  either by zmodload -R or by redefining the alias.[0m
[38;2;248;248;242m                     Chains of aliases (i.e. where the first resolved name is itself  an[0m
[38;2;248;248;242m                     alias) are valid so long as these are not circular.  As the aliases[0m
[38;2;248;248;242m                     take the same format as module names, they may include path separa‐[0m
[38;2;248;248;242m                     tors:   in  this  case, there is no requirement for any part of the[0m
[38;2;248;248;242m                     path named to exist as the alias will be resolved first.  For exam‐[0m
[38;2;248;248;242m                     ple, `any/old/alias' is always a valid alias.[0m

[38;2;248;248;242m                     Dependencies added to aliased modules are actually added to the re‐[0m
[38;2;248;248;242m                     solved module; these remain if the alias is removed.  It  is  valid[0m
[38;2;248;248;242m                     to  create an alias whose name is one of the standard shell modules[0m
[38;2;248;248;242m                     and which resolves to a different module.  However, if a module has[0m
[38;2;248;248;242m                     dependencies, it will not be possible to use the module name as  an[0m
[38;2;248;248;242m                     alias  as the module will already be marked as a loadable module in[0m
[38;2;248;248;242m                     its own right.[0m

[38;2;248;248;242m                     Apart from the above, aliases can be used in the  zmodload  command[0m
[38;2;248;248;242m                     anywhere  module  names are required.  However, aliases will not be[0m
[38;2;248;248;242m                     shown in lists of loaded modules with a bare `zmodload'.[0m

[38;2;248;248;242m              zmodload -R modalias ...[0m
[38;2;248;248;242m                     For each modalias argument that was previously defined as a  module[0m
[38;2;248;248;242m                     alias  via  zmodload -A, delete the alias.  If any was not defined,[0m
[38;2;248;248;242m                     an error is caused and the remainder of the line is ignored.[0m

[38;2;248;248;242m              Note that zsh makes no distinction between modules that were  linked  into[0m
[38;2;248;248;242m              the  shell  and  modules  that  are loaded dynamically. In both cases this[0m
[38;2;248;248;242m              builtin command has to be used to make available the  builtins  and  other[0m
[38;2;248;248;242m              things defined by modules (unless the module is autoloaded on these defin‐[0m
[38;2;248;248;242m              itions).  This is true even for systems that don't support dynamic loading[0m
[38;2;248;248;242m              of modules.[0m
