[38;2;248;248;242mNAME[0m
[38;2;248;248;242m       zshoptions - zsh options[0m

[38;2;248;248;242mSPECIFYING OPTIONS[0m
[38;2;248;248;242m       Options  are primarily referred to by name.  These names are case insensitive and[0m
[38;2;248;248;242m       underscores are ignored.  For  example,  `allexport'  is  equivalent  to  `A__ll‐[0m
[38;2;248;248;242m       eXP_ort'.[0m

[38;2;248;248;242m       The sense of an option name may be inverted by preceding it with `no', so `setopt[0m
[38;2;248;248;242m       No_Beep' is equivalent to `unsetopt beep'.  This inversion can only be done once,[0m
[38;2;248;248;242m       so  `nonobeep'  is  not a synonym for `beep'.  Similarly, `tify' is not a synonym[0m
[38;2;248;248;242m       for `nonotify' (the inversion of `notify').[0m

[38;2;248;248;242m       Some options also have one or more single letter names.  There are  two  sets  of[0m
[38;2;248;248;242m       single  letter  options:  one used by default, and another used to emulate sh/ksh[0m
[38;2;248;248;242m       (used when the SH_OPTION_LETTERS option is set).  The single letter  options  can[0m
[38;2;248;248;242m       be used on the shell command line, or with the set, setopt and unsetopt builtins,[0m
[38;2;248;248;242m       as normal Unix options preceded by `-'.[0m

[38;2;248;248;242m       The  sense  of  the single letter options may be inverted by using `+' instead of[0m
[38;2;248;248;242m       `-'.  Some of the single letter option names refer to an  option  being  off,  in[0m
[38;2;248;248;242m       which  case  the inversion of that name refers to the option being on.  For exam‐[0m
[38;2;248;248;242m       ple, `+n' is the short name of `exec', and `-n' is the short name of  its  inver‐[0m
[38;2;248;248;242m       sion, `noexec'.[0m

[38;2;248;248;242m       In  strings  of  single letter options supplied to the shell at startup, trailing[0m
[38;2;248;248;242m       whitespace will be ignored; for example the string `-f    ' will be treated  just[0m
[38;2;248;248;242m       as  `-f',  but the string `-f i' is an error.  This is because many systems which[0m
[38;2;248;248;242m       implement the `#!' mechanism for calling scripts do  not  strip  trailing  white‐[0m
[38;2;248;248;242m       space.[0m

[38;2;248;248;242m       It  is  possible for options to be set within a function scope.  See the descrip‐[0m
[38;2;248;248;242m       tion of the option LOCAL_OPTIONS below.[0m

[38;2;248;248;242mDESCRIPTION OF OPTIONS[0m
[38;2;248;248;242m       In the following list, options set by default in all emulations are  marked  <D>;[0m
[38;2;248;248;242m       those set by default only in csh, ksh, sh, or zsh emulations are marked <C>, <K>,[0m
[38;2;248;248;242m       <S>, <Z> as appropriate.  When listing options (by `setopt', `unsetopt', `set -o'[0m
[38;2;248;248;242m       or  `set  +o'), those turned on by default appear in the list prefixed with `no'.[0m
[38;2;248;248;242m       Hence (unless KSH_OPTION_PRINT is set), `setopt' shows all options whose settings[0m
[38;2;248;248;242m       are changed from the default.[0m

[38;2;248;248;242m   Changing Directories[0m
[38;2;248;248;242m       AUTO_CD (-J)[0m
[38;2;248;248;242m              If a command is issued that can't be executed as a normal command, and the[0m
[38;2;248;248;242m              command is the name of a directory, perform the cd command to that  direc‐[0m
[38;2;248;248;242m              tory.   This  option  is  only applicable if the option SHIN_STDIN is set,[0m
[38;2;248;248;242m              i.e. if commands are being read from standard input.  The  option  is  de‐[0m
[38;2;248;248;242m              signed  for  interactive use; it is recommended that cd be used explicitly[0m
[38;2;248;248;242m              in scripts to avoid ambiguity.[0m

[38;2;248;248;242m       AUTO_PUSHD (-N)[0m
[38;2;248;248;242m              Make cd push the old directory onto the directory stack.[0m

[38;2;248;248;242m       CDABLE_VARS (-T)[0m
[38;2;248;248;242m              If the argument to a cd command (or an implied cd with the AUTO_CD  option[0m
[38;2;248;248;242m              set)  is  not  a directory, and does not begin with a slash, try to expand[0m
[38;2;248;248;242m              the expression as if it were preceded by a `~' (see the section  `Filename[0m
[38;2;248;248;242m              Expansion').[0m

[38;2;248;248;242m       CD_SILENT[0m
[38;2;248;248;242m              Never  print the working directory after a cd (whether explicit or implied[0m
[38;2;248;248;242m              with the AUTO_CD option set). cd normally  prints  the  working  directory[0m
[38;2;248;248;242m              when  the  argument given to it was -, a stack entry, or the name of a di‐[0m
[38;2;248;248;242m              rectory found under CDPATH.  Note  that  this  is  distinct  from  pushd's[0m
[38;2;248;248;242m              stack-printing behaviour, which is controlled by PUSHD_SILENT. This option[0m
[38;2;248;248;242m              overrides the printing-related effects of POSIX_CD.[0m

[38;2;248;248;242m       CHASE_DOTS[0m
[38;2;248;248;242m              When  changing  to  a directory containing a path segment `..' which would[0m
[38;2;248;248;242m              otherwise be treated as canceling the previous segment  in  the  path  (in[0m
[38;2;248;248;242m              other  words,  `foo/..'  would be removed from the path, or if `..' is the[0m
[38;2;248;248;242m              first part of the path, the last part of  the  current  working  directory[0m
[38;2;248;248;242m              would  be  removed),  instead  resolve the path to the physical directory.[0m
[38;2;248;248;242m              This option is overridden by CHASE_LINKS.[0m

[38;2;248;248;242m              For example, suppose /foo/bar is a link to the directory /alt/rod.   With‐[0m
[38;2;248;248;242m              out  this  option  set,  `cd /foo/bar/..' changes to /foo; with it set, it[0m
[38;2;248;248;242m              changes to /alt.  The same applies if the current  directory  is  /foo/bar[0m
[38;2;248;248;242m              and  `cd ..' is used.  Note that all other symbolic links in the path will[0m
[38;2;248;248;242m              also be resolved.[0m

[38;2;248;248;242m       CHASE_LINKS (-w)[0m
[38;2;248;248;242m              Resolve symbolic links to their true values when changing directory.  This[0m
[38;2;248;248;242m              also has the effect of CHASE_DOTS,  i.e.  a  `..'  path  segment  will  be[0m
[38;2;248;248;242m              treated  as  referring  to the physical parent, even if the preceding path[0m
[38;2;248;248;242m              segment is a symbolic link.[0m

[38;2;248;248;242m       POSIX_CD <K> <S>[0m
[38;2;248;248;242m              Modifies the behaviour of cd, chdir and pushd commands to make  them  more[0m
[38;2;248;248;242m              compatible with the POSIX standard. The behaviour with the option unset is[0m
[38;2;248;248;242m              described  in  the documentation for the cd builtin in zshbuiltins(1).  If[0m
[38;2;248;248;242m              the option is set, the shell does not test for directories beneath the lo‐[0m
[38;2;248;248;242m              cal directory (`.') until  after  all  directories  in  cdpath  have  been[0m
[38;2;248;248;242m              tested,  and  the  cd and chdir commands do not recognise arguments of the[0m
[38;2;248;248;242m              form `{+|-}n' as directory stack entries.[0m

[38;2;248;248;242m              Also, if the option is set, the conditions under which  the  shell  prints[0m
[38;2;248;248;242m              the  new directory after changing to it are modified.  It is no longer re‐[0m
[38;2;248;248;242m              stricted to interactive shells (although printing of the  directory  stack[0m
[38;2;248;248;242m              with  pushd is still limited to interactive shells); and any use of a com‐[0m
[38;2;248;248;242m              ponent of CDPATH, including a `.' but excluding an empty component that is[0m
[38;2;248;248;242m              otherwise treated as `.', causes the directory to be printed.[0m

[38;2;248;248;242m       PUSHD_IGNORE_DUPS[0m
[38;2;248;248;242m              Don't push multiple copies of the same directory onto the directory stack.[0m

[38;2;248;248;242m       PUSHD_MINUS[0m
[38;2;248;248;242m              Exchanges the meanings of `+' and `-' when used with a number to specify a[0m
[38;2;248;248;242m              directory in the stack.[0m

[38;2;248;248;242m       PUSHD_SILENT (-E)[0m
[38;2;248;248;242m              Do not print the directory stack after pushd or popd.[0m

[38;2;248;248;242m       PUSHD_TO_HOME (-D)[0m
[38;2;248;248;242m              Have pushd with no arguments act like `pushd $HOME'.[0m

[38;2;248;248;242m   Completion[0m
[38;2;248;248;242m       ALWAYS_LAST_PROMPT <D>[0m
[38;2;248;248;242m              If unset, key functions that list completions try to return  to  the  last[0m
[38;2;248;248;242m              prompt  if  given a numeric argument. If set these functions try to return[0m
[38;2;248;248;242m              to the last prompt if given no numeric argument.[0m

[38;2;248;248;242m       ALWAYS_TO_END[0m
[38;2;248;248;242m              If a completion is performed with the cursor within a  word,  and  a  full[0m
[38;2;248;248;242m              completion  is inserted, the cursor is moved to the end of the word.  That[0m
[38;2;248;248;242m              is, the cursor is moved to the end of the word if either a single match is[0m
[38;2;248;248;242m              inserted or menu completion is performed.[0m

[38;2;248;248;242m       AUTO_LIST (-9) <D>[0m
[38;2;248;248;242m              Automatically list choices on an ambiguous completion.[0m

[38;2;248;248;242m       AUTO_MENU <D>[0m
[38;2;248;248;242m              Automatically use menu completion after the second consecutive request for[0m
[38;2;248;248;242m              completion, for example by pressing the tab key repeatedly. This option is[0m
[38;2;248;248;242m              overridden by MENU_COMPLETE.[0m

[38;2;248;248;242m       AUTO_NAME_DIRS[0m
[38;2;248;248;242m              Any parameter that is set to the absolute name of a directory  immediately[0m
[38;2;248;248;242m              becomes  a  name for that directory, that will be used by the `%~' and re‐[0m
[38;2;248;248;242m              lated prompt sequences, and will be available when completion is performed[0m
[38;2;248;248;242m              on a word starting with `~'.  (Otherwise, the parameter must  be  used  in[0m
[38;2;248;248;242m              the form `~param' first.)[0m

[38;2;248;248;242m       AUTO_PARAM_KEYS <D>[0m
[38;2;248;248;242m              If  a  parameter  name was completed and a following character (normally a[0m
[38;2;248;248;242m              space) automatically inserted, and the next  character  typed  is  one  of[0m
[38;2;248;248;242m              those that have to come directly after the name (like `}', `:', etc.), the[0m
[38;2;248;248;242m              automatically  added  character  is  deleted,  so that the character typed[0m
[38;2;248;248;242m              comes immediately after the parameter name.  Completion in a brace  expan‐[0m
[38;2;248;248;242m              sion  is  affected  similarly: the added character is a `,', which will be[0m
[38;2;248;248;242m              removed if `}' is typed next.[0m

[38;2;248;248;242m       AUTO_PARAM_SLASH <D>[0m
[38;2;248;248;242m              If a parameter is completed whose content is the name of a directory, then[0m
[38;2;248;248;242m              add a trailing slash instead of a space.[0m

[38;2;248;248;242m       AUTO_REMOVE_SLASH <D>[0m
[38;2;248;248;242m              When the last character resulting from a completion is  a  slash  and  the[0m
[38;2;248;248;242m              next  character  typed  is  a word delimiter, a slash, or a character that[0m
[38;2;248;248;242m              ends a command (such as a semicolon or an ampersand), remove the slash.[0m

[38;2;248;248;242m       BASH_AUTO_LIST[0m
[38;2;248;248;242m              On an ambiguous completion, automatically list choices when the completion[0m
[38;2;248;248;242m              function is called  twice  in  succession.   This  takes  precedence  over[0m
[38;2;248;248;242m              AUTO_LIST.   The  setting of LIST_AMBIGUOUS is respected.  If AUTO_MENU is[0m
[38;2;248;248;242m              set, the menu behaviour will then start with the third press.   Note  that[0m
[38;2;248;248;242m              this will not work with MENU_COMPLETE, since repeated completion calls im‐[0m
[38;2;248;248;242m              mediately cycle through the list in that case.[0m

[38;2;248;248;242m       COMPLETE_ALIASES[0m
[38;2;248;248;242m              Prevents aliases on the command line from being internally substituted be‐[0m
[38;2;248;248;242m              fore  completion is attempted.  The effect is to make the alias a distinct[0m
[38;2;248;248;242m              command for completion purposes.[0m

[38;2;248;248;242m       COMPLETE_IN_WORD[0m
[38;2;248;248;242m              If unset, the cursor is set to the  end  of  the  word  if  completion  is[0m
[38;2;248;248;242m              started. Otherwise it stays there and completion is done from both ends.[0m

[38;2;248;248;242m       GLOB_COMPLETE[0m
[38;2;248;248;242m              When  the current word has a glob pattern, do not insert all the words re‐[0m
[38;2;248;248;242m              sulting from the expansion but generate matches as for completion and  cy‐[0m
[38;2;248;248;242m              cle through them like MENU_COMPLETE. The matches are generated as if a `*'[0m
[38;2;248;248;242m              was  added  to  the  end  of the word, or inserted at the cursor when COM‐[0m
[38;2;248;248;242m              PLETE_IN_WORD is set.  This actually uses pattern matching, not  globbing,[0m
[38;2;248;248;242m              so  it  works  not only for files but for any completion, such as options,[0m
[38;2;248;248;242m              user names, etc.[0m

[38;2;248;248;242m              Note that when the pattern matcher is used, matching control (for example,[0m
[38;2;248;248;242m              case-insensitive or anchored matching) cannot be  used.   This  limitation[0m
[38;2;248;248;242m              only  applies  when the current word contains a pattern; simply turning on[0m
[38;2;248;248;242m              the GLOB_COMPLETE option does not have this effect.[0m

[38;2;248;248;242m       HASH_LIST_ALL <D>[0m
[38;2;248;248;242m              Whenever a command completion or spelling correction  is  attempted,  make[0m
[38;2;248;248;242m              sure  the  entire command path is hashed first.  This makes the first com‐[0m
[38;2;248;248;242m              pletion slower but avoids false reports of spelling errors.[0m

[38;2;248;248;242m       LIST_AMBIGUOUS <D>[0m
[38;2;248;248;242m              This option works when AUTO_LIST or BASH_AUTO_LIST is also set.  If  there[0m
[38;2;248;248;242m              is an unambiguous prefix to insert on the command line, that is done with‐[0m
[38;2;248;248;242m              out a completion list being displayed; in other words, auto-listing behav‐[0m
[38;2;248;248;242m              iour  only  takes  place  when  nothing would be inserted.  In the case of[0m
[38;2;248;248;242m              BASH_AUTO_LIST, this means that the list will be delayed to the third call[0m
[38;2;248;248;242m              of the function.[0m

[38;2;248;248;242m       LIST_BEEP <D>[0m
[38;2;248;248;242m              Beep on an ambiguous completion.  More accurately, this forces the comple‐[0m
[38;2;248;248;242m              tion widgets to return status 1 on an ambiguous completion,  which  causes[0m
[38;2;248;248;242m              the  shell to beep if the option BEEP is also set; this may be modified if[0m
[38;2;248;248;242m              completion is called from a user-defined widget.[0m

[38;2;248;248;242m       LIST_PACKED[0m
[38;2;248;248;242m              Try to make the completion list smaller (occupying less lines) by printing[0m
[38;2;248;248;242m              the matches in columns with different widths.[0m

[38;2;248;248;242m       LIST_ROWS_FIRST[0m
[38;2;248;248;242m              Lay out the matches in completion lists sorted horizontally, that is,  the[0m
[38;2;248;248;242m              second match is to the right of the first one, not under it as usual.[0m

[38;2;248;248;242m       LIST_TYPES (-X) <D>[0m
[38;2;248;248;242m              When  listing  files  that are possible completions, show the type of each[0m
[38;2;248;248;242m              file with a trailing identifying mark.[0m

[38;2;248;248;242m       MENU_COMPLETE (-Y)[0m
[38;2;248;248;242m              On an ambiguous completion, instead of listing possibilities  or  beeping,[0m
[38;2;248;248;242m              insert  the  first  match  immediately.  Then when completion is requested[0m
[38;2;248;248;242m              again, remove the first match and insert  the  second  match,  etc.   When[0m
[38;2;248;248;242m              there  are  no  more  matches,  go  back  to  the  first  one  again.  re‐[0m
[38;2;248;248;242m              verse-menu-complete may be used to loop through the list in the other  di‐[0m
[38;2;248;248;242m              rection. This option overrides AUTO_MENU.[0m

[38;2;248;248;242m       REC_EXACT (-S)[0m
[38;2;248;248;242m              If the string on the command line exactly matches one of the possible com‐[0m
[38;2;248;248;242m              pletions,  it  is accepted, even if there is another completion (i.e. that[0m
[38;2;248;248;242m              string with something else added) that also matches.[0m

[38;2;248;248;242m   Expansion and Globbing[0m
[38;2;248;248;242m       BAD_PATTERN (+2) <C> <Z>[0m
[38;2;248;248;242m              If a pattern for filename generation is badly formed, print an error  mes‐[0m
[38;2;248;248;242m              sage.  (If this option is unset, the pattern will be left unchanged.)[0m

[38;2;248;248;242m       BARE_GLOB_QUAL <Z>[0m
[38;2;248;248;242m              In  a  glob  pattern,  treat  a trailing set of parentheses as a qualifier[0m
[38;2;248;248;242m              list, if it contains no `|', `(' or (if special) `~' characters.  See  the[0m
[38;2;248;248;242m              section `Filename Generation'.[0m

[38;2;248;248;242m       BRACE_CCL[0m
[38;2;248;248;242m              Expand  expressions  in braces which would not otherwise undergo brace ex‐[0m
[38;2;248;248;242m              pansion to a lexically ordered list of all the characters.  See  the  sec‐[0m
[38;2;248;248;242m              tion `Brace Expansion'.[0m

[38;2;248;248;242m       CASE_GLOB <D>[0m
[38;2;248;248;242m              Make  globbing  (filename  generation) sensitive to case.  Note that other[0m
[38;2;248;248;242m              uses of patterns are always sensitive to case.  If the  option  is  unset,[0m
[38;2;248;248;242m              the presence of any character which is special to filename generation will[0m
[38;2;248;248;242m              cause case-insensitive matching.  For example, cvs(/) can match the direc‐[0m
[38;2;248;248;242m              tory  CVS  owing  to  the presence of the globbing flag (unless the option[0m
[38;2;248;248;242m              BARE_GLOB_QUAL is unset).[0m

[38;2;248;248;242m       CASE_MATCH <D>[0m
[38;2;248;248;242m              Make regular expressions using the  zsh/regex  module  (including  matches[0m
[38;2;248;248;242m              with =~) sensitive to case.[0m

[38;2;248;248;242m       CASE_PATHS[0m
[38;2;248;248;242m              If  CASE_PATHS is not set (the default), CASE_GLOB affects the interpreta‐[0m
[38;2;248;248;242m              tion of every path component, whenever a special character appears in  any[0m
[38;2;248;248;242m              component.   When CASE_PATHS is set, file path components that do not con‐[0m
[38;2;248;248;242m              tain special filename generation characters are always sensitive to  case,[0m
[38;2;248;248;242m              thus  restricting NO_CASE_GLOB to components that contain globbing charac‐[0m
[38;2;248;248;242m              ters.[0m

[38;2;248;248;242m              Note that if  the  filesystem  itself  is  not  sensitive  to  case,  then[0m
[38;2;248;248;242m              CASE_PATHS has no effect.[0m

[38;2;248;248;242m       CSH_NULL_GLOB <C>[0m
[38;2;248;248;242m              If  a  pattern  for filename generation has no matches, delete the pattern[0m
[38;2;248;248;242m              from the argument list; do not report an error unless all the patterns  in[0m
[38;2;248;248;242m              a command have no matches.  Overrides NOMATCH.[0m

[38;2;248;248;242m       EQUALS <Z>[0m
[38;2;248;248;242m              Perform = filename expansion.  (See the section `Filename Expansion'.)[0m

[38;2;248;248;242m       EXTENDED_GLOB[0m
[38;2;248;248;242m              Treat  the  `#',  `~'  and `^' characters as part of patterns for filename[0m
[38;2;248;248;242m              generation, etc.  (An initial unquoted `~' always produces named directory[0m
[38;2;248;248;242m              expansion.)[0m

[38;2;248;248;242m       FORCE_FLOAT[0m
[38;2;248;248;242m              Constants in arithmetic evaluation will be treated as floating point  even[0m
[38;2;248;248;242m              without  the  use of a decimal point; the values of integer variables will[0m
[38;2;248;248;242m              be converted to floating point when used in arithmetic expressions.  Inte‐[0m
[38;2;248;248;242m              gers in any base will be converted.[0m

[38;2;248;248;242m       GLOB (+F, ksh: +f) <D>[0m
[38;2;248;248;242m              Perform filename generation (globbing).  (See the section `Filename Gener‐[0m
[38;2;248;248;242m              ation'.)[0m

[38;2;248;248;242m       GLOB_ASSIGN <C>[0m
[38;2;248;248;242m              If this option is set, filename generation (globbing) is performed on  the[0m
[38;2;248;248;242m              right  hand side of scalar parameter assignments of the form `name=pattern[0m
[38;2;248;248;242m              (e.g. `foo=*').  If the result has more than one word the  parameter  will[0m
[38;2;248;248;242m              become an array with those words as arguments. This option is provided for[0m
[38;2;248;248;242m              backwards  compatibility  only:  globbing is always performed on the right[0m
[38;2;248;248;242m              hand side of array assignments of the form `name=(value)' (e.g. `foo=(*)')[0m
[38;2;248;248;242m              and this form is recommended for clarity; with this option set, it is  not[0m
[38;2;248;248;242m              possible to predict whether the result will be an array or a scalar.[0m

[38;2;248;248;242m       GLOB_DOTS (-4)[0m
[38;2;248;248;242m              Do not require a leading `.' in a filename to be matched explicitly.[0m

[38;2;248;248;242m       GLOB_STAR_SHORT[0m
[38;2;248;248;242m              When  this  option is set and the default zsh-style globbing is in effect,[0m
[38;2;248;248;242m              the pattern `**/*' can be abbreviated to `**' and the pattern `***/*'  can[0m
[38;2;248;248;242m              be abbreviated to ***.  Hence `**.c' finds a file ending in .c in any sub‐[0m
[38;2;248;248;242m              directory,  and `***.c' does the same while also following symbolic links.[0m
[38;2;248;248;242m              A / immediately after the `**' or `***' forces the pattern to  be  treated[0m
[38;2;248;248;242m              as the unabbreviated form.[0m

[38;2;248;248;242m       GLOB_SUBST <C> <K> <S>[0m
[38;2;248;248;242m              Treat  any characters resulting from parameter expansion as being eligible[0m
[38;2;248;248;242m              for filename expansion and filename generation, and any characters result‐[0m
[38;2;248;248;242m              ing from command substitution as being eligible for  filename  generation.[0m
[38;2;248;248;242m              Braces (and commas in between) do not become eligible for expansion.[0m

[38;2;248;248;242m       HIST_SUBST_PATTERN[0m
[38;2;248;248;242m              Substitutions  using  the  :s  and :& history modifiers are performed with[0m
[38;2;248;248;242m              pattern matching instead of string matching.  This occurs wherever history[0m
[38;2;248;248;242m              modifiers are valid, including glob qualifiers and  parameters.   See  the[0m
[38;2;248;248;242m              section `Modifiers' in zshexpn(1).[0m

[38;2;248;248;242m       IGNORE_BRACES (-I) <S>[0m
[38;2;248;248;242m              Do not perform brace expansion.  For historical reasons this also includes[0m
[38;2;248;248;242m              the effect of the IGNORE_CLOSE_BRACES option.[0m

[38;2;248;248;242m       IGNORE_CLOSE_BRACES[0m
[38;2;248;248;242m              When  neither  this  option  nor  IGNORE_BRACES is set, a sole close brace[0m
[38;2;248;248;242m              character `}' is syntactically significant at any point on a command line.[0m
[38;2;248;248;242m              This has the effect that no semicolon or newline is necessary  before  the[0m
[38;2;248;248;242m              brace  terminating a function or current shell construct.  When either op‐[0m
[38;2;248;248;242m              tion is set, a closing brace is syntactically significant only in  command[0m
[38;2;248;248;242m              position.  Unlike IGNORE_BRACES, this option does not disable brace expan‐[0m
[38;2;248;248;242m              sion.[0m

[38;2;248;248;242m              For example, with both options unset a function may be defined in the fol‐[0m
[38;2;248;248;242m              lowing fashion:[0m

[38;2;248;248;242m                     args() { echo $# }[0m

[38;2;248;248;242m              while if either option is set, this does not work and something equivalent[0m
[38;2;248;248;242m              to the following is required:[0m

[38;2;248;248;242m                     args() { echo $#; }[0m

[38;2;248;248;242m       KSH_GLOB <K>[0m
[38;2;248;248;242m              In  pattern  matching,  the interpretation of parentheses is affected by a[0m
[38;2;248;248;242m              preceding `@', `*', `+', `?' or `!'.  See the  section  `Filename  Genera‐[0m
[38;2;248;248;242m              tion'.[0m

[38;2;248;248;242m       MAGIC_EQUAL_SUBST[0m
[38;2;248;248;242m              All  unquoted  arguments of the form `anything=expression' appearing after[0m
[38;2;248;248;242m              the command name have filename expansion (that is, where expression has  a[0m
[38;2;248;248;242m              leading  `~' or `=') performed on expression as if it were a parameter as‐[0m
[38;2;248;248;242m              signment.  The argument is not otherwise treated specially; it  is  passed[0m
[38;2;248;248;242m              to  the  command as a single argument, and not used as an actual parameter[0m
[38;2;248;248;242m              assignment.  For example, in echo foo=~/bar:~/rod, both occurrences  of  ~[0m
[38;2;248;248;242m              would be replaced.  Note that this happens anyway with typeset and similar[0m
[38;2;248;248;242m              statements.[0m

[38;2;248;248;242m              This  option  respects  the  setting  of the KSH_TYPESET option.  In other[0m
[38;2;248;248;242m              words, if both options are in effect, arguments looking  like  assignments[0m
[38;2;248;248;242m              will not undergo word splitting.[0m

[38;2;248;248;242m       MARK_DIRS (-8, ksh: -X)[0m
[38;2;248;248;242m              Append  a trailing `/' to all directory names resulting from filename gen‐[0m
[38;2;248;248;242m              eration (globbing).[0m

[38;2;248;248;242m       MULTIBYTE <D>[0m
[38;2;248;248;242m              Respect multibyte characters when found in strings.  When this  option  is[0m
[38;2;248;248;242m              set,  strings  are examined using the system library to determine how many[0m
[38;2;248;248;242m              bytes form a character, depending on the current locale.  This affects the[0m
[38;2;248;248;242m              way characters are counted in pattern matching, parameter values and vari‐[0m
[38;2;248;248;242m              ous delimiters.[0m

[38;2;248;248;242m              The option is on by default if the shell was compiled with  MULTIBYTE_SUP‐[0m
[38;2;248;248;242m              PORT; otherwise it is off by default and has no effect if turned on.[0m

[38;2;248;248;242m              If  the  option is off a single byte is always treated as a single charac‐[0m
[38;2;248;248;242m              ter.  This setting is designed purely for examining strings known to  con‐[0m
[38;2;248;248;242m              tain  raw  bytes or other values that may not be characters in the current[0m
[38;2;248;248;242m              locale.  It is not necessary to unset the option merely because the  char‐[0m
[38;2;248;248;242m              acter set for the current locale does not contain multibyte characters.[0m

[38;2;248;248;242m              The  option does not affect the shell's editor,  which always uses the lo‐[0m
[38;2;248;248;242m              cale to determine multibyte characters.  This is because the character set[0m
[38;2;248;248;242m              displayed by the terminal emulator is independent of shell settings.[0m

[38;2;248;248;242m       NOMATCH (+3) <C> <Z>[0m
[38;2;248;248;242m              If a pattern for filename generation has no matches, print an  error,  in‐[0m
[38;2;248;248;242m              stead  of leaving it unchanged in the argument list.  This also applies to[0m
[38;2;248;248;242m              file expansion of an initial `~' or `='.[0m

[38;2;248;248;242m       NULL_GLOB (-G)[0m
[38;2;248;248;242m              If a pattern for filename generation has no matches,  delete  the  pattern[0m
[38;2;248;248;242m              from the argument list instead of reporting an error.  Overrides NOMATCH.[0m

[38;2;248;248;242m       NUMERIC_GLOB_SORT[0m
[38;2;248;248;242m              If  numeric  filenames  are matched by a filename generation pattern, sort[0m
[38;2;248;248;242m              the filenames numerically rather than lexicographically.[0m

[38;2;248;248;242m       RC_EXPAND_PARAM (-P)[0m
[38;2;248;248;242m              Array expansions of the form `foo${xx}bar', where the parameter xx is  set[0m
[38;2;248;248;242m              to  (a b c), are substituted with `fooabar foobbar foocbar' instead of the[0m
[38;2;248;248;242m              default `fooa b cbar'.  Note that an empty array will therefore cause  all[0m
[38;2;248;248;242m              arguments to be removed.[0m

[38;2;248;248;242m       REMATCH_PCRE[0m
[38;2;248;248;242m              If  set,  regular  expression  matching  with  the  =~  operator  will use[0m
[38;2;248;248;242m              Perl-Compatible Regular Expressions from the PCRE library.  (The  zsh/pcre[0m
[38;2;248;248;242m              module  must  be available.)  If not set, regular expressions will use the[0m
[38;2;248;248;242m              extended regexp syntax provided by the system libraries.[0m

[38;2;248;248;242m       SH_GLOB <K> <S>[0m
[38;2;248;248;242m              Disables the special meaning of `(', `|', `)' and '<' for globbing the re‐[0m
[38;2;248;248;242m              sult of parameter and command substitutions,  and  in  some  other  places[0m
[38;2;248;248;242m              where  the shell accepts patterns.  If SH_GLOB is set but KSH_GLOB is not,[0m
[38;2;248;248;242m              the shell allows the interpretation of subshell  expressions  enclosed  in[0m
[38;2;248;248;242m              parentheses  in  some  cases  where  there  is no space before the opening[0m
[38;2;248;248;242m              parenthesis, e.g. !(true) is interpreted as if there were  a  space  after[0m
[38;2;248;248;242m              the !.  This option is set by default if zsh is invoked as sh or ksh.[0m

[38;2;248;248;242m       UNSET (+u, ksh: +u) <K> <S> <Z>[0m
[38;2;248;248;242m              Treat  unset parameters as if they were empty when substituting, and as if[0m
[38;2;248;248;242m              they were zero when reading  their  values  in  arithmetic  expansion  and[0m
[38;2;248;248;242m              arithmetic commands.  Otherwise they are treated as an error.[0m

[38;2;248;248;242m       WARN_CREATE_GLOBAL[0m
[38;2;248;248;242m              Print  a  warning message when a global parameter is created in a function[0m
[38;2;248;248;242m              by an assignment or in math context.  This often indicates that a  parame‐[0m
[38;2;248;248;242m              ter  has not been declared local when it should have been.  Parameters ex‐[0m
[38;2;248;248;242m              plicitly declared global from within a function using typeset  -g  do  not[0m
[38;2;248;248;242m              cause  a warning.  Note that there is no warning when a local parameter is[0m
[38;2;248;248;242m              assigned to in a nested function, which may also indicate an error.[0m

[38;2;248;248;242m       WARN_NESTED_VAR[0m
[38;2;248;248;242m              Print a warning message when an existing parameter from an enclosing func‐[0m
[38;2;248;248;242m              tion scope, or global, is set in a function by an assignment  or  in  math[0m
[38;2;248;248;242m              context.  Assignment to shell special parameters does not cause a warning.[0m
[38;2;248;248;242m              This is the companion to WARN_CREATE_GLOBAL as in this case the warning is[0m
[38;2;248;248;242m              only  printed  when  a  parameter  is not created.  Where possible, use of[0m
[38;2;248;248;242m              typeset -g to set the parameter suppresses the error, but note  that  this[0m
[38;2;248;248;242m              needs  to be used every time the parameter is set.  To restrict the effect[0m
[38;2;248;248;242m              of this option to a single function scope, use `functions -W'.[0m

[38;2;248;248;242m              For example, the following code produces a warning for the assignment  in‐[0m
[38;2;248;248;242m              side the function nested as that overrides the value within toplevel[0m

[38;2;248;248;242m                     toplevel() {[0m
[38;2;248;248;242m                       local foo="in fn"[0m
[38;2;248;248;242m                       nested[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     nested() {[0m
[38;2;248;248;242m                          foo="in nested"[0m
[38;2;248;248;242m                     }[0m
[38;2;248;248;242m                     setopt warn_nested_var[0m
[38;2;248;248;242m                     toplevel[0m

[38;2;248;248;242m   History[0m
[38;2;248;248;242m       APPEND_HISTORY <D>[0m
[38;2;248;248;242m              If this is set, zsh sessions will append their history list to the history[0m
[38;2;248;248;242m              file,  rather  than  replace it. Thus, multiple parallel zsh sessions will[0m
[38;2;248;248;242m              all have the new entries from their history lists  added  to  the  history[0m
[38;2;248;248;242m              file,  in  the  order that they exit.  The file will still be periodically[0m
[38;2;248;248;242m              re-written to trim it when the number of lines grows 20% beyond the  value[0m
[38;2;248;248;242m              specified by $SAVEHIST (see also the HIST_SAVE_BY_COPY option).[0m

[38;2;248;248;242m       BANG_HIST (+K) <C> <Z>[0m
[38;2;248;248;242m              Perform  textual  history expansion, csh-style, treating the character `!'[0m
[38;2;248;248;242m              specially.[0m

[38;2;248;248;242m       EXTENDED_HISTORY <C>[0m
[38;2;248;248;242m              Save each command's beginning timestamp (in seconds since the  epoch)  and[0m
[38;2;248;248;242m              the  duration  (in  seconds) to the history file.  The format of this pre‐[0m
[38;2;248;248;242m              fixed data is:[0m

[38;2;248;248;242m              `: <beginning time>:<elapsed seconds>;<command>'.[0m

[38;2;248;248;242m       HIST_ALLOW_CLOBBER[0m
[38;2;248;248;242m              Add `|' to output redirections in the history.  This allows history refer‐[0m
[38;2;248;248;242m              ences to clobber files even when CLOBBER is unset.[0m

[38;2;248;248;242m       HIST_BEEP <D>[0m
[38;2;248;248;242m              Beep in ZLE when a widget attempts to access a history entry  which  isn't[0m
[38;2;248;248;242m              there.[0m

[38;2;248;248;242m       HIST_EXPIRE_DUPS_FIRST[0m
[38;2;248;248;242m              If  the  internal  history  needs to be trimmed to add the current command[0m
[38;2;248;248;242m              line, setting this option will cause the oldest history event that  has  a[0m
[38;2;248;248;242m              duplicate  to  be  lost  before  losing a unique event from the list.  You[0m
[38;2;248;248;242m              should be sure to set the value of HISTSIZE to a larger number than  SAVE‐[0m
[38;2;248;248;242m              HIST  in  order to give you some room for the duplicated events, otherwise[0m
[38;2;248;248;242m              this option will behave just like HIST_IGNORE_ALL_DUPS  once  the  history[0m
[38;2;248;248;242m              fills up with unique events.[0m

[38;2;248;248;242m       HIST_FCNTL_LOCK[0m
[38;2;248;248;242m              When writing out the history file, by default zsh uses ad-hoc file locking[0m
[38;2;248;248;242m              to avoid known problems with locking on some operating systems.  With this[0m
[38;2;248;248;242m              option  locking  is  done  by means of the system's fcntl call, where this[0m
[38;2;248;248;242m              method is available.  On recent operating systems this may provide  better[0m
[38;2;248;248;242m              performance,  in  particular  avoiding  history  corruption when files are[0m
[38;2;248;248;242m              stored on NFS.[0m

[38;2;248;248;242m       HIST_FIND_NO_DUPS[0m
[38;2;248;248;242m              When searching for history entries in the line editor, do not display  du‐[0m
[38;2;248;248;242m              plicates  of  a line previously found, even if the duplicates are not con‐[0m
[38;2;248;248;242m              tiguous.[0m

[38;2;248;248;242m       HIST_IGNORE_ALL_DUPS[0m
[38;2;248;248;242m              If a new command line being added to the history list duplicates an  older[0m
[38;2;248;248;242m              one,  the  older  command  is removed from the list (even if it is not the[0m
[38;2;248;248;242m              previous event).[0m

[38;2;248;248;242m       HIST_IGNORE_DUPS (-h)[0m
[38;2;248;248;242m              Do not enter command lines into the history list if they are duplicates of[0m
[38;2;248;248;242m              the previous event.[0m

[38;2;248;248;242m       HIST_IGNORE_SPACE (-g)[0m
[38;2;248;248;242m              Remove command lines from the history list when the first character on the[0m
[38;2;248;248;242m              line is a space, or when one of the expanded aliases  contains  a  leading[0m
[38;2;248;248;242m              space.   Only  normal aliases (not global or suffix aliases) have this be‐[0m
[38;2;248;248;242m              haviour.  Note that the command lingers in the internal history until  the[0m
[38;2;248;248;242m              next  command is entered before it vanishes, allowing you to briefly reuse[0m
[38;2;248;248;242m              or edit the line.  If you want to make it vanish right away without enter‐[0m
[38;2;248;248;242m              ing another command, type a space and press return.[0m

[38;2;248;248;242m       HIST_LEX_WORDS[0m
[38;2;248;248;242m              By default, shell history that is read in from files is split  into  words[0m
[38;2;248;248;242m              on  all white space.  This means that arguments with quoted whitespace are[0m
[38;2;248;248;242m              not correctly handled, with the consequence that references  to  words  in[0m
[38;2;248;248;242m              history  lines  that  have  been read from a file may be inaccurate.  When[0m
[38;2;248;248;242m              this option is set, words read in from a history file are divided up in  a[0m
[38;2;248;248;242m              similar fashion to normal shell command line handling.  Although this pro‐[0m
[38;2;248;248;242m              duces  more accurately delimited words, if the size of the history file is[0m
[38;2;248;248;242m              large this can be slow.  Trial and error is necessary to decide.[0m

[38;2;248;248;242m       HIST_NO_FUNCTIONS[0m
[38;2;248;248;242m              Remove function definitions from the history list.  Note that the function[0m
[38;2;248;248;242m              lingers in the internal history until the next command is  entered  before[0m
[38;2;248;248;242m              it vanishes, allowing you to briefly reuse or edit the definition.[0m

[38;2;248;248;242m       HIST_NO_STORE[0m
[38;2;248;248;242m              Remove  the  history  (fc  -l) command from the history list when invoked.[0m
[38;2;248;248;242m              Note that the command lingers in the internal history until the next  com‐[0m
[38;2;248;248;242m              mand  is entered before it vanishes, allowing you to briefly reuse or edit[0m
[38;2;248;248;242m              the line.[0m

[38;2;248;248;242m       HIST_REDUCE_BLANKS[0m
[38;2;248;248;242m              Remove superfluous blanks from each command line being added to  the  his‐[0m
[38;2;248;248;242m              tory list.[0m

[38;2;248;248;242m       HIST_SAVE_BY_COPY <D>[0m
[38;2;248;248;242m              When  the  history file is re-written, we normally write out a copy of the[0m
[38;2;248;248;242m              file named $HISTFILE.new and then rename it over the old one.  However, if[0m
[38;2;248;248;242m              this option is unset, we instead truncate the old history file  and  write[0m
[38;2;248;248;242m              out  the new version in-place.  If one of the history-appending options is[0m
[38;2;248;248;242m              enabled, this option only has an effect when  the  enlarged  history  file[0m
[38;2;248;248;242m              needs  to be re-written to trim it down to size.  Disable this only if you[0m
[38;2;248;248;242m              have special needs, as doing so makes it possible to lose history  entries[0m
[38;2;248;248;242m              if zsh gets interrupted during the save.[0m

[38;2;248;248;242m              When  writing out a copy of the history file, zsh preserves the old file's[0m
[38;2;248;248;242m              permissions and group information, but will refuse to write out a new file[0m
[38;2;248;248;242m              if it would change the history file's owner.[0m

[38;2;248;248;242m       HIST_SAVE_NO_DUPS[0m
[38;2;248;248;242m              When writing out the history file, older  commands  that  duplicate  newer[0m
[38;2;248;248;242m              ones are omitted.[0m

[38;2;248;248;242m       HIST_VERIFY[0m
[38;2;248;248;242m              Whenever  the user enters a line with history expansion, don't execute the[0m
[38;2;248;248;242m              line directly; instead, perform history expansion and reload the line into[0m
[38;2;248;248;242m              the editing buffer.[0m

[38;2;248;248;242m       INC_APPEND_HISTORY[0m
[38;2;248;248;242m              This option works like APPEND_HISTORY except that new  history  lines  are[0m
[38;2;248;248;242m              added to the $HISTFILE incrementally (as soon as they are entered), rather[0m
[38;2;248;248;242m              than  waiting  until the shell exits.  The file will still be periodically[0m
[38;2;248;248;242m              re-written to trim it when the number of lines grows 20% beyond the  value[0m
[38;2;248;248;242m              specified by $SAVEHIST (see also the HIST_SAVE_BY_COPY option).[0m

[38;2;248;248;242m       INC_APPEND_HISTORY_TIME[0m
[38;2;248;248;242m              This  option  is a variant of INC_APPEND_HISTORY in which, where possible,[0m
[38;2;248;248;242m              the history entry is written out to the file after  the  command  is  fin‐[0m
[38;2;248;248;242m              ished,  so that the time taken by the command is recorded correctly in the[0m
[38;2;248;248;242m              history file in EXTENDED_HISTORY format.  This means that the history  en‐[0m
[38;2;248;248;242m              try  will  not  be available immediately from other instances of the shell[0m
[38;2;248;248;242m              that are using the same history file.[0m

[38;2;248;248;242m              This option is only useful if  INC_APPEND_HISTORY  and  SHARE_HISTORY  are[0m
[38;2;248;248;242m              turned off.  The three options should be considered mutually exclusive.[0m

[38;2;248;248;242m       SHARE_HISTORY <K>[0m

[38;2;248;248;242m              This  option  both  imports  new  commands from the history file, and also[0m
[38;2;248;248;242m              causes your typed commands to be appended to the history file (the  latter[0m
[38;2;248;248;242m              is  like specifying INC_APPEND_HISTORY, which should be turned off if this[0m
[38;2;248;248;242m              option is in effect).  The history lines are also output  with  timestamps[0m
[38;2;248;248;242m              ala EXTENDED_HISTORY (which makes it easier to find the spot where we left[0m
[38;2;248;248;242m              off reading the file after it gets re-written).[0m

[38;2;248;248;242m              By  default, history movement commands visit the imported lines as well as[0m
[38;2;248;248;242m              the local lines, but you can toggle this  on  and  off  with  the  set-lo‐[0m
[38;2;248;248;242m              cal-history  zle binding.  It is also possible to create a zle widget that[0m
[38;2;248;248;242m              will make some commands ignore imported commands, and some include them.[0m

[38;2;248;248;242m              If you find that you want more control over when  commands  get  imported,[0m
[38;2;248;248;242m              you  may  wish  to  turn  SHARE_HISTORY off, INC_APPEND_HISTORY or INC_AP‐[0m
[38;2;248;248;242m              PEND_HISTORY_TIME (see above) on, and then manually import commands  when‐[0m
[38;2;248;248;242m              ever you need them using `fc -RI'.[0m

[38;2;248;248;242m   Initialisation[0m
[38;2;248;248;242m       ALL_EXPORT (-a, ksh: -a)[0m
[38;2;248;248;242m              All parameters subsequently defined are automatically exported.[0m

[38;2;248;248;242m       GLOBAL_EXPORT <Z>[0m
[38;2;248;248;242m              If this option is set, passing the -x flag to the builtins declare, float,[0m
[38;2;248;248;242m              integer,  readonly  and typeset (but not local) will also set the -g flag;[0m
[38;2;248;248;242m              hence parameters exported to the environment will not be made local to the[0m
[38;2;248;248;242m              enclosing function, unless they were already or the flag +g is  given  ex‐[0m

[38;2;248;248;242m   Scripts and Functions[0m
[38;2;248;248;242m       ALIAS_FUNC_DEF <S>[0m
[38;2;248;248;242m              By default, zsh does not allow the definition of functions using the `name[0m
[38;2;248;248;242m              ()' syntax if name was expanded as an alias: this causes an  error.   This[0m
[38;2;248;248;242m              is usually the desired behaviour, as otherwise the combination of an alias[0m
[38;2;248;248;242m              and a function based on the same definition can easily cause problems.[0m

[38;2;248;248;242m              When this option is set, aliases can be used for defining functions.[0m

[38;2;248;248;242m              For  example,  consider the following definitions as they might occur in a[0m
[38;2;248;248;242m              startup file.[0m

[38;2;248;248;242m                     alias foo=bar[0m
[38;2;248;248;242m                     foo() {[0m
[38;2;248;248;242m                       print This probably does not do what you expect.[0m
[38;2;248;248;242m                     }[0m

[38;2;248;248;242m              Here, foo is expanded as an alias to bar before the () is encountered,  so[0m
[38;2;248;248;242m              the  function  defined  would be named bar.  By default this is instead an[0m
[38;2;248;248;242m              error in native mode.  Note that quoting any part of the function name, or[0m
[38;2;248;248;242m              using the keyword function, avoids the problem, so is recommended when the[0m
[38;2;248;248;242m              function name can also be an alias.[0m

[38;2;248;248;242m       C_BASES[0m
[38;2;248;248;242m              Output hexadecimal numbers in the standard C format,  for  example  `0xFF'[0m
[38;2;248;248;242m              instead  of the usual `16#FF'.  If the option OCTAL_ZEROES is also set (it[0m
[38;2;248;248;242m              is not by default), octal numbers will be treated similarly and hence  ap‐[0m
[38;2;248;248;242m              pear  as `077' instead of `8#77'.  This option has no effect on the choice[0m
[38;2;248;248;242m              of the output base, nor on the output of bases other than hexadecimal  and[0m
[38;2;248;248;242m              octal.   Note  that these formats will be understood on input irrespective[0m
[38;2;248;248;242m              of the setting of C_BASES.[0m

[38;2;248;248;242m       C_PRECEDENCES[0m
[38;2;248;248;242m              This alters the precedence of arithmetic operators to be more like  C  and[0m
[38;2;248;248;242m              other  programming  languages;  the  section ARITHMETIC EVALUATION in zsh‐[0m
[38;2;248;248;242m              misc(1) has an explicit list.[0m

[38;2;248;248;242m       DEBUG_BEFORE_CMD <D>[0m
[38;2;248;248;242m              Run the DEBUG trap before each command; otherwise it  is  run  after  each[0m
[38;2;248;248;242m              command.  Setting this option mimics the behaviour of ksh 93; with the op‐[0m
[38;2;248;248;242m              tion unset the behaviour is that of ksh 88.[0m

[38;2;248;248;242m       ERR_EXIT (-e, ksh: -e)[0m
[38;2;248;248;242m              If  a  command  has a non-zero exit status, execute the ZERR trap, if set,[0m
[38;2;248;248;242m              and exit.  This is disabled while running initialization scripts.[0m

[38;2;248;248;242m              The behaviour is also disabled inside DEBUG traps.  In this case  the  op‐[0m
[38;2;248;248;242m              tion  is  handled specially: it is unset on entry to the trap.  If the op‐[0m
[38;2;248;248;242m              tion DEBUG_BEFORE_CMD is set, as it is by default, and the option ERR_EXIT[0m
[38;2;248;248;242m              is found to have been set on exit, then the command for  which  the  DEBUG[0m
[38;2;248;248;242m              trap  is being executed is skipped.  The option is restored after the trap[0m
[38;2;248;248;242m              exits.[0m

[38;2;248;248;242m              Non-zero status in a command list containing && or || is ignored for  com‐[0m
[38;2;248;248;242m              mands not at the end of the list.  Hence[0m

[38;2;248;248;242m                     false && true[0m

[38;2;248;248;242m              When this option is set, only the ASCII characters a to z, A to Z, 0 to  9[0m
[38;2;248;248;242m              and _ may be used in identifiers (names of shell parameters and modules).[0m

[38;2;248;248;242m              In  addition, setting this option limits the effect of parameter substitu‐[0m
[38;2;248;248;242m              tion with no braces, so that the expression $# is treated as the parameter[0m
[38;2;248;248;242m              $# even if followed by a valid parameter name.  When it is unset, zsh  al‐[0m
[38;2;248;248;242m              lows  expressions of the form $#name to refer to the length of $name, even[0m
[38;2;248;248;242m              for special variables, for example in expressions such as $#- and $#*.[0m

[38;2;248;248;242m              Another difference is that with the option  set  assignment  to  an  unset[0m
[38;2;248;248;242m              variable  in  arithmetic  context  causes  the variable to be created as a[0m
[38;2;248;248;242m              scalar rather than a numeric type.  So after `unset t; (( t = 3 ))'. with‐[0m
[38;2;248;248;242m              out POSIX_IDENTIFIERS set t has integer type, while with  it  set  it  has[0m
[38;2;248;248;242m              scalar type.[0m

[38;2;248;248;242m              When  the option is unset and multibyte character support is enabled (i.e.[0m
[38;2;248;248;242m              it is compiled in and the option MULTIBYTE is set), then additionally  any[0m
[38;2;248;248;242m              alphanumeric  characters in the local character set may be used in identi‐[0m
[38;2;248;248;242m              fiers.  Note that scripts and functions written with this feature are  not[0m
[38;2;248;248;242m              portable,  and  also  that  both  options must be set before the script or[0m
[38;2;248;248;242m              function is parsed; setting them during execution is not sufficient as the[0m
[38;2;248;248;242m              syntax variable=value has already been parsed as a command rather than  an[0m
[38;2;248;248;242m              assignment.[0m

[38;2;248;248;242m              If  multibyte character support is not compiled into the shell this option[0m
[38;2;248;248;242m              is ignored; all octets with the top bit set may be  used  in  identifiers.[0m
[38;2;248;248;242m              This is non-standard but is the traditional zsh behaviour.[0m

[38;2;248;248;242m       POSIX_STRINGS <K> <S>[0m
[38;2;248;248;242m              This  option  affects processing of quoted strings.  Currently it only af‐[0m
[38;2;248;248;242m              fects the behaviour of null characters, i.e. character 0 in  the  portable[0m
[38;2;248;248;242m              character set corresponding to US ASCII.[0m

[38;2;248;248;242m              When  this  option  is not set, null characters embedded within strings of[0m
[38;2;248;248;242m              the form $'...' are treated as ordinary characters. The entire  string  is[0m
[38;2;248;248;242m              maintained  within the shell and output to files where necessary, although[0m
[38;2;248;248;242m              owing to restrictions of the library interface the string is truncated  at[0m
[38;2;248;248;242m              the  null  character in file names, environment variables, or in arguments[0m
[38;2;248;248;242m              to external programs.[0m

[38;2;248;248;242m              When this option is set, the $'...' expression is truncated  at  the  null[0m
[38;2;248;248;242m              character.  Note that remaining parts of the same string beyond the termi‐[0m
[38;2;248;248;242m              nation of the quotes are not truncated.[0m

[38;2;248;248;242m              For  example,  the command line argument a$'b\0c'd is treated with the op‐[0m
[38;2;248;248;242m              tion off as the characters a, b, null, c, d, and with the option on as the[0m
[38;2;248;248;242m              characters a, b, d.[0m

[38;2;248;248;242m       POSIX_TRAPS <K> <S>[0m
[38;2;248;248;242m              When this option is set, the usual zsh behaviour of  executing  traps  for[0m
[38;2;248;248;242m              EXIT on exit from shell functions is suppressed.  In that case, manipulat‐[0m
[38;2;248;248;242m              ing  EXIT  traps  always alters the global trap for exiting the shell; the[0m
[38;2;248;248;242m              LOCAL_TRAPS option is ignored for the EXIT trap.[0m

[38;2;248;248;242m              Also, a return statement executed in a trap with no argument  passes  back[0m
[38;2;248;248;242m              from  the  function  the value from the surrounding context, not from code[0m
[38;2;248;248;242m              executed within the trap.[0m

[38;2;248;248;242m              Furthermore, if a trap is set to be ignored, this state  persists  when  a[0m
[38;2;248;248;242m              subshell  is  entered.  Without the option, the trap would be reset to its[0m
[38;2;248;248;242m              default state at this point.[0m

[38;2;248;248;242m       SH_FILE_EXPANSION <K> <S>[0m
[38;2;248;248;242m              Perform filename expansion (e.g., ~ expansion) before parameter expansion,[0m
[38;2;248;248;242m              command substitution, arithmetic expansion and brace expansion.   If  this[0m
[38;2;248;248;242m              option  is  unset,  it  is performed after brace expansion, so things like[0m
[38;2;248;248;242m              `~$USERNAME' and `~{pfalstad,rc}' will work.[0m

[38;2;248;248;242m       SH_NULLCMD <K> <S>[0m
[38;2;248;248;242m              Do not use the values of NULLCMD and READNULLCMD when doing  redirections,[0m
[38;2;248;248;242m              use `:' instead (see the section `Redirection').[0m

[38;2;248;248;242m       SH_OPTION_LETTERS <K> <S>[0m
[38;2;248;248;242m              If  this  option is set the shell tries to interpret single letter options[0m
[38;2;248;248;242m              (which are used with set and setopt) like ksh does.  This also affects the[0m
[38;2;248;248;242m              value of the - special parameter.[0m

[38;2;248;248;242m       SH_WORD_SPLIT (-y) <K> <S>[0m
[38;2;248;248;242m              Causes field splitting to be performed on unquoted  parameter  expansions.[0m
[38;2;248;248;242m              Note  that  this  option has nothing to do with word splitting.  (See zsh‐[0m
[38;2;248;248;242m              expn(1).)[0m

[38;2;248;248;242m       TRAPS_ASYNC[0m
[38;2;248;248;242m              While waiting for a program to exit, handle signals and run traps  immedi‐[0m
[38;2;248;248;242m              ately.   Otherwise the trap is run after a child process has exited.  Note[0m
[38;2;248;248;242m              this does not affect the point at which traps are run for any  case  other[0m
[38;2;248;248;242m              than when the shell is waiting for a child process.[0m

[38;2;248;248;242m   Shell State[0m
[38;2;248;248;242m       INTERACTIVE (-i, ksh: -i)[0m
[38;2;248;248;242m              This  is  an interactive shell.  This option is set upon initialisation if[0m
[38;2;248;248;242m              the standard input is a tty and commands are being read from standard  in‐[0m
[38;2;248;248;242m              put.   (See the discussion of SHIN_STDIN.)  This heuristic may be overrid‐[0m
[38;2;248;248;242m              den by specifying a state for this option on the command line.  The  value[0m
[38;2;248;248;242m              of this option can only be changed via flags supplied at invocation of the[0m
[38;2;248;248;242m              shell.  It cannot be changed once zsh is running.[0m

[38;2;248;248;242m       LOGIN (-l, ksh: -l)[0m
[38;2;248;248;242m              This  is  a  login shell.  If this option is not explicitly set, the shell[0m
[38;2;248;248;242m              becomes a login shell if the first character of the argv[0] passed to  the[0m
[38;2;248;248;242m              shell is a `-'.[0m

[38;2;248;248;242m       PRIVILEGED (-p, ksh: -p)[0m
[38;2;248;248;242m              Turn  on  privileged mode. Typically this is used when script is to be run[0m
[38;2;248;248;242m              with elevated privileges. This should be done as follows directly with the[0m
[38;2;248;248;242m              -p option to zsh so that it takes effect during startup.[0m

[38;2;248;248;242m                     #!/bin/zsh -p[0m

[38;2;248;248;242m              The option is enabled automatically  on  startup  if  the  effective  user[0m
[38;2;248;248;242m              (group) ID is not equal to the real user (group) ID. In this case, turning[0m
[38;2;248;248;242m              the  option  off  causes the effective user and group IDs to be set to the[0m
[38;2;248;248;242m              real user and group IDs. Be aware that if that fails the shell may be run‐[0m
[38;2;248;248;242m              ning with different IDs than was intended so a  script  should  check  for[0m
[38;2;248;248;242m              failure and act accordingly, for example:[0m

[38;2;248;248;242m                     unsetopt privileged || exit[0m

[38;2;248;248;242m              The PRIVILEGED option disables sourcing user startup files.  If zsh is in‐[0m
[38;2;248;248;242m              voked  as `sh' or `ksh' with this option set, /etc/suid_profile is sourced[0m
[38;2;248;248;242m              (after /etc/profile on interactive shells). Sourcing  ~/.profile  is  dis‐[0m
[38;2;248;248;242m              abled  and the contents of the ENV variable is ignored. This option cannot[0m
[38;2;248;248;242m              be changed using the -m option of setopt and unsetopt, and changing it in‐[0m
[38;2;248;248;242m              side a function always changes it globally regardless of the LOCAL_OPTIONS[0m
[38;2;248;248;242m              option.[0m

[38;2;248;248;242m       RESTRICTED (-r)[0m
[38;2;248;248;242m              Enables restricted mode.  This option cannot be  changed  using  unsetopt,[0m
[38;2;248;248;242m              and  setting it inside a function always changes it globally regardless of[0m
[38;2;248;248;242m              the LOCAL_OPTIONS option.  See the section `Restricted Shell'.[0m

[38;2;248;248;242m       SHIN_STDIN (-s, ksh: -s)[0m
[38;2;248;248;242m              Commands are being read from the standard input.  Commands are  read  from[0m
[38;2;248;248;242m              standard  input if no command is specified with -c and no file of commands[0m
[38;2;248;248;242m              is specified.  If SHIN_STDIN is set explicitly on the  command  line,  any[0m
[38;2;248;248;242m              argument  that  would  otherwise have been taken as a file to run will in‐[0m
[38;2;248;248;242m              stead be treated as a normal positional parameter.  Note that  setting  or[0m
[38;2;248;248;242m              unsetting  this option on the command line does not necessarily affect the[0m
[38;2;248;248;242m              state the option will have while the shell is running - that is purely  an[0m
[38;2;248;248;242m              indicator of whether or not commands are actually being read from standard[0m
[38;2;248;248;242m              input.  The value of this option can only be changed via flags supplied at[0m
[38;2;248;248;242m              invocation of the shell.  It cannot be changed once zsh is running.[0m

[38;2;248;248;242m       SINGLE_COMMAND (-t, ksh: -t)[0m
[38;2;248;248;242m              If  the shell is reading from standard input, it exits after a single com‐[0m
[38;2;248;248;242m              mand has been executed.  This also makes the shell non-interactive, unless[0m
[38;2;248;248;242m              the INTERACTIVE option is explicitly set on the command line.   The  value[0m
[38;2;248;248;242m              of this option can only be changed via flags supplied at invocation of the[0m
[38;2;248;248;242m              shell.  It cannot be changed once zsh is running.[0m

[38;2;248;248;242m   Zle[0m
[38;2;248;248;242m       BEEP (+B) <D>[0m
[38;2;248;248;242m              Beep on error in ZLE.[0m

[38;2;248;248;242m       COMBINING_CHARS[0m
[38;2;248;248;242m              Assume that the terminal displays combining characters correctly.  Specif‐[0m
[38;2;248;248;242m              ically,  if  a  base  alphanumeric  character  is  followed by one or more[0m
[38;2;248;248;242m              zero-width punctuation characters, assume that the  zero-width  characters[0m
[38;2;248;248;242m              will  be  displayed as modifications to the base character within the same[0m
[38;2;248;248;242m              width.  Not all terminals  handle  this.   If  this  option  is  not  set,[0m
[38;2;248;248;242m              zero-width characters are displayed separately with special mark-up.[0m

[38;2;248;248;242m              If  this  option  is set, the pattern test [[:WORD:]] matches a zero-width[0m
[38;2;248;248;242m              punctuation character on the assumption that it will be used as part of  a[0m
[38;2;248;248;242m              word  in combination with a word character.  Otherwise the base shell does[0m
[38;2;248;248;242m              not handle combining characters specially.[0m

[38;2;248;248;242m       EMACS  If ZLE is loaded, turning on this option  has  the  equivalent  effect  of[0m
[38;2;248;248;242m              `bindkey -e'.  In addition, the VI option is unset.  Turning it off has no[0m
[38;2;248;248;242m              effect.   The  option  setting  is  not  guaranteed to reflect the current[0m
[38;2;248;248;242m              keymap.  This option is provided for compatibility; bindkey is the  recom‐[0m
[38;2;248;248;242m              mended interface.[0m

[38;2;248;248;242m       OVERSTRIKE[0m
[38;2;248;248;242m              Start up the line editor in overstrike mode.[0m

[38;2;248;248;242m       SINGLE_LINE_ZLE (-M) <K>[0m
[38;2;248;248;242m              Use single-line command line editing instead of multi-line.[0m

[38;2;248;248;242m              Note that although this is on by default in ksh emulation it only provides[0m
[38;2;248;248;242m              superficial  compatibility with the ksh line editor and reduces the effec‐[0m
[38;2;248;248;242m              tiveness of the zsh line editor.  As it has no  effect  on  shell  syntax,[0m
[38;2;248;248;242m              many users may wish to disable this option when using ksh emulation inter‐[0m
[38;2;248;248;242m              actively.[0m

[38;2;248;248;242m       VI     If  ZLE  is  loaded,  turning  on this option has the equivalent effect of[0m
[38;2;248;248;242m              `bindkey -v'.  In addition, the EMACS option is unset.  Turning it off has[0m
[38;2;248;248;242m              no effect.  The option setting is not guaranteed to  reflect  the  current[0m
[38;2;248;248;242m              keymap.   This option is provided for compatibility; bindkey is the recom‐[0m
[38;2;248;248;242m              mended interface.[0m

[38;2;248;248;242m       ZLE (-Z)[0m
[38;2;248;248;242m              Use the zsh line editor.  Set by default in interactive  shells  connected[0m
[38;2;248;248;242m              to a terminal.[0m
