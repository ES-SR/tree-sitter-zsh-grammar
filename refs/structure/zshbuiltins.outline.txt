L1 SHELL BUILTIN COMMANDS
L2   Some shell builtin commands take options as described in individual
L2   entries; these are often referred to in the list below as ‘flags' to avoid
L2   confusion with shell options, which may also have an effect on the
L2   behaviour of builtin commands.  In this introductory section, ‘option'
L2   always has the meaning of an option to a command that should be familiar
L2   to most command line users.
L2   Typically, options are single letters preceded by a hyphen (-).  Options
L2   that take an argument accept it either immediately following the option
L2   letter or after white space, for example ‘print -C3 {1..9}' or ‘print -C 3
L2   {1..9}' are equivalent.  Arguments to options are not the same as
L2   arguments to the command; the documentation indicates which is which.
L2   Options that do not take an argument may be combined in a single word, for
L2   example ‘print -rca -- *' and ‘print -r -c -a -- *' are equivalent.
L2   Some shell builtin commands also take options that begin with ‘+' instead
L2   of ‘-'.  The list below makes clear which commands these are.
L2   Options (together with their individual arguments, if any) must appear in
L2   a group before any non-option arguments; once the first non-option
L2   argument has been found, option processing is terminated.
L2   All builtin commands other than ‘echo' and precommand modifiers, even
L2   those that have no options, can be given the argument ‘--' to terminate
L2   option processing.  This indicates that the following words are non-option
L2   arguments, but is otherwise ignored.  This is useful in cases where
L2   arguments to the command may begin with ‘-'.  For historical reasons, most
L2   builtin commands (including ‘echo') also recognize a single ‘-' in a
L2   separate word for this purpose; note that this is less standard and use of
L2   ‘--' is recommended.
